# 谈判与议价

> 智能体就资源、价格、任务分配和条款进行谈判。2026 年的基准测试结论是明确的：NegotiationArena（arXiv:2402.05863）表明 LLM 可以通过人格操纵（"绝望"策略）将收益提高约 20%；《衡量谈判能力》（arXiv:2402.15813）表明买方比卖方更难，且规模扩大并无帮助——他们的 **OG-Narrator**（确定性报价生成器 + LLM 叙述器）将成交率从 26.67% 提升到 88.88%；大规模自主谈判竞赛（arXiv:2503.06416）运行了约 18 万次谈判，发现**隐藏思维链**的智能体通过向对手隐藏推理过程而获胜；Bhattacharya 等人 2025 年基于哈佛谈判项目指标的研究将 Llama-3 评为最有效、Claude-3 最激进、GPT-4 最公平。本课时实现合约网协议（FIPA 的前身，第 02 课）、构建一个类 LLM 的买方/卖方、运行 OG-Narrator 风格的解耦，并测量每种结构性选择对成交率的影响。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**先修：** 第 16 阶段 · 02（FIPA-ACL 遗产）、第 16 阶段 · 09（并行群体网络）
**时间：** 约 75 分钟

## 问题

两个智能体需要就价格达成一致。如果完全依赖纯语言提示，2024-2026 年的 LLM 成交率低得惊人（在 arXiv:2402.15813 中严格参数化的议价场景下约为 27%）。规模扩大并不能解决这个问题：GPT-4 在谈判能力上并不比 GPT-3.5 有结构性优势；它只是在谈判的*语言*方面更好。

根本问题在于 LLM 混淆了两个任务——决定报价和叙述报价。OG-Narrator 将这两个任务分离：确定性报价生成器计算数值动作；LLM 只负责叙述。成交率跃升至约 89%。

这反映了一个经典的多智能体发现：将机制与通信层解耦才是制胜之道。合约网协议（FIPA，1996；Smith，1980）是参考任务市场机制。将 LLM 接入叙述插槽，你就得到了一个现代化的 LLM 驱动的任务市场。

## 概念

### 合约网，一段话说完

Smith 1980 年的合约网协议：一个**管理者**广播**征求建议书（cfp）**；**投标者**用包含其报价的**提议**消息回应；管理者选出获胜者，向获胜者发送**接受提议**，向失败者发送**拒绝提议**。获胜者执行工作。可选消息：**拒绝**（投标者拒绝提议）。FIPA 将其正式化为 `fipa-contract-net` 交互协议。

### 为什么 OG-Narrator 能赢

《衡量语言模型的谈判能力》（arXiv:2402.15813）观察到：

- LLM 经常违反谈判规则（以荒谬的价格报价，忽略对方的 ZOPA）。
- 它们锚定能力差（接受糟糕的首次报价；提出象征性而非战略性的还价）。
- 仅靠规模无法修复这些问题。更大的模型产生更似合理的语言，但策略错误相似。

OG-Narrator 的解耦方式：

```
           ┌──────────────────┐        ┌──────────────────┐
  状态  → │ 报价生成器        │ 价格 → │  LLM 叙述器      │ → 消息
           │  （确定性）       │        │  （撰写类人的     │
           │                  │        │   附带说明）       │
           └──────────────────┘        └──────────────────┘
```

报价生成器是一个经典的谈判策略：Rubinstein 议价模型、Zeuthen 策略，或基于价格的简单针锋相对。LLM 负责叙述。消息包含确定性价格和自然语言框架。

成交率跃升的原因：

- 价格保持在议价区间内。
- 锚定是战略性的，而非情绪化的。
- LLM 做它擅长的事：写作。

### NegotiationArena 的发现

arXiv:2402.05863 提供了权威基准。主要发现：

- LLM 可以通过采用人格（"我周五前必须卖掉这个"）将收益提高约 20%——人格操纵是一种真实存在的策略。
- 公平/合作型智能体容易被对抗型智能体利用；防御需要明确的反制姿态。
- 对称配对在约 40% 的基准场景中收敛到不平等的结果。

这不是"LLM 是糟糕的谈判者"。而是"LLM 谈判方式太像人类，包括可被利用的部分"。

### 思维链隐藏

大规模自主谈判竞赛（arXiv:2503.06416）运行了约 18 万次跨多种 LLM 策略的谈判。获胜者向对手隐藏了他们的推理过程：

- 如果一个智能体将"我只出到 75 美元；我的保留价格是 70 美元"打印到公开可见的草稿区，对手就能读到它。
- 获胜者私下计算策略；输出通道仅包含报价和最少的必要叙述。

这是经典博弈论（Aumann 1976 年关于理性和信息）在 2026 年的回响：暴露你的私有估值会付出代价。LLM 不会直觉到这一点，并乐于在推理痕迹中输入它们的保留价格，而这些痕迹变得对对手可见。

工程启示：将私有草稿上下文与公开消息上下文分开。这不是可选项。

### Bhattacharya 等人 2025 年——模型排名

基于哈佛谈判项目指标（原则性谈判、BATNA 尊重、利益互惠）：

- **Llama-3** 在达成交易方面最有效（成交率 + 收益）。
- **Claude-3** 是最激进的谈判者（高锚定、晚让步）。
- **GPT-4** 是最公平的（不同配对间的收益方差最小）。

这是 2025 年的快照。重点不是哪个模型在 2026 年 4 月获胜——而是不同的基础模型具有持久的谈判风格。异构集成（第 15 课）将其作为一种多样性来源。

### 通过合约网 + LLM 进行任务分配

LLM 多智能体对合约网的现代重用：

1. 管理者智能体将任务分解为单元。
2. 向工作智能体广播带有任务描述的 `cfp`。
3. 每个工作者返回一个报价：`(价格, 预计完成时间, 置信度)`，其中价格可以是代币、计算单元或美元。
4. 管理者选出获胜者（单个或多个，取决于任务）并授予任务。
5. 被拒绝的工作者可以自由投标其他任务。

这在超过 100 个工作者的规模下表现良好，因为协调是广播-响应式的，而非同步聊天。已在生产中使用：Microsoft Agent Framework 的编排模式、一些 LangGraph 实现。

### LLM 利益相关者交互式谈判

NeurIPS 2024（https://proceedings.neurips.cc/paper_files/paper/2024/file/984dd3db213db2d1454a163b65b84d08-Paper-Datasets_and_Benchmarks_Track.pdf）引入了具有**秘密分数**和**最低接受阈值**的多方可评分游戏。每个利益相关者都有私有效用；LLM 必须从消息中推断它们。这是两方议价向 N 方联盟形成的推广。与具有异构工作者能力的生产任务市场相关。

### 叙述 vs 机制规则

在所有 2024-2026 年的谈判基准中，一致性的工程规则是：

> 让 LLM 叙述。不要让 LLM 计算报价。

如果报价需要是一个数字（价格、预计完成时间、数量），从谈判状态确定性地生成它，并让 LLM 产生框架。如果报价需要是一个提议结构（任务分解、角色分配），让 LLM 起草它，但在发送前根据模式和约束检查进行验证。

## 构建

`code/main.py` 实现了：

- `ContractNetManager`、`ContractNetTask`、`Bid`——管理者 + 投标者，广播 cfp，收集提议，授予任务。
- `og_narrator_bargain(state, rng)`——OG-Narrator 买方：确定性地向中点让步（Zeuthen 风格）。
- `seller_response(state, rng)`——确定性卖方还价策略（两种风格的结构化真实情况）。
- `naive_llm_bargain(state, rng)`——模拟全 LLM 议价者：以高方差选择价格，经常在 ZOPA 之外。
- 测量：1000 次试验的成交率，每次试验采样新的保留价格。

运行：

```
python3 code/main.py
```

预期输出：朴素 LLM 成交率约 65-75%；OG-Narrator 成交率约 85-95%；15-25 个百分点的差距是解耦报价生成与叙述的结构性优势。外加一个带有三个投标者和一个任务的合约网任务市场分配示例。

## 使用

`outputs/skill-bargainer-designer.md` 设计了一个议价协议：谁生成报价（确定性或 LLM），谁叙述，私有草稿区如何与公开消息分开，以及如何监控成交率。

## 投产

生产议价检查清单：

- **分开草稿区。** 私有状态永远不应到达对手的上下文。这是不可协商的。
- **确定性报价生成。** 价格、数量、预计完成时间：计算，不要提示。
- **根据模式验证所有传入报价。** 在协议边界拒绝超出 ZOPA 的报价。
- **限制轮次。** 最多 3-5 轮；死锁时上报调解人。
- **持续测量成交率和收益方差。** 成交率下降是一种症状——通常是提示漂移或对手方攻击。
- **记录所有被拒绝的提议**及其确定性理由。对于合约网管理者，失败的投标者需要理解原因。

## 练习

1. 运行 `code/main.py`。确认 OG-Narrator 在成交率上优于朴素 LLM。高出多少？
2. 实现**基于人格的收益改进**（arXiv:2402.05863）——买方仅在叙述中采用"本周急于购买"人格，报价生成器不变。成交率或收益是否改变？
3. 实现思维链**隐藏**：维护一个不传递给对手的私有草稿字符串。如果你意外泄露它（通过交换通道模拟）会发生什么？
4. 将合约网扩展为带有保留价格的 N 投标者拍卖。当所有投标都超过保留价格时，管理者如何在最低价格和最高质量之间决定？你选择哪种授予规则？为什么？
5. 阅读 Bhattacharya 等人 2025 年关于哈佛谈判项目指标的文章。实现两种不同风格（激进 vs 公平）的议价者。测量对称和非对称配对下的收益方差。

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|------|------------|----------|
| 合约网 | "任务市场" | Smith 1980，FIPA 1996。cfp + 提议 + 接受/拒绝。权威任务市场。 |
| ZOPA | "可能协议区间" | 买方最高价和卖方最低价的重叠部分。之外的报价无法成交。 |
| BATNA | "谈判协议的最佳替代方案" | 如果本次交易失败，你的 fallback。设定你的保留价格。 |
| OG-Narrator | "报价生成器 + 叙述器" | 解耦：确定性报价，LLM 叙述。 |
