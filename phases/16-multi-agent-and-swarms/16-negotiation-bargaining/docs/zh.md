# 谈判与议价

> 智能体就资源、价格、任务分配和条款进行谈判。2026年的基准测试集很明确：NegotiationArena（arXiv:2402.05863）表明LLM可以通过角色操控（"绝望"）将收益提高约20%；"测量议价能力"（arXiv:2402.15813）表明买方比卖方更难，而且规模没有帮助——他们的**OG-Narrator**（确定性报价生成器 + LLM叙述者）将成交率从26.67%提高到88.88%；大规模自主谈判竞赛（arXiv:2503.06416）运行了约18万次谈判，发现**隐藏思维链**的智能体通过向对手隐藏推理而获胜；Bhattacharya等人2025年基于哈佛谈判项目指标的排名显示，Llama-3最有效，Claude-3最激进，GPT-4最公平。本课程实现了合同网协议（FIPA的前身，第02课），连接LLM风格的买方/卖方，运行OG-Narrator风格的分解，并测量每种结构选择下成交率的变化。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置知识：** 第16阶段 · 02（FIPA-ACL传承），第16阶段 · 09（并行群体网络）
**时间：** 约75分钟

## 问题

两个智能体需要在价格上达成一致。如果仅依靠纯语言提示，2024-2026年的LLM以 surprisingly 低的成交率完成交易（在arXiv:2402.15813中严格参数化的议价中约为27%）。规模无法解决这个问题：GPT-4在议价方面并不比GPT-3.5有结构性优势；它只是更擅长议价的*语言*。

根本问题在于LLM混淆了两项工作——决定报价和叙述报价。OG-Narrator将这两者分开：确定性报价生成器计算数字变动；LLM只负责叙述。成交率跃升至约89%。

这反映了经典多智能体的一个发现：将机制与通信层解耦会获胜。合同网协议（FIPA，1996；Smith，1980）是参考任务市场机制。将LLM插入叙述槽位，你就得到了一个现代LLM驱动的任务市场。

## 核心概念

### 合同网，一段话

Smith 1980年的合同网协议：**管理者**广播**招标（cfp）**；**投标者**用包含其报价的**提议**消息回应；管理者选择获胜者并向获胜者发送**接受提议**，向失败者发送**拒绝提议**。获胜者执行工作。可选消息：**拒绝**（投标者拒绝提议）。FIPA将其编纂为`fipa-contract-net`交互协议。

### 为什么OG-Narrator获胜

"测量语言模型的议价能力"（arXiv:2402.15813）观察到：

- LLM经常违反议价规则（以荒谬的价格报价，忽略对方的ZOPA）。
- 它们的锚定很差（接受糟糕的首个报价；还价时给出象征性而非战略性的金额）。
- 仅靠规模无法解决这些问题。更大的模型用类似的战略错误产生更合理的语言。

OG-Narrator分解：

```
           ┌──────────────────┐        ┌──────────────────┐
  状态  → │ 报价生成器       │ 价格 → │  LLM叙述者       │ → 消息
           │  (确定性的)      │        │  (撰写           │
           │                  │        │   人类风格的     │
           └──────────────────┘        │   伴随文本)      │
                                       └──────────────────┘
```

报价生成器是经典谈判策略：Rubinstein议价模型、Zeuthen策略，或简单的价格上的以牙还牙。LLM负责叙述。消息包含确定性价格和自然语言框架。

成交率跃升是因为：
- 价格保持在议价区内。
- 锚定是战略性的，不是情绪化的。
- LLM做它擅长的事：写作。

### NegotiationArena发现

arXiv:2402.05863提供了规范基准。主要发现：

- LLM可以通过采用角色（"我绝望地要在周五前卖掉这个"）将收益提高约20%——角色操控是真实战术。
- 公平/合作的智能体被对抗性的利用；防御需要明确的反姿态。
- 对称配对在约40%的基准场景中收敛到不平等结果。

这不是"LLM是糟糕的谈判者"。而是"LLM谈判太像人类了，包括可利用的部分"。

### 思维链隐藏

大规模自主谈判竞赛（arXiv:2503.06416）在许多LLM策略中运行了约18万次谈判。获胜者向对手隐藏了他们的推理：

- 如果一个智能体在公开可见的草稿中打印"我只出到75美元；我的保留价是70美元"，对手会读到它。
- 获胜者私下计算策略；输出通道只包含报价和最低限度的必要叙述。

这是2026年对经典博弈论的回应（Aumann 1976关于理性和信息）：透露你的私人估值会损失收益。LLM不直观理解这一点，并乐于在推理痕迹中输入它们的保留价，而这些痕迹对对手可见。

工程要点：将私人草稿上下文与公共消息上下文分开。不是可选的。

### Bhattacharya等人2025——模型排名

基于哈佛谈判项目指标（原则性谈判、BATNA尊重、利益互惠）：

- **Llama-3**在达成交易方面最有效（成交率 + 收益）。
- **Claude-3**是最激进的谈判者（高锚定、晚让步）。
- **GPT-4**是最公平的（跨配对收益方差最小）。

这是2025年的快照。重点不是哪个模型在2026年4月获胜——而是不同基础模型有持久的谈判风格。异构集成（第15课）将其作为多样性来源。

### 通过合同网 + LLM进行任务分配

合同网用于LLM多智能体的现代重用：

1. 管理者智能体将任务分解为单元。
2. 向工作者智能体广播带有任务描述的`cfp`。
3. 每个工作者返回报价：`(价格, 预计时间, 置信度)`，其中价格可以是token、计算单元或美元。
4. 管理者选择获胜者（单个或多个，取决于任务）并授予。
5. 被拒绝的工作者可以自由地对其他任务投标。

这可以很好地扩展到100多个工作者，因为协调是广播-回应，不是同步聊天。用于生产：Microsoft Agent Framework的编排模式，一些LangGraph实现。

### LLM利益相关者交互谈判

NeurIPS 2024（https://proceedings.neurips.cc/paper_files/paper/2024/file/984dd3db213db2d1454a163b65b84d08-Paper-Datasets_and_Benchmarks_Track.pdf）引入了带有**秘密分数**和**最低接受阈值**的多方可评分游戏。每个利益相关者有私人效用；LLM必须从消息中推断它们。这是两方议价向N方联盟形成的推广。与具有异构工作者能力的生产任务市场相关。

### 叙述 vs 机制规则

在所有2024-2026年的谈判基准中，一致的工程规则是：

> 让LLM叙述。不要让LLM计算报价。

如果报价需要是数字（价格、预计时间、数量），从谈判状态确定性生成，并让LLM产生框架。如果报价需要是提议结构（任务分解、角色分配），让LLM起草，但在发送前根据模式和约束检查进行验证。

## 构建它

`code/main.py`实现：

- `ContractNetManager`、`ContractNetTask`、`Bid` —— 管理者 + 投标者，广播cfp，收集提议，授予。
- `og_narrator_bargain(state, rng)` —— OG-Narrator买方：确定性的Zeuthen式向中点让步。
- `seller_response(state, rng)` —— 确定性卖方还价策略（两种风格的结构基本事实）。
- `naive_llm_bargain(state, rng)` —— 模拟全LLM议价者：以高方差选择价格，经常在ZOPA之外。
- 测量：在1000次试验中的成交率，每次试验采样新的保留价。

运行：

```
python3 code/main.py
```

预期输出：朴素LLM成交率约65-75%；OG-Narrator成交率约85-95%；15-25个百分点的差距是将报价生成与叙述分解的结构优势。加上一个合同网任务市场分配示例，有三个投标者和一个任务。

## 交付它

生产议价检查清单：

- **分离草稿。** 私人状态永远不会到达对手的上下文。这是不可协商的。
- **确定性报价生成。** 价格、数量、预计时间：计算，不要提示。
- **根据模式验证所有传入报价。** 在协议边界拒绝ZOPA之外的报价。
- **限制轮次。** 最多3-5轮；僵局时升级到调解人。
- **持续测量成交率和收益方差。** 下降的成交率是症状——通常是提示漂移或对手方攻击。
- **记录所有被拒绝的提议及确定性理由。** 对于合同网管理者，失败的投标者需要理解为什么。

## 练习

1. 运行`code/main.py`。确认OG-Narrator在成交率上击败朴素LLM。差距多少？
2. 实现**基于角色的收益改进**（arXiv:2402.05863）—— 买方仅在叙述中采用"本周绝望购买"的角色，报价生成器不变。成交率或收益变化吗？
3. 实现思维链**隐藏**：维护一个不传递给对手的私人草稿字符串。如果你不小心泄露它（通过交换通道模拟）会发生什么？
4. 将合同网扩展到带保留价的N投标者拍卖。当所有报价都超过保留价时，管理者如何在最低价和最高质量之间决定？你选择哪种授予规则，为什么？
5. 阅读Bhattacharya等人2025年关于哈佛谈判项目指标。实现两种不同风格的议价者（激进 vs 公平）。测量对称和非对称配对下的收益方差。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| 合同网 | "任务市场" | Smith 1980，FIPA 1996。cfp + 提议 + 接受/拒绝。规范任务市场。 |
| ZOPA | "可能协议区" | 买方最高和卖方最低之间的重叠。之外的报价无法成交。 |
| BATNA | "谈判协议的最佳替代方案" | 如果这笔交易失败，你的后备方案。设定你的保留价。 |
| OG-Narrator | "报价生成器 + 叙述者" | 分解：确定性报价，LLM叙述。 |
| Zeuthen策略 | "风险最小化让步" | 基于风险限制让步的经典报价生成器。 |
| Rubinstein议价 | "交替报价均衡" | 带折现的无限期议价的博弈论模型。 |
| CoT隐藏 | "隐藏你的推理" | arXiv:2503.06416的获胜者保持私人草稿；公共通道只显示报价。 |
| 角色操控 | "情绪姿态" | arXiv:2402.05863：绝望/紧迫角色带来约20%收益增益。 |
