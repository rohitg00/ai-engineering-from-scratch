# 失败模式——MAST、群体思维、单一文化、级联错误

> 2026年的参考分类法是**MAST**（Cemri等人，NeurIPS 2025，arXiv:2503.13657），源自7个最先进的开源MAS的1642个执行追踪，显示**41-86.7%的失败率**。三个根本原因类别：**规范问题**（41.77%）——角色模糊、任务定义不清；**协调失败**（36.94%）——通信崩溃、状态失步；**验证缺口**（21.30%）——缺少验证、缺乏质量检查。**群体思维**家族（arXiv:2508.05687）补充：单一文化崩溃（相同基础模型→相关失败）、从众偏见（智能体强化彼此的错误）、心智理论缺陷、混合动机动态、级联可靠性失败。级联示例：重试风暴，支付失败触发订单重试，触发库存重试，压垮库存服务（数秒内10倍负载——需要熔断器）。记忆投毒：一个智能体的幻觉进入共享内存，下游智能体将其视为事实；准确性逐渐衰减，使根因诊断痛苦。**STRATUS**（NeurIPS 2025）报告通过专门的检测/诊断/验证智能体实现1.5倍缓解成功改善。本课程将失败模式视为一等工程目标。

**类型：** 学习
**语言：** Python（标准库）
**前置知识：** 第16阶段 · 13（共享内存），第16阶段 · 14（共识与BFT），第16阶段 · 15（投票与辩论拓扑）
**时间：** 约75分钟

## 问题

多智能体系统在真实任务上41-86.7%的时间失败（Cemri等人2025年在7个开源MAS上测量了这一点）。这不是通过"只需添加更多智能体"就能调试的。失败有结构性原因。MAST分类法给你类别。本课程将每个类别映射到具体的检测、诊断和缓解模式，使数字不再看起来随意。

2026年的生产实践是将失败模式视为设计输入。你的架构不是"足够好"，直到你能指向每个MAST类别并命名你部署的缓解措施。

## 概念

### MAST类别

**规范问题（41.77%的失败）。** 智能体的任务定义不够紧密。示例：

- 角色模糊：两个智能体都认为自己是审查者。
- 任务欠指定："总结这个"，当用户想要特定角度时。
- 成功标准隐式：智能体无法判断它是否成功。

缓解措施：
- 编写显式角色契约。每个智能体的提示说明它做什么*和不做什么*。
- 每个任务的验收测试。在智能体开始之前，定义"完成看起来像X"。
- 飞行前规范检查：一个单独的智能体在分派前审查任务定义。

**协调失败（36.94%）。** 通信或状态崩溃。

示例：
- 两个智能体在没有同步的情况下更新共享状态。
- 智能体之间的消息丢失（队列失败、超时）。
- 状态漂移：智能体A认为任务完成；智能体B仍在执行。

缓解措施：
- 带乐观并发的版本化共享状态。
- 关键消息的显式确认（重试直到确认）。
- 定期状态同步检查点；早期检测漂移。

**验证缺口（21.30%）。** 没有对输出的独立检查。

示例：
- 一个智能体声称成功；没有人验证。
- 智能体链每个都信任前一个的输出。
- 对涌现组合行为的测试覆盖缺失。

缓解措施：
- 独立验证者智能体（第13课）。只读，独立源访问。
- 显式交接契约："A的输出必须通过检查器C才能开始B"。
- 用于事后分析的结果日志。

### 群体思维家族（arXiv:2508.05687）

当智能体同质化或模仿彼此时五种相关失败：

**单一文化崩溃。** 相同基础模型或训练数据→相关错误。当三个智能体共享一个LLM时，它们共享其幻觉。

**从众偏见。** 智能体向最大声或最自信的同伴调整，即使错误。

**ToM缺陷。** 智能体无法对彼此的信念建模；协调崩溃（第18课）。

**混合动机动态。** 具有部分对齐激励的智能体漂向妥协中间，这谁都不满意。

**级联可靠性失败。** 一个组件的错误模式触发依赖组件中的错误模式。

### 级联示例——重试风暴

经典的2026年事件模式：

```
支付服务10%的请求失败
   ↓
订单智能体重试支付（指数退避但天真）
   ↓
每次重试是新的订单-库存检查
   ↓
库存服务看到2倍正常负载
   ↓
库存服务开始超时
   ↓
每个订单重试库存检查
   ↓
库存服务看到10倍正常负载
   ↓
集群宕机
```

修复是经典的：**熔断器**。当下游错误率超过阈值时，用缓存或默认结果短路。加上每个请求的封顶重试预算。

熔断器是你直接从分布式系统借用而无需修改的少数多智能体失败缓解措施之一。

### 记忆投毒（重访）

来自第13课：一个智能体的幻觉成为共享内存事实；下游智能体在投毒事实上推理。用MAST术语，这是共享内存层的验证缺口。

逐渐准确性衰减是症状。你不会得到崩溃；你得到难以根因的缓慢漂移。

缓解：仅追加日志、来源、不可写验证者。已在第13课中涵盖。

### STRATUS——用于失败检测的专门智能体

STRATUS（NeurIPS 2025）报告当你部署时1.5倍缓解成功改善：

- **检测智能体。** 观察症状模式（高分歧、重试峰值、准确性漂移）。
- **诊断智能体。** 给定症状，从MAST分类法推断可能的根本原因。
- **验证智能体。** 应用缓解后，检查症状是否清除。

这是应用于智能体系统的SRE风格事件响应。三个角色都可以是具有专门提示的LLM智能体。

### 失败模式审计

2026年最佳实践是年度（或每个主要版本）失败模式审计：

1. **追踪采样。** 收集约1000个真实执行追踪。
2. **分类。** 对每个追踪的失败，映射到MAST + 群体思维类别。
3. **计算按类别的失败率。** 哪些类别主导你的系统？
4. **排名缓解措施。** 哪个修复会消除最多失败？
5. **选择2-3个缓解措施。** 实施；下季度重新审计。

纪律比具体选择更重要。没有审计，失败融入噪音，永远不会被系统解决。

### 当系统静默失败时

最危险的失败类别是静默正确性失败。大声失败的系统（崩溃、异常、警报）可以被监控。产生合理但错误输出的系统无法通过异常日志检测。这就是为什么验证缺口虽然只占21.30%，却是每次失败最昂贵的类别。

投资于：
- 基于采样的人工审查。
- 黄金数据集回归测试。
- 重要输出上的跨智能体交叉检查。

### 失败 vs 缓慢失败

有些失败是即时的；有些是缓慢的。即时失败（超时、模式不匹配、认证错误）检测便宜。缓慢失败（记忆投毒、单一文化漂移、角色模糊）检测和预防昂贵。

2026年的工程动作：检测缓慢失败代理，以便你可以在漂移成为可见错误之前捕获它。同意率、重试率、输出长度分布和连续智能体版本之间的编辑距离都是有用的代理。

## 交付它

生产中的失败模式纪律：

- **每季度MAST审计。** 不是年度。随着系统增长，类别会变化。
- **到处熔断器。** 对每个出站调用到任何依赖服务。默认在5-10%错误率时打开阈值。
- **黄金数据集。** 小、高质量、人工审计。每周对它们进行回归测试。
- **STRATUS三人组。** 检测 + 诊断 + 验证智能体监控生产。从仅检测智能体开始；当症状嘈杂时添加诊断。
- **失败预算。** 按类别明确的失败率SLO。超出预算触发停止发布对话。

2. 实现一个**缓慢失败代理**：3个并行智能体的同意率。当它急剧下降时，触发警报。通过逐渐相关智能体输出模拟单一文化漂移。
3. 阅读Cemri等人（arXiv:2503.13657）。选择他们的7个MAS系统之一并映射其前3个失败类别。这些与MAST预测的比较如何？

5. 为你知道的特定多智能体系统设计一个STRATUS风格的检测-诊断-验证三人组。检测观察哪些症状？诊断推荐哪些缓解措施？验证如何确认它们有效？

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| MAST | "2026年分类法" | Cemri 2025；3个根类别 + 14种子类型的失败。 |
| 规范问题 | "角色模糊" | 任务或角色定义不足；智能体不知道做什么。 |
| 协调失败 | "状态漂移" | 智能体之间的通信或同步崩溃。 |
| 验证缺口 | "没人检查" | 未经独立验证接受的输出。 |
| 群体思维家族 | "同质化失败" | 单一文化、从众、ToM缺陷、混合动机、级联。 |
| 单一文化崩溃 | "相同模型，相同幻觉" | 来自共享基础模型或训练数据的相关错误。 |
| 重试风暴 | "级联错误放大" | 一个失败触发重试，放大下游负载。 |
| 熔断器 | "错误率时快速失败" | 错误率超过阈值时打开；用默认短路。 |
| STRATUS | "事件响应三人组" | 检测 + 诊断 + 验证智能体。1.5倍缓解成功。 |
| 记忆投毒 | "幻觉传播" | 共享内存事实被污染；下游智能体在毒药上推理。 |
