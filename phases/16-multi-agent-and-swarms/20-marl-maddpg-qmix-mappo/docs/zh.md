# MARL —— MADDPG、QMIX、MAPPO

> 多智能体协调的强化学习传承，在2026年仍然为LLM智能体系统提供信息。**MADDPG**（Lowe等人，NeurIPS 2017，arXiv:1706.02275）引入了集中式训练、分散式执行（CTDE）：每个critic在训练期间看到所有智能体的状态和动作；测试时只有本地actor运行。适用于合作、竞争和混合设置。**QMIX**（Rashid等人，ICML 2018，arXiv:1803.11485）是具有单调混合网络的价值分解；每个智能体的Q值组合成联合Q值，因此`argmax`干净地分布——在星际争霸多智能体挑战（SMAC）上占主导地位。**MAPPO**（Yu等人，NeurIPS 2022，arXiv:2103.01955）是具有集中式价值函数的PPO；在粒子世界、SMAC、Google Research Football、Hanabi上" surprisingly 有效"，只需最少调参。这些为必须分散行动的智能体团队训练策略提供了基础。MAPPO是**2026年合作MARL的默认基线**。本课程从一个小网格世界玩具构建每个算法，在接触LLM智能体训练之前将这三个想法烙印在肌肉记忆中。

**类型：** 学习
**语言：** Python（标准库，小型无NumPy实现）
**前置知识：** 第09阶段（强化学习），第16阶段 · 09（并行群体网络）

## 问题

LLM智能体系统越来越多地为智能体间协调训练策略：何时退让、何时行动、调用哪个同伴。告诉你如何训练这些策略的文献是多智能体强化学习（MARL），它早于LLM浪潮，有一小套主导算法。

没有模式词汇阅读MARL论文是痛苦的。集中式训练与分散式执行（CTDE）、价值分解和集中式critic不是流行语——它们是对特定问题的特定答案：

- 独立RL（每个智能体单独学习）从每个智能体的角度看是非平稳的。不好。
- 集中式RL（一个智能体控制所有）无法扩展且违反执行约束。
- CTDE获得两者最佳：用全局信息训练，用本地策略部署。

## 概念

### 论文使用的三种环境

- **粒子世界（多智能体粒子环境）。** 简单2D物理，合作/竞争任务。MADDPG的原始测试平台。
- **星际争霸多智能体挑战（SMAC）。** 合作微管理，部分观察。QMIX的测试平台。离散动作，连续状态。
- **Google Research Football、Hanabi、MPE。** MAPPO基线。

不同环境有不同的动作/观察类型。算法相应选择。

### MADDPG（2017）—— CTDE模式

每个智能体`i`有一个actor`mu_i(o_i)`，将其自己的观察映射到动作。每个智能体还有一个critic`Q_i(x, a_1, ..., a_n)`，在训练期间看到所有观察和所有动作。actor通过策略梯度根据critic的评估更新。

```
actor更新：    grad_theta_i J = E[grad_theta mu_i(o_i) * grad_a_i Q_i(x, a_1..n) 在 a_i=mu_i(o_i) 处]
critic更新：   给定下一状态联合估计的Q_i(x, a_1..n)上的TD
```

为什么CTDE：在训练时，我们知道每个人的动作；我们用它来减少每个critic的方差。在部署时，每个智能体只看到`o_i`并调用`mu_i(o_i)`。

失败模式：critic随N个智能体增长（输入包括所有动作）。没有近似无法扩展到约10个智能体以上。

### QMIX（2018）—— 价值分解

仅合作。全局奖励是每个智能体Q值的单调函数之和：

```
Q_tot(tau, a) = f(Q_1(tau_1, a_1), ..., Q_n(tau_n, a_n)),   df/dQ_i >= 0
```

单调性保证`argmax_a Q_tot`可以通过每个智能体独立选择`argmax_{a_i} Q_i`来计算。这正是你需要的**分散式执行属性**。在训练时，混合网络从每个智能体的Q值产生`Q_tot`。

为什么QMIX在SMAC上获胜：合作星际争霸微管理有同构智能体、局部观察、全局奖励——完美适合价值分解。

失败模式：单调性约束是限制性的；有些任务有非单调可分解的奖励结构（一个智能体为团队牺牲）。扩展（QTRAN、QPLEX）放宽了这一点。

### MAPPO（2022）—— 被忽视的默认选择

多智能体PPO：具有集中式价值函数的PPO。每个智能体有自己的策略；所有智能体共享（或有每个智能体的）看到完整状态的价值函数。Yu等人2022年在五个基准上针对MADDPG、QMIX及其扩展对MAPPO进行了基准测试，发现：

- MAPPO在粒子世界、SMAC、Google Research Football、Hanabi、MPE上匹配或击败离线MARL方法。
- 需要最少超参数调优。
- 训练稳定；跨种子可复现。

直到这篇论文，社区低估了在线MARL。2026年，MAPPO是合作MARL的默认基线；任何新方法必须击败它。

### 为什么LLM智能体工程师应该关心

三个直接用途：

1. **路由器训练。** 元智能体选择哪个子智能体处理任务。这是一个有N个分散子智能体和一个集中路由器的MARL问题。MAPPO适合。
2. **角色涌现。** 在生成式智能体模拟中，训练智能体随时间采用互补角色是伪装的MARL问题。QMIX风格的价值分解按构造强制互补性。
3. **多智能体工具使用。** 当智能体共享工具并竞争预算时，通过CTDE训练它们产生可部署的尊重资源约束的本地策略。

实际警告：2026年，大多数生产LLM智能体系统提示它们的策略而不是训练它们。MARL在以下情况出现：（a）大量交互数据，（b）清晰的奖励信号，（c）愿意投资训练基础设施。

### CTDE作为超越RL的设计模式

即使没有训练，CTDE也是一个有用的架构模式：

- 在*设计*期间，假设完整团队可见性。
- 在*运行时*，强制执行分散式执行：每个智能体只看到`o_i`。

该模式强制你保持每个智能体状态显式，并提前考虑部分可观察性。许多生产多智能体系统到处静默假设共享状态——CTDE纪律防止这一点。

### 非平稳性问题

当多个智能体同时学习时，每个智能体的环境（包括其他智能体的策略）是非平稳的。经典单智能体RL证明失效。本课程中的MARL算法都解决这个问题：

- MADDPG：全局critic看到所有动作，因此其价值估计是平稳的。
- QMIX：价值分解将学习移到联合Q空间，其中最优性定义良好。
- MAPPO：集中式价值函数抑制其他策略变化的方差。

在LLM智能体系统中，非平稳性表现为"我的智能体上个月工作正常，现在上游那个智能体变了，我的就行为异常了。"用CTDE训练MARL是原则性修复；提示级修复更快但不够持久。

### 本课程不涵盖的内容

训练实际网络是第09阶段的主题。本课程构建脚本化策略版本，演示CTDE、价值分解和集中式价值模式，无需梯度更新。目标是在你拿起完整MARL库（PyMARL、MARLlib、RLlib多智能体）之前将模式内化。

## 构建它

`code/main.py`实现三个模式演示，都在一个微小的2智能体合作网格世界上：

- 环境：4x4网格上的2个智能体，一个奖励颗粒。奖励=1如果任何智能体到达颗粒；任务结束。
- `IndependentAgents` —— 每个智能体将其他视为环境。基线。
- `MADDPGStyle` —— 集中式critic计算联合价值；actor策略从中更新。脚本化策略改进。
- `QMIXStyle` —— 具有单调混合器的价值分解。
- `MAPPOStyle` —— 集中式价值函数；策略针对共享基线更新。

所有四个运行相同的回合并报告平均到达目标的步数。CTDE变体比独立基线收敛到更短的路径。

运行：

```
python3 code/main.py
```

预期输出：独立智能体平均约6步；CTDE变体收敛到约3.5步（4x4网格的最优是3）。尽管是脚本化策略，模式差异也显现出来。

`outputs/skill-marl-picker.md`是一个技能，为给定多智能体任务选择MARL算法：合作 vs 竞争、同构 vs 异构、动作空间类型、规模、奖励信号。

## 交付它

生产中MARL很少见。当你确实使用它时：

- **从MAPPO开始。** 2022年的论文确立了这是基线；先复现它节省数周追逐更花哨方法的时间。
- **记录每个智能体的观察和动作流。** 没有每个智能体追踪调试MARL是无望的。
- **将训练代码与执行代码分离。** CTDE是一种纪律；让执行路径真的只看到`o_i`。
- **奖励塑造警告。** MARL对奖励设计非常敏感。塑造中的一个协调漏洞，智能体就会学会利用它。运行对抗性测试。
- **对于LLM智能体**，首先考虑提示级策略。只有当交互数据 + 奖励信号 + 基础设施都存在时，才投资MARL训练。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| MARL | "多智能体RL" | 多智能体系统的强化学习。 |
| CTDE | "集中式训练，分散式执行" | 用全局信息训练；用本地策略部署。 |
| MADDPG | "多智能体DDPG" | CTDE，每个智能体的critic看到所有观察 + 动作。 |
| QMIX | "价值分解" | 每个智能体Q值的单调混合。合作。 |
| MAPPO | "多智能体PPO" | 具有集中式价值函数的PPO。2026年默认基线。 |
| 价值分解 | "个体Q值之和" | 联合Q表示为每个智能体Q值的单调函数。 |
| 非平稳性 | "移动目标" | 每个智能体的环境随其他智能体学习而变化。核心MARL问题。 |
| 在线/离线 | "从当前/回放缓冲区学习" | PPO是在线的（MAPPO）；DDPG和Q-learning是离线的。 |
| SMAC | "星际争霸多智能体挑战" | 合作微管理基准；QMIX的主场。 |
