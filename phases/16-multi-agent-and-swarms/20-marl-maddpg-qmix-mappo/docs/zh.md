# MARL — MADDPG、QMIX、MAPPO

> 多智能体协调的强化学习遗产，在 2026 年仍然为 LLM 智能体系统提供信息。**MADDPG**（Lowe 等人，NeurIPS 2017，arXiv:1706.02275）引入了集中训练、分散执行（CTDE）：每个评论者在训练期间看到所有智能体的状态和行为；在测试时仅运行本地行动者。适用于合作、竞争和混合设置。**QMIX**（Rashid 等人，ICML 2018，arXiv:1803.11485）是具有单调混合网络的价值分解；每个智能体的 Q 组合成联合 Q，因此 `argmax` 干净地分布——在 StarCraft 多智能体挑战（SMAC）中占主导地位。**MAPPO**（Yu 等人，NeurIPS 2022，arXiv:2103.01955）是带有集中价值函数的 PPO；在粒子世界、SMAC、Google Research Football、Hanabi 上"出奇地有效"，只需最少的调优。这些支撑着必须分散行动的智能体团队的训练策略。MAPPO 是 **2026 年默认的合作型 MARL 基线**。本课从一个小型网格世界玩具构建每一个，并在接触 LLM 智能体训练之前将这三个想法融入肌肉记忆。

**类型：** 学习
**语言：** Python（标准库，小型无 NumPy 实现）
**前置要求：** 第 09 阶段（强化学习）、第 16 阶段 · 09（并行群体网络）
**时间：** 约 90 分钟

## 问题

LLM 智能体系统越来越多地训练智能体间协调策略：何时让步、何时行动、调用哪个对等节点。告诉你如何训练此类策略的文献是多智能体强化学习（MARL），它早于 LLM 浪潮，并且有一小部分主导算法。

在没有模式词汇的情况下阅读 MARL 论文是痛苦的。带有分散执行的集中训练（CTDE）、价值分解和集中评论者不是流行语——它们是对特定问题的具体回答：

- 独立 RL（每个智能体单独学习）从每个智能体的角度来看是非平稳的。不好。
- 集中 RL（一个智能体控制所有）无法扩展并违反执行约束。
- CTDE 兼得两者之长：使用全局信息训练，使用本地策略部署。

## 概念

### 论文使用的三种环境

- **粒子世界（multi-agent particle env）。** 具有合作/竞争任务的简单 2D 物理。MADDPG 的原始测试平台。
- **StarCraft 多智能体挑战（SMAC）。** 合作微操作，部分可观察性。QMIX 的测试平台。离散动作，连续状态。
- **Google Research Football、Hanabi、MPE。** MAPPO 基线。

不同的环境具有不同的动作/观察类型。算法相应地选择。

### MADDPG（2017）— CTDE 模式

每个智能体 `i` 有一个行动者 `mu_i(o_i)`，将其自己的观察映射到动作。每个智能体还有一个评论者 `Q_i(x, a_1, ..., a_n)`，在训练期间看到所有观察和所有动作。行动者通过对评论者的评估进行策略梯度更新。

```
actor update:    grad_theta_i J = E[grad_theta mu_i(o_i) * grad_a_i Q_i(x, a_1..n) at a_i=mu_i(o_i)]
critic update:   TD on Q_i(x, a_1..n) given next-state joint estimate
```

为何 CTDE：在训练时，我们知道每个人的动作；我们利用它来减少每个评论者的方差。在部署时，每个智能体只能看到 `o_i` 并调用 `mu_i(o_i)`。

失效模式：评论者随 N 个智能体而增长（输入包括所有动作）。在没有近似的情况下，无法扩展到约 10 个智能体以上。

### QMIX（2018）— 价值分解

仅限合作。全局奖励是每个智能体 Q 值的单调函数之和：

```
Q_tot(tau, a) = f(Q_1(tau_1, a_1), ..., Q_n(tau_n, a_n)),   df/dQ_i >= 0
```

单调性保证 `argmax_a Q_tot` 可以通过每个智能体独立选择 `argmax_{a_i} Q_i` 来计算。这**正是你需要的分散执行属性**。在训练时，混合网络从每个智能体的 Q 产生 `Q_tot`。

为何 QMIX 在 SMAC 上胜出：合作的 StarCraft 微操作具有同构智能体、本地 obs、全局奖励——非常适合价值分解。

失效模式：单调性约束是限制性的；某些任务具有不是单调可分解的奖励结构（一个智能体为团队牺牲）。扩展（QTRAN、QPLEX）放宽了这一点。

### MAPPO（2022）— 被忽视的默认设置

多智能体 PPO：带有集中价值函数的 PPO。每个智能体有自己的策略；所有智能体共享（或拥有每个智能体的）价值函数，这些函数看到完整状态。Yu 等人 2022 在五个基准上将 MAPPO 与 MADDPG、QMIX 及其扩展进行了基准测试，发现：

- MAPPO 在粒子世界、SMAC、Google Research Football、Hanabi、MPE 上匹配或击败离策略 MARL 方法。
- 需要最少的超参数调优。
- 训练稳定；跨种子可重现。

在这篇论文之前，社区低估了在策略 MARL。在 2026 年，MAPPO 是合作型 MARL 的默认基线；任何新方法都必须击败它。

### 为何 LLM 智能体工程师应该关心

三个直接用途：

1. **路由器训练。** 一个元智能体选择哪个子智能体处理任务。这是一个具有 N 个分散子智能体和一个集中路由器的 MARL 问题。MAPPO 适合。
2. **角色涌现。** 在生成式智能体模拟中，训练智能体随时间采用互补角色是一个伪装的 MARL 问题。QMIX 风格的价值分解通过构造强制互补性。
3. **多智能体工具使用。** 当智能体共享工具并竞争预算时，通过 CTDE 训练它们会产生可部署的本地策略，这些策略尊重资源约束。

实践警告：在 2026 年，大多数生产 LLM 智能体系统提示其策略而不是训练它们。当你有 (a) 大量交互数据、(b) 明确的奖励信号和 (c) 投资训练基础设施的意愿时，MARL 才会介入。

### 超越 RL 的 CTDE 作为设计模式

即使没有训练，CTDE 也是一种有用的架构模式：

- 在*设计*期间，假设完全的团队可见性。
- 在*运行时*，强制分散执行：每个智能体只能看到 `o_i`。

该模式迫使你保持每个智能体的状态明确，并预先考虑部分可观察性。许多生产多智能体系统默默地假设到处都是共享状态——CTDE 纪律防止了这一点。

### 非平稳性问题

当多个智能体同时学习时，每个智能体的环境（包括其他人的策略）是非平稳的。经典的单智能体 RL 证明被打破了。本课中的 MARL 算法都解决了这个问题：

- MADDPG：全局评论者看到所有动作，因此其价值估计是平稳的。
- QMIX：价值分解将学习转移到一个联合 Q 空间，其中最优性得到了很好的定义。
- MAPPO：集中价值函数抑制了来自其他人策略变化的方差。

在 LLM 智能体系统中，非平稳性表现为"我的智能体上个月工作，现在那个上游智能体改变了，我的行为不端。" 用 CTDE 训练 MARL 是有原则的修复；提示级修复更快但不那么持久。

### 本课不涵盖的内容

训练实际网络是第 09 阶段的话题。本课构建脚本化策略版本，演示 CTDE、价值分解和集中价值模式，无需梯度更新。目标是在你使用完整的 MARL 库（PyMARL、MARLlib、RLlib multi-agent）之前内化这些模式。

## 动手实践

`code/main.py` 实现了三个模式演示，全部在一个小型 2 智能体合作网格世界上：

- 环境：4x4 网格上的 2 个智能体，一个奖励 pellets。如果任何智能体到达 pellet，奖励 = 1；任务完成。
- `IndependentAgents` — 每个智能体将其他人视为环境。基线。
- `MADDPGStyle` — 集中评论者计算联合价值；行动者策略从中更新。脚本化策略改进。
- `QMIXStyle` — 带有单调混合器的价值分解。
- `MAPPOStyle` — 集中价值函数；策略针对共享基线进行更新。

所有四个运行相同的回合并报告平均步数到目标。CTDE 变体比独立基线收敛到更短的路径。

运行：

```
python3 code/main.py
```

预期输出：独立智能体平均需要约 6 步；CTDE 变体收敛到约 3.5 步（4x4 网格的最优值是 3）。尽管是脚本化策略，模式差异仍然显示出来。

## 使用

`outputs/skill-marl-picker.md` 是一个为给定多智能体任务选择 MARL 算法的技能：合作 vs 竞争、同构 vs 异构、动作空间类型、规模、奖励信号。

## 部署

生产中的 MARL 是罕见的。当你确实使用它时：

- **从 MAPPO 开始。** 2022 年的论文将其确立为基线；首先重现它可以节省数周追逐更复杂方法的时间。
- **记录每个智能体的观察和动作流。** 没有每个智能体的追踪，调试 MARL 是没有希望的。
- **将训练代码与执行代码分开。** CTDE 是一种纪律；让执行路径真正只能看到 `o_i`。
- **奖励塑形警告。** MARL 对奖励设计极其敏感。塑形中的一个协调错误，智能体就会学会利用它。运行对抗测试。
- **对于 LLM 智能体**，首先考虑提示级策略。仅在交互数据 + 奖励信号 + 基础设施都存在时，才投资 MARL 训练。

## 练习

2. 实现一个竞争变体：两个智能体，一个 pellet，只有第一个到达的才能获得奖励。哪种模式干净地处理竞争？历史上 MADDPG。
3. 阅读 MADDPG（arXiv:1706.02275）第 3 节。用你自己的话以伪代码形式符号化地实现确切的评论者更新规则。
4. 阅读 MAPPO（arXiv:2103.01955）。为何作者认为集中价值 + PPO 在他们的基准上击败了离策略 MARL？列出三个最强的主张。
5. 将 CTDE 作为设计模式应用于假设的 LLM 智能体系统（例如，研究智能体 + 摘要器 + 编码者）。在设计时可用但在运行时不可用的联合信息是什么？

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|------|----------------|------------------------|
| MARL | "多智能体 RL" | 多智能体系统的强化学习。 |
| CTDE | "集中训练、分散执行" | 使用全局信息训练；使用本地策略部署。 |
| MADDPG | "多智能体 DDPG" | 带有看到所有观察 + 动作的每个智能体评论者的 CTDE。 |
| QMIX | "价值分解" | 每个智能体 Q 的单调混合。合作型。 |
| MAPPO | "多智能体 PPO" | 带有集中价值函数的 PPO。2026 年默认基线。 |
| 价值分解（Value decomposition） | "单个 Q 的总和" | 联合 Q 表示为每个智能体 Q 的单调函数。 |
| 非平稳性（Non-stationarity） | "移动目标" | 随着其他人学习，每个智能体的环境都会发生变化。核心 MARL 问题。 |
| 在策略 / 离策略（On-policy / off-policy） | "从当前 / 回放中学习" | PPO 是在策略（MAPPO）；DDPG 和 Q-learning 是离策略。 |
| SMAC | "StarCraft 多智能体挑战" | 合作微操作基准；QMIX 的本土阵地。 |
