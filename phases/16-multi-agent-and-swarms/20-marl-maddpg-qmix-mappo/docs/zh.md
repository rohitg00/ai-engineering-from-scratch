# 20 · 多智能体强化学习——MADDPG、QMIX、MAPPO

> 多智能体协调的强化学习血脉，直到 2026 年仍在为大语言模型（LLM）智能体系统提供理论参考。**MADDPG**（Lowe 等人，NeurIPS 2017，arXiv:1706.02275）提出了「集中式训练、分散式执行（Centralized Training, Decentralized Execution，CTDE）」：训练时每个评论家（critic）能看到所有智能体的状态与动作；测试时只运行各自的本地行动者（actor）。它适用于合作型、竞争型以及混合型场景。**QMIX**（Rashid 等人，ICML 2018，arXiv:1803.11485）是一种带「单调混合网络（monotonic mixing network）」的「价值分解（value decomposition）」方法；将各智能体的 Q 值组合成联合 Q 值，使得 `argmax` 能够干净地分解到每个智能体——在「星际争霸多智能体挑战赛（StarCraft Multi-Agent Challenge，SMAC）」上长期占据主导。**MAPPO**（Yu 等人，NeurIPS 2022，arXiv:2103.01955）是带「集中式价值函数（centralized value function）」的 PPO；在粒子世界、SMAC、Google Research Football、Hanabi 上仅需极少调参便「出人意料地有效」。这些方法支撑着对必须分散行动的智能体团队的策略训练。MAPPO 是 **2026 年合作型 MARL 的默认基线**。本课从一个小型网格世界玩具示例出发逐一构建这三种方法，在接触 LLM 智能体训练之前把这三个思想固化为肌肉记忆。

**类型：** 学习
**语言：** Python（标准库，小型、不依赖 NumPy 的实现）
**前置：** 第 09 阶段（强化学习）、第 16 阶段 · 09（并行集群网络）
**时长：** 约 90 分钟

## 问题

LLM 智能体系统越来越多地为智能体间的协调训练策略：何时延迟交给他人、何时自己行动、调用哪个同伴。告诉你如何训练这类策略的文献体系，就是「多智能体强化学习（Multi-Agent Reinforcement Learning，MARL）」，它早于 LLM 浪潮，并拥有一小批占主导地位的算法。

在缺乏模式词汇的情况下读 MARL 论文是痛苦的。集中式训练与分散式执行（CTDE）、价值分解、集中式评论家都不是流行术语——它们是针对特定问题的特定答案：

- 独立强化学习（每个智能体单独学习）从各智能体视角看是非平稳的。糟糕。
- 集中式强化学习（一个智能体控制所有智能体）无法扩展，且违反执行约束。
- CTDE 兼得两者之长：用全局信息训练，用本地策略部署。

## 概念

### 论文使用的三种环境

- **粒子世界（multi-agent particle env）。** 简单的二维物理环境，含合作/竞争任务。MADDPG 最初的测试平台。
- **星际争霸多智能体挑战赛（SMAC）。** 合作型微操作，部分可观测。QMIX 的测试平台。离散动作、连续状态。
- **Google Research Football、Hanabi、MPE。** MAPPO 的基线环境。

不同环境有不同的动作/观测类型。算法据此进行选择。

### MADDPG（2017）——CTDE 模式

每个智能体 `i` 拥有一个行动者 `mu_i(o_i)`，将其自身观测映射为动作。每个智能体还拥有一个评论家 `Q_i(x, a_1, ..., a_n)`，在训练时能看到所有观测与所有动作。行动者通过针对评论家评估的策略梯度进行更新。

```
actor update:    grad_theta_i J = E[grad_theta mu_i(o_i) * grad_a_i Q_i(x, a_1..n) at a_i=mu_i(o_i)]
critic update:   TD on Q_i(x, a_1..n) given next-state joint estimate
```

为何用 CTDE：训练时我们知道每个人的动作，便用它来降低各评论家的方差。部署时每个智能体只看到 `o_i` 并调用 `mu_i(o_i)`。

失效模式：评论家随智能体数量 N 增长（输入包含所有动作）。若不做近似，无法扩展到约 10 个以上的智能体。

### QMIX（2018）——价值分解

仅适用于合作型。全局奖励是各智能体 Q 值的某个单调函数：

```
Q_tot(tau, a) = f(Q_1(tau_1, a_1), ..., Q_n(tau_n, a_n)),   df/dQ_i >= 0
```

单调性保证了 `argmax_a Q_tot` 可以由每个智能体独立地选择 `argmax_{a_i} Q_i` 来计算。这**恰恰是你所需要的分散式执行性质**。训练时，一个混合网络从各智能体的 Q 值生成 `Q_tot`。

为何 QMIX 在 SMAC 上取胜：合作型星际争霸微操作拥有同质智能体、本地观测、全局奖励——与价值分解完美契合。

失效模式：单调性约束具有限制性；某些任务的奖励结构并非单调可分解（比如某个智能体为团队牺牲自己）。其扩展方法（QTRAN、QPLEX）放宽了这一约束。

### MAPPO（2022）——被忽视的默认选择

多智能体 PPO：带集中式价值函数的 PPO。每个智能体拥有自己的策略；所有智能体共享（或各自拥有）能看到完整状态的价值函数。Yu 等人于 2022 年在五个基准上将 MAPPO 与 MADDPG、QMIX 及其扩展进行了对比，发现：

- 在粒子世界、SMAC、Google Research Football、Hanabi、MPE 上，MAPPO 与离策略（off-policy）MARL 方法持平或更优。
- 所需的超参数调优极少。
- 训练稳定；跨随机种子可复现。

在此论文之前，社区一直低估了在策略（on-policy）MARL。到了 2026 年，MAPPO 成为合作型 MARL 的默认基线；任何新方法都必须击败它。

### LLM 智能体工程师为何应当关注

三个直接用途：

1. **路由器训练。** 一个元智能体（meta-agent）选择由哪个子智能体处理某项任务。这是一个具有 N 个分散式子智能体与一个集中式路由器的 MARL 问题。MAPPO 适用。
2. **角色涌现。** 在生成式智能体仿真中，训练智能体随时间采用互补角色，本质上是一个伪装过的 MARL 问题。QMIX 式的价值分解通过构造方式强制实现互补性。
3. **多智能体工具使用。** 当智能体共享工具并争夺预算时，通过 CTDE 训练它们可产出可部署的、尊重资源约束的本地策略。

实务提醒：到 2026 年，大多数生产级 LLM 智能体系统是为其策略编写提示词（prompt），而非训练它们。MARL 适用于以下情形：（a）拥有大量交互数据，（b）有清晰的奖励信号，以及（c）愿意投入训练基础设施。

### CTDE 作为超越强化学习的设计模式

即便不进行训练，CTDE 仍是一种有用的架构模式：

- 在*设计*阶段，假定团队全局可见。
- 在*运行时*，强制分散式执行：每个智能体只看到 `o_i`。

该模式迫使你保持每个智能体的状态显式化，并提前考虑部分可观测性。许多生产级多智能体系统会默默地假定处处共享状态——CTDE 的纪律可以避免这一点。

### 非平稳性问题

当多个智能体同时学习时，每个智能体的环境（其中包含他人的策略）都是非平稳的。经典的单智能体强化学习证明随之失效。本课中的各 MARL 算法都应对了这一点：

- MADDPG：全局评论家能看到所有动作，因此其价值估计是平稳的。
- QMIX：价值分解将学习转移到一个最优性有良好定义的联合 Q 空间中。
- MAPPO：集中式价值函数抑制了来自他人策略变化的方差。

在 LLM 智能体系统中，非平稳性表现为「我的智能体上个月还能用，现在上游那个智能体一改，我的就出问题了」。用 CTDE 训练 MARL 是原则性的修复方法；提示词层面的修复更快，但更不持久。

### 本课不涵盖的内容

训练实际网络是第 09 阶段的主题。本课构建的是脚本化策略（scripted-policy）版本，在不做梯度更新的情况下演示 CTDE、价值分解与集中式价值这三种模式。目标是在你拿起一个完整的 MARL 库（PyMARL、MARLlib、RLlib 多智能体）之前，先把这些模式内化。

## 动手构建

`code/main.py` 实现了三种模式演示，全部基于一个微型的双智能体合作型网格世界：

- 环境：4x4 网格上有 2 个智能体、一个奖励豆（pellet）。任一智能体到达豆即奖励 = 1，任务结束。
- `IndependentAgents`——每个智能体把其他智能体当作环境。基线。
- `MADDPGStyle`——集中式评论家计算联合价值；行动者策略据此更新。脚本化的策略改进。
- `QMIXStyle`——带单调混合器的价值分解。
- `MAPPOStyle`——集中式价值函数；策略针对共享基线进行更新。

四者运行相同的回合（episode）并报告平均「到达目标步数」。CTDE 各变体会收敛到比独立基线更短的路径。

运行：

```
python3 code/main.py
```

预期输出：独立智能体平均约需 6 步；CTDE 各变体收敛至约 3.5 步（4x4 网格的最优值为 3）。尽管使用的是脚本化策略，模式差异依然显现。

## 应用它

`outputs/skill-marl-picker.md` 是一个技能（skill），针对给定的多智能体任务挑选 MARL 算法：合作 vs 竞争、同质 vs 异质、动作空间类型、规模、奖励信号。

## 交付上线

MARL 在生产中很罕见。当你确实要用时：

- **从 MAPPO 开始。** 2022 年的论文已将其确立为基线；先复现它能省下数周追逐更花哨方法的时间。
- **记录每个智能体的观测与动作流。** 没有逐智能体的轨迹，调试 MARL 是没指望的。
- **将训练代码与执行代码分离。** CTDE 是一种纪律；让执行路径真正只看到 `o_i`。
- **奖励塑形警告。** MARL 对奖励设计极其敏感。塑形中一处协调缺陷，智能体就会学会去钻空子。务必运行对抗性测试。
- **对于 LLM 智能体**，先考虑提示词层面的策略。只有当交互数据 + 奖励信号 + 基础设施三者俱全时，才投入 MARL 训练。

## 练习

1. 运行 `code/main.py`。测量独立智能体与 MAPPO 式智能体之间的「到达目标步数」差距。在 6x6 网格上，该差距是扩大还是缩小？
2. 实现一个竞争型变体：两个智能体、一个豆，只有第一个到达者获得奖励。哪种模式能干净地处理竞争？历史上是 MADDPG。
3. 阅读 MADDPG（arXiv:1706.02275）第 3 节。用你自己的话以伪代码符号化地实现精确的评论家更新规则。
4. 阅读 MAPPO（arXiv:2103.01955）。作者为何论证「集中式价值 + PPO」在其基准上胜过离策略 MARL？列出最有力的三条论点。
5. 把 CTDE 作为设计模式应用到一个假想的 LLM 智能体系统（例如：研究智能体 + 摘要智能体 + 编码智能体）。在设计时可获得、而在运行时不可获得的联合信息是什么？

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|----------------|------------------------|
| MARL | “多智能体强化学习” | 面向多智能体系统的强化学习。 |
| CTDE | “集中式训练、分散式执行” | 用全局信息训练；用本地策略部署。 |
| MADDPG | “多智能体 DDPG” | 每个智能体的评论家能看到所有观测 + 动作的 CTDE。 |
| QMIX | “价值分解” | 对各智能体 Q 值的单调混合。合作型。 |
| MAPPO | “多智能体 PPO” | 带集中式价值函数的 PPO。2026 年默认基线。 |
| 价值分解 | “各个 Q 值之和” | 联合 Q 表示为各智能体 Q 值的单调函数。 |
| 非平稳性 | “移动的目标” | 随他人学习，每个智能体的环境也在变化。MARL 的核心问题。 |
| 在策略 / 离策略 | “从当前学 / 从回放学” | PPO 是在策略（MAPPO）；DDPG 与 Q-learning 是离策略。 |
| SMAC | “星际争霸多智能体挑战赛” | 合作型微操作基准；QMIX 的本土阵地。 |
