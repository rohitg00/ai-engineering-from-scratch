# 投票、自一致性与辩论拓扑#

> 最便宜的聚合：采样 N 个独立智能体，多数投票。Wang et al. 2022 自一致性使用一个模型采样 N 次来完成这个。多智能体通过**异构**智能体扩展它以逃避单一文化 —— 不同的模型、不同的提示、不同的温度、不同的上下文。 beyond 多数投票，辩论拓扑很重要：MultiAgentBench (arXiv:2503.01935, ACL 2025) 评估了星型 / 链型 / 树型 / 图型协调，并发现**图型对研究最好**，超过约 4 个智能体时有"协调税"。AgentVerse (ICLR 2024) 记录了两种涌现模式 —— 志愿者行为和从众行为 —— 并且从众既是一种特征（找到共识）也是一种风险（群体思维，课程 24）。本课映射拓扑空间，构建每个变体，并衡量协调税。

**类型：** 学习 + 构建
**语言：** Python (标准库)
**前置条件：** 第 16 阶段 · 07 (心智社会与辩论), 第 16 阶段 · 14 (共识与 BFT)
**时间：** ~75 分钟

## 问题#

辩论可以改进准确性 (Du et al., arXiv:2305.14325)。它也可以降低准确性。辩论是否有帮助取决于四个结构选择：

1. 谁与谁交谈（拓扑）。
2. 多少轮（Du 2023：轮次和智能体都独立重要）。
3. 智能体是否异构（不同的基础模型打破单一文化）。
4. 是否存在对抗性声音（钢铁人 vs. 稻草人）。

直接在整个任务上实施"运行 5 个智能体并投票"的团队通常比单个智能体更差。失败不是随机的。它们跟踪拓扑和异构性。本课是拓扑地图。

## 概念#

### 自一致性，单模型基线#

Wang et al. 2022 ("Self-Consistency Improves Chain of Thought Reasoning") 在同一模型温度 >0 时采样 N 次，并对推理路径答案进行多数投票。在 GSM8K 上的结果：与单次贪婪解码相比，N=40 个样本有实质性收益。自一致性是多智能体投票的单智能体前身。

限制：自一致性使用一个基础模型。错误是通过构建相关的。如果模型有系统偏见，所有 N 个样本都共享它。

### 多智能体投票，异构扩展#

用 N 个*不同的*智能体替换 N 个样本。不同的基础模型 (Claude, GPT, Llama)，不同的提示，不同的工具访问。好处：不相关的错误。成本：不同的智能体成本不同；协调它们增加了开销。

异构辩论的规范 2026 年名称是 **A-HMAD** —— 对抗性异构多智能体辩论。没有普遍采用，但论文使用这个术语表示"不同模型辩论，这减少了单一文化崩溃的相关错误。"

### 四种拓扑#

```
星型                链型               树型                图型

    ┌─A─┐           A─B─C─D         ┌──A──┐              A───B
    │   │                           │     │              │ × │
    B   C                           B     C              D───C
    │   │                          / \   / \
    D   E                         D   E F   G           (全连接)
```

星型：一个中心，所有其他只与中心交谈。等同于没有反向通道的监督者-工作者。
链型：线性，每个智能体看到前一个的输出。类似管道。
树型：分层，由分层智能体系统使用（课程 06）。
图型：任意到任意。包括全连接集团和任意 DAG。

### 协调税 (MultiAgentBench)#

MultiAgentBench (MARBLE, ACL 2025, arXiv:2503.01935) 在包括研究、编码和规划的套件上基准测试了星型、链型、树型、图型。关键测量结果：

- **图型**拓扑在研究任务上获胜。信息流是任意到任意的；智能体可以相互批评。
- **星型**在快速回答事实任务上获胜。中心过滤并整合。
- **链型**在逐步管道上获胜（分阶段精炼）。
- **协调税**在图型拓扑中出现在约 4 个智能体之后。挂钟和令牌成本的增长快于质量。

4 智能体上限是经验性的，不是根本性的。它反映了 2026 年 LLM 上下文容量：每个智能体的上下文都充满了同行输出，并且一旦每个人都可以看到每个人，添加智能体 N+1 的边际价值就会下降。

### 多智能体辩论策略（"我们应该去 MAD 吗？"）#

arXiv:2311.17371 是 2023 年 MAD 策略的调查。其他复制的关键发现：*结构 similar* 于自一致性的 MAD 变体（独立采样 + 聚合）在使用相同预算时通常表现不如自一致性。当智能体真正异构并且辩论具有对抗结构（一个智能体反对）时，MAD 最有帮助。

### AgentVerse 涌现模式#

AgentVerse (ICLR 2024, https://proceedings.iclr.cc/paper_files/paper/2024/file/578e65cdee35d00c708d4c64bce32971-Paper-Conference.pdf) 记录了即使没有显式设计，多智能体辩论也会涌现的两种行为：

- **志愿者。** 智能体主动提供帮助（"我可以采取下一步"）无需提示。有用：它将工作分配给最有能力的智能体来处理子任务。
- **从众。** 智能体调整其立场以匹配批评者，即使批评者是错误的。这是辩论等效于谄媚（课程 14）。

从众是为什么辩论直到一致奖励恶霸的原因。有限的轮次加上单独的裁判可以缓解这个问题。

### 异构性：实际移动准确性的旋钮#

2024-2026 年实践文献中的一种模式：将你的 N 个智能体中的一个换成不同的基础模型，比将 N 增加 1 带来的准确性提升更大。直觉是单一文化 —— 每个新的独立错误源比相关的额外样本更有价值。

在极限情况下，异构性击败数量性。在大多数具有清晰基本事实的任务上，三个不同的模型击败一个模型的五个副本。

### 陪审团方法#

Sibyl 框架（在 Minsky-LM 文献中引用）形式化了一个"陪审团" —— 一组小型专业智能体，通过在每个阶段投票来精炼答案。与纯多数投票不同，陪审团有角色：一个智能体盘问，一个提供上下文，一个评分 plausibility。陪审团方法是纯投票（便宜，易于单一文化）和完整 MAD（昂贵，易于从众）之间的中点。

### 投票与辩论主导的地方#

- 问题有基本事实（事实、数学、代码行为）。投票收敛是有意义的。
- 智能体可以访问不同的来源或工具（异构性可用）。
- 轮次是有限的（通常 2-3）并且有单独的裁判或验证器。
- 预算允许 3-5 个智能体。在图型拓扑中超过 5-7 个，协调税占主导。

### 投票与辩论伤害的地方#

- 问题是意见形状的。智能体收敛到看起来最自信的答案，而不是最正确的。
- 所有智能体共享一个基础模型。单一文化使共识变得无意义。
- 轮次是无限制的。从众每次都获胜。
- 任务是简单的。在 N=5 时使用具有自一致性的单个智能体更便宜且同样准确。

## 构建它#

`code/main.py` 实现：

- `run_star(agents, hub, question)` —— 中心轮询每个工作者，聚合。
- `run_chain(agents, question)` —— 顺序精炼。
- `run_tree(root, children, question)` —— 分层，深度为 2 的聚合。
- `run_graph(agents, question, rounds)` —— 全到全辩论，有限轮次。
- 脚本化异构性拨号：每个智能体有一个 `error_bias` 表示其系统性错误。
- 一个测量工具，在 N=3, 5, 7 时运行每个拓扑，并报告（准确性、total_tokens、wallclock_simulated）。

运行：

```
python3 code/main.py
```

预期输出：拓扑 × N → （准确性、令牌、延迟）的表格。图型在 N=3-5 时在研究风格任务上获胜；星型在快速事实任务上获胜；图型在 N=7 时显示协调税（延迟膨胀快于准确性）。

## 使用它#

`outputs/skill-topology-picker.md` 是一个技能，读取任务描述并推荐拓扑（星型 / 链型 / 树型 / 图型）、N（智能体数量）、异构性配置文件（要使用的基模型）和轮次限制。

## 发布它#

对于任何集成：

- 从**自一致性在 N=5** 开始，使用一个强基础模型。这是便宜的基线。
- 如果准确性很重要，升级到**异构投票在 N=3**。衡量增量。
- 仅当任务有结构（研究、多步骤）并且有限轮次可行时，才升级到**辩论拓扑**。
- 总是记录少数聚类。当少数持续正确时，你有一个多样性信号。
- 基准测试挂钟和令牌以及准确性。"以 10 倍成本获得更好的准确性"是一个业务决策。

## 关键术语#

| 术语 | 人们的说法 | 实际含义 |
|------|----------------|------------------------|
| 自一致性 | "采样 N 次，投票" | Wang 2022。单模型，N 个温度>0 样本，对推理路径进行多数投票。 |
| 异构性 | "不同模型" | 不同基础模型或提示系列的集成。打破单一文化。 |
| MAD | "多智能体辩论" | 智能体跨轮次交换批评的通用术语。参见 Du 2023。 |
| A-HMAD | "对抗性异构 MAD" | 强调不同模型 + 对抗性结构的 MAD 变体。 |
| 拓扑 | "谁与谁交谈" | 星型、链型、树型、图型。确定信息流。 |
| 协调税 | "收益递减" | 在图型上约 4 个智能体以上，成本增长快于质量。 |
| 志愿者行为 | "主动帮助" | AgentVerse 涌现模式：智能体主动采取步骤。 |
| 从众行为 | "压力下的协议" | AgentVerse 涌现模式：智能体与批评者保持一致。 |
| 陪审团 | "小型专业小组" | Sibyl 风格的集成，具有角色（审查员、上下文、评分员）。 |
