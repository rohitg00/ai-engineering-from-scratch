# 15 · 投票、自一致性与辩论拓扑

> 最廉价的聚合方式：采样 N 个独立智能体，进行多数投票。Wang 等人 2022 年提出的自一致性（self-consistency）用同一个模型采样 N 次实现了这一点。多智能体把它扩展为采用**异构（heterogeneous）**智能体来摆脱单一文化（monoculture）——不同的模型、不同的提示词、不同的温度、不同的上下文。除多数投票之外，辩论拓扑（debate topology）也很关键：MultiAgentBench（arXiv:2503.01935，ACL 2025）评估了星形 / 链式 / 树形 / 图形协调方式，发现**图形最适合研究类任务**，并在超过约 4 个智能体后出现「协调税（coordination tax）」。AgentVerse（ICLR 2024）记录了两种涌现模式——志愿行为（volunteer behaviors）与从众行为（conformity behaviors）——而从众既是一项特性（达成共识），也是一种风险（群体思维 groupthink，见第 24 课）。本课绘制拓扑空间的地图，构建每一种变体，并测量协调税。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置：** 阶段 16 · 07（心智社会与辩论），阶段 16 · 14（共识与 BFT）
**时长：** 约 75 分钟

## 问题

辩论可以提升准确率（Du 等人，arXiv:2305.14325）。它也可能损害准确率。辩论是否有益，取决于四个结构性选择：

1. 谁与谁对话（拓扑）。
2. 多少轮（Du 2023：轮数与智能体数量各自独立地起作用）。
3. 智能体是否异构（不同的基座模型打破单一文化）。
4. 是否存在对抗性声音（强论证 steel-manning 对比稻草人式攻击 straw-manning）。

把「跑 5 个智能体然后投票」硬塞到任务上的团队，往往反而比单个智能体表现更差。这些失败并非随机。它们与拓扑和异构性相关。本课就是这张拓扑地图。

## 概念

### 自一致性，单模型基线

Wang 等人 2022 年（《Self-Consistency Improves Chain of Thought Reasoning》）在温度 > 0 下对同一个模型采样 N 次，并对各推理路径得出的答案进行多数投票。在 GSM8K 上的结果是：相比单次贪婪解码，N=40 个样本带来了显著提升。自一致性是多智能体投票的单智能体前身。

局限：自一致性只用一个基座模型。从构造上看，误差是相关的。如果模型存在系统性偏差，那么 N 个样本全都会共享这一偏差。

### 多智能体投票，异构扩展

把 N 个样本替换为 N 个*不同*的智能体。不同的基座模型（Claude、GPT、Llama），不同的提示词，不同的工具访问权限。好处：误差互不相关。代价：不同智能体的开销不同；协调它们会增加额外成本。

异构辩论在 2026 年的标准称呼是 **A-HMAD**——对抗式异构多智能体辩论（Adversarial Heterogeneous Multi-Agent Debate）。尚未被普遍采用，但论文用这个术语来指代「不同模型相互辩论，从而减少单一文化坍缩带来的相关误差」。

### 四种拓扑

```
star                chain               tree                graph

    ┌─A─┐           A─B─C─D         ┌──A──┐              A───B
    │   │                           │     │              │ × │
    B   C                           B     C              D───C
    │   │                          / \   / \
    D   E                         D   E F   G           (fully connected)
```

星形（Star）：一个中枢，其余所有智能体只与中枢对话。等价于没有旁路通道的「主管—工作者」结构。
链式（Chain）：线性，每个智能体看到上一个智能体的输出。类似流水线。
树形（Tree）：层级化，被层级式智能体系统采用（见第 06 课）。
图形（Graph）：任意对任意。包括全连接团（clique）与任意有向无环图（DAG）。

### 协调税（MultiAgentBench）

MultiAgentBench（MARBLE，ACL 2025，arXiv:2503.01935）在一套涵盖研究、编码与规划的任务集上，对星形、链式、树形、图形进行了基准测试。关键测量结果：

- **图形**拓扑在研究类任务上胜出。信息任意对任意流动；智能体可以相互批评。
- **星形**在快速作答的事实类任务上胜出。中枢负责过滤与整合。
- **链式**在分步流水线（分阶段精炼）上胜出。
- 在图形拓扑中，超过约 4 个智能体后会出现**协调税**。墙钟时间与 token 成本的增长快于质量的提升。

4 智能体上限是经验性的，并非根本性的。它反映了 2026 年大语言模型的上下文容量：每个智能体的上下文会被同伴的输出填满，一旦人人都能看到所有人，新增第 N+1 个智能体的边际价值就会下降。

### 多智能体辩论策略（《我们该不该走向 MAD？》）

arXiv:2311.17371 是 2023 年关于 MAD 策略的综述。被他人复现的关键发现：在使用相同预算时，那些在*结构上类似于*自一致性（独立采样 + 聚合）的 MAD 变体，往往表现不如自一致性。当智能体真正异构、且辩论具备对抗性结构（有一个智能体唱反调）时，MAD 才最有帮助。

### AgentVerse 的涌现模式

AgentVerse（ICLR 2024，https://proceedings.iclr.cc/paper_files/paper/2024/file/578e65cdee35d00c708d4c64bce32971-Paper-Conference.pdf）记录了两种即便没有显式设计也会从多智能体辩论中涌现出来的行为：

- **志愿（Volunteer）。** 某个智能体在未被要求的情况下主动提供帮助（「我可以接手下一步」）。有用之处：它把工作分配给最有能力完成某个子任务的智能体。
- **从众（Conformity）。** 某个智能体调整自己的立场以与批评者保持一致，即便批评者是错的。这是辩论中等价于谄媚（sycophancy）的现象（见第 14 课）。

从众正是「辩论到达成一致」会奖励霸凌者的原因。设定有界轮数并配备一个独立的裁判可以缓解这一点。

### 异构性：真正撬动准确率的旋钮

2024–2026 年实践文献中的一个模式：把你 N 个智能体中的一个换成不同的基座模型，所带来的准确率提升，比把 N 增加 1 还要大。直觉在于单一文化——每一个新的独立误差来源，都比再加一个相关样本更有价值。

在极限情形下，异构性胜过数量。在大多数有清晰真值的任务上，三个不同的模型胜过同一模型的五份副本。

### 陪审团方法

Sibyl 框架（在 Minsky-LLM 文献中被引用）将「陪审团（jury）」形式化——一小组专门化的智能体在每个阶段通过投票来精炼答案。与单纯的多数投票不同，陪审团带有角色：一个智能体负责交叉质询，一个负责提供上下文，一个负责评估可信度。陪审团方法介于单纯投票（便宜、易陷入单一文化）与完整 MAD（昂贵、易陷入从众）之间的中间地带。

### 何时「带辩论的投票」占优

- 问题有真值（事实、数学、代码行为）。投票收敛是有意义的。
- 智能体可以访问不同的来源或工具（具备异构性）。
- 轮数有界（通常 2–3 轮）且配有独立的裁判或验证者。
- 预算允许 3–5 个智能体。在图形拓扑上超过 5–7 个时，协调税开始主导。

### 何时「带辩论的投票」有害

- 问题偏主观。智能体会收敛到看起来最自信、而非最正确的那个答案。
- 所有智能体共享同一基座模型。单一文化使共识失去意义。
- 轮数无界。从众每次都会获胜。
- 任务简单。一个采用 N=5 自一致性的单智能体更便宜，且同样准确。

## 动手构建

`code/main.py` 实现了：

- `run_star(agents, hub, question)` —— 中枢轮询每个工作者，进行聚合。
- `run_chain(agents, question)` —— 顺序精炼。
- `run_tree(root, children, question)` —— 带深度为 2 的聚合的层级结构。
- `run_graph(agents, question, rounds)` —— 全对全辩论，轮数有界。
- 一个脚本化的异构性旋钮：每个智能体都有一个 `error_bias`，表示其系统性错误程度。
- 一套测量框架，在 N=3、5、7 下运行每种拓扑，并报告 (accuracy, total_tokens, wallclock_simulated)。

运行：

```
python3 code/main.py
```

预期输出：一张「拓扑 × N → (accuracy, tokens, latency)」的表格。在研究风格的任务上，图形在 N=3–5 时胜出；在快速事实类任务上，星形胜出；图形在 N=7 时显现协调税（延迟膨胀快于准确率提升）。

## 实际运用

`outputs/skill-topology-picker.md` 是一个技能（skill），它读取任务描述，并推荐一种拓扑（star / chain / tree / graph）、一个 N（智能体数量）、一份异构性画像（要使用的基座模型）以及一个轮数上限。

## 投入生产

对任何集成（ensemble）而言：

- 从使用一个强基座模型的 **N=5 自一致性**起步。这是廉价基线。
- 如果准确率很重要，升级为 **N=3 的异构投票**。测量两者的差值（delta）。
- 仅当任务具备结构（研究、多步骤）且有界轮数可行时，才升级为**辩论拓扑**。
- 始终记录少数派簇（minority cluster）。当某个少数派持续正确时，你就得到了一个多样性信号。
- 在测量准确率的同时，对墙钟时间与 token 进行基准测试。「以 10 倍成本换更好的准确率」是一个商业决策。

## 练习

1. 运行 `code/main.py`。绘制图形拓扑的协调税曲线：准确率对 N、token 对 N。曲线在哪个 N 处出现拐点？
2. 实现 A-HMAD：三个故意带有不同偏差的智能体。在第 14 课的单一文化攻击下，全部相同偏差的基线与 A-HMAD 相比表现如何？
3. 给图形拓扑添加一个「裁判（judge）」角色，它不投票，只对最终共识打分。这会改变涌现出的从众行为吗？
4. 阅读 AgentVerse 论文（ICLR 2024）。识别你的实现中表现最强烈的那种涌现行为。你能否通过修改提示词诱发相反的行为？
5. 阅读 MultiAgentBench（arXiv:2503.01935）第 4 节（拓扑实验）。用你的测量框架在论文中的一个任务上复现「图形在研究任务上胜出」的结果。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|----------------|------------------------|
| 自一致性（Self-consistency） | 「采样 N 次，投票」 | Wang 2022。单模型，N 个温度>0 的样本，对推理路径进行多数投票。 |
| 异构性（Heterogeneity） | 「不同的模型」 | 由不同基座模型或不同提示词族组成的集成。打破单一文化。 |
| MAD | 「多智能体辩论」 | 指智能体跨多轮交换批评意见的通用术语。参见 Du 2023。 |
| A-HMAD | 「对抗式异构 MAD」 | 强调不同模型 + 对抗性结构的 MAD 变体。 |
| 拓扑（Topology） | 「谁与谁对话」 | 星形、链式、树形、图形。决定信息流动方式。 |
| 协调税（Coordination tax） | 「收益递减」 | 在图形上超过约 4 个智能体时，成本增长快于质量提升。 |
| 志愿行为（Volunteer behavior） | 「未被要求的帮助」 | AgentVerse 的涌现模式：某个智能体主动提出接手某一步。 |
| 从众行为（Conformity behavior） | 「压力下的附和」 | AgentVerse 的涌现模式：某个智能体向批评者靠拢。 |
| 陪审团（Jury） | 「小型专门化评审组」 | Sibyl 风格的集成，带有角色（质询者、上下文、打分者）。 |
