# 投票、自洽性与辩论拓扑

> 最便宜的聚合：采样N个独立智能体，多数投票。Wang等人2022年的自洽性用温度>0对同一个模型采样N次来实现。多智能体通过**异构**智能体扩展它以逃离单一文化——不同模型、不同提示、不同温度、不同上下文。超越多数投票，辩论拓扑很重要：MultiAgentBench（arXiv:2503.01935，ACL 2025）评估了星形/链形/树形/图协调，发现**图对研究最好**，超过约4个智能体有"协调税"。AgentVerse（ICLR 2024）记录两种涌现模式——自愿行为和从众行为——从众既是特征（找到共识）也是风险（群体思维，第24课）。本课程映射拓扑空间，构建每个变体，并测量协调税。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置知识：** 第16阶段 · 07（心智社会与辩论），第16阶段 · 14（共识与BFT）
**时间：** 约75分钟

## 问题

辩论可以提高准确性（Du等人，arXiv:2305.14325）。它也可以降低它。辩论是否有帮助取决于四个结构选择：

1. 谁与谁交谈（拓扑）。
2. 多少轮（Du 2023：轮次和智能体都独立重要）。
3. 智能体是否异构（不同基础模型打破单一文化）。
4. 是否存在对抗性声音（钢铁人 vs 稻草人）。

将"运行5个智能体并投票"附加到任务上的团队经常相对于单智能体退步。失败不是随机的。它们跟踪拓扑和异构性。本课程是拓扑地图。

## 核心概念

### 自洽性，单模型基线

Wang等人2022（"自洽性改进思维链推理"）在温度>0时对同一模型采样N次，并对推理路径答案进行多数投票。GSM8K上的结果：N=40样本相对于单次贪婪解码有实质性增益。自洽性是单智能体投票的前驱。

限制：自洽性使用一个基础模型。错误按构造相关。如果模型有系统性偏见，所有N个样本共享它。

### 多智能体投票，异构扩展

用N个*不同*智能体替换N个样本。不同基础模型（Claude、GPT、Llama）、不同提示、不同工具访问。好处：不相关错误。成本：不同智能体成本不同；协调它们增加开销。

异构辩论的2026年规范名称是**A-HMAD**——对抗性异构多智能体辩论。不是普遍采用，但论文使用该术语表示"不同模型辩论，减少来自单一文化崩溃的相关错误"。

### 四种拓扑

```
星形                链形               树形                图

    ┌─A─┐           A─B─C─D         ┌──A──┐              A───B
    │   │                           │     │              │ × │
    B   C                           B     C              D───C
    │   │                          / \   / \
    D   E                         D   E F   G           (全连接)
```

星形：一个中心，其他只与中心交谈。等价于没有后通道的监督者-工作者。
链形：线性，每个智能体看到前一个的输出。类似流水线。
树形：层次化，用于层次智能体系统（第06课）。
图：任意对任意。包括全连接团和任意DAG。

### 协调税（MultiAgentBench）

MultiAgentBench（MARBLE，ACL 2025，arXiv:2503.01935）在包括研究、编码和规划的任务套件上基准测试了星形、链形、树形、图。关键测量结果：

- **图**拓扑在研究任务上获胜。信息流任意对任意；智能体可以相互批判。
- **星形**在快速回答事实任务上获胜。中心过滤和整合。
- **链形**在逐步流水线上获胜（分段细化）。
- **协调税**在图拓扑超过约4个智能体时出现。挂钟和token成本增长快于质量。

4智能体上限是经验的，不是基本的。它反映2026年LLM上下文容量：每个智能体的上下文被同伴的输出填满，一旦每个人能看到每个人，添加智能体N+1的边际价值下降。

### 多智能体辩论策略（"我们应该走向MAD吗？"）

arXiv:2311.17371是2023年MAD策略的综述。被其他人复制的关键发现：与自洽性*结构相似*的MAD变体（独立采样 + 聚合）在使用相同预算时经常表现不如自洽性。MAD在智能体真正异构且辩论有对抗性结构（一个智能体争论反对）时帮助最大。

### AgentVerse涌现模式

AgentVerse（ICLR 2024，https://proceedings.iclr.cc/paper_files/paper/2024/file/578e65cdee35d00c708d4c64bce32971-Paper-Conference.pdf）记录两种即使无需明确设计也从多智能体辩论中出现的行为：

- **自愿。** 一个智能体主动提供帮助（"我可以进行下一步"）未经提示。有用：它将工作分配给最有能力的智能体处理子任务。
- **从众。** 一个智能体调整其立场以匹配批评者，即使批评者是错的。这是辩论中等价于谄媚（第14课）的行为。

从众是为什么辩论直到同意奖励欺凌者的原因。有界轮次与单独法官缓解。

### 异构性：实际移动准确性的旋钮

2024-2026年实践文献中的模式：将N个智能体中的一个替换为不同基础模型，比将N增加1给出更大的准确性提升。直觉是单一文化——每个新的独立错误源比额外相关样本更有价值。

在极限情况下，异构性击败数量。三个不同模型在大多数有干净基本事实的任务上击败五个同一模型的副本。

### 陪审团方法

Sibyl框架（在Minsky-LLM文献中引用）形式化"陪审团"——一小套专业智能体，通过每阶段投票来细化答案。与简单多数投票不同，陪审团有角色：一个智能体交叉审问，一个提供上下文，一个评分合理性。陪审团方法是简单投票（便宜、单一文化倾向）和完整MAD（昂贵、从众倾向）之间的中点。

### 投票-辩论何时占主导

- 问题有基本事实（事实、数学、代码行为）。投票收敛有意义。
- 智能体可以访问不同来源或工具（异构性可用）。
- 轮次有界（典型2-3轮）且有单独法官或验证者。
- 预算允许3-5个智能体。在图拓扑上超过5-7个，协调税占主导。

### 投票-辩论何时有害

- 问题是意见型。智能体收敛到看起来最自信的答案，不是最正确的。
- 所有智能体共享基础模型。单一文化使共识无意义。
- 轮次无界。从众每次都赢。
- 任务简单。单智能体在N=5时自洽性更便宜且同样准确。

## 构建它

`code/main.py`实现：

- `run_star(agents, hub, question)` —— 中心轮询每个工作者，聚合。
- `run_chain(agents, question)` —— 顺序细化。
- `run_tree(root, children, question)` —— 层次化，深度-2聚合。
- `run_graph(agents, question, rounds)` —— 全对全辩论，有界轮次。
- 脚本化异构性拨盘：每个智能体有`error_bias`指示其系统性错误。
- 测量工具，在N=3, 5, 7时运行每种拓扑并报告（准确性、总token、模拟挂钟）。

运行：

```
python3 code/main.py
```

预期输出：拓扑 × N →（准确性、token、延迟）的表格。图在N=3-5时在研究风格任务上获胜；星形在快速事实任务上获胜；图在N=7时显示协调税（延迟膨胀快于准确性）。

## 使用它

`outputs/skill-topology-picker.md`是一个技能，读取任务描述并推荐拓扑（星形/链形/树形/图）、N（智能体数量）、异构性配置文件（使用的基础模型）和轮次边界。

## 交付它

对于任何集成：

- 从使用一个强基础模型的**N=5自洽性**开始。它是便宜的基线。
- 如果准确性重要，升级到**N=3异构投票**。测量增量。
- 只有当任务有结构（研究、多步骤）且有界轮次可行时，才升级到**辩论拓扑**。
- 始终记录少数集群。当少数持续正确时，你有多样性信号。
- 基准测试挂钟和token以及准确性。"10倍成本下更好的准确性"是商业决策。

5. 阅读MultiAgentBench（arXiv:2503.01935）第4节（拓扑实验）。在你的工具上复制论文中一个任务的"图赢研究"结果。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| 自洽性 | "采样N次，投票" | Wang 2022。单模型，N个温度>0样本，对推理路径多数投票。 |
| 异构性 | "不同模型" | 不同基础模型或提示家族的集成。打破单一文化。 |
| MAD | "多智能体辩论" | 智能体在多轮中交换批评的通用术语。参见Du 2023。 |
| A-HMAD | "对抗性异构MAD" | 强调不同模型 + 对抗性结构的MAD变体。 |
| 拓扑 | "谁与谁交谈" | 星形、链形、树形、图。决定信息流。 |
| 协调税 | "收益递减" | 在图上超过约4个智能体，成本增长快于质量。 |
| 自愿行为 | "未经提示的帮助" | AgentVerse涌现模式：智能体主动提出进行一步。 |
| 从众行为 | "压力下同意" | AgentVerse涌现模式：智能体与批评者对齐。 |
| 陪审团 | "小型专业小组" | Sibyl风格集成，有角色（审查者、上下文、评分者）。 |
