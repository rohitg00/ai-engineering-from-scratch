# 07 · 心智社会与多智能体辩论

> 明斯基（Minsky）1986 年的命题——智能是一群专家组成的社会——每隔十年就被重新发现一次。2023 年 Du 等人把它变成了一个具体算法：多个 LLM 实例各自提出答案，阅读彼此的答案，相互批判，然后更新。经过 N 轮迭代，它们收敛到一个共识，并在六项推理与事实性任务上击败了零样本「思维链（Chain-of-Thought，CoT）」和「反思（reflection）」。两个发现至关重要：**多个智能体**和**多个轮次**各自独立地做出了贡献。社会胜过单智能体的独白；多轮交换胜过单次投票。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置：** 阶段 16 · 04（基元模型）
**时长：** 约 60 分钟

## 问题

「自洽性（self-consistency）」——对一个模型采样多次并取多数答案——是你能加挂的最廉价的推理增强手段。它有效，但很快就会饱和。你可以把采样数翻倍，却看不到下一次有意义的跃升。

辩论打破了这种饱和。不再是从一个模型独立采样 N 次，而是让 N 个智能体阅读彼此的推理并修正。样本之间的相关性下降了（它们不再是独立同分布的），而且在独立同分布投票自信地出错的地方，收敛点往往是正确的。

## 概念

### Du 等人 2023 年的算法

出自 arXiv:2305.14325（ICML 2024）：

1. N 个智能体中的每一个都对问题产生一个初始答案。
2. 对于第 r = 2..R 轮：每个智能体看到其他智能体在第 r-1 轮的答案，并被要求「考虑这些答案，给出你更新后的答案。」
3. 经过 R 轮后，对最终答案进行多数投票。

该论文在 MMLU、GSM8K、传记、MATH 以及事实性基准上进行了测试。辩论始终优于 CoT 和「自我反思（Self-Reflection）」。

### 两个独立的旋钮

来自同一篇论文的消融实验：

- **仅增加智能体数量**（1 轮，对 N 个智能体多数投票）在大多数任务上优于单智能体，但会进入平台期。
- **仅增加轮次数量**（1 个智能体看到它自己先前的推理）几乎没有帮助——这正是反思已知的弱点。
- **两者结合**才产生巨大的跃升。多个智能体之间的多轮交换驱动了增益。

### 它为何有效

两个机制：

1. **暴露于分歧。** 当一个智能体看到另一个智能体的推理链得出了不同的结论时，它必须要么为自己辩护，要么更新。无论哪种方式，第 r+1 轮的上下文都比第 r 轮更丰富。
2. **相关误差的削减。** 在自洽性中，所有样本都来自同一个模型，因此误差是相关的——你会平均出一个自信却错误的答案。不同的模型或不同的随机种子会去相关。不同的*辩论观点*则进一步去相关。

### 异构辩论

A-HMAD 及相关的后续工作为不同的智能体使用*不同的基座模型*。Llama + Claude + GPT 进行辩论可以减少「单一文化崩溃（monoculture collapse）」（第 26 课），因为一个模型家族的相关误差不会被其他模型共享。

缺点：参与辩论的弱模型可能会把共识拖向它的错误答案（参见《Should we be going MAD?》，arXiv:2311.17371）。

### NLSOM——129 智能体的扩展

Zhuge 等人（《Mindstorms in Natural Language-Based Societies of Mind》，arXiv:2305.17066）把这一思路扩展到了 129 成员的社会。结果是：随着规模扩大，专业化与自组织得以涌现，并且系统在诸如视觉问答这样的任务上优于单智能体。

### 失败模式

- **奉承级联（Sycophancy cascade）。** 所有智能体都听从听起来最自信的那个智能体。辩论崩塌为最大声的那个声音。提示智能体扮演对抗性角色（「必须有一个智能体论证反方立场」）会有帮助。
- **话题漂移（Topic drift）。** 多轮辩论会逐渐偏离最初的问题。缓解方式：每一轮都重新注入问题。
- **算力爆炸（Compute blowup）。** N 个智能体 × R 轮 = N·R 次 LLM 调用，且每次调用的上下文都在增长。5 智能体、5 轮的辩论就是 25 次调用，且上下文不断增长。每个问题的成本可能超过单次 CoT 调用的 10 倍。

## 动手构建

`code/main.py` 在一道数学题上运行 3 智能体 × 3 轮的辩论，其中每个智能体以一个不同的（可能错误的）答案开始。智能体是脚本化的——每个智能体通过对邻居答案按脚本化的置信度加权平均来「更新」。收敛过程在逐轮日志中清晰可见。

该演示展示了两个关键效应：

- 单次交换就能让智能体更接近正确答案。
- 第 2 轮之后的额外轮次显示出收益递减（与 Du 等人的平台期一致）。

运行：

```
python3 code/main.py
```

## 实际运用

`outputs/skill-debate-configurator.md` 为一个新任务配置辩论：智能体数量、轮次数量、异构性（同一模型 vs 混合）、角色分配（对称 vs 单一对抗）。它还会在你运行之前估算 token 成本。

## 上线部署

如果你要上线辩论系统：

- **将轮次上限设为 3。** Du 等人表明 3 轮就能捕获大部分增益。再多只是成本，而非质量。
- **将智能体上限设为 5。** 超过 5 个，上下文膨胀和成本就会占主导。
- **默认异构。** 池中至少有两个不同的基座模型。
- **设置一个对抗槽位。** 让一个智能体被提示无论如何都要反对。这能打破奉承。
- **记录每一轮。** 隐藏中间轮次的辩论系统无法被调试或审计。

## 练习

1. 运行 `code/main.py`，然后把轮次数量设为 5，观察收益递减。在第几轮额外的收敛停止了？
2. 加入第四个具有对抗性角色的智能体：始终与当前多数意见相反。这会打破还是改善收敛？
3. 绘制（打印）每一轮的一致性分数（处于多数答案的智能体所占比例）。它何时达到 1.0，而这是否等同于「正确」？
4. 阅读 Du 等人论文第 4 节的消融实验。使用这份代码复现「仅智能体」vs「仅轮次」vs「两者」的结果。
5. 阅读《Should we be going MAD?》（arXiv:2311.17371），列出两种超越轮转制（round-robin）的辩论变体——例如裁判主导（judge-led）、辩论链（chain-of-debate）、对抗式（adversarial）。

## 关键术语

| 术语 | 人们怎么说 | 它实际的含义 |
|------|----------------|------------------------|
| 心智社会（Society of Mind） | 「明斯基的想法」 | 智能即相互作用的专家；1986 年的框架如今通过 LLM 辩论被操作化。 |
| 多智能体辩论（Multi-agent debate） | 「智能体在争论」 | N 个智能体提出方案、相互批判、在 R 轮内修正，再多数投票。 |
| 共识（Consensus） | 「它们达成一致」 | 不是认知上的真理——只是处于多数答案的比例。可能自信却错误。 |
| 轮次（Rounds） | 「交换步骤」 | 一轮 = 每个智能体读取其他智能体并更新一次。 |
| 异构辩论（Heterogeneous debate） | 「混合模型家族」 | 使用不同的基座模型来去相关误差。 |
| 奉承级联（Sycophancy cascade） | 「所有人都同意那个大嗓门」 | 一种辩论失败：智能体不论对错都听从最自信的智能体。 |
| NLSOM | 「129 智能体社会」 | 基于自然语言的心智社会；Zhuge 等人扩展后的版本。 |
| 相关误差（Correlated error） | 「同一个模型，同一个 bug」 | 自洽性为何饱和的原因；跨不同观点的辩论可去相关。 |
