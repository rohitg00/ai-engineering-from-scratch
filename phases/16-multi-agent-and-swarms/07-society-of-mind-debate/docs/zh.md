# 心智社会与多智能体辩论

> Minsky 1986年的前提——智能是一个专家的集合——每十年都会被重新发现。2023年，Du等人将其转化为一个具体算法：多个LLM实例提出答案，阅读彼此的答案，批判并更新。经过N轮，它们收敛到一个共识，在六个推理和事实性任务上击败零样本CoT和反思。两个发现很重要：**多个智能体**和**多轮**都独立贡献。心智社会击败单智能体独白；多轮交换击败一次性投票。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置知识：** 第16阶段 · 04（原语模型）
**时间：** 约60分钟

## 问题

自洽性——从一个模型多次采样并取多数答案——是你能附加的最便宜的推理改进。它有效，但很快饱和。你可以将样本翻倍，却看不到另一个有意义的跳跃。

辩论打破饱和。不是从一个模型进行N次独立采样，而是N个智能体阅读彼此的推理并修订。样本之间的相关性下降（它们不再是i.i.d.），收敛点通常是正确的，而i.i.d.投票是自信的错误。

## 核心概念

### Du等人2023年算法

来自arXiv:2305.14325（ICML 2024）：

1. N个智能体中的每一个产生问题的初始答案。
2. 对于轮次r = 2..R：每个智能体被展示其他智能体的r-1轮答案，并被问到"考虑到这些，给出你的更新答案。"
3. 经过R轮，对最终答案进行多数投票。

该论文在MMLU、GSM8K、传记、MATH和事实性基准上测试。辩论始终击败CoT和自我反思。

### 两个独立旋钮

同一篇论文的消融实验：

- **仅智能体数量**（1轮，N的多数投票）在大多数任务上击败单智能体，但达到平台期。
- **仅轮次数**（1个智能体看到自己的先前推理）几乎无帮助——反思的已知弱点。
- **两者一起**产生大的跳跃。多个智能体之间的多轮交换驱动增益。

### 为什么有效

两种机制：

1. **暴露于分歧。** 当智能体看到另一个智能体的推理链与不同结论时，它必须要么证明要么更新。无论哪种方式，r+1轮的上下文都比r轮更丰富。
2. **相关错误减少。** 在自洽性中，所有样本来自同一模型，所以错误相关——你平均到一个自信的错误答案。不同模型或不同种子去相关。不同的*辩论观点*进一步去相关。

### 异构辩论

A-HMAD和相关后续工作对不同智能体使用*不同的基础模型*。Llama + Claude + GPT辩论减少了单一文化崩溃（第26课），因为一个模型家族的相关错误不被其他模型共享。

缺点：参与辩论的弱模型可以将共识拖向其错误答案（参见"我们应该走向MAD吗？"，arXiv:2311.17371）。

### NLSOM——129智能体扩展

Zhuge等人（"基于自然语言的心智社会中的头脑风暴"，arXiv:2305.17066）将这一想法扩展到129成员社会。结果：专业化与自组织随规模涌现，系统在视觉问答等任务上优于单智能体。

### 失败模式

- **谄媚级联。** 所有智能体都听从听起来最自信的智能体。辩论崩溃为最大声音。提示对抗性角色（"一个智能体必须争论反方立场"）有帮助。
- **主题漂移。** 多轮辩论偏离原始问题。缓解：每轮重新注入问题。
- **计算爆炸。** N个智能体 × R轮 = N·R次LLM调用，每次上下文都在增长。5智能体、5轮辩论是25次调用，上下文不断增长。每个问题的成本可能超过单CoT调用的10倍。

## 构建它

`code/main.py`在一个数学问题上运行3智能体 × 3轮辩论，每个智能体以不同的（可能错误的）答案开始。智能体是脚本化的——每个通过以脚本化置信度加权平均邻居的答案来"更新"。收敛在逐轮日志中可见。

演示显示两个关键效果：

- 单轮交换使智能体更接近正确答案。
- 第2轮之后的额外轮次显示收益递减（与Du等人的平台期匹配）。

运行：

```
python3 code/main.py
```

## 使用它

`outputs/skill-debate-configurator.md`为新任务配置辩论：智能体数量、轮次数、异构性（相同模型 vs 混合）、角色分配（对称 vs 一个对抗性）。它还估计运行前的token成本。

## 交付它

如果你交付辩论：

- **将轮次限制为3。** Du等人显示3轮捕获大部分增益。更多是成本，不是质量。
- **将智能体限制为5。** 超过5，上下文膨胀和成本占主导。
- **默认异构。** 池中至少两个不同的基础模型。
- **对抗性槽位。** 一个智能体被提示无论如何都不同意。打破谄媚。
- **记录每轮。** 隐藏中间轮的辩论系统无法调试或审计。

## 练习

1. 运行`code/main.py`，然后将轮次数设置为5并观察收益递减。在哪一轮额外的收敛停止？
2. 添加一个具有对抗性角色的第四个智能体：始终与当前多数意见不同意。这会打破还是改善收敛？
3. 绘制（打印）每轮的同意分数（多数答案上的智能体比例）。它何时达到1.0，这是否等同于"正确"？
4. 阅读Du等人第4节消融实验。使用此代码复制"仅智能体" vs "仅轮次" vs "两者"的结果。
5. 阅读"我们应该走向MAD吗？"（arXiv:2311.17371）并列出轮询之外的两种辩论变体——例如，法官主导、辩论链、对抗性。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| 心智社会 | "Minsky的想法" | 智能作为交互专家；1986年的框架现在通过LLM辩论可操作化。 |
| 多智能体辩论 | "智能体争论" | N个智能体提出、相互批判、在R轮中修订、多数投票。 |
| 共识 | "他们同意" | 不是认识论真理——只是多数答案上的比例。可能是自信的错误。 |
| 轮次 | "交换步骤" | 一轮 = 每个智能体阅读其他智能体并更新一次。 |
| 异构辩论 | "混合模型家族" | 使用不同的基础模型来去相关错误。 |
| 谄媚级联 | "每个人都同意大声的那个" | 辩论失败，智能体无论正确性如何都听从最自信的智能体。 |
| NLSOM | "129智能体社会" | 基于自然语言的心智社会；Zhuge等人的扩展版本。 |
| 相关错误 | "相同模型，相同bug" | 为什么自洽性饱和；不同观点之间的辩论去相关。 |
