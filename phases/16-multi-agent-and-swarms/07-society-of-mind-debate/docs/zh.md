# 心智社会与多智能体辩论

> Minsky 1986 年的前提——智能是专家的社会——每十年被重新发现一次。2023 年，Du 等人将其转化为具体算法：多个 LLM 实例提出答案，阅读彼此的答案，批评并更新。经过 N 轮后，它们收敛到一个在六个推理和事实性任务上击败零样本思维链（CoT）和反思的共识。两个发现很重要：both **多个智能体**和**多轮**独立贡献。社会击败单智能体独白；多轮交换击败单次投票。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置条件：** 阶段 16.04（原语模型）
**时长：** 约 60 分钟

## 问题背景

自洽性（Self-consistency）——从一个模型采样多次并取多数答案——是你能够添加的最便宜的推理改进。它有效，但饱和得很快。你可以将样本加倍，却看不到另一个有意义的跳跃。

辩论打破了饱和。不是从一个模型进行 N 次独立采样，而是 N 个智能体阅读彼此的推理并修订。样本之间的相关性降低了（它们不再是独立同分布的），并且收敛点通常是正确的，而独立同分布投票却自信地错了。

## 概念讲解

### Du 等人 2023 年算法

来自 arXiv:2305.14325（ICML 2024）：

1. N 个智能体中的每一个都产生对问题的初始答案。
2. 对于轮次 r = 2..R：每个智能体都被展示其他智能体的第 r-1 轮答案，并被问到"考虑这些，给出你更新的答案。"
3. R 轮后，对最终答案进行多数投票。

该论文在 MMLU、GSM8K、传记、MATH 和事实性基准上进行了测试。辩论一致击败 CoT 和自反思。

### 两个独立旋钮

同一篇论文的消融：

- **仅智能体数量**（1 轮，N 的多数投票）在大多数任务上击败单智能体，但进入平台期。
- **仅轮次数**（1 个智能体看到自己的先前推理）几乎没有帮助——反思的已知弱点。
- **两者一起**产生大的跳跃。多个智能体之间的多轮交换推动了增益。

### 为什么它有效

两种机制：

1. **接触分歧。** 当一个智能体看到另一个具有不同结论的智能体的推理链时，它必须要么辩护要么更新。无论哪种方式，第 r+1 轮的上下文都比第 r 轮更丰富。
2. **相关错误减少。** 在自洽性中，所有样本来自同一模型，所以错误是相关的——你平均得到一个自信的错误答案。不同模型或不同种子去相关。不同的*辩论观点*进一步去相关。

### 异构辩论

A-HMAD 和相关后续研究对不同的智能体使用*不同的基础模型*。Llama + Claude + GPT 辩论减少了单一文化崩溃（第 26 课），因为一个模型家族的相关错误不会被其他模型共享。

缺点：参与辩论的弱模型可能会将共识拖向其错误答案（参见"我们是否应该走向 MAD？"，arXiv:2311.17371）。

### NLSOM — 129 智能体扩展

Zhuge 等人（"自然语言基心智社会中的思维风暴"，arXiv:2305.17066）将这个想法扩展到 129 个成员的社会。结果：专业化和自组织随着规模出现，并且系统在视觉问答等任务上优于单智能体。

### 失败模式

- **谄媚级联（Sycophancy cascade）。** 所有智能体都听从听起来最自信的智能体。辩论崩溃为最大声的声音。针对对抗角色提示（"一个智能体必须论证反方立场"）有帮助。
- **主题漂移。** 多轮辩论 from 原始问题漂移。缓解：每轮重新注入问题。
- **计算爆炸。** N 个智能体 × R 轮 = N·R 次 LLM 调用，每个都有增长的上下文。一个 5 智能体、5 轮辩论是 25 次调用，上下文不断增长。每个问题的成本可能超过单次 CoT 调用的 10 倍。

## 构建实现

`code/main.py` 在一个数学问题上运行 3 智能体 × 3 轮辩论，其中每个智能体以不同的（可能是错误的）答案开始。智能体被编写脚本——每个都通过按脚本化置信度加权邻居答案来"更新"。收敛在逐轮日志中可见。

演示显示两个关键效果：

- 单轮交换使智能体更接近正确答案。
- 第 2 轮之后的额外轮次显示收益递减（匹配 Du 等人的平台期）。

运行：

```
python3 code/main.py
```

## 实际应用

`outputs/skill-debate-configurator.md` 为新任务配置辩论：智能体数量、轮次数、异构性（相同模型 vs 混合）、角色分配（对称 vs 一对抗）。它还在你运行之前估计令牌成本。

## 部署实现

如果你部署辩论：

- **将轮次上限设为 3。** Du 等人显示 3 轮捕获大部分增益。更多是为了成本，而不是质量。
- **将智能体上限设为 5。** 超过 5 个，上下文膨胀和成本占主导。
- **默认异构。** 池中至少有两个不同的基础模型。
- **对抗性槽位。** 一个被提示无论如何都不同意的智能体。打破谄媚。
- **记录每一轮。** 隐藏中间轮的辩论系统无法调试或审计。

## 练习

1. 运行 `code/main.py`，然后将轮次计数设为 5 并观察收益递减。在哪一轮额外收敛停止？
2. 添加第四个具有对抗角色的智能体：总是不同意当前多数。这会打破还是改善收敛？
3. 绘制（打印）每轮的协议分数（多数答案上的智能体分数）。它何时达到 1.0，这等同于"正确"吗？
4. 阅读 Du 等人第 4 节的消融。使用此代码复制"仅智能体"vs"仅轮次"vs"两者"结果。
5. 阅读"我们是否应该走向 MAD？"（arXiv:2311.17371）并列出超越轮询的两种辩论变体——例如，法官主导、辩论链、对抗性。

## 关键术语

| 术语 | 人们说的 | 它实际意味着什么 |
|------|----------------|------------------------|
| Society of Mind（心智社会） | "Minsky 的想法" | 作为交互专家智能；1986 年框架现在通过 LLM 辩论操作化。 |
| Multi-agent debate（多智能体辩论） | "智能体争论" | N 个智能体提议、相互批评、在 R 轮中修订、多数投票。 |
| Consensus（共识） | "他们同意" | 不是认知真理——只是多数答案上的分数。可能会自信地错。 |
| Rounds（轮次） | "交换步骤" | 一轮 = 每个智能体阅读其他智能体并更新的次数。 |
| Heterogeneous debate（异构辩论） | "混合模型家族" | 使用不同的基础模型去相关错误。 |
| Sycophancy cascade（谄媚级联） | "每个人都同意大声的那个" | 辩论失败，其中智能体无论如何都听从最自信的智能体。 |
| NLSOM | "129 智能体社会" | 自然语言心智社会；Zhuge 等人的扩展版本。 |
| Correlated error（相关错误） | "相同模型，相同错误" | 自洽性饱和的原因；跨不同观点的辩论去相关。 |
