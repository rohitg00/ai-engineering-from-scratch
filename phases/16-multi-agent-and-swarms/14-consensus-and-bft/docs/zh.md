# 智能体的共识与拜占庭容错

> 经典分布式系统BFT遇见随机LLM。2025-2026年出现了三个研究方向：**CP-WBFT**（arXiv:2511.10400）通过置信度探测加权每个投票；**DecentLLMs**（arXiv:2507.14928）采用无领导者并行工作者提议和几何中值聚合；**WBFT**（arXiv:2505.05103）将加权投票与层次结构聚类结合，将节点分为核心和边缘。来自"AI智能体能达成一致吗？"（arXiv:2603.01213）的诚实实证结果是，即使标量一致性今天也很脆弱——一个欺骗性智能体就能破坏混合智能体。BFT是必要但不充分的。本课程构建一个最小BFT协议，注入三种智能体特定攻击（拜占庭谎言、谄媚性从众、相关错误单一文化），并测量每种共识变体如何应对。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置知识：** 第16阶段 · 07（心智社会与辩论），第16阶段 · 13（共享内存）
**时间：** 约75分钟

## 问题

你有N个LLM智能体，每个产生一个答案。它们不同意。多数投票选择了错误的，因为两个智能体是相关的（相同基础模型、相同训练数据、相同失败模式）。第三个智能体碰巧以一种新颖的方式错了——所以多数是虚假多数。

现在添加一个欺骗性智能体：它故意撒谎。或一个谄媚性智能体：它同意最后一个说话的人。在经典BFT中，假设拜占庭节点是分数`f < n/3`且行为任意。2026年的现实是，LLM节点即使诚实也是随机的，跨模型相关，并相互影响输出。你不能将它们视为独立的伯努利投票者。

经典BFT（PBFT，1999）不是错的——它是不完整的。它处理任意位翻转。它不处理"三个诚实智能体共享幻觉，因为它们共享训练数据。"本课程从PBFT的基础构建，并叠加三个2025-2026年的改编。

## 核心概念

### 经典BFT给你什么

实用拜占庭容错（Castro & Liskov，OSDI 1999）容忍`f < n/3`拜占庭节点。协议有三个阶段（预准备、准备、提交）和两个原语（签名消息、法定证书）。在`n >= 3f + 1`诚实或恶意节点之间对单个值达成一致。

保证很强，但假设：

1. **独立故障。** 拜占庭不协调。
2. **诚实节点真正诚实。** 诚实输出的正确性不是问题；协议只对齐分歧。
3. **问题有基本事实答案。** 对错误事实的共识仍然是共识。

LLM智能体违反所有三个。两个运行相同基础模型的智能体共享故障。"诚实"的LLM仍然幻觉。在模糊问题上，"真相"是智能体决定的——没有外部神谕。

### 三种LLM特定攻击

**拜占庭谎言。** 一个智能体输出故意错误的答案。如果`f < n/3`，经典BFT处理这个。

**谄媚性从众。** 一个智能体在投票前读取其他智能体的答案，并与最后一个说话的人对齐。不是恶意的，但与最大声音相关。经典BFT不防止这个，因为智能体通过每个签名检查。

**相关错误单一文化。** 三个智能体共享一个基础模型。它们幻觉相同的错误答案。多数是错的。经典BFT不帮助，因为所有三个"诚实"地同意。

### 2025-2026年的回应

**CP-WBFT**（arXiv:2511.10400）—— 置信度探测加权BFT。每个投票者附加一个置信度探测到其答案（自报告概率，或单独校准模型的预测）。投票权重随置信度缩放。报告在完整图上+85.71% BFT改进。缓解：谄媚性从众（从众智能体倾向于对其自愿立场有低置信度）。

**DecentLLMs**（arXiv:2507.14928）—— 无领导者。工作者智能体并行提议，评估者智能体评分提议，最终答案是评分位置的几何中值。当`f < n/2`时稳健。缓解：拜占庭谎言和相关错误（几何中值对异常值稳健，并拉向密集集群，不是模型偏置的平均值）。

**WBFT**（arXiv:2505.05103）—— 带层次结构聚类的加权BFT。投票权重由响应质量加上从历史学习的信任分数分配。将智能体聚类为核心和边缘；核心智能体必须先达成共识，边缘智能体跟随。缓解：可扩展性（核心共识小且快）和部分单一文化（核心可以选择多样性）。

### 实证："AI智能体能达成一致吗？"（arXiv:2603.01213）

论文测量标量一致性（LLM智能体在单个数值上达成一致）跨多个前沿模型。发现令人不安：

- 即使没有对手，LLM智能体在许多基准上对标量问题的分歧率超过30%。
- 采用欺骗性角色的单个智能体可以将混合智能体共识从诚实基线拉偏40+百分点。
- 分歧率与模型多样性相关——异构集成比同构集成分歧更多（好：不相关错误）但也漂移更慢（坏：达成一致时间更长）。

结论：BFT给你对齐输出的机制，但不告诉你对齐的输出是否正确。与验证（第16阶段 · 08角色专业化）、多样性（第16阶段 · 15辩论变体）和评估者智能体（第16阶段 · 24基准）结合。

### 核心协议，精简版

LLM智能体的最小BFT轮次：

```
1. 任务到达；每个智能体i产生答案a_i
2. 每个智能体附加置信度探测c_i在[0, 1]
3. 聚合器从所有n个智能体收集(a_i, c_i)
4. 聚合器按语义集群分组（等价答案）
5. 聚合器计算每个集群C的权重：
     w(C) = sum_{i in C} c_i
6. 获胜者 = 最大权重的集群，如果最大 > 阈值 * sum(c_i)
   否则：重试或升级
7. 少数集群记录来源用于事后审计
```

语义聚类步骤是LLM特定的转折。两个答案"研究报告4.2%"和"4.2%改进"是同一个集群。朴素的字符串相等检查会错过这个。生产中，使用便宜的嵌入模型或显式规范化。

### 阈值调优

`阈值`参数决定何时接受何时重试。太低：你接受弱多数。太高：你从不接受任何东西。经验范围：n=5-7智能体为0.5-0.67，n更小则更高。低于阈值，升级到人类或不同智能体集成。

### 共识不帮助的地方

- **模糊问题。** 如果问题没有基本事实，共识是意见。就这么称呼它。
- **复合问题。** "写代码并解释它"——两个答案。独立投票每个。
- **对抗性多轮。** 如果智能体可以观察先前轮次并模仿（Du 2023辩论），它们开始相互同意，无论真相。限制轮次（通常2-3轮）。

## 构建它

`code/main.py`实现：

- `AgentVoter` —— 带有（答案，置信度）的脚本化策略。
- `MajorityVote` —— 经典多数。
- `CPWBFT` —— 带语义聚类的置信度加权投票。
- `DecentLLMs` —— 评分提议上的几何中值聚合。
- `Scenario` —— 在三种攻击模式下运行每个聚合器。

实现的攻击模式：

1. `byzantine`：一个智能体以高置信度撒谎。
2. `sycophancy`：一个智能体复制它看到的第一个答案，匹配置信度。
3. `monoculture`：三个智能体共享错误答案（相关错误）以中等置信度。

运行：

```
python3 code/main.py
```

预期输出：（攻击，聚合器）→ 最终答案的表格，正确答案高亮。多数在单一文化情况下失败。CPWBFT的置信度加权缓解谄媚。DecentLLMs的几何中值在单一文化少于一半人口时拉向诚实集群。

## 交付它

在交付任何共识机制之前：

- **用至少上述三种模式攻击测试。** 你的协议应该可预测地失败，不是静默地。
- **记录每个少数集群及来源。** 少数集群是你相关错误的早期预警系统。
- **强制执行有界轮次。** 没有"一直辩论直到达成一致"——那奖励谄媚。
- **将一致性与正确性分离。** 共识输出到验证者；验证者独立于集成。
- **监控一致率。** 急剧上升意味着从众偏见；急剧下降意味着模型漂移。

## 练习

1. 运行`code/main.py`。确认多数在单一文化攻击下失败，但当单一文化置信度低于0.7时CPWBFT部分缓解它。
2. 添加第四种攻击模式：**静默弃权** —— 一个智能体拒绝回答（"我不知道"）。每个聚合器应如何处理弃权？实现你的选择。
3. 将语义聚类从字符串规范化切换到嵌入相似性（使用任何开源嵌入模型）。谄媚攻击发生什么？
4. 阅读CP-WBFT（arXiv:2511.10400）。实现置信度探测校准步骤（单独的校准模型检查每个智能体的自报告置信度）。测量单一文化场景上的准确性增益。
5. 阅读"AI智能体能达成一致吗？"（arXiv:2603.01213）。复制简化标量一致性实验：三个智能体，一个标量问题，欺骗性角色提示。CPWBFT或DecentLLMs抓住它吗？

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| BFT | "拜占庭容错" | Castro-Liskov 1999协议，在`f < n/3`任意故障下达成共识。 |
| 拜占庭 | "任何坏行为" | 可以撒谎、丢弃消息、静默失败的节点——任何但不是安全崩溃。 |
| 置信度探测 | "你有多确定？" | 附加到投票的自报告或校准器预测概率。 |
| 语义聚类 | "相同答案，不同词语" | 在计票前分组等价答案。 |
| 几何中值 | "稳健中心" | 最小化到样本点距离之和的点。对异常值稳健，不像均值。 |
| 单一文化 | "相同模型，相同失败" | 智能体共享训练数据或基础模型时的相关错误。 |
| 谄媚性从众 | "同意大声的声音" | 智能体的投票偏向第一个/最大声说话的人。 |
| 核心/边缘 | "层次BFT" | WBFT分割：小核心共识先，边缘节点跟随。限制延迟。 |
