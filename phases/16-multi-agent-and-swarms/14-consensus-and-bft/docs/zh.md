# 共识与拜占庭容错（BFT）用于智能体

> 经典分布式系统 BFT 遇见随机 LLM。在 2025-2026 年间出现了三个研究方向：**CP-WBFT** (arXiv:2511.10400) 通过置信探针权衡每个投票；**DecentLLMs** (arXiv:2507.14928) 通过并行工作者提议和几何中位数聚合实现无领导者；**WBFT** (arXiv:2505.05103) 将加权投票与层次结构聚类结合以分割核心和边缘节点。"Can AI Agents Agree?" (arXiv:2603.01213) 的诚实实证结果是，即使是标量一致性在今天也是脆弱的 —— 单个欺骗性智能体可以破坏混合智能体。BFT 是必要的，但还不够。本课构建了一个最小的 BFT 协议，注入三种智能体特定攻击（拜占庭谎言、谄媚从众、相关错误单一文化），并衡量每种共识变体的应对能力。

**类型：** 学习 + 构建
**语言：** Python (标准库)
**前置条件：** 第 16 阶段 · 07 (心智社会与辩论), 第 16 阶段 · 13 (共享内存)
**时间：** ~75 分钟

## 问题

你有 N 个 LLM 智能体，每个都产生一个答案。它们不同意。多数投票选择了错误的一个，因为两个智能体是相关的（相同的基础模型、相同的训练数据、相同的失败模式）。第三个智能体恰好以新颖的方式出错 —— 所以多数是虚假的多数。

现在添加一个欺骗性智能体：它故意说谎。或者一个谄媚性智能体：它同意最后一个发言者。在经典 BFT 中，假设是拜占庭节点占比 `f < n/3` 且行为任意。2026 年的现实是，LLM 节点即使诚实也是随机的，跨模型相关，并受彼此输出的影响。你不能将它们视为独立的伯努利投票者。

经典 BFT (PBFT, 1999) 没有错 —— 它是不完整的。它处理任意位翻转。它不处理"三个诚实的智能体共享一个幻觉，因为它们共享训练数据。" 本课从 PBFT 的基础构建，并分层三种 2025-2026 年的适应。

## 概念

### 经典 BFT 给你的东西

实用拜占庭容错 (Castro & Liskov, OSDI 1999) 容忍 `f < n/3` 个拜占庭节点。该协议有三个阶段（预准备、准备、提交）和两个基元（签名消息、仲裁证书）。在 `n >= 3f + 1` 个诚实或恶意节点中对单个值达成一致。

保证是强的，但假设：

1. **独立故障。** 拜占庭节点不协作。
2. **诚实节点真正诚实。** 诚实输出的正确性不是问题；协议只对齐分歧。
3. **问题有基本真实答案。** 关于错误事实的共识仍然是共识。

LLM 智能体违反所有三个假设。运行相同基础模型的两个智能体共享故障。一个"诚实的" LLM 仍然产生幻觉。在模棱两可的问题上，"真相"是智能体决定的 —— 没有外部预言机。

### 三种 LLM 特定攻击

**拜占庭谎言。** 一个智能体输出故意错误的答案。如果 `f < n/3`，经典 BFT 处理这个。

**谄媚从众。** 一个智能体在投票前读取其他人的答案，并与最后一个发言者保持一致。不是恶意的，但与最大声的声音相关。经典 BFT 不防止这个，因为智能体通过每个签名检查。

**相关错误单一文化。** 三个智能体共享一个基础模型。它们产生相同的错误答案。多数是错误的。经典 BFT 没有帮助，因为所有三个"诚实地"同意。

### 2025-2026 年的回应

**CP-WBFT** (arXiv:2511.10400) —— 置信探针加权 BFT。每个投票者将其答案附加一个置信探针（自我报告的概率，或单独的校准模型的预测）。投票权重随置信度缩放。报告在完全图上 +85.71% BFT 改进。缓解措施：谄媚从众（从众智能体往往对其自愿立场有低置信度）。

**DecentLLMs** (arXiv:2507.14928) —— 无领导者。工作者智能体并行提议，评估者智能体对提议评分，最终答案是评分位置的几何中位数。当 `f < n/2` 时鲁棒。缓解措施：拜占庭谎言和相关错误（几何中位数对异常值鲁棒，并拉向密集集群，而不是模型偏置的平均值）。

**WBFT** (arXiv:2505.05103) —— 带层次结构聚类的加权 BFT。投票权重由响应质量加上从历史学习的信任分数分配。将智能体聚类为核心和边缘；核心智能体必须首先达成共识，边缘智能体跟随。缓解措施：可扩展性（核心共识小且快）和部分针对单一文化（核心可以选择多样性）。

### 实证："Can AI Agents Agree?" (arXiv:2603.01213)

该论文衡量了跨多个前沿模型的标量一致性（LLM 智能体在单个数值上达成一致）。发现令人不安：

- 即使没有对手，LLM 智能体在许多基准上的标量问题上也以超过 30% 的比率不同意。
- 单个采用欺骗性人格的智能体可以将混合智能体共识拉离诚实基线 40+ 个百分点。
- 不一致率与模型多样性相关 —— 异构集成比同质集成更不同意（好：不相关错误）但达成一致也更慢（坏：更长的达成一致时间）。

要点：BFT 给你对齐输出的机制，但它不告诉你对齐的输出是否正确。与验证结合（第 16 阶段 · 08 角色专业化）、多样性（第 16 阶段 · 15 辩论变体）和评估者智能体（第 16 阶段 · 24 基准）。

### 核心协议，简化版

LLM 智能体的最小 BFT 轮次：

```
1. 任务到达；每个智能体 i 产生答案 a_i
2. 每个智能体附加置信探针 c_i 在 [0,1] 中
3. 聚合器从所有 n 个智能体收集 (a_i, c_i)
4. 聚合器按语义聚类分组（等效答案）
5. 聚合器计算每个聚类的权重 C：
     w(C) = sum_{i 在 C 中} c_i
6. 获胜者 = 具有最大权重的聚类，如果最大值 > 阈值 * sum(c_i)
   否则：重试或升级
7. 少数聚类记录出处以供事后审计
```

语义聚类步骤是 LLM 特定的转折。两个答案"研究报告 4.2%"和"4.2% 改进"是同一个聚类。天真的字符串相等检查会错过这个。在生产中，使用便宜的嵌入模型或显式规范化。

### 阈值调优

`阈值` 参数决定何时接受和何时重试。太低：你接受弱势多数。太高：你永远不接受任何东西。经验范围：`n=5-7` 个智能体为 0.5-0.67，较小的 `n` 更高。低于阈值，升级到人类或不同的智能体集成。

### 共识没有帮助的地方

- **模棱两可的问题。** 如果问题没有基本事实，共识是一种意见。称之为那样。
- **复合问题。** "写代码并解释它" —— 两个答案。独立对每个进行投票。
- **对抗性多轮。** 如果智能体可以观察先前的轮次并模仿（Du 2023 辩论），它们开始相互同意，而不管真相。限制轮次（通常 2-3）。

## 构建它

`code/main.py` 实现：

- `AgentVoter` —— 带有 (答案, 置信度) 的脚本化策略。
- `MajorityVote` —— 经典复数。
- `CPWBFT` —— 带语义聚类的置信加权投票。
- `DecentLLMs` —— 评分提议上的几何中位数聚合。
- `Scenario` —— 在三种攻击模式下运行每个聚合器。

实现的攻击模式：

1. `byzantine`：一个智能体以高置信度说谎。
2. `sycophancy`：一个智能体复制它看到的第一个答案，匹配置信度。
3. `monoculture`：三个智能体共享一个错误答案（相关错误）并具有中等置信度。

运行：

```
python3 code/main.py
```

预期输出：一个（攻击, 聚合器）-> 最终答案的表格，并突出显示正确答案。复数在单一文化情况下失败。CPWBFT 的置信加权缓解了谄媚从众。当单一文化少于人口的一半时，DecentLLMs 的几何中位数拉向诚实的聚类。

## 使用它

`outputs/skill-consensus-designer.md` 为多智能体集成设计共识协议：聚类方法、加权、阈值以及低于阈值的轮次的升级策略。

## 发布它

在发布任何共识机制之前：

- **至少用上述三种模式进行攻击测试。** 你的协议应该可预测地失败，而不是静默地。
- **记录每个少数聚类**的出处。少数聚类是你对相关错误的早期预警系统。
- **强制执行有限轮次。** 不要"继续辩论直到达成一致" —— 那奖励谄媚从众。
- **将一致性与正确性分离。** 共识输出进入验证器；验证器独立于集成。
- **监控一致率。** 急剧上升意味着从众偏见；急剧下降意味着模型漂移。

## 练习

1. 运行 `code/main.py`。确认复数在单一文化攻击中失败，但当单一文化置信度低于 0.7 时，CPWBFT 部分缓解了它。
2. 添加第四种攻击模式：**静默弃权** —— 一个智能体拒绝回答（"我不知道"）。每个聚合器应该如何处理弃权？实现你的选择。
3. 将语义聚类从字符串规范化交换为嵌入相似性（使用任何开源嵌入模型）。谄媚从众攻击会发生什么？
4. 阅读 CP-WBFT (arXiv:2511.10400)。实现置信探针校准步骤（一个单独的校准模型检查每个智能体的自我报告置信度）。衡量单一文化场景上的准确性增益。
5. 阅读"Can AI Agents Agree?" (arXiv:2603.01213)。重现一个简化的标量一致性实验：三个智能体，一个标量问题，欺骗性人格提示。CPWBFT 或 DecentLLMs 捕获它吗？

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|------|----------------|------------------------|
| BFT | "拜占庭容错" | Castro-Liskov 1999 协议，用于 `f < n/3` 任意故障的共识。 |
| 拜占庭 | "任何不良行为" | 可以撒谎、丢弃消息、静默失败的节点 —— 除了安全崩溃外的任何事情。 |
| 置信探针 | "你有多确定？" | 附加到投票的自我报告或校准器预测的概率。 |
| 语义聚类 | "相同答案，不同词语" | 在计数投票之前将等效答案分组。 |
| 几何中位数 | "鲁棒中心" | 最小化到样本点距离之和的点。对异常值鲁棒，不同于平均值。 |
| 单一文化 | "相同模型，相同失败" | 当智能体共享训练数据或基础模型时的相关错误。 |
| 谄媚从众 | "同意最大声的声音" | 智能体的投票偏向首先/最大声发言的人。 |
| 核心/边缘 | "层次结构 BFT" | WBFT 分割：小型核心共识优先，边缘节点跟随。限制延迟。 |
