# 14 · 面向智能体的共识与拜占庭容错

> 经典分布式系统的「拜占庭容错（BFT，Byzantine Fault Tolerance）」遇上随机性的大语言模型。2025—2026 年涌现出三条研究方向：**CP-WBFT**（arXiv:2511.10400）通过置信度探针为每一票加权；**DecentLLMs**（arXiv:2507.14928）走无领导者路线，让工作者并行提案并用几何中位数聚合；**WBFT**（arXiv:2505.05103）将加权投票与「层级结构聚类（Hierarchical Structure Clustering）」结合，将节点划分为核心（Core）与边缘（Edge）。论文《Can AI Agents Agree?》（arXiv:2603.01213）给出的诚实实证结论是：即便只是标量层面的一致，如今也很脆弱——单个欺骗性智能体就能攻破「智能体混合（Mixture-of-Agents）」。BFT 是必要的，但并不充分。本课构建一个最小化的 BFT 协议，注入三种智能体特有的攻击（拜占庭式撒谎、谄媚式从众、相关性错误的单一文化），并衡量每种共识变体的应对能力。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置：** 阶段 16 · 07（心智社会与辩论），阶段 16 · 13（共享内存）
**时长：** 约 75 分钟

## 问题

你有 N 个 LLM 智能体，每个都产出一个答案。它们彼此不一致。多数投票（majority vote）选错了，因为有两个智能体彼此相关（同一基础模型、同一训练数据、同一失败模式）。第三个智能体恰好以一种新颖的方式出错——于是多数派成了一个虚假的多数派。

现在再加入一个欺骗性智能体：它故意撒谎。或者一个谄媚式智能体：它附和最后发言的那个人。在经典 BFT 中，假设是拜占庭节点占比为 `f < n/3` 且行为任意。而 2026 年的现实是：LLM 节点即便诚实也是随机的，跨模型相互关联，并受彼此输出的影响。你不能把它们当作独立的伯努利（Bernoulli）投票者。

经典 BFT（PBFT，1999）并没有错——它只是不完整。它能处理任意的比特翻转，却处理不了「三个诚实的智能体因共享训练数据而共享同一个幻觉」。本课从 PBFT 的基础出发，叠加三项 2025—2026 年的适配方案。

## 概念

### 经典 BFT 给你什么

「实用拜占庭容错（Practical Byzantine Fault Tolerance）」（Castro & Liskov，OSDI 1999）容忍 `f < n/3` 个拜占庭节点。该协议有三个阶段（pre-prepare、prepare、commit）和两个原语（签名消息、法定人数证书 quorum certificate）。它在 `n >= 3f + 1` 个诚实或恶意节点之间就单个值达成一致。

这些保证很强，但有以下假设：

1. **故障相互独立。** 拜占庭节点之间不协同。
2. **诚实节点是真正诚实的。** 诚实输出的正确性不在考量范围内；协议只对齐分歧。
3. **问题存在一个真值答案。** 即便对错误事实达成共识，那也仍是共识。

LLM 智能体违反了全部三条。运行同一基础模型的两个智能体共享故障。一个「诚实」的 LLM 仍会产生幻觉。而对于歧义问题，「真值」就是智能体们决定的结果——并不存在外部的预言机（oracle）。

### 三种 LLM 特有的攻击

**拜占庭式撒谎（Byzantine lie）。** 一个智能体故意输出错误答案。若 `f < n/3`，经典 BFT 能处理这种情况。

**谄媚式从众（Sycophantic conformity）。** 一个智能体在投票前先读取其他人的答案，并附和最后发言者。它并非恶意，但会与最大的声音相关联。经典 BFT 无法阻止这种行为，因为该智能体能通过每一项签名校验。

**相关性错误的单一文化（Correlated-error monoculture）。** 三个智能体共享一个基础模型。它们产生同一个错误的幻觉。多数派是错的。经典 BFT 无能为力，因为这三者都「诚实地」一致。

### 2025—2026 年的应对方案

**CP-WBFT**（arXiv:2511.10400）——置信度探针加权 BFT（Confidence-Probed Weighted BFT）。每个投票者为其答案附加一个置信度探针（自报的概率，或一个独立校准模型的预测）。投票权重随置信度缩放。在完全图上报告了 +85.71% 的 BFT 改进。缓解对象：谄媚式从众（从众的智能体对其自愿提出的立场往往置信度偏低）。

**DecentLLMs**（arXiv:2507.14928）——无领导者。工作者智能体并行提案，评估者智能体为提案打分，最终答案是已打分立场的几何中位数（geometric median）。当 `f < n/2` 时具备鲁棒性。缓解对象：拜占庭式撒谎与相关性错误（几何中位数对离群点鲁棒，会被拉向密集簇，而非偏向模型的平均值）。

**WBFT**（arXiv:2505.05103）——带层级结构聚类的加权 BFT。投票权重由回答质量加上一个从历史中学习到的信任分数共同决定。将智能体聚类为核心（Core）与边缘（Edge）；核心智能体必须先达成共识，边缘智能体随后跟随。缓解对象：可扩展性（核心共识规模小、速度快），并部分缓解单一文化（核心可以为多样性而精选）。

### 实证：《Can AI Agents Agree?》（arXiv:2603.01213）

该论文在多个前沿模型上衡量标量一致性（LLM 智能体在单个数值上达成一致）。结论令人不安：

- 即便没有对手，LLM 智能体在标量问题上的分歧率在许多基准上也超过 30%。
- 单个采用欺骗性人设的智能体，就能把「智能体混合」的共识从诚实基线拉偏 40 个百分点以上。
- 分歧率与模型多样性相关——异构集成比同构集成分歧更多（好处：错误不相关），但漂移也更慢（坏处：达成一致所需时间更长）。

要点：BFT 为你提供对齐输出的机制，但它并不能告诉你被对齐的那个输出是否正确。请与验证（阶段 16 · 08 角色专门化）、多样性（阶段 16 · 15 辩论变体）以及评估者智能体（阶段 16 · 24 基准）结合使用。

### 核心协议，精简版

面向 LLM 智能体的最小化 BFT 一轮流程：

```
1. task arrives; each agent i produces answer a_i
2. each agent attaches confidence probe c_i in [0, 1]
3. aggregator collects (a_i, c_i) from all n agents
4. aggregator groups by semantic cluster (equivalent answers)
5. aggregator computes weight for each cluster C:
     w(C) = sum_{i in C} c_i
6. winner = cluster with max weight, if max > threshold * sum(c_i)
   else: retry or escalate
7. minority clusters logged with provenance for post-hoc audit
```

语义聚类步骤是 LLM 特有的关键转折。两个答案「the study reports 4.2%」和「4.2% improvement」属于同一个簇。朴素的字符串相等检查会错过这一点。在生产中，请使用一个廉价的嵌入模型或显式的规范化（canonicalization）。

### 阈值调优

`threshold` 参数决定何时接受、何时重试。太低：你会接受脆弱的多数派。太高：你永远接受不了任何东西。经验范围：对 `n=5-7` 个智能体取 0.5—0.67，`n` 更小时取更高值。低于阈值时，升级到人工或换一组智能体集成。

### 共识帮不上忙的场景

- **歧义问题。** 如果问题没有真值，共识就只是一种意见。请如实称呼它。
- **复合问题。** 「写代码并解释它」——两个答案。对每一个分别投票。
- **对抗式多轮。** 如果智能体能观察先前轮次并模仿（Du 2023 辩论），它们就会不顾真值地开始彼此附和。请限定轮数（通常 2—3 轮）。

## 动手构建

`code/main.py` 实现了：

- `AgentVoter`——一个带（answer, confidence）的脚本化策略。
- `MajorityVote`——经典的相对多数（plurality）。
- `CPWBFT`——带语义聚类的置信度加权投票。
- `DecentLLMs`——对已打分提案做几何中位数聚合。
- `Scenario`——在三种攻击模式下运行每个聚合器。

已实现的攻击模式：

1. `byzantine`：一个智能体以高置信度撒谎。
2. `sycophancy`：一个智能体复制它看到的第一个答案，并匹配其置信度。
3. `monoculture`：三个智能体共享一个错误答案（相关性错误），置信度中等。

运行：

```
python3 code/main.py
```

预期输出：一张 (attack, aggregator) -> final answer 的表格，并高亮正确答案。相对多数在单一文化场景下失败。CPWBFT 的置信度加权缓解了谄媚。当单一文化人群不足半数时，DecentLLMs 的几何中位数会拉向诚实簇。

`outputs/skill-consensus-designer.md` 为一个多智能体集成设计共识协议：聚类方法、加权方式、阈值，以及在低于阈值轮次时的升级策略。

## 交付上线

在交付任何共识机制之前：

- **至少用上述三种模式做攻击测试。** 你的协议应当可预测地失败，而非悄无声息地失败。
- **记录每一个少数派簇**及其溯源（provenance）。少数派簇是你针对相关性错误的预警系统。
- **强制限定轮数。** 不要「一直辩论到达成一致」——那会奖励谄媚。
- **将一致性与正确性分开。** 共识输出交给一个验证器；验证器独立于该集成之外。
- **监控一致率。** 急剧上升意味着从众偏差；急剧下降意味着模型漂移。

## 练习

1. 运行 `code/main.py`。确认相对多数在单一文化攻击下失败，而当单一文化置信度低于 0.7 时，CPWBFT 能部分缓解它。
2. 添加第四种攻击模式：**沉默弃权（silent abstention）**——一个智能体拒绝作答（「我不知道」）。每个聚合器应如何处理弃权？实现你的选择。
3. 把语义聚类从字符串规范化换成嵌入相似度（使用任意开源嵌入模型）。谄媚攻击会发生什么变化？
4. 阅读 CP-WBFT（arXiv:2511.10400）。实现置信度探针校准步骤（一个独立校准模型核对每个智能体自报的置信度）。衡量它在单一文化场景下带来的准确率增益。
5. 阅读《Can AI Agents Agree?》（arXiv:2603.01213）。复现一个简化的标量一致性实验：三个智能体、一个标量问题、欺骗性人设提示词。CPWBFT 或 DecentLLMs 能否抓住它？

## 关键术语

| 术语 | 人们常说 | 它实际的含义 |
|------|----------------|------------------------|
| BFT | 「拜占庭容错」 | Castro-Liskov 1999 协议，用于在 `f < n/3` 个任意故障下达成共识。 |
| 拜占庭（Byzantine） | 「任何不良行为」 | 一个可以撒谎、丢弃消息、静默失败的节点——除了安全崩溃外什么都可能干。 |
| 置信度探针（Confidence probe） | 「你有多确定？」 | 附加在一票上的、自报或由校准器预测的概率。 |
| 语义聚类（Semantic clustering） | 「同一答案，不同措辞」 | 在计票前对等价答案进行分组。 |
| 几何中位数（Geometric median） | 「鲁棒的中心」 | 使到各样本点距离之和最小的点。与均值不同，它对离群点鲁棒。 |
| 单一文化（Monoculture） | 「同一模型，同样的失败」 | 当智能体共享训练数据或基础模型时出现的相关性错误。 |
| 谄媚式从众（Sycophantic conformity） | 「附和大声的那一方」 | 一个智能体的投票偏向最先/最大声发言的人。 |
| 核心/边缘（Core/Edge） | 「层级 BFT」 | WBFT 的划分：先由小规模核心达成共识，边缘节点跟随。可限定延迟。 |
