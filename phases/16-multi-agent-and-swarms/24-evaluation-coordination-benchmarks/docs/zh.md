# 评估与协调基准

> 五个2025-2026年基准覆盖多智能体评估空间。**MultiAgentBench / MARBLE**（ACL 2025，arXiv:2503.01935）用里程碑KPI评估星形/链形/树形/图拓扑；**图对研究最好**，认知规划增加约3%里程碑达成。**COMMA**评估多模态非对称信息协调；包括GPT-4o在内的最先进模型难以击败随机基线。**MedAgentBoard**（arXiv:2505.12371）覆盖四个医学任务类别，经常发现多智能体不占优于单LLM。**AgentArch**（arXiv:2509.10769）基准测试结合工具使用 + 记忆 + 编排的企业智能体架构。**SWE-bench Pro**（[arXiv:2509.16941](https://arxiv.org/abs/2509.16941)）有跨越商业应用、B2B服务和开发者工具的41个仓库的1865个问题；前沿模型在Pro上得分约23% vs Verified上70%+——对污染的 reality check。Claude Opus 4.7（2026年4月）据报道在Pro上达到**64.3%**，使用显式智能体团队协调（尚无Anthropic主要来源发布——视为初步）；Verdent（智能体脚手架）在Verified上达到**76.1% pass@1**（[Verdent技术报告](https://www.verdent.ai/blog/swe-bench-verified-technical-report)）。**AAAI 2026 Bridge Program WMAC**（https://multiagents.org/2026/）是2026年社区焦点。本课程基于MARBLE的指标，运行拓扑-指标扫描，并钉住"仅通过SWE-bench Verified不是泛化证据"的规则。

**类型：** 学习
**语言：** Python（标准库）
**前置知识：** 第16阶段 · 15（投票与辩论拓扑），第16阶段 · 23（失败模式）
**时间：** 约75分钟

## 问题

当论文声称"我们的多智能体系统更好"时，问题是：比什么好，在什么上，如何测量？2023-2024年多智能体评估时代是混乱的——每个人都选择自己的指标、自己的基线和自己的任务集。2025-2026年基准施加了结构。

没有共享基准，你无法有意义地比较两个多智能体系统。更糟的是，没有留出基准，前沿模型可以污染。SWE-bench Verified到2025年中期在训练语料中部分污染；前沿分数膨胀；Pro被设计为无污染的 reality check。

本课程列举五个规范2026基准，命名每个测量什么，并教你怀疑地阅读基准声明。

## 概念

### MultiAgentBench（MARBLE）—— ACL 2025

arXiv:2503.01935。在研究、编码和规划任务上评估四种协调拓扑（星形、链形、树形、图）。基于里程碑的KPI追踪部分进展而不是仅最终成功。

测量结果：

- **图**拓扑对研究场景最好；支持任意对任意批判。
- **链形**对逐步细化编码最好。
- **星形**对快速事实整合最好。
- **协调税**在图上超过约4个智能体时出现。
- **认知规划**在各拓扑上增加约3%里程碑达成。

用于：你想苹果对苹果比较协调拓扑。MARBLE仓库（https://github.com/ulab-uiuc/MARBLE）提供评估器。

### COMMA —— 多模态非对称信息

覆盖智能体有不同观察模态且必须在不完全信息共享下协调的任务。报告的结果令人不安：包括GPT-4o在内的前沿模型在COMMA的智能体-智能体协作上难以击败**随机基线**。信号是多智能体模态训练不足和评估不足——LLM处理单模态合作合理；多模态协调崩溃。

用于：你的系统有多模态或非对称信息协调。COMMA的零结果是测量前声称的警告。

### MedAgentBoard —— 领域压力测试

arXiv:2505.12371。四个医学任务类别：诊断、治疗规划、报告生成、患者沟通。比较多智能体 vs 单LLM vs 传统基于规则的系统。

发现：多智能体在大多数类别上不占优于单LLM。多智能体优势狭窄——当子任务清晰可分时（诊断 + 治疗），任务分解有帮助；当协调开销超过专业化收益时（报告生成），它有害。

用于：你的领域有明确的单LLM基线。如果MedAgentBoard的教训泛化，许多提出的多智能体系统过度工程化。

### AgentArch —— 企业架构

arXiv:2509.10769。工具使用、记忆和编排分层在一起的企业设置。基准隔离每层的贡献：添加工具帮助多少？添加记忆？添加多智能体编排？

用于：你正在设计企业智能体栈并需要证明每层。AgentArch帮助避免购买你无法测量价值的功能。

### SWE-bench Pro —— reality check

arXiv:2509.16941。跨越商业应用、B2B服务和开发者工具的41个仓库的1865个问题。设计为**无污染**，使用更晚的训练截止。前沿模型在Pro上得分约23% vs Verified上70%+。差距是污染信号。

2026年4月分数：
- Claude Opus 4.7在Pro上：**64.3%**（据报道使用显式智能体团队协调；尚无Anthropic主要来源发布——视为初步）。
- Verdent（智能体脚手架）在Verified上：**76.1% pass@1**（[技术报告](https://www.verdent.ai/blog/swe-bench-verified-technical-report)）。
- 没有智能体脚手架的前沿模型在Pro上的原始分数：约23-35%（[SWE-bench Pro论文](https://arxiv.org/abs/2509.16941)）。

要点："我们击败SWE-bench Verified"不再是能力的证据。Pro是当前门槛测试。智能体团队脚手架在Pro上产生可测量的增益（约30-40点增量），这是2026年多智能体协调最强经验论据之一。

### AAAI 2026 WMAC

AAAI 2026 Bridge Program——多智能体协调研讨会（https://multiagents.org/2026/）。2026年多智能体AI研究的社区焦点。接受的论文和研讨会论文集是评估新方法的规范场所；对生产决策，优先于arXiv预印本的WMAC接受声明。

### 怀疑地阅读基准声明——2026检查清单

当有人声称多智能体结果时：

1. **哪个基准，哪个分割？** SWE-bench Verified vs Pro很重要。在错误分割上报告的数字毫无价值。
2. **污染检查。** 基准是否在模型训练截止后发布？如果不是，谨慎对待。
3. **基线比较。** vs 单LLM基线，vs 随机，vs 先前多智能体工作。不是"vs 同一系统的未调优版本"。
4. **统计显著性。** N次试验，p值，置信区间。前沿模型是高方差的；单次运行误导。
5. **任务多样性。** 一个任务还是多个？泛化对生产很重要。
6. **成本披露。** 每任务token，挂钟。90%的解决方案以20倍成本是商业决策，不是能力声明。

### 没有一个基准测量好的内容

- **长程协调。** 数天挂钟交互。所有当前基准运行短。
- **对抗韧性。** 当一个智能体恶意或妥协时会发生什么？
- **部署下的漂移。** 基准是静态的；生产分布变化。
- **成本归一化性能。** 大多数基准报告原始准确性，不是每美元准确性。

为你实际关心的轴构建自己的内部基准通常是正确的举动。

## 构建它

`code/main.py`是非交互式演练：

- 在玩具任务上模拟3个多智能体系统。
- 为每个计算MARBLE风格里程碑指标。
- 通过从"训练"集保留任务运行污染检查。
- 显式与随机基线比较。
- 打印基准声明记分卡。

运行：

```bash
python3 code/main.py
```

预期输出：系统记分卡，包含原始准确性、里程碑达成、每任务成本、vs随机基线增量，以及污染检查注释。

## 使用它

`outputs/skill-benchmark-reader.md`阅读任何多智能体基准声明并应用审查检查清单。输出：等级和注意事项。

## 交付它

生产评估纪律：

- **构建反映你实际生产分布的内部基准。** 公共基准提供信息但不替代。
- **在每个比较中包含随机基线。** 如果你在协调任务上无法以很大幅度击败随机，任务可能是病态的。
- **报告准确性旁边的成本。** Token成本和挂钟。运维团队需要两者。
- **每季度重建基准。** 生产分布变化；陈旧的基准误导。
- **避免已发布基准的过拟合。** 如果你的团队专门优化SWE-bench Pro数字，你会在生产上回归。

## 练习

1. 运行`code/main.py`。识别三个模拟系统中哪个有最好的每里程碑成本。它与最高原始准确性系统匹配吗？
2. 阅读MultiAgentBench（arXiv:2503.01935）。对于你自己的任务领域，决定MARBLE会推荐四种拓扑中的哪一种。从论文结果论证。
3. 阅读SWE-bench Pro论文。什么具体使其抗污染？相同技术可以应用到你关心的其他基准吗？
4. 阅读COMMA关于多模态协调的发现。设计一个你可以添加到内部基准的简单多模态协调任务。什么算作有用信号？
5. 将基准声明检查清单应用于一篇近期多智能体论文的标题结果。你会给该声明什么等级？
