# 21 · 智能体经济、代币激励与声誉

> 长时程自主智能体（METR 提出的 1 小时到 8 小时工作曲线）需要经济行动能力。新兴的 **5 层技术栈** 是：**DePIN（去中心化物理基础设施，DePIN）**（物理算力）→ **身份（Identity）**（W3C DID + 声誉资本）→ **认知（Cognition）**（RAG + MCP）→ **结算（Settlement）**（账户抽象）→ **治理（Governance）**（智能体 DAO）。生产级的智能体激励网络包括 **Bittensor**（TAO 子网奖励特定任务模型）、**Fetch.ai / ASI Alliance**（ASI-1 Mini LLM + FET 代币），以及 **Gonka**（基于 transformer 的工作量证明，将算力重新分配到有生产价值的 AI 任务上）。学术工作方面：AAMAS 2025 的去中心化 LaMAS 使用 **Shapley 值信用归因（Shapley-value credit attribution）** 公平地奖励有贡献的智能体；Google Research 的「大语言模型机制设计（Mechanism design for large language models）」提出在单调聚合下采用次价支付的 **代币拍卖（token auctions）**。本课构建一个最小化的智能体市场，把 Shapley 值信用归因应用到一条多智能体流水线上，并运行一场次价代币拍卖，让这套博弈论机制落到实处。

## 问题

当多个智能体共同创造价值、却需要被单独奖励时，多智能体系统就会变得复杂。经典的分配机制——平均分配、最后贡献者通吃——要么不公平，要么容易被钻空子。基于联盟的奖励方式（即 Shapley 值）在设计上是公平的，但计算代价高昂。2025-2026 年的文献推动了一些有用的近似方法：Shapley 采样、单调聚合拍卖，以及从已确认贡献中累积的链上声誉。

除了信用归因之外，该领域已经转向真正的经济智能体：Bittensor TAO 奖励挖矿算力以微调特定子网的模型，Fetch.ai/ASI 用 FET 代币奖励 ASI-1 Mini LLM 的使用，Gonka 将 transformer 工作量证明重新分配到有生产价值的 AI 任务上。能够自主交易的智能体如今已经存在；问题在于如何对齐激励。

本课把智能体经济视为一类特定的问题家族——信用归因、机制设计与声誉——并用最少的数学逐个构建，以便这些理念能够真正记住。

## 概念

### 5 层智能体经济技术栈

1. **DePIN（物理算力）。** 出租 GPU、存储、带宽的去中心化基础设施。Bittensor 子网、Render Network、Akash。它本身并非智能体专用；智能体只是使用它。
2. **身份。** W3C 去中心化标识符（Decentralized Identifiers，DID）为每个智能体提供独立于任何平台的持久 ID。声誉累积到 DID 上。智能体网络协议（Agent Network Protocol，ANP）将 DID 作为发现层。
3. **认知。** 智能体的推理循环：LLM + RAG + MCP。这正是其他阶段所构建的内容。
4. **结算。** 账户抽象（ERC-4337）让智能体能够用自己的余额支付 gas，而无需持有 ETH。智能体可以为服务、彼此或算力付费。
5. **治理。** 智能体 DAO：人类*和*智能体共同对协议变更投票的治理结构，投票权与声誉挂钩。

并非每个生产系统都用到全部五层。Bittensor 用到第 1、2 层，部分用到第 3 层，部分用到第 4 层，完全不用第 5 层。OpenAI 的智能体除第 3 层外都不用。这套技术栈是一张参考地图，而非硬性要求。

### Bittensor、Fetch.ai、Gonka——它们究竟在做什么

**Bittensor（TAO）。** 子网是专门化的任务（语言建模、图像生成、预测）。矿工提交模型输出。验证者对其排名；按质押权重打分来分配 TAO 奖励。每个子网都有自己的评估方式。其经济学要义是：为特定任务的输出质量付费，而非为消耗的算力付费。

**Fetch.ai / ASI Alliance。** ASI-1 Mini LLM 运行在 Fetch.ai 的网络上；用户支付 FET 代币以获得推理。这里「智能体即对等节点」的叙事更强：Fetch 上的一个智能体可以调用另一个智能体完成任务，并用 FET 付费。

**Gonka。** Transformer 工作量证明：所谓「工作」就是 transformer 的前向传播。矿工通过运行那些已知正确输出（来自训练数据）的推理任务来赚取收益。这是一种资源生产型的 PoW，而非基于哈希的 PoW。

截至 2026 年 4 月，这三者都已达到生产级水平。它们的收益分配方式各不相同。Bittensor 奖励的是相对于子网验证者的质量；Fetch 奖励的是由付费用户衡量的效用；Gonka 奖励的是可验证的推理工作。

### Shapley 值信用归因

三个智能体协作完成一项任务。输出得分为 0.8。各自贡献了多少？

Shapley 值：满足四条公理（效率、对称性、线性、空贡献）的唯一信用分配。对于智能体 `i`：

```
shapley(i) = (1/N!) * sum over all orderings O of (v(S_i_O ∪ {i}) - v(S_i_O))
```

其中 `S_i_O` 是排序 `O` 中位于 `i` 之前的智能体集合。实践中：枚举所有排列，记录每个智能体在每个排列中的边际贡献，然后取平均。

对于 N=3 个智能体，有 6 个排列。对于 N=10，则有 360 万个——所以实践中你会对排序进行采样，而非全部枚举。

### 用于聚合的次价拍卖

Google Research（「大语言模型机制设计」）提出用次价代币拍卖来聚合 LLM 输出。设定：N 个智能体各自提出一份补全；每个智能体对被选中都有一个私有估值。拍卖者选取估值最高的提案，并支付*第二高*的估值。在单调聚合（价值取决于选中了哪个提案，而非有多少人出价）下，这是真实可信的——智能体会按其真实估值出价。

这对 LLM 系统为何重要：你可以把补全任务外包给多个定价不同的智能体；拍卖会选出最优者并公平付费，而智能体没有谎报估值的动机。

### 声誉资本

绑定到 DID 的声誉分数从已确认的贡献中累积。一条简单的更新规则：

```
rep(i, t+1) = alpha * rep(i, t) + (1 - alpha) * contribution_quality(i, t)
```

其中衰减因子 `alpha` 接近 1。声誉具有以下特性：

- 在做路由决策时读取成本低廉（「把困难任务发给高声誉智能体」）。
- 伪造成本高昂（随时间累积，且绑定到 DID）。
- 可以被罚没（slash）：未通过验证的贡献会被扣减。

### AAMAS 2025 去中心化 LaMAS

LaMAS 方案（AAMAS 2025）结合了：DID 身份、Shapley 值信用归因，以及一套简单的拍卖机制。其核心论点是：将信用归因这一步去中心化，可使系统具备可审计性，并免疫于单点操纵。

### 经济机制在哪里会失效

- **价格预言机操纵。** 如果信用函数可以被钻空子，智能体就会钻空子。每一种机制都需要对抗性测试。
- **女巫攻击（Sybil attacks）。** 一个运营者启动 N 个虚假智能体来抬高自己的贡献。DID 能减缓但无法阻止这一点；声誉的伪造成本才是缓解手段。
- **验证成本。** 信用归因的公平程度取决于验证者。如果验证廉价（小型 LLM），它会被钻空子；如果验证昂贵（人工评审团），系统就无法扩展。
- **监管悬置。** 智能体经济与金融监管相交叉。截至 2026 年，Bittensor、Fetch 和 Gonka 在某些司法管辖区都处于法律灰色地带运营。

### 智能体经济何时有意义

- **由异构运营者组成的开放网络。** 没有任何单一团队控制所有智能体。
- **可验证的输出。** 没有验证，信用归因就只是猜测。
- **长时程工作流。** 一次性任务无法从声誉累积中获益。
- **在你的司法管辖区内，代币化支付在法律上是可行的。**

在封闭的企业系统中，经济机制让位于更简单的分配方式（由管理者分派工作，指标内部统计）。经济学文献主要适用于开放网络。

## 上线交付

2026 年运营一套智能体经济：

- **从声誉起步，而非代币。** 声誉实现成本低，且单独使用就有价值；代币会增加法律与经济上的复杂性。
- **先验证，再奖励。** 永远不要在没有独立验证步骤的情况下分配信用。自我报告的质量会滋生女巫游戏。
- **用 Shapley 采样，而非精确 Shapley。** 采样 100-1000 个排序；精确枚举无法扩展。
- **为衰减因子设上限、为声誉设下限。** 无限制的衰减会抹去合法贡献者；过慢的衰减则会奖励陈旧的高声誉智能体。
- **对机制进行对抗性审计。** 在开放网络之前运行红队场景。每一种机制都有其博弈论；你要做的是先于攻击者找出漏洞。

## 练习

1. 运行 `code/main.py`。确认各 Shapley 值之和等于总价值（效率公理）。修改价值函数；Shapley 分配是否按预期方向变化？
2. 实现 Shapley *采样*（对 K 个排序做蒙特卡洛）。K 如何影响近似精度？对 N=4 与精确结果做比较。
3. 在拍卖前实现一个联盟组建步骤：智能体可以合并为团队并作为整体出价。哪些联盟会形成？其结果在帕累托意义上是否优于单独出价？
4. 阅读 Google Research 的机制设计博文。找出一个一旦被违反就会破坏真实性的假设。在 LLM 场景下，这种失效模式看起来是怎样的？
5. 阅读 AAMAS 2025 去中心化 LaMAS 论文。在一个合成任务上对 10 个智能体实现他们的 Shapley 步骤。精确计算需要多长时间？采样 100 次能逼近到何种程度？

## 关键术语

| 术语 | 人们怎么说 | 它实际指什么 |
|------|----------------|------------------------|
| DePIN | 「去中心化物理基础设施」 | 代币激励的算力/存储/带宽。Bittensor、Akash、Render。 |
| DID | 「去中心化标识符」 | 可移植 ID 的 W3C 规范。智能体声誉绑定到 DID，而非平台。 |
| ERC-4337 | 「账户抽象」 | 能够代付 gas 的合约账户，使智能体支付成为可能。 |
| Shapley 值 | 「公平的信用归因」 | 满足效率、对称性、线性、空贡献的唯一分配。 |
| 次价拍卖 | 「Vickrey 拍卖」 | 真实可信的机制：获胜者支付第二高出价。与单调聚合兼容。 |
| 声誉资本 | 「累积的质量分数」 | 绑定 DID、由已确认贡献累积而来的分数；随时间衰减。 |
| 智能体 DAO | 「智能体 + 人类共同治理」 | 把智能体作为一等投票者的 DAO，投票权与声誉挂钩。 |
| TAO / FET / GPU 信用额度 | 「代币计价单位」 | Bittensor 的 TAO、Fetch.ai 的 FET，以及各类 DePIN 代币。 |
