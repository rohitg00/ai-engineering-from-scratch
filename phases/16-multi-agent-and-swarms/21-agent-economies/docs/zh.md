# 智能体经济、代币激励、声誉

> 长期自主智能体（METR 的 1 小时到 8 小时工作曲线）需要经济代理。新兴的 **5 层堆栈** 是：**DePIN**（物理计算）→ **身份**（W3C DID + 声誉资本）→ **认知**（RAG + MCP）→ **结算**（账户抽象）→ **治理**（智能体 DAO）。生产级智能体激励网络包括 **Bittensor**（TAO 子网奖励特定任务的模型）、**Fetch.ai / ASI Alliance**（ASI-1 Mini LLM + FET 代币）和 **Gonka**（基于 transformer 的 PoW，将计算重新分配给生产性 AI 任务）。学术工作：AAMAS 2025 的去中心化 LaMAS 使用**沙普利值信用归因**来公平奖励贡献智能体；Google Research "大型语言模型的机制设计" 提出了在单调聚合下具有第二价格支付的**代币拍卖**。本课构建了一个最小的智能体市场，将沙普利值信用归因应用于多智能体管道，并运行第二价格代币拍卖，以便博弈论机制具体地落地。

**类型：** 学习
**语言：** Python（标准库）
**前置要求：** 第 16 阶段 · 16（谈判与讨价还价）、第 16 阶段 · 09（并行群体网络）
**时间：** 约 75 分钟

## 问题

当智能体共同产生价值但需要单独奖励时，多智能体系统变得复杂。经典机制——平均分配、最后贡献者通吃——是不公平的或可被博弈的。通过沙普利值进行的基于联盟的奖励是构造公平的，但计算成本高。2025-2026 年的文献推动了有用的近似：沙普利采样、单调聚合拍卖和来自确认贡献的链上声誉。

除了信用归因，该领域已转向实际的经济智能体：Bittensor TAO 奖励挖矿计算以微调子网特定模型，Fetch.ai/ASI 用 FET 代币奖励 ASI-1 Mini LLM 使用，Gonka 将 transformer 工作量证明重新分配给生产性 AI 任务。自主交易的智能体今天存在；问题是如何调整激励。

本课将智能体经济视为一个特定问题族——信用归因、机制设计和声誉——并用最少的数学构建每一个，以便这些想法留下深刻印象。

## 概念

### 5 层智能体经济堆栈

1. **DePIN（物理计算）。** 租用 GPU、存储、带宽的去中心化基础设施。Bittensor 子网、Render Network、Akash。不是智能体特定的；智能体使用它。
2. **身份。** W3C 去中心化标识符（DID）为每个智能体提供一个独立于任何平台的持久 ID。声誉累积到 DID。智能体网络协议（ANP）使用 DID 作为发现层。
3. **认知。** 智能体的推理循环：LLM + RAG + MCP。这是其他阶段构建的内容。
4. **结算。** 账户抽象（ERC-4337）允许智能体从自己的余额支付 gas，而无需持有 ETH。智能体可以为服务、彼此或计算付费。
5. **治理。** 智能体 DAO：人类 *和* 智能体对协议变更进行投票的治理结构，投票权与声誉挂钩。

并非每个生产系统都使用所有五层。Bittensor 使用 1、2、部分 3、部分 4，不使用 5。OpenAI 智能体除 3 外不使用任何层。该堆栈是参考地图，不是要求。

### Bittensor、Fetch.ai、Gonka — 运行的内容

**Bittensor (TAO)。** 子网是专门任务（语言建模、图像生成、预测）。矿工提交模型输出。验证者对其进行排名；质押加权评分分配 TAO 奖励。每个子网都有自己的评估。经济教训：为任务特定的输出质量付费，而不是使用的计算。

**Fetch.ai / ASI Alliance。** ASI-1 Mini LLM 在 Fetch.ai 的网络上运行；用户为推理支付 FET 代币。这里的智能体即对等节点叙事更强烈：Fetch 上的智能体可以为任务调用另一个智能体并以 FET 支付。

**Gonka。** Transformer 工作量证明："工作"是 transformer 的前向传递。矿工通过运行具有已知正确输出（来自训练数据）的推理任务来赚钱。资源生产性 PoW 而不是基于哈希的 PoW。

截至 2026 年 4 月，所有这三个都是生产级别的。收益分配不同。Bittensor 相对于子网验证者奖励质量；Fetch 奖励由付费用户衡量的实用性；Gonka 奖励可验证的推理工作。

### 沙普利值信用归因

三个智能体在一项任务上协作。输出得分为 0.8。谁贡献了什么？

沙普利值：满足四个公理（效率、对称性、线性、零）的唯一信用分配。对于智能体 `i`：

```
shapley(i) = (1/N!) * sum over all orderings O of (v(S_i_O ∪ {i}) - v(S_i_O))
```

其中 `S_i_O` 是排序 `O` 中 `i` 之前的智能体集合。在实践中：枚举所有排列，记录每个排列中每个智能体的边际贡献，取平均。

对于 N=3 个智能体，有 6 种排列。对于 N=10，有 360 万——所以在实践中，你对排序进行采样而不是枚举。

### 用于聚合的第二价格拍卖

Google Research（"大型语言模型的机制设计"）提出了用于聚合 LLM 输出的第二价格代币拍卖。设置：N 个智能体每个都提出一个完成；每个都有被选中的私有价值。拍卖者选择价值最高的提案并支付*第二高*的价值。在单调聚合下（价值取决于选择哪个提案，而不是出价多少），这是真实的——智能体出价其真实价值。

为什么这对 LLM 系统很重要：你可以将完成任务外包给具有不同定价的多个智能体；拍卖选择最佳 + 公平支付，并且智能体没有动机误报。

### 声誉资本

DID 绑定的声誉分数从确认的贡献中累积。一个简单的更新规则：

```
rep(i, t+1) = alpha * rep(i, t) + (1 - alpha) * contribution_quality(i, t)
```

衰减因子 `alpha` 接近 1。声誉：

- 用于路由决策读取便宜（"将困难任务发送给高声誉智能体"）。
- 伪造昂贵（随时间累积，绑定到 DID）。
- 可以被削减：未通过验证的贡献会减去。

### AAMAS 2025 去中心化 LaMAS

LaMAS 提案（AAMAS 2025）结合了：DID 身份、沙普利值信用归因和简单的拍卖机制。关键主张：去中心化信用归因步骤使系统可审计并免受单点操纵。

### 经济学在何处瓦解

- **价格预言机操纵。** 如果信用函数可以被博弈，智能体就会博弈它。每个机制都需要对抗测试。
- **女巫攻击。** 一个运营商启动 N 个假智能体以增加自己的贡献。DID 减慢但不能阻止这一点；声誉伪造成本是缓解措施。
- **验证成本。** 信用归因只与验证者一样公平。如果验证便宜（小型 LLM），它可以被博弈；如果昂贵（人类小组），系统无法扩展。
- **监管悬垂。** 智能体经济与金融监管相交。截至 2026 年，Bittensor、Fetch 和 Gonka 在一些司法管辖区都在法律灰色地带运营。

### 智能体经济何时有意义

- **具有异构运营商的开放网络。** 没有单个团队控制所有智能体。
- **可验证输出。** 没有验证，信用归因是猜测。
- **长期工作流。** 一次性任务不会从声誉累积中受益。
- **代币化支付在你的司法管辖区是合法的。**

在封闭的企业系统中，经济学让位于更简单的分配（管理者分配工作，指标是内部的）。经济学文献主要应用于开放网络。

## 部署

在 2026 年运行智能体经济：

- **从声誉开始，而不是代币。** 声誉实施便宜且单独具有价值；代币增加了法律和经济的复杂性。
- **在奖励之前验证。** 永远不要在没有独立验证步骤的情况下分配信用。自我报告的质量会累积女巫博弈。
- **沙普利采样，而不是沙普利精确。** 采样 100-1000 个排序；精确枚举无法扩展。
- **限制衰减因子和声誉下限。** 无界衰减会抹去合法的贡献者；过慢的衰减会奖励陈旧的声誉高的智能体。
- **对抗性地审计机制。** 在开放网络之前运行红队场景。每个机制都有博弈论；你想找到漏洞，而不是让攻击者找到。

## 练习

1. 运行 `code/main.py`。确认沙普利值总和为总价值（效率公理）。更改价值函数；沙普利分配是否按预期方向变化？
2. 实现沙普利*采样*（K 个排序上的蒙特卡洛）。K 如何影响近似精度？与 N=4 的精确值进行比较。
3. 在拍卖之前实现联盟形成步骤：智能体可以合并为团队并作为一个单元出价。哪些联盟形成？结果是否比单独出价帕累托更优？
4. 阅读 Google Research 机制设计帖子。找出一个如果被违反会破坏真实性的假设。在 LLM 设置中，该失效模式看起来像什么？
5. 阅读 AAMAS 2025 去中心化 LaMAS 论文。在合成任务上针对 10 个智能体实现他们的沙普利步骤。精确计算需要多长时间？100 次抽取采样有多接近？

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|------|----------------|------------------------|
| DePIN | "去中心化物理基础设施" | 代币激励的计算/存储/带宽。Bittensor、Akash、Render。 |
| DID | "去中心化标识符" | 可移植 ID 的 W3C 规范。智能体声誉绑定到 DID，而不是平台。 |
| ERC-4337 | "账户抽象" | 可以赞助 gas 的合约账户，启用智能体支付。 |
| 沙普利值（Shapley value） | "公平信用归因" | 满足效率、对称性、线性、零的唯一分配。 |
| 第二价格拍卖（Second-price auction） | "维克里拍卖" | 真实机制：赢家支付第二高出价。单调聚合兼容。 |
| 声誉资本（Reputation capital） | "累积质量分数" | 来自确认贡献的 DID 绑定分数；随时间衰减。 |
| 智能体 DAO（Agentic DAO） | "智能体 + 人类治理" | 智能体投票者作为一等公民的 DAO，投票权与声誉挂钩。 |
| TAO / FET / GPU 积分 | "代币面额" | Bittensor TAO、Fetch.ai FET、各种 DePIN 代币。 |
