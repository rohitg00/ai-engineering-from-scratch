## 问题

当智能体共同产生价值但需要单独奖励时，多智能体系统变得复杂。经典机制——均分、最后贡献者全拿——不公平或可博弈。通过Shapley值的联盟奖励按构造是公平的，但计算昂贵。2025-2026年文献推动有用的近似：Shapley采样、单调聚合拍卖和从确认贡献中积累的链上声誉。

超越信用归因，该领域转向实际经济智能体：Bittensor TAO奖励挖掘计算以微调子网特定模型，Fetch.ai/ASI奖励使用FET代币的ASI-1 Mini LLM使用，Gonka将Transformer工作量证明重新分配给生产性AI任务。自主交易的智能体今天已经存在；问题是如何对齐激励。

本课程将智能体经济视为特定问题族——信用归因、机制设计和声誉——并用最小数学构建每个，使想法扎根。

## 概念

### 5层智能体经济栈

1. **DePIN（物理计算）。** 租赁GPU、存储、带宽的去中心化基础设施。Bittensor子网、Render Network、Akash。不是智能体特定的；智能体使用它。
2. **身份。** W3C去中心化标识符（DID）给每个智能体一个独立于任何平台的持久ID。声誉积累到DID。智能体网络协议（ANP）使用DID作为发现层。
3. **认知。** 智能体的推理循环：LLM + RAG + MCP。这是其他阶段构建的内容。
4. **结算。** 账户抽象（ERC-4337）让智能体从自己的余额支付gas，无需持有ETH。智能体可以为服务、彼此或计算付费。
5. **治理。** 智能体DAO：人类*和*智能体对协议变更投票的治理结构，投票权与声誉绑定。

不是每个生产系统都使用全部五层。Bittensor使用1、2、部分3、部分4、不使用5。OpenAI智能体除了3都不使用。该栈是参考地图，不是要求。

### Bittensor、Fetch.ai、Gonka——运行中的系统

**Bittensor（TAO）。** 子网是专业化任务（语言建模、图像生成、预测）。矿工提交模型输出。验证者排名；质押加权评分分配TAO奖励。每个子网有自己的评估。经济学教训：为任务特定输出质量付费，不是使用的计算。

**Fetch.ai / ASI联盟。** ASI-1 Mini LLM运行在Fetch.ai网络上；用户用FET代币支付推理。这里的智能体即对等体叙事更强：Fetch上的智能体可以为任务调用另一个并支付FET。

**Gonka。** Transformer工作量证明："工作"是Transformer的前向传播。矿工通过运行具有已知正确输出（来自训练数据）的推理任务来赚取。资源生产性PoW，不是基于哈希的PoW。

截至2026年4月，三者都是生产级的。收益分配不同。Bittensor奖励相对于子网验证者的质量；Fetch奖励按付费用户衡量的效用；Gonka奖励可验证的推理工作。

### Shapley值信用归因

三个智能体合作一个任务。输出得分0.8。谁贡献了什么？

Shapley值：满足四个公理（效率、对称性、线性性、零性）的唯一信用分配。对于智能体`i`：

```
shapley(i) = (1/N!) * 对所有排序O求和 (v(S_i_O ∪ {i}) - v(S_i_O))
```

其中`S_i_O`是排序`O`中`i`之前的智能体集合。实践中：枚举所有排列，记录每个智能体在每个排列中的边际贡献，取平均。

对于N=3个智能体，有6种排列。对于N=10，360万——所以实践中你采样排序而不是枚举。

### 聚合的二价拍卖

Google Research（"大型语言模型的机制设计"）提出用于聚合LLM输出的二价Token拍卖。设置：N个智能体每个提出一个完成；每个对被选中有一个私人价值。拍卖者选择最高价值提议并支付*第二高*价值。在单调聚合下（价值取决于选择哪个提议，不是多少投标），这是真实的——智能体出价其真实价值。

为什么这对LLM系统重要：你可以将完成外包给多个不同定价的智能体；拍卖选择最好的 + 公平支付，智能体没有动机虚报。

### 声誉资本

DID绑定的声誉分数从确认贡献中积累。简单更新规则：

```
rep(i, t+1) = alpha * rep(i, t) + (1 - alpha) * contribution_quality(i, t)
```

衰减因子`alpha`接近1。声誉：

- 对路由决策便宜可读（"将困难任务发送给高声誉智能体"）。
- 昂贵伪造（随时间积累，绑定到DID）。
- 可以被削减：未通过验证的贡献减去。

### AAMAS 2025去中心化LaMAS

LaMAS提案（AAMAS 2025）结合：DID身份、Shapley值信用归因和简单拍卖机制。关键主张：去中心化信用归因步骤使系统可审计且免疫单点操纵。

### 经济学在哪里崩溃

- **价格预言机操纵。** 如果信用函数可被博弈，智能体会博弈它。每个机制都需要对抗性测试。
- **女巫攻击。** 一个操作者启动N个假智能体来膨胀自己的贡献。DID减缓但不阻止；伪造声誉的成本是缓解措施。
- **验证成本。** 信用归因只与验证者一样公平。如果验证便宜（小LLM），它可被博弈；如果昂贵（人类小组），系统无法扩展。
- **监管悬置。** 智能体经济与金融监管相交。截至2026年，Bittensor、Fetch和Gonka在某些司法管辖区都在法律灰色地带运营。

### 智能体经济何时有意义

- **具有异构操作者的开放网络。** 没有单一团队控制所有智能体。
- **可验证输出。** 没有验证，信用归因是猜测。
- **长程工作流。** 一次性任务不受益于声誉积累。
- **代币化支付在你的司法管辖区法律上可行。**

在封闭企业系统中，经济让位于更简单的分配（管理者分配工作，指标是内部的）。经济学文献主要适用于开放网络。

## 交付它

2026年运行智能体经济：

- **从声誉开始，不是代币。** 声誉实现便宜且单独有价值；代币增加法律和经济复杂性。
- **奖励前验证。** 没有独立验证步骤绝不分配信用。自我报告的质量积累女巫游戏。
- **Shapley采样，不是Shapley精确。** 采样100-1000个排序；精确枚举无法扩展。
- **限制衰减因子和声誉下限。** 无界衰减抹除合法贡献者；太慢衰减奖励陈旧高声誉智能体。
- **对抗性审计机制。** 在开放网络前运行红队场景。每个机制都有博弈论；你想找到漏洞，不是攻击者。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| DePIN | "去中心化物理基础设施" | 代币激励的计算/存储/带宽。Bittensor、Akash、Render。 |
| DID | "去中心化标识符" | 可移植ID的W3C规范。智能体声誉绑定到DID，不是平台。 |
| ERC-4337 | "账户抽象" | 可以赞助gas的合约账户，实现智能体支付。 |
| Shapley值 | "公平信用归因" | 满足效率、对称性、线性性、零性的唯一分配。 |
| 二价拍卖 | "Vickrey拍卖" | 真实机制：获胜者支付第二高出价。兼容单调聚合。 |
| 声誉资本 | "积累的质量分数" | 来自确认贡献的DID绑定分数；随时间衰减。 |
| 智能体DAO | "智能体 + 人类治理" | 智能体投票者作为一等公民的DAO，投票权与声誉绑定。 |
| TAO / FET / GPU积分 | "代币面额" | Bittensor TAO、Fetch.ai FET、各种DePIN代币。 |
