# 用于 LLM 的群体优化（PSO、ACO）

> 生物启发式优化正在 LLM 领域回归。**LMPSO**（arXiv:2504.09247）使用 PSO，其中每个粒子的速度是一个提示，LLM 生成下一个候选者；在结构化序列输出（数学表达式、程序）上表现良好。**模型群体**（arXiv:2410.11163）将每个 LLM 专家视为模型权重流形上的 PSO 粒子，并报告在 9 个数据集上，仅用 200 个实例就比 12 个基线平均提高了 **13.3%**。**群体提示**（ICAART 2025）将 PSO + 灰狼混合用于提示优化。**AMRO**-S（arXiv:2603.12933）是受 ACO 启发的费洛蒙专家，用于多智能体 LLM 路由——**4.7 倍加速**，可解释的路由证据，质量门控的异步更新将推理与学习解耦。本课时在提示参数空间上实现 PSO，在智能体路由上实现 ACO，测量为什么这些经典算法适合 LLM 时代，以及它们何时不适合。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**先修：** 第 16 阶段 · 09（并行群体网络）、第 16 阶段 · 14（共识与 BFT）
**时间：** 约 75 分钟

## 问题

你有一个在你的任务评估上得分 62% 的提示。你想改进它。天真的做法是无需梯度的手动调整，但扩展性差。强化学习需要奖励信号和足够的 rollout 来进行训练。通过提示进行反向传播并不真正可能——提示是一个离散字符串，而不是可微参数。

经典的生物启发式优化——用于连续搜索空间的 PSO 和用于路径选择的 ACO——正是为这种机制设计的：无需梯度、基于种群、每次评估成本低。将它们与 LLM 配对用于无需梯度的搜索步骤，你就得到了一个出乎意料地实用的优化器。

相同的模式适用于多智能体系统中的智能体*路由*。ACO 风格的费洛蒙轨迹记录哪个智能体在哪种任务类型上工作得最好，让路由器利用该轨迹，并使费洛蒙衰减，以便可以重新发现路由。

## 概念

### PSO 复习（Kennedy & Eberhart 1995）

粒子群优化：连续搜索空间中的粒子种群。每个粒子具有位置 `x_i` 和速度 `v_i`。每次迭代：

```
v_i <- w * v_i + c1 * r1 * (p_best_i - x_i) + c2 * r2 * (g_best - x_i)
x_i <- x_i + v_i
评估 fitness(x_i)
如果改进则更新 p_best_i
如果全局最佳则更新 g_best
```

其中 `p_best` 是粒子自身的最佳值，`g_best` 是种群的最佳值，`w, c1, c2` 是惯性 + 认知 + 社会权重，`r1, r2` 是随机因子。

### 在 LLM 输出上的 PSO——LMPSO

arXiv:2504.09247 使 PSO 适用于 LLM 生成的结构化输出（数学表达式、程序）。每个粒子是一个候选输出。速度是一个*提示*，描述如何向个人或全局最佳修改当前输出。LLM 根据速度提示生成新输出。速度的"惯性"是一个类似"进行小的增量更改"的提示。

这在以下情况下效果良好：

- 输出是结构化的（可解析、可评估）。
- 适应度是自动的（测试运行、算术评估）。
- 种群规模小（约 10-30 个粒子），因此总 LLM 调用量保持可控。

这在以下情况下效果不好：适应度需要人工审查——每次迭代的成本变得过高。

### 模型群体

arXiv:2410.11163 将 PSO 从输出层移到了*模型*层。每个"粒子"是一个专家 LLM（参数）。种群通过无需梯度的更新将参数移向集体最佳。报告称：在 9 个数据集上，仅每次迭代 200 个实例，就比 12 个基线平均提高了 13.3%。

关键的洞见是，LLM 专家模型在共享参数流形（适配器权重、LoRA 增量）中已经在附近。在这个低维子空间上的 PSO 是廉价且有效的。

### ACO 复习（Dorigo 1992）

蚁群优化：蚂蚁遍历图；每条路径都有一个费洛蒙轨迹。蚂蚁移动概率按费洛蒙强度加权。完成任务的蚂蚁沉积与解决方案质量成比例的费洛蒙。费洛蒙随时间衰减。

### AMRO-S——用于智能体路由的 ACO

arXiv:2603.12933 使用 ACO 进行多智能体路由。每个任务类型是一个"目的地"；每个智能体是一条可能的路由。费洛蒙加强产生良好输出的路由。主要贡献：

- **可解释的路由证据。** 费洛蒙强度是人类可读的信号。
- **质量门控的异步更新。** 费洛蒙仅在质量检查通过后更新，将推理从学习中解耦。
- 在多智能体路由基准上实现**4.7 倍加速**。

质量门很重要：没有它，快速但错误的智能体会累积费洛蒙，系统会锁定在错误的路由上。

### 何时对 LLM 使用 PSO / ACO

**在以下情况下使用 PSO：**

- 搜索空间是连续的或映射到连续参数（提示嵌入、LoRA 权重、数字生成参数）。
- 适应度是廉价且自动的。
- 种群可以很小（10-30）。

**在以下情况下使用 ACO：**

- 你有路由或路径选择问题。
- 决策随时间强化（相同的任务类型会回来）。
- 你需要路由决策的可解释证据。

**在以下情况下不要使用两者：**

- 适应度需要人工审查（每次迭代成本过高）。
- 搜索空间是离散的且组合性的，PSO 无法覆盖（改用遗传算法）。
- 实时决策需要严格的延迟（PSO/ACO 相对于单次启发式收敛缓慢）。

### 为什么生物启发式仍然获胜

基于梯度的方法需要可微信号。LLM 输出和路由决策并非微不足道地可微。伪梯度方法（强化学习路由器、DPO 风格的提示调优器）有效，但需要昂贵的训练。

PSO 和 ACO 仅需要一个*评估器*函数。如果你能对候选输出或路由决策评分，你就可以在空间中优化。这使得适用性门槛低得多。

### 实际限制

- **种群预算。** N 个粒子 × T 次迭代 × 每次评估成本。对于每次调用约 0.02 美元的 LLM 评估，一个 20 粒子的 PSO 运行 50 次迭代成本约 20 美元。相应地做计划。
- **探索与利用。** 费洛蒙衰减率和 PSO 惯性权衡；衰减太快 → 忘记解决方案；太慢 → 卡在早期局部最优。
- **灾难性漂移。** 如果适应度景观转移（新数据分布），两种算法都可能收敛然后发散。监控最佳适应度稳定性。

## 构建

`code/main.py` 实现了：

- `LMPSO`——在数字提示参数（温度、top_k 权重）上的 PSO。每个粒子的"LLM 生成"被模拟为一个脚本化的适应度函数。运行该算法 30 次迭代，并显示 g_best 收敛。
- `AMRO_S`——ACO 风格的路由。3 个智能体，4 个任务类型，费洛蒙矩阵，100 个路由任务。打印随时间变化的（任务类型 → 智能体选择）分布以显示轨迹形成。
- 比较：相同任务流上的随机路由 vs ACO 路由。测量质量和延迟。

运行：

```
python3 code/main.py
```

预期输出：
- LMPSO：g_best 适应度在 30 次迭代中从随机改进到接近最优。
- AMRO-S：费洛蒙表在每个任务类型的正确智能体上稳定；ACO 路由在质量上比随机路由好约 30-40%，并且还减少了延迟（更少的重试）。

`outputs/skill-swarm-optimizer.md` 有助于在 PSO、ACO、遗传算法和基于梯度的优化器之间为 LLM / 智能体优化问题进行选择。

## 投产

- **从小处开始。** 10-20 个粒子，20-50 次迭代。仅当收敛曲线显示明确增益时才扩大规模。
- **记录每次迭代的费洛蒙或 g_best。** 没有轨迹调试群体优化器是痛苦的。
- **质量门控更新。** 特别是 for ACO 路由：快速且错误的智能体不得累积费洛蒙。
- **在分布转移时重置衰减。** 当你的评估分布变化时，过时的费洛蒙是陈旧的；重置或暂时加倍衰减率。
- **限制每次迭代的成本。** 发出每次迭代成本的指标。每次迭代成本 500 美元且获得 0.5% 收益的 PSO 是不可发布的。

## 练习

1. 运行 `code/main.py`。观察 LMPSO 收敛。改变种群大小 5、10、20、50。收敛时间在什么规模下饱和？
2. 实现一个"灾难性漂移"实验：在迭代 30 后，改变适应度函数。PSO 适应速度有多快？重置 `p_best` 有帮助吗？
3. 向 AMRO-S 添加质量门：仅在评估分数 > 0.7 的运行中沉积费洛蒙。与无门控版本相比，这如何改变收敛？
4. 阅读 LMPSO（arXiv:2504.09247）。将论文的"速度作为提示"映射回你的数字速度。模拟中丢失了什么，保留了什么？
5. 阅读 AMRO-S（arXiv:2603.12933）。实现解耦的"推理快速路径"，具有异步费洛蒙更新。在持续负载下，这如何改变系统延迟？

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|------|------------|----------|
| PSO | "粒子群优化" | Kennedy-Eberhart 1995。基于种群的无需梯度优化器。 |
| ACO | "蚁群优化" | Dorigo 1992。通过费洛蒙轨迹进行路径/路由优化。 |
| LMPSO | "带 LLM 生成的 PSO" | arXiv:2504.09247。速度是提示；LLM 产生候选者。 |
| 模型群体 | "在专家权重上的 PSO" | arXiv:2410.11163。在模型权重子空间上的无需梯度更新。 |
| AMRO-S | "用于智能体路由的 ACO" | arXiv:2603.12933。任务类型 × 智能体上的费洛蒙矩阵。 |
| p_best / g_best | "个人 / 全局最佳" | 迄今为止找到的每个粒子和种群范围的最佳解决方案。 |
| 费洛蒙 | "路由记忆" | 边上的强度；随时间衰减；在质量上沉积。 |
| 质量门控更新 | "仅从良好运行中学习" | 费洛蒙沉积以质量检查为条件。 |
| 灾难性漂移 | "分布转移" | 适应度景观变化；旧的 p_best 和费洛蒙变得陈旧。 |
