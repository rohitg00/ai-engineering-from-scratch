# 19 · 面向 LLM 的群体智能优化（PSO、ACO）

> 受生物启发的优化算法正在 LLM 领域强势回归。**LMPSO**（arXiv:2504.09247）采用「粒子群优化（PSO）」，其中每个粒子的速度是一个提示词，由 LLM 生成下一个候选解；它在结构化序列输出（数学表达式、程序）上效果良好。**Model Swarms**（arXiv:2410.11163）把每个 LLM 专家当作模型权重流形上的一个 PSO 粒子，在 9 个数据集上仅用 200 个实例就比 12 个基线取得了 **13.3% 的平均增益**。**SwarmPrompt**（ICAART 2025）将 PSO 与灰狼算法（Grey Wolf）混合用于提示词优化。**AMRO-S**（arXiv:2603.12933）是受「蚁群优化（ACO）」启发、用于多智能体 LLM 路由的信息素专家机制——实现 **4.7 倍加速**、可解释的路由证据，以及将推理与学习解耦的质量门控异步更新。本课在提示词参数空间上实现 PSO，在智能体路由上实现 ACO，量化分析这些经典算法为何契合 LLM 时代，以及何时并不契合。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置：** 阶段 16 · 09（并行群体网络），阶段 16 · 14（共识与 BFT）
**时长：** 约 75 分钟

## 问题

你有一个提示词，在任务评测上得分 62%。你想改进它。最朴素的做法是无梯度的手工调整，但这种方式扩展性很差。强化学习需要奖励信号和足够多的 rollout 来训练。而通过提示词做反向传播基本不可行——提示词是一个离散字符串，而非可微参数。

经典的受生物启发的优化算法——用于连续搜索空间的 PSO、用于路径选择的 ACO——恰恰就是为这种场景设计的：无梯度、基于种群、单次评估开销低。把它们与 LLM 配对来执行无梯度的搜索步骤，你就得到了一个出乎意料地实用的优化器。

同样的模式也适用于多智能体系统中的智能体*路由*。一条 ACO 式的信息素轨迹记录了哪个智能体在哪种任务类型上表现最好，让路由器去利用这条轨迹，并通过让信息素衰减，使得路由可以被重新发现。

## 概念

### PSO 复习（Kennedy 与 Eberhart 1995）

粒子群优化：在连续搜索空间中有一群粒子。每个粒子有位置 `x_i` 和速度 `v_i`。每次迭代：

```
v_i <- w * v_i + c1 * r1 * (p_best_i - x_i) + c2 * r2 * (g_best - x_i)
x_i <- x_i + v_i
evaluate fitness(x_i)
update p_best_i if improved
update g_best if global best
```

其中 `p_best` 是粒子自身的最优解，`g_best` 是整个群体的最优解，`w, c1, c2` 分别是惯性、认知和社会权重，`r1, r2` 是随机因子。

### 在 LLM 输出上的 PSO——LMPSO

arXiv:2504.09247 把 PSO 适配到 LLM 生成的结构化输出（数学表达式、程序）上。每个粒子是一个候选输出。速度是一段*提示词*，描述如何把当前输出朝着个体最优/全局最优修改。LLM 根据这段速度提示词生成新的输出。速度的「惯性」是一段类似「做出微小的增量改动」的提示词。

它在以下情况下效果良好：
- 输出是结构化的（可解析、可评估）。
- 适应度是自动计算的（测试运行、算术求值）。
- 种群规模小（约 10-30 个粒子），从而 LLM 总调用次数保持可控。

当适应度需要人工评审时，它效果不佳——每次迭代的成本会变得高得难以承受。

### Model Swarms

arXiv:2410.11163 把 PSO 从输出层挪到了*模型*层。每个「粒子」是一个专家 LLM（参数）。群体通过一次无梯度更新，把参数朝着集体最优解移动。论文报告：在 9 个数据集上比 12 个基线取得 13.3% 的平均增益，每次迭代仅用 200 个实例。

关键洞见是：LLM 专家模型本来就在一个共享的参数流形（适配器权重、LoRA 增量）中彼此邻近。在这个低维子空间上做 PSO 既便宜又有效。

### ACO 复习（Dorigo 1992）

蚁群优化：蚂蚁在一张图上穿行；每条路径都有一条信息素轨迹。蚂蚁的移动概率按信息素强度加权。完成任务的蚂蚁会按解的质量成比例地释放信息素。信息素随时间衰减。

### AMRO-S——用于智能体路由的 ACO

arXiv:2603.12933 把 ACO 用于多智能体路由。每种任务类型是一个「目的地」；每个智能体是一条可能的路由。信息素会强化那些产生优质输出的路由。主要贡献：

- **可解释的路由证据。** 信息素强度是一个人类可读的信号。
- **质量门控的异步更新。** 信息素仅在通过质量检查后才更新，从而将推理与学习解耦。
- 在多智能体路由基准上实现 **4.7 倍加速**。

质量门控很重要：没有它，又快又错的智能体会累积信息素，系统会锁定在糟糕的路由上。

### 何时为 LLM 使用 PSO / ACO

**在以下情况使用 PSO：**
- 搜索空间是连续的，或可映射到连续参数（提示词嵌入、LoRA 权重、数值型生成参数）。
- 适应度便宜且自动。
- 种群可以很小（10-30）。

**在以下情况使用 ACO：**
- 你面对一个路由或路径选择问题。
- 决策会随时间不断强化（相同的任务类型反复出现）。
- 你需要为路由决策提供可解释的证据。

**在以下情况两者都不要用：**
- 适应度需要人工评审（每次迭代太昂贵）。
- 搜索空间以 PSO 无法覆盖的方式呈离散和组合的特性（改用遗传算法）。
- 实时决策需要严格的低延迟（相对于单遍启发式方法，PSO/ACO 收敛较慢）。

### 为何受生物启发的方法依然胜出

基于梯度的方法需要可微的信号。LLM 的输出和路由决策都不是轻易可微的。伪梯度方法（强化学习训练的路由器、DPO 风格的提示词调优器）可行，但需要昂贵的训练。

PSO 和 ACO 只需要一个*评估器*函数。只要你能给一个候选输出或一个路由决策打分，就能在这个空间上做优化。这大大降低了适用门槛。

### 实际限制

- **种群预算。** N 个粒子 × T 次迭代 × 单次评估成本。对于约 $0.02/次调用的 LLM 评估，一个 20 粒子的 PSO 运行 50 次迭代成本约为 $20。请据此规划。
- **探索 vs 利用。** 信息素衰减率与 PSO 惯性之间存在权衡；衰减太快 → 遗忘解；太慢 → 困在早期局部最优。
- **灾难性漂移。** 如果适应度地形发生变化（新的数据分布），这两种算法都可能先收敛再发散。请监控最优适应度的稳定性。

## 动手构建

`code/main.py` 实现了：

- `LMPSO`——在数值型提示词参数（temperature、top_k 权重）上做 PSO。每个粒子的「LLM 生成」被模拟为一个脚本化的适应度函数。该算法运行 30 次迭代并展示 g_best 的收敛过程。
- `AMRO_S`——ACO 式路由。3 个智能体、4 种任务类型、信息素矩阵、100 个被路由的任务。打印（task_type → 智能体选择）随时间变化的分布，以展示轨迹的形成。
- 对比：在同一任务流上比较随机路由与 ACO 路由。测量质量和延迟。

运行：

```
python3 code/main.py
```

预期输出：
- LMPSO：g_best 适应度在 30 次迭代中从随机值改善到接近最优。
- AMRO-S：信息素表稳定在每种任务类型对应的正确智能体上；ACO 路由在质量上比随机路由高约 30-40%，并且也降低了延迟（更少的重试）。

## 上线交付

- **从小处着手。** 10-20 个粒子，20-50 次迭代。仅当收敛曲线显示明显增益时才扩大规模。
- **每次迭代记录信息素或 g_best。** 没有轨迹日志，调试群体优化器会非常痛苦。
- **质量门控更新。** 尤其对 ACO 路由而言：又快又错的智能体绝不能累积信息素。
- **分布漂移时重置衰减。** 当你的评测分布发生变化时，老旧的信息素已经失效；请重置或临时把衰减率翻倍。
- **限制每次迭代的成本。** 输出一个「每次迭代成本」指标。一个每次迭代花费 $500、却只带来 0.5% 增益的 PSO 是不可交付的。

## 练习

1. 运行 `code/main.py`。观察 LMPSO 的收敛过程。变化种群规模为 5、10、20、50。在哪个规模下，收敛所需时间趋于饱和？
2. 实现一个「灾难性漂移」实验：在第 30 次迭代后改变适应度函数。PSO 适应得有多快？重置 `p_best` 是否有帮助？
3. 给 AMRO-S 添加一个质量门控：仅在评测得分 > 0.7 的运行上才释放信息素。与无门控版本相比，这如何改变收敛？
4. 阅读 LMPSO（arXiv:2504.09247）。把论文中「速度即提示词」的概念对应回你的数值型速度。在模拟中丢失了什么，又保留了什么？
5. 阅读 AMRO-S（arXiv:2603.12933）。实现解耦的「推理快速路径」与异步信息素更新。在持续负载下，这如何改变系统延迟？

## 关键术语

| 术语 | 人们怎么说 | 它实际指什么 |
|------|----------------|------------------------|
| PSO | 「粒子群优化」 | Kennedy-Eberhart 1995。基于种群的无梯度优化器。 |
| ACO | 「蚁群优化」 | Dorigo 1992。通过信息素轨迹做路径/路由优化。 |
| LMPSO | 「带 LLM 生成的 PSO」 | arXiv:2504.09247。速度是一段提示词；LLM 产生候选解。 |
| Model Swarms | 「在专家权重上做 PSO」 | arXiv:2410.11163。在模型参数子空间上做无梯度更新。 |
| AMRO-S | 「用于智能体路由的 ACO」 | arXiv:2603.12933。在任务类型 × 智能体上的信息素矩阵。 |
| p_best / g_best | 「个体最优 / 全局最优」 | 至今发现的单粒子级与全群级的最优解。 |
| Pheromone（信息素） | 「路由记忆」 | 边上的强度；随时间衰减；按质量释放。 |
| Quality-gated update（质量门控更新） | 「只从优质运行中学习」 | 信息素释放以质量检查为条件。 |
| Catastrophic drift（灾难性漂移） | 「分布漂移」 | 适应度地形发生变化；旧的 p_best 和信息素变得失效。 |
