给定面向用户的任务（通常是分类或基于技能的路由），生成可映射到 OpenAI Swarm 或 OpenAI Agents SDK 的交接拓扑。

生成：

1. **代理名单。** 每个代理：名称、一句话目的、工具以及它可以交接给哪些其他代理。
2. **交接函数。** 每代理的工具签名。每个交接函数返回目标 Agent。
3. **上下文传输策略。** 在每个交接边上：完整历史、最后 N 条消息或摘要快照。证明。
4. **护栏。** 每代理的输入验证（允许什么提示触发到敏感专家的交接）、需要时的交接认证。
5. **循环检测。** 检测 ping-pong 的规则（例如，"A 交接给 B；B 交接回 A"连续发生超过一次）。
6. **回退行为。** 如果交接目标缺失（移除的代理、认证失败），哪个代理处理会话。
7. **会话/内存计划。** 是否使用 Agents SDK 会话、调用者管理的内存或根本没有内存。

硬性拒绝：

- 任何没有循环检测的交接设计。
- 将完整历史传递给具有不同工具权限的专家的交接函数（安全风险）。
- 假设 Swarm 的无状态行为但随后需要多轮内存的设计 —— 改用 Agents SDK 会话。

拒绝规则：

- 如果任务需要并行执行，拒绝 Swarm 并推荐监督者（Lesson 05）。
- 如果任务需要确定性审计/重放，拒绝并推荐 LangGraph 静态图。
- 如果任务是阶段的简单 DAG（研究 → 代码 → 审查），推荐 CrewAI Sequential。

输出：一页交接简报。以关于提示注入如何触发不需要的交接以及什么护栏阻止它的安全注释结束。
