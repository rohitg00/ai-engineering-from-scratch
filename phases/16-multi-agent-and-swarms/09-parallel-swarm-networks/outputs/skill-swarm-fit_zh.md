给定任务及其吞吐量/确定性要求，推荐群体或监督者并列出具体的队列和护栏选择。

生成：

1. **任务独立性检查。** 子任务是独立的还是相互依赖的？仅当独立性高时，群体才适合。
2. **持续时间分布。** 均匀 vs 可变。群体主要在可变持续时间工作负载上获胜。
3. **排序要求。** 严格、宽松或无。群体不保留顺序；监督者保留。
4. **可调试性需求。** 高（金融、医疗）→ 监督者。中等 → 带有每任务跟踪 ID 的群体。
5. **队列选择。** 演示用内存（`queue.Queue`）；生产用 Kafka / Redis Streams / NATS / 持久 DB 支持。
6. **工作者设计要求。** 必须是幂等的；必须发出每任务跟踪；必须处理背压。
7. **反饥饿计划。** 优先级老化、工作者专业化、有界队列。
8. **可观测性计划。** 每任务 ID、开始/结束事件、结果池模式。

硬性拒绝：

- 对具有硬排序要求的任务的群体推荐。
- 没有幂等工作者的群体。
- 生产中没有持久队列的群体。

拒绝规则：

- 如果任务每秒少于 10 个独立单元，拒绝群体并推荐监督者。在低吞吐量下，群体开销不合理。
- 如果可观测性需求需要单一连贯跟踪（审计、合规），拒绝群体并推荐 LangGraph 确定性图。

输出：一页架构简报。以适配裁决开头，以目标吞吐量的具体消息代理推荐结束。
