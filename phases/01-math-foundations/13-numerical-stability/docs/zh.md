# 数值稳定性

> 浮点数是抽象漏洞。它会在训练过程中给你带来麻烦，而你毫无察觉。

**类型:** 构建  
**语言:** Python  
**前置条件:** 阶段 1, 课程 01-04  
**时间:** ~120 分钟

## 学习目标

- 使用最大值相减技巧实现数值稳定的 softmax 和对数求和指数（log-sum-exp）
- 识别浮点计算中的上溢（overflow）、下溢（underflow）和灾难性抵消（catastrophic cancellation）
- 使用中心差分法验证解析梯度与数值梯度的一致性
- 解释为什么 bfloat16 比 float16 更适合训练，以及损失缩放（loss scaling）如何防止梯度下溢

## 问题

你的模型训练了三个小时，然后损失变成 NaN。你加了一个 print 语句。在第 9,000 步时 logits 正常。在第 9,001 步它们变成了 `inf`。到第 9,002 步每个梯度都是 `nan`，训练终止。

或者：你的模型训练完成，但准确率比论文声称的低 2%。你检查了一切。架构匹配。超参数匹配。数据匹配。问题在于论文使用了 float32，而你使用了 float16 却没有正确的缩放。三十二位累积的舍入误差悄悄地吞噬了你的准确率。

或者：你从头实现了交叉熵损失。它在小的 logits 上工作正常。当 logits 超过 100 时，它返回 `inf`。softmax 上溢了，因为 `exp(100)` 大于 float32 能表示的范围。每个机器学习框架都用两行技巧来处理这个问题。你不知道这个技巧存在。

数值稳定性不是一个理论问题。它是成功训练和静默失败之间的区别。你最终调试的每一个严重的机器学习 bug 都归结于浮点数。

## 概念

### IEEE 754：计算机如何存储实数

计算机按照 IEEE 754 标准将实数存储为浮点值。一个浮点数有三个部分：符号位、指数和尾数（有效数字）。

```
float32 布局（共 32 位）：
[1 符号] [8 指数] [23 尾数]

数值 = (-1)^符号 * 2^(指数 - 127) * 1.尾数
```

尾数决定了精度（有多少位有效数字）。指数决定了范围（数字可以有多大或多小）。

```
格式       比特   指数   尾数   十进制位数   范围（近似）
float64    64     11     52     ~15-16       ± 1.8e308
float32    32     8      23     ~7-8         ± 3.4e38
float16    16     5      10     ~3-4         ± 65,504
bfloat16   16     8      7      ~2-3         ± 3.4e38
```

float32 给你大约 7 位十进制数字的精度。这意味着它可以区分 1.0000001 和 1.0000002，但不能区分 1.00000001 和 1.00000002。7 位之后，一切都是舍入噪声。

float16 给你大约 3 位精度。它能表示的最大数是 65,504。对于机器学习来说，这个数字小得令人不安，因为 logits、梯度和激活函数经常超过这个值。

bfloat16 是 Google 解决 float16 范围问题的答案。它拥有与 float32 相同的 8 位指数（相同的范围，高达 3.4e38），但只有 7 位尾数（精度比 float16 还低）。对于训练神经网络，范围比精度更重要，所以 bfloat16 通常胜出。

### 为什么 0.1 + 0.2 != 0.3

数字 0.1 不能在二进制浮点数中精确表示。在二进制中，它是一个循环小数：

```
0.1 的二进制 = 0.0001100110011001100110011...（无限循环）
```

float32 将其截断为 23 位尾数。存储的值大约是 0.100000001490116。类似地，0.2 存储为大约 0.200000002980232。它们的和是 0.300000004470348，而不是 0.3。

```
在 Python 中：
>>> 0.1 + 0.2
0.30000000000000004

>>> 0.1 + 0.2 == 0.3
假
```

这对机器学习很重要，因为：

1. 像 `if loss < threshold` 这样的损失比较可能给出错误的结果
2. 累积许多小值（成千上万步的梯度更新）会偏离真实的和
3. 如果你用 `==` 比较浮点数，校验和和可重复性测试会失败

解决方法：永远不要用 `==` 比较浮点数。使用 `abs(a - b) < epsilon` 或 `math.isclose()`。

### 灾难性抵消

当你减去两个几乎相等的浮点数时，有效数字会抵消，你只剩下被提升为首位的舍入噪声。

```
a = 1.0000001    （在 float32 中存储为 1.00000011920929）
b = 1.0000000    （在 float32 中存储为 1.00000000000000）

真实差值： 0.0000001
计算值：   0.00000011920929

相对误差：19.2%
```

一次减法就产生了 19% 的相对误差。在机器学习中，这发生在你：

- 计算具有大平均值的数据的方差时：当 `E[x]` 很大时，`E[x^2] - E[x]^2`
- 减去几乎相等的对数概率时
- 使用过小的 epsilon 计算有限差分梯度时

解决方法：重新排列公式，避免减去大的、几乎相等的数字。对于方差，使用 Welford 算法或先对数据中心化。对于对数概率，始终在对数空间中进行计算。

### 上溢和下溢

当结果太大无法表示时发生上溢。当结果太小（比最小可表示正数更接近零）时发生下溢。

```
float32 边界：
  最大值：       3.4028235e+38
  最小正数（规格化）：1.175e-38
  最小正数（非规格化）：1.401e-45
  上溢：      任何 > 3.4e38 的值变成 inf
  下溢：      任何 < 1.4e-45 的值变成 0.0
```

`exp()` 函数是机器学习中上溢的主要来源：

```
exp(88.7)  = 3.40e+38   （勉强适合 float32）
exp(89.0)  = inf         （上溢）
exp(-87.3) = 1.18e-38   （勉强高于下溢）
exp(-104)  = 0.0         （下溢到零）
```

`log()` 函数会遇到相反方向的问题：

```
log(0.0)   = -inf
log(-1.0)  = nan
log(1e-45) = -103.3      （没问题）
log(1e-46) = -inf        （输入下溢到 0，然后 log(0) = -inf）
```

在机器学习中，`exp()` 出现在 softmax、sigmoid 和概率计算中。`log()` 出现在交叉熵、对数似然和 KL 散度中。如果没有正确的技巧，`log(exp(x))` 的组合是一个雷区。

### 对数求和指数技巧

直接计算 `log(sum(exp(x_i)))` 在数值上很危险。如果任何 `x_i` 很大，`exp(x_i)` 会溢出。如果所有 `x_i` 都非常负，每个 `exp(x_i)` 都会下溢到零，而 `log(0)` 是 `-inf`。

技巧：在取指数之前减去最大值。

```
log(sum(exp(x_i))) = max(x) + log(sum(exp(x_i - max(x))))
```

为什么这样有效：减去 `max(x)` 后，最大的指数是 `exp(0) = 1`。不可能发生上溢。和中至少有一项是 1，因此和至少为 1，且 `log(1) = 0`。不可能下溢到 `-inf`。

证明：

```
log(sum(exp(x_i)))
= log(sum(exp(x_i - c + c)))                    （加上并减去 c）
= log(sum(exp(x_i - c) * exp(c)))               （exp(a+b) = exp(a)*exp(b)）
= log(exp(c) * sum(exp(x_i - c)))               （提出因子 exp(c)）
= c + log(sum(exp(x_i - c)))                    （log(a*b) = log(a) + log(b)）
```

设 `c = max(x)`，则上溢被消除。

这个技巧出现在机器学习的各个地方：
- Softmax 归一化
- 交叉熵损失计算
- 序列模型中的对数概率求和
- 高斯混合模型
- 变分推理

### 为什么 Softmax 需要最大值相减技巧

Softmax 将 logits 转换为概率：

```
softmax(x_i) = exp(x_i) / sum(exp(x_j))
```

没有这个技巧，logits 为 [100, 101, 102] 会导致上溢：

```
exp(100) = 2.69e43
exp(101) = 7.31e43
exp(102) = 1.99e44
和       = 2.99e44

这些会溢出 float32 吗（最大 ~3.4e38）？不，2.69e43 < 3.4e38？实际上：
exp(88.7) 已经到了 float32 的限制。
exp(100) 在 float32 中是 inf。
```

使用这个技巧，减去 max(x) = 102：

```
exp(100 - 102) = exp(-2) = 0.135
exp(101 - 102) = exp(-1) = 0.368
exp(102 - 102) = exp(0)  = 1.000
和 = 1.503

softmax = [0.090, 0.245, 0.665]
```

概率完全相同。计算是安全的。这不是一个优化。这是正确性的要求。

### NaN 和 Inf：检测与预防

`nan`（非数字）和 `inf`（无穷大）在计算中病毒式地传播。梯度更新中的一个 `nan` 会使权重变成 `nan`，进而使每个后续输出变成 `nan`。训练在一两步之内就死亡了。

`inf` 出现的方式：
- 对大的正数取 `exp()`
- 除以零：`1.0 / 0.0`
- 累积中的 `float32` 上溢

`nan` 出现的方式：
- `0.0 / 0.0`
- `inf - inf`
- `inf * 0`
- 对负数取 `sqrt()`
- 对负数取 `log()`
- 任何涉及已有 `nan` 的算术运算

检测：

```python
import math

math.isnan(x)       # 如果 x 是 nan 则返回 True
math.isinf(x)       # 如果 x 是 +inf 或 -inf 则返回 True
math.isfinite(x)    # 如果 x 既不是 nan 也不是 inf 则返回 True
```

预防策略：

1. 对 `exp()` 的输入进行裁剪：`exp(clamp(x, -80, 80))`
2. 在分母中添加 epsilon：`x / (y + 1e-8)`
3. 在 `log()` 内部添加 epsilon：`log(x + 1e-8)`
4. 使用稳定的实现（log-sum-exp，稳定 softmax）
5. 梯度裁剪以防止权重爆炸
6. 在调试期间，每次前向传播后检查是否有 `nan`/`inf`

### 数值梯度检查

解析梯度（来自反向传播）可能有 bug。数值梯度检查通过使用有限差分计算梯度来验证它们。

中心差分公式：

```
df/dx ~= (f(x + h) - f(x - h)) / (2h)
```

这是 O(h^2) 精度，远好于仅 O(h) 精度的前向差分 `(f(x+h) - f(x)) / h`。

选择 h：太大则近似不准确。太小则灾难性抵消会破坏结果。典型的 h 是 `1e-5` 到 `1e-7`。

检查：计算解析梯度和数值梯度之间的相对差异。

```
relative_error = |grad_analytical - grad_numerical| / max(|grad_analytical|, |grad_numerical|, 1e-8)
```

经验法则：
- relative_error < 1e-7：完美，梯度正确
- relative_error < 1e-5：可接受，可能正确
- relative_error > 1e-3：有问题
- relative_error > 1：梯度完全错误

在实现新层或损失函数时，一定要检查梯度。PyTorch 提供了 `torch.autograd.gradcheck()` 用于此目的。

### 混合精度训练

现代 GPU 具有专门的硬件（Tensor Cores），可以以比 float32 快 2-8 倍的速度计算 float16 矩阵乘法。混合精度训练利用了这一点：

```
1. 保留 float32 主副本的权重
2. 前向传播使用 float16（快速）
3. 用 float32 计算损失（防止上溢）
4. 反向传播使用 float16（快速）
5. 将梯度缩放到 float32
6. 更新 float32 主权重
```

纯 float16 训练的问题：梯度通常非常小（1e-8 或更小）。float16 对任何小于约 6e-8 的值都会下溢到零。你的模型停止学习，因为所有梯度更新都是零。

解决方法是损失缩放：

```
1. 将损失乘以一个大的缩放因子（例如 1024）
2. 反向传播计算（损失 * 1024）的梯度
3. 所有梯度都放大了 1024 倍（推至超过 float16 下溢阈值）
4. 在更新权重之前将梯度除以 1024
5. 净效果：相同的更新，但没有下溢
```

动态损失缩放自动调整缩放因子。从一个大的值（65536）开始。如果梯度上溢到 `inf`，则将其减半。如果经过 N 步没有上溢，则将其加倍。

### bfloat16 vs float16：为什么 bfloat16 在训练中胜出

```
float16:   [1 符号] [5 指数]  [10 尾数]
bfloat16:  [1 符号] [8 指数]  [7 尾数]
```

float16 精度更高（10 位尾数 vs 7 位），但范围有限（最大约 65,504）。bfloat16 精度较低，但范围与 float32 相同（最大约 3.4e38）。

对于训练神经网络：

- 在训练峰值期间，激活函数和 logits 经常超过 65,504。float16 会溢出；bfloat16 可以处理。
- 使用 float16 时需要损失缩放，但使用 bfloat16 时通常不需要，因为其范围覆盖了梯度大小范围。
- bfloat16 是对 float32 的简单截断：丢弃尾数的低 16 位。转换是微不足道的，且指数部分无损。

float16 更适用于推理，因为此时值有界且精度更重要。bfloat16 更适用于训练，因为范围更重要。这就是 TPU 和现代 NVIDIA GPU（A100、H100）原生支持 bfloat16 的原因。

### 梯度裁剪

梯度爆炸发生在梯度通过许多层呈指数增长时（常见于 RNN、深层网络和 Transformer）。一个大的梯度可以在一步之内破坏所有权重。

两种裁剪类型：

**按值裁剪：** 独立地裁剪每个梯度元素。

```
grad = clamp(grad, -max_val, max_val)
```

简单，但可能改变梯度向量的方向。

**按范数裁剪：** 缩放整个梯度向量，使其范数不超过阈值。

```
如果 ||grad|| > max_norm:
    grad = grad * (max_norm / ||grad||)
```

保持梯度的方向。这就是 `torch.nn.utils.clip_grad_norm_()` 所做的。这是标准选择。

典型值：Transformer 使用 `max_norm=1.0`，强化学习使用 `max_norm=0.5`，更简单的网络使用 `max_norm=5.0`。

梯度裁剪不是黑客手段。它是一种安全机制。没有它，一个异常的批量就可以产生足够大的梯度，毁掉数周的训练。

### 归一化层作为数值稳定器

批归一化（Batch Normalization）、层归一化（Layer Normalization）和 RMS 归一化通常被介绍为有助于训练收敛的正则化器。它们也是数值稳定器。

没有归一化，激活函数可以通过层指数级增长或缩小：

```
第 1 层：值在 [0, 1]
第 5 层：值在 [0, 100]
第 10 层：值在 [0, 10,000]
第 50 层：值在 [0, inf]
```

归一化在每一层重新居中和缩放激活函数：

```
LayerNorm(x) = (x - mean(x)) / (std(x) + epsilon) * gamma + beta
```

`epsilon`（通常为 1e-5）防止所有激活相同时除以零。学习的参数 `gamma` 和 `beta` 让网络恢复它需要的任何尺度。

这使值在整个网络中保持在数值安全的范围内，防止前向传播中的上溢和反向传播中的梯度爆炸。
