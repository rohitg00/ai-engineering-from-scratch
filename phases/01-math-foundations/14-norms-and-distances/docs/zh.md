# 范数与距离（Norms and Distances）

> 你的距离函数定义了什么叫“相似”。选错了，下游的一切都会出问题。

**类型：** 构建
**语言：** Python
**前置条件：** 阶段1，课程01（线性代数直觉）、02（向量、矩阵与运算）
**时长：** ~90分钟

## 学习目标

- 从零实现 L1、L2、余弦、马氏、杰卡德和编辑距离函数
- 为给定机器学习任务选择适当的距离度量，并解释为什么其他方案会失败
- 将 L1 和 L2 范数与 LASSO 和岭正则化及其几何约束区域联系起来
- 演示同一数据集在不同度量下如何产生不同最近邻

## 问题

你有两个向量。它们可能是词嵌入（word embeddings），可能是用户画像，也可能是像素数组。你需要知道：它们有多接近？

答案完全取决于你选择的距离函数。两个数据点在一个度量下可能是最近邻，在另一个度量下可能相距甚远。你的 KNN 分类器、推荐引擎、向量数据库、聚类算法、损失函数——它们都依赖这个选择。选错了，你的模型就会朝着错误的方向优化。

没有全局通用的最佳距离。L2 适合空间数据，余弦相似度（Cosine Similarity）主导 NLP，杰卡德（Jaccard）处理集合，编辑距离（Edit Distance）处理字符串，马氏距离（Mahalanobis Distance）考虑相关性，沃瑟斯坦距离（Wasserstein Distance）移动概率质量。每一种都编码了关于“相似”的不同假设。

本课程从零构建每一种主要的距离函数，告诉你每种方法在何时是合适的工具，并演示相同的数据如何根据你使用的度量产生完全不同的最近邻。

## 概念

### 范数：衡量向量的大小

范数（Norm）衡量一个向量的“大小”。两个向量之间的每个距离函数都可以写成它们差的范数：d(a, b) = ||a - b||。因此理解范数就是理解距离。

### L1 范数（曼哈顿距离）

L1 范数对各分量的绝对值求和。

```
||x||_1 = |x_1| + |x_2| + ... + |x_n|
```

之所以叫曼哈顿距离（Manhattan distance），是因为它测量你在一个只能沿坐标轴移动的城市网格中走的距离。没有对角线。

```
点 A = (1, 1)
点 B = (4, 5)

L1 距离 = |4-1| + |5-1| = 3 + 4 = 7

在网格上，你向东走 3 个街区，向北走 4 个街区。
```

何时使用 L1：
- 高维稀疏数据（文本特征、独热编码）
- 当你希望对异常值稳健（单个巨大差异不会主导）
- 特征选择问题（L1 正则化（Regularization）促进稀疏性）

与 L1 正则化（Lasso）的联系：在损失函数中加入 ||w||_1 会惩罚权重的绝对值之和。这会将小权重推至恰好为零，实现自动特征选择。L1 惩罚在权重空间中创建菱形约束区域，菱形的角落在坐标轴上，使得一些权重为零。

与损失函数的联系：平均绝对误差（MAE）是预测值与目标之间 L1 距离的平均值。它对所有误差线性惩罚，因此与 MSE 相比对异常值更稳健。

### L2 范数（欧氏距离）

L2 范数是直线距离。各分量平方和的平方根。

```
||x||_2 = sqrt(x_1^2 + x_2^2 + ... + x_n^2)
```

这是你在几何课上学到的距离。n 维空间中的勾股定理。

```
点 A = (1, 1)
点 B = (4, 5)

L2 距离 = sqrt((4-1)^2 + (5-1)^2) = sqrt(9 + 16) = sqrt(25) = 5.0

直线，对角线穿过网格。
```

何时使用 L2：
- 低到中维度的连续数据
- 当特征尺度可比时
- 物理距离（空间数据、传感器读数）
- 像素级图像相似度

与 L2 正则化（Ridge）的联系：在损失函数中加入 ||w||_2^2 会惩罚大权重。与 L1 不同，它不会将权重推至零。它会按比例将所有权重向零收缩。L2 惩罚创建圆形约束区域，因此在坐标轴上没有角点。权重变小但很少恰好为零。

与损失函数的联系：均方误差（MSE）是 L2 距离平方的平均值。平方对大误差的惩罚比小误差更重。

```
MAE (L1 损失):  |y - y_hat|         线性惩罚。对异常值鲁棒。
MSE (L2 损失):  (y - y_hat)^2       二次惩罚。对异常值敏感。
```

### Lp 范数：一般族

L1 和 L2 是 Lp 范数的特例：

```
||x||_p = (|x_1|^p + |x_2|^p + ... + |x_n|^p)^(1/p)
```

不同的 p 值会产生不同形状的“单位球”（所有离原点距离为 1 的点的集合）：

```
p=1:    菱形形状    （角在轴上）
p=2:    圆/球体    （通常的圆球）
p=3:    超椭圆    （圆角方形）
p=inf:  正方形/超立方体（边与轴平行）
```

### L-无穷范数（切比雪夫距离）

当 p 趋向无穷时，Lp 范数收敛到最大绝对值分量。

```
||x||_inf = max(|x_1|, |x_2|, ..., |x_n|)
```

两点之间的距离由它们相差最大的单一维度决定。所有其他维度被忽略。

```
点 A = (1, 1)
点 B = (4, 5)

L-inf 距离 = max(|4-1|, |5-1|) = max(3, 4) = 4
```

何时使用 L-无穷：
- 当任何一个维度的最坏情况偏差很重要时
- 棋盘（国际象棋中的王按 L-无穷移动：任意方向一步代价为 1）
- 制造公差（所有维度必须在规格内）

### 余弦相似度和余弦距离

余弦相似度（Cosine Similarity）测量两个向量之间的角度，忽略它们的大小。

```
cos_sim(a, b) = (a . b) / (||a||_2 * ||b||_2)
```

取值范围从 -1（相反方向）到 +1（相同方向）。垂直的向量余弦相似度为 0。

余弦距离（Cosine Distance）将其转换为距离：cosine_distance = 1 - cosine_similarity。取值范围从 0（方向相同）到 2（方向相反）。

```
a = (1, 0)    b = (1, 1)

cos_sim = (1*1 + 0*1) / (1 * sqrt(2)) = 1/sqrt(2) = 0.707
cos_dist = 1 - 0.707 = 0.293
```

为什么余弦在 NLP 和嵌入中占主导地位：在文本中，文档长度不应影响相似度。一篇关于猫的文档，如果长度是另一篇关于猫的文档的两倍，仍然应该是“相似”的。余弦相似度忽略大小（长度），只关心方向。具有相同词分布但长度不同的两篇文档指向同一方向，余弦相似度为 1.0。

何时使用余弦相似度：
- 文本相似度（TF-IDF 向量、词嵌入、句子嵌入）
- 任何大小是噪声而方向是信号的领域
- 推荐系统（用户偏好向量）
- 嵌入搜索（向量数据库几乎总是使用余弦或点积（Dot Product））

### 点积相似度 vs 余弦相似度

两个向量的点积是：

```
a . b = a_1*b_1 + a_2*b_2 + ... + a_n*b_n
      = ||a|| * ||b|| * cos(角)
```

余弦相似度是用两个大小归一化后的点积。当两个向量都已单位归一化（大小 = 1）时，点积和余弦相同。

```
如果 ||a|| = 1 且 ||b|| = 1：
    a . b = cos(a 与 b 之间的夹角)
```

它们何时不同：点积包含大小信息。大小更大的向量获得更高的点积分数。这在某些检索系统中很重要，其中你希望“热门”项目排名更高。大小充当隐式的质量或重要性信号。

```
a = (3, 0)    b = (1, 0)    c = (0, 1)

dot(a, b) = 3     dot(a, c) = 0
cos(a, b) = 1.0   cos(a, c) = 0.0

两者在方向上一致，但点积还反映大小。
```

实际中：
- 需要纯方向相似度时使用余弦相似度
- 当大小携带有意义的信息时使用点积
- 许多向量数据库（Pinecone、Weaviate、Qdrant）让你在它们之间选择
- 如果你的嵌入是 L2 归一化的，那么选择无关紧要

### 马氏距离（Mahalanobis Distance）

欧氏距离平等对待所有维度。但如果特征相关或具有不同尺度，L2 会给出误导性结果。

马氏距离（Mahalanobis Distance）考虑了数据的协方差结构。

```
d_M(x, y) = sqrt((x - y)^T * S^(-1) * (x - y))
```

其中 S 是数据的协方差矩阵。

直观解释：马氏距离首先对数据进行去相关和归一化（白化（Whitening）），然后在该变换空间中计算 L2 距离。如果 S 是单位矩阵（不相关、单位方差的特征），马氏距离退化为欧氏距离。

```
示例：身高和体重是相关的。
身高 6'2"、体重 180 磅的人并不异常。
身高 5'0"、体重 180 磅的人则是异常。

欧氏距离可能会说它们离均值一样远。
马氏距离正确地将第二个识别为离群点，因为它考虑了身高-体重的相关性。
```

何时使用马氏距离：
- 离群点检测（离均值马氏距离大的点是离群点）
- 特征具有不同尺度和相关性时的分类
- 当你有足够的数据来估计可靠的协方差矩阵时
- 制造中的质量控制（多变量过程监控）

### 杰卡德相似度（Jaccard Similarity）（用于集合）

杰卡德相似度（Jaccard Similarity）测量两个集合的重叠程度。

```
J(A, B) = |A ∩ B| / |A ∪ B|
```

取值范围从 0（无重叠）到 1（相同集合）。杰卡德距离（Jaccard distance）= 1 - 杰卡德相似度。

```
A = {cat, dog, fish}
B = {cat, bird, fish, snake}

交集 = {cat, fish}              大小 = 2
并集 = {cat, dog, fish, bird, snake}  大小 = 5

杰卡德相似度 = 2/5 = 0.4
杰卡德距离 = 0.6
```

何时使用杰卡德：
- 比较标签、类别或特征的集合
- 基于词存在（而非频率）的文档相似度
- 近重复检测（MinHash 近似杰卡德）
- 比较二值特征向量（存在/不存在数据）
- 评估分割模型（交并比 = 杰卡德）

### 编辑距离（Levenshtein 距离）

编辑距离（Edit Distance）计算将一个字符串转换为另一个字符串所需的最小单字符操作数。操作包括：插入、删除或替换。

```
"kitten" -> "sitting"

kitten -> sitten  (替换 k -> s)
sitten -> sittin  (替换 e -> i)
sittin -> sitting (插入 g)

编辑距离 = 3
```

使用动态规划计算。填充一个矩阵，其中条目 (i, j) 是字符串 A 的前 i 个字符和字符串 B 的前 j 个字符之间的编辑距离。

```
        ""  s  i  t  t  i  n  g
    ""   0  1  2  3  4  5  6  7
    k    1  1  2  3  4  5  6  7
    i    2  2  1  2  3  4  5  6
    t    3  3  2  1  2  3  4  5
    t    4  4  3  2  1  2  3  4
    e    5  5  4  3  2  2  3  4
    n    6  6  5  4  3  3  2  3
```

何时使用编辑距离：
- 拼写检查和纠正
- DNA 序列比对（带加权操作）
- 模糊字符串匹配
- 杂乱文本数据的去重

### KL 散度（不是距离，但被当作距离使用）

KL 散度（KL Divergence）衡量一个概率分布与另一个的差异。将在课程 09 中介绍，但它属于这里的讨论，因为人们将其当作“距离”使用，尽管它不是。

```
D_KL(P || Q) = sum(p(x) * log(p(x) / q(x)))
```

关键性质：KL 散度不是对称的。

```
D_KL(P || Q) != D_KL(Q || P)
```

这意味着它不满足距离度量的基本要求。它也不满足三角不等式。它是一种散度，不是距离。

前向 KL（D_KL(P || Q)）是“均值寻求”：Q 试图覆盖 P 的所有模式。
反向 KL（D_KL(Q || P)）是“模式寻求”：Q 聚焦于 P 的单一模式。

当你看到 KL 散度时：
- VAE（ELBO 中的 KL 项将潜在分布推向先验）
- 知识蒸馏（学生试图匹配教师的分布）
- RLHF（KL 惩罚使微调后的模型接近基础模型）
- 策略梯度方法（约束策略更新）

### 沃瑟斯坦距离（Earth Mover's Distance）

沃瑟斯坦距离（Wasserstein Distance）测量将一个概率分布变换为另一个所需的最小“功”。可以想象为：如果一个分布是一堆土，另一个是一个坑，你需要移动多少土以及移动多远？

```
W(P, Q) = inf over all transport plans gamma of E[d(x, y)]
```

对于一维分布，它简化为累积分布函数绝对差值的积分：

```
W_1(P, Q) = integral |CDF_P(x) - CDF_Q(x)| dx
```

为什么沃瑟斯坦距离重要：
- 它是真正的度量（对称、满足三角不等式）
- 即使分布没有重叠，它也提供梯度（KL 散度趋于无穷）
- 这一性质使其成为 Wasserstein GAN（WGAN）的核心，解决了原始 GAN 的训练不稳定性

```
无重叠的分布：

P: [1, 0, 0, 0, 0]    Q: [0, 0, 0, 0, 1]

KL 散度：无穷大（对零取对数）
Wasserstein：4（移动所有质量 4 个箱子）

Wasserstein 提供了有意义的梯度。KL 则没有。
```

何时使用沃瑟斯坦距离：
- GAN 训练（WGAN、WGAN-GP）
- 比较可能不重叠的分布
- 最优传输问题
- 图像检索（比较颜色直方图）

### 为什么不同任务需要不同的距离

| 任务 | 最佳距离 | 原因 |
|------|---------|------|
| 文本相似度 | 余弦 | 大小是噪声，方向是意义 |
| 图像像素比较 | L2 | 空间关系重要，特征尺度可比 |
| 稀疏高维特征 | L1 | 稳健，不会放大罕见的巨大差异 |
| 集合重叠（标签、类别） | 杰卡德 | 数据天然是集合值，而非向量值 |
| 字符串匹配 | 编辑距离 | 操作对应人类编辑直觉 |
| 离群点检测 | 马氏距离 | 考虑特征相关性和尺度 |
| 比较分布 | KL 散度 | 衡量使用 Q 代替 P 损失的信息 |
| GAN 训练 | 沃瑟斯坦 | 即使分布不重叠也提供梯度 |
| 嵌入（向量数据库） | 余弦或点积 | 嵌入训练将意义编码在方向中 |
| 推荐 | 点积 | 大小可以编码流行度或置信度 |
| DNA 序列 | 加权编辑距离 | 替换代价因核苷酸对而异 |
| 制造品控 | L-无穷 | 任何维度的最坏情况偏差重要 |

### 与损失函数的联系

损失函数是应用于预测值与目标的距离函数。

```
损失函数       使用的距离          行为
MSE            L2 平方             严重惩罚大误差
MAE            L1                  平等惩罚所有误差
Huber 损失    大误差用 L1，       两者优点：对异常值鲁棒，
              小误差用 L2         在零附近有平滑梯度
交叉熵          KL 散度             衡量分布不匹配
合页损失         max(0, margin - d)  仅惩罚低于边界的点
