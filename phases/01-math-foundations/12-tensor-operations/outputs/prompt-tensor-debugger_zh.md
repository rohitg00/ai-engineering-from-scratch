我的深度学习代码中有一个张量形状错误。帮我修复它。

**错误消息：** [在此粘贴错误]

**我的张量形状：**
- [名称]: [形状]
- [名称]: [形状]

**我尝试执行的操作：** [描述它]

---

调试时，请遵循这个确切的过程：

**步骤 1：识别操作类型。**
什么操作产生了错误？将其映射到以下之一：
- 矩阵乘法 / 线性层（内部维度必须匹配）
- 广播（从右对齐，每个维度必须相等或为 1）
- 连接（除了连接维度外，所有维度匹配）
- 卷积（期望特定秩和通道位置）
- 重塑（必须保留总元素数）

**步骤 2：写出形状约定。**
对于识别的操作，显式写出期望的形状：
```
matmul(A, B): A 是 (..., m, k), B 是 (..., k, n) -> (..., m, n)
broadcast(A, B): 右对齐，每对必须（相等）或（一个是 1）
cat([A, B], dim=d): 除了维度 d 外，所有维度匹配
Linear(in_f, out_f): 输入最后一个维度必须等于 in_f
Conv2d(in_c, out_c, k): 输入必须是 (B, in_c, H, W)
```

**步骤 3：找到不匹配。**
将实际形状与约定进行比较。识别违反规则的确切维度。

**步骤 4：选择最小修复。**
从这个表中选择：

| 症状 | 修复 |
|---|---|
| 缺少批次维度 | `.unsqueeze(0)` |
| 缺少通道维度 | `.unsqueeze(1)` |
| 额外的大小为 1 的维度 | `.squeeze(dim)` |
| 矩阵乘法内部维度错误 | `.transpose(-1, -2)` 或检查权重形状 |
| 需要从 NHWC 转为 NCHW | `.permute(0, 3, 1, 2)` |
| 需要从 NCHW 转为 NHWC | `.permute(0, 2, 3, 1)` |
| 为线性层展平空间维度 | `.flatten(1)` 或 `.reshape(B, -1)` |
| 拆分多头：(B,T,D) 到 (B,H,T,D/H) | `.reshape(B, T, H, D//H).transpose(1, 2)` |
| 合并多头：(B,H,T,D/H) 到 (B,T,D) | `.transpose(1, 2).reshape(B, T, H*(D//H))` |
| 带有 .view() 的非连续张量 | `.contiguous().view(...)` 或使用 `.reshape(...)` |

**步骤 5：验证修复。**
在每个步骤显示结果形状。确认在任何重塑中保留了总元素数。确认操作的形状约定现在满足。

**步骤 6：检查静默 bug。**
即使形状匹配，也要验证：
- 广播沿着预期的轴发生（不是偶然的）
- 归约正在对正确的维度求和
- 批次维度（维度 0）在整个前向传递中存活
- 当维度顺序重要时，使用转置 + 重塑（不仅仅是重塑）

将您的回答格式化为：
```
OPERATION: [什么操作失败了]
EXPECTED: [形状约定]
ACTUAL: [提供了什么形状]
MISMATCH: [哪个维度，为什么]
FIX: [确切代码]
RESULT: [修复后的形状]
```
