# 分类基线指南

在尝试复杂模型前，先用逻辑回归建立基线。它几秒就能训练好，输出概率，且完全可解释。 surprisingly 多的现实世界问题根本不需要更花哨的方法。

## 决策清单

1. 决策边界可能是线性的吗？
   - 是：逻辑回归可能就足够了
   - 否：你仍然需要它作为基线来衡量改进

2. 你有多少特征？
   - 少于 50：标准逻辑回归即可
   - 50 到 10,000：添加 L2 正则化（Ridge）
   - 超过 10,000（如 TF-IDF 文本特征）：使用 L1 正则化（Lasso）或 LinearSVC

3. 数据集不平衡吗？
   - 低于 5:1 比例：可能不需要调整
   - 5:1 到 50:1：在 sklearn 中使用 `class_weight="balanced"`
   - 超过 50:1：结合类别权重与适当指标（精确率、召回率或 F1）

4. 特征尺度不同吗？
   - 逻辑回归前始终标准化。它使用基于梯度的优化，未缩放的特征会减慢收敛或扭曲决策边界。

5. 有缺失值吗？
   - 拟合前填充。逻辑回归无法处理 NaN。
   - 数值列用中位数填充，类别列用众数填充。

## 逻辑回归足够好的情况

- 特征关系大多线性的二分类
- 你需要概率输出（不仅是类别标签）
- 需要可解释性（系数指示特征重要性的方向和标准化后的相对大小）
- 训练数据小（数百到低数千样本）
- 你需要快速模型进行实时服务（推理时单次点积）
- 监管或合规要求需要可解释性
