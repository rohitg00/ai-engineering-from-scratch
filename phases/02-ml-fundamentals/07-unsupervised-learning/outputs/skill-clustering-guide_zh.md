# 聚类算法选择指南

聚类没有单一的"最佳"算法。正确的选择取决于聚类形状、是否知道聚类数量、数据中有多少噪声以及数据集有多大。

## 决策清单

1. 你知道聚类数量吗？
   - 是：K-Means 或 GMM
   - 否：DBSCAN（自动发现聚类），或层次聚类（在不同层级切割树状图）

2. 聚类是什么形状？
   - 大致球形（ blob 状）：K-Means
   - 不同大小的椭圆形：GMM
   - 任意形状（新月形、环形、链状）：DBSCAN
   - 嵌套或层次：层次聚类

3. 数据包含噪声或异常值吗？
   - 是：DBSCAN（显式标记噪声点）或 GMM（低概率点为异常值）
   - 否：K-Means 即可

4. 你需要软分配（概率）吗？
   - 是：GMM 给出每个聚类的 P(cluster | data point)
   - 否：K-Means 或 DBSCAN 给出硬分配

5. 数据集有多大？
   - 少于 10,000：任何算法都适用
   - 10,000 到 1,000,000：K-Means（快），Mini-Batch K-Means（更快）
   - 超过 1,000,000：Mini-Batch K-Means 或 BIRCH。层次聚类太慢。

## 何时使用每种方法

**K-Means**：默认起点。快（O(n * k * 迭代次数)），简单，对许多问题足够好。使用肘部法或轮廓分数选择 K。限制：假设球形聚类，对初始化敏感（使用 K-Means++ 或多次运行），不能很好地处理变化的聚类大小。

**DBSCAN**：最适合发现任意形状的聚类并自动检测异常值。两个参数：eps（邻域半径）和 min_samples（最小密度）。不需要指定 K。限制：当聚类密度差异很大时困难，调整 eps 可能棘手。使用 k-距离图估计 eps：计算每个点到其第 k 近邻的距离，排序，寻找肘部。

**层次（凝聚）**：构建合并树。当你想探索多粒度聚类结构时有用（在不同高度切割树状图）。Ward 连接最适合紧凑聚类。单连接找到细长聚类但对噪声敏感。限制：O(n²) 内存和 O(n³) 时间，所以对大数据集不实用。

**GMM（高斯混合模型）**：软聚类，概率分配。将每个聚类建模为具有自己均值和协方差的高斯分布。当聚类是椭圆形或重叠时比 K-Means 更好。使用 BIC（贝叶斯信息准则）选择组件数量。限制：假设高斯分布，在非凸形状上可能失败，对初始化敏感。

## 评估聚类质量（无标签）

| 指标 | 测量内容 | 范围 | 何时使用 |
|------|----------|------|----------|
| 轮廓分数 | 内聚 vs 分离 | -1 到 1（越高越好） | 比较 K 值或算法 |
| 惯性（簇内平方和） | 聚类紧密度 | 0 到 inf（越低越好） | K-Means 的肘部法 |
| BIC / AIC | 带复杂度惩罚的模型拟合 | 越低越好 | 选择 GMM 组件数 |
| Calinski-Harabasz 指数 | 组间与组内方差比 | 越高越好 | 快速比较 |
| Davies-Bouldin 指数 | 聚类间平均相似度 | 越低越好 | 惩罚重叠聚类 |

## 常见错误

- 不缩放特征就运行 K-Means（较大尺度的特征主导距离计算）
- 在 2D 中肉眼挑选 K 而实际数据是高维的（使用轮廓分数）
- 在非球形聚类上使用 K-Means（新月形或环形数据需要 DBSCAN）
- 设置 DBSCAN eps 太大（所有东西在一个聚类中）或太小（所有东西都是噪声）
- 将聚类标签视为真实值（聚类是探索性的；用领域知识验证）
- 在超过 20,000 个点的数据集上运行层次聚类（内存和时间爆炸）
