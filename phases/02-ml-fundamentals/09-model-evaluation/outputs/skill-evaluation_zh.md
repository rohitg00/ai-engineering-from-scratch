# 模型评估策略

正确评估任何机器学习模型的清单。按此顺序避免最常见的评估错误。

## 步骤 1：正确分割数据

- 在任何预处理（缩放、填充、编码）之前分割
- 分类任务使用分层分割
- 保留一个测试集，最后只接触一次
- 小数据集使用 5 折或 10 折交叉验证代替单次分割
- 时间序列使用基于时间的分割（永远不要打乱）

## 步骤 2：选择正确的指标

### 分类

| 情况 | 使用此指标 | 原因 |
|------|------------|------|
| 类别平衡，简单比较 | 准确率 | 易于解释，类别相等时有意义 |
| 假阳性代价高（垃圾邮件过滤、欺诈警报） | 精确率 | 测量标记项目中有多少确实是正例 |
| 假阴性代价高（癌症筛查、安全） | 召回率 | 测量你捕获了多少实际正例 |
| 需要平衡精确率和召回率 | F1 分数 | 调和平均，惩罚极端不平衡 |
| 跨阈值比较模型 | AUC-ROC | 独立于阈值的排序质量 |
| 不平衡数据 | F1、AUC-ROC 或 PR-AUC | 准确率在不平衡类别时有误导性 |

### 回归

| 情况 | 使用此指标 | 原因 |
|------|------------|------|
| 标准回归，异常值可接受 | RMSE | 与目标相同单位，惩罚大误差 |
| 异常值稳健的评估 | MAE | 平等对待所有误差，不被异常值主导 |
| 不同尺度比较模型 | R² | 归一化 0-1 尺度（解释方差比例） |
| 业务需要金额 | MAE 或 RMSE | 直接可解释为误差大小 |

## 步骤 3：建立基线

评估模型前，计算基线性能：
- 分类：多数类预测器（始终预测最常见的类别）
- 回归：始终预测训练目标的均值
- 任何无法击败这些基线的模型都没有在学习

## 步骤 4：交叉验证

- 使用 K 折（K=5 或 K=10）获得稳定估计
- 分类使用分层 K 折
- 报告折间的均值和标准差
- 均值=0.85 且 std=0.02 的模型比均值=0.87 且 std=0.10 的更可信

## 步骤 5：统计比较模型

- 不要在没有检查显著性的情况下选择平均分数最高的模型
- 使用交叉验证折间的配对 t 检验
- 如果 |t| < 2.78（K=5，df=4，p<0.05），差异可能由偶然造成
- 当性能差异不显著时，考虑更简单的模型

## 步骤 6：检查常见错误

- 数据泄露：任何测试数据信息是否流入训练？（分割前缩放、目标派生特征）
- 类别不平衡：准确率是否掩盖了少数类的不良表现？
- 过拟合：训练与验证性能差距是否大？
- 评估次数过多：你是否看了测试集超过一次？

## 步骤 7：报告最终性能

- 在训练+验证组合上训练
- 在保留的测试集上恰好评估一次
- 报告所选指标，尽可能带置信区间
- 说明与基线的比较（比随机/均值好多少）
