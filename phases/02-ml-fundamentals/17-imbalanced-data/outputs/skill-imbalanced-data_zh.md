# 不平衡数据策略

处理不平衡分类的决策清单。按此顺序为你的问题选择正确的方法。

## 步骤 1：测量不平衡

- 统计每类样本数
- 计算不平衡比率（多数 / 少数）
- 轻度：比率 < 3:1（如 70/30）
- 中度：比率 3:1 到 20:1（如 95/5）
- 重度：比率 > 20:1（如 99/1）

## 步骤 2：选择正确的指标

对不平衡数据集优先使用精确率/召回率/F1 而非准确率。根据你的问题选择：

| 情况 | 主要指标 | 次要指标 |
|------|----------|----------|
| 漏掉正例代价很高（欺诈、疾病） | 召回率 | F2 分数 |
| 误报警代价高（垃圾邮件过滤、推荐） | 精确率 | F0.5 分数 |
| 两者大致同等重要 | F1 分数 | MCC |
| 需要单一排序指标 | AUPRC | AUC-ROC |
| 需要跨数据集比较 | MCC | AUPRC |

## 步骤 3：选择重平衡策略

### 按不平衡严重程度

| 不平衡 | 首先尝试 | 第二次尝试 | 避免 |
|--------|----------|------------|------|
| 轻度 (< 3:1) | 类别权重 | 阈值调优 | 过采样（不必要） |
| 中度 (3:1 到 20:1) | SMOTE + 类别权重 | 顶部阈值调优 | 欠采样（数据损失太大） |
| 重度 (> 20:1) | SMOTE + 类别权重 + 阈值 | 平衡装袋集成 | 单独欠采样 |

### 按数据集大小

| 数据集大小 | 首选策略 | 原因 |
|-----------|----------|------|
| < 1,000 样本 | 过采样或 SMOTE | 不能承受丢失多数数据 |
| 1,000 - 10,000 | SMOTE + 阈值调优 | 足够的少数样本用于 k-NN |
| > 10,000 | 类别权重或欠采样 | 快速，足够的少数数据 |

## 步骤 4：应用技术

### 类别权重（始终先尝试）
- 在 sklearn 中：`class_weight='balanced'`
- 不需要修改数据
- 适用于任何基于损失的模型
- 在期望中等价于过采样

### SMOTE
- 仅应用于训练数据（永远不要测试/验证）
- 使用 k=5 邻居（默认）
- 与类别权重结合以获得最佳结果
- 注意边界附近的噪声合成点

### 阈值调优
- 训练模型，在验证集上获取预测概率
- 从 0.05 到 0.95 扫描阈值
- 选择最大化所选指标的阈值
- 始终在验证数据上调整，永远不要测试数据

## 步骤 5：正确验证

- 使用分层交叉验证（在每折中保留类别比率）
- 在原始（未重采样）测试集上报告指标
- 分割前永远不要应用 SMOTE -- 仅在训练折上
- 与"始终预测多数"基线比较

## 步骤 6：避免的常见错误

- 在训练/测试分割前将 SMOTE 应用于整个数据集（数据泄露）
- 使用准确率作为评估指标
- 不先尝试类别权重（最简单的方法，通常足够）
- 过采样然后交叉验证（合成点泄露到折间）
- 忽视阈值调优（免费性能，无需重新训练）
- 在小数据集上使用随机欠采样（丢弃太多数据）

## 快速决策树

1. 不平衡比率 < 3:1？ -> 仅尝试类别权重
2. 数据集 > 10,000 样本？ -> 类别权重 + 阈值调优
3. 数据集 < 1,000 样本？ -> SMOTE + 类别权重
4. 否则 -> SMOTE + 类别权重 + 阈值调优
5. 仍然不够好？ -> 平衡装袋集成
