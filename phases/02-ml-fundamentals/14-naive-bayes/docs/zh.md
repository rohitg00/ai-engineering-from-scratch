# 朴素贝叶斯（Naive Bayes）

> “朴素”假设是错误的，但它依然有效。这正是它的美妙之处。

-   比较多项式朴素贝叶斯（Multinomial）、伯努利朴素贝叶斯（Bernoulli）和高斯朴素贝叶斯（Gaussian）变体，并为给定的特征类型选择合适的变体

## 问题

你需要对文本进行分类。将电子邮件分为垃圾邮件或非垃圾邮件。将客户评论分为正面或负面。将支持工单分类到不同类别。你有数千个特征（每个单词一个），但训练数据有限。

大多数分类器在此处会失效。逻辑回归需要足够的样本来可靠地估计数千个权重。决策树每次只在一个单词上进行分裂，并会严重过拟合。在 10,000 维空间中的 KNN（K近邻）毫无意义，因为每个点与其他所有点的距离都相等。

朴素贝叶斯可以处理这种情况。它做出了一个数学上错误的假设（即在给定类别的情况下，每个特征与其他所有特征相互独立），但在文本分类上，尤其是在训练集较小的情况下，其表现仍优于“更聪明”的模型。它只需对数据进行一次遍历即可完成训练。它可以扩展到数百万个特征。它能产生概率估计（尽管由于独立性假设，这些概率通常校准不佳）。

理解为什么一个错误的假设能带来良好的预测，会教给你一些关于机器学习的根本性知识：最好的模型不是最正确的模型，而是对你的数据具有最佳偏差-方差权衡的模型。

## 概念

### 贝叶斯定理（快速回顾）

贝叶斯定理翻转了条件概率：

```
P(类别 | 特征) = P(特征 | 类别) * P(类别) / P(特征)
```

我们想要 `P(类别 | 特征)` —— 给定文档中的单词，该文档属于某个类别的概率。我们可以通过以下方式计算：
- `P(特征 | 类别)` —— 在此类别的文档中看到这些单词的似然（likelihood）
- `P(类别)` —— 类别的先验概率（prior probability）（总体上垃圾邮件有多常见？）
- `P(特征)` —— 证据（evidence），对于所有类别相同，因此在比较时可以忽略

`P(类别 | 特征)` 最高的类别获胜。

### 朴素独立性假设

精确计算 `P(特征 | 类别)` 需要估计所有特征一起的联合概率（joint probability）。对于一个包含 10,000 个单词的词汇表，你需要估计一个在 2^10,000 种可能组合上的分布。这是不可能的。

朴素假设：在给定类别的情况下，每个特征条件独立。

```
P(w1, w2, ..., wn | 类别) = P(w1 | 类别) * P(w2 | 类别) * ... * P(wn | 类别)
```

我们不再估计一个不可能的联合分布，而是估计 n 个简单的每特征分布。每个分布只需要一个计数。

这个假设显然是错误的。在任何文档中，“机器”（machine）和“学习”（learning）这两个词都不是独立的。但是分类器不需要正确的概率估计。它需要正确的排名 —— 哪个类别的概率最高。独立性假设引入了系统性误差，但这些误差对所有类别的影响相似，因此排名保持不变。

### 为什么它仍然有效

三个原因：

1.  **排名优于校准。** 分类只需要排名最高的类别正确即可。即使 P(垃圾邮件) = 0.99999，而真实概率为 0.7，分类器仍然正确地将它判为垃圾邮件。我们不需要正确的概率。我们需要的是正确的胜者。

2.  **高偏差，低方差。** 独立性假设是一个强先验。它严重约束了模型，从而防止过拟合。在训练数据有限的情况下，一个略有错误但稳定的模型，要优于一个理论上正确但极不稳定的模型。这就是偏差-方差权衡在起作用。

3.  **特征冗余相互抵消。** 相关特征提供了冗余的证据。分类器会重复计算这些证据，但它也会为正确的类别重复计算。如果“机器”和“学习”总是同时出现，那么两者都为“科技”类别提供了证据。朴素贝叶斯将它们计算了两次，但它是为正确的类别计算了两次。

第四个实际原因：朴素贝叶斯非常快。训练只需对数据进行一次遍历来统计频率。预测是矩阵乘法。你可以在几秒钟内对一百万篇文档进行训练。这种速度意味着你可以比使用较慢模型时更快地迭代、尝试更多特征集并运行更多实验。

### 逐步数学推导

让我们通过一个具体示例来追踪。假设我们有两个类别：垃圾邮件和非垃圾邮件。我们的词汇表有三个单词：“免费”（free）、“金钱”（money）、“会议”（meeting）。

训练数据：
-   垃圾邮件提到“免费”80次，“金钱”60次，“会议”10次（总共150个词）
-   非垃圾邮件提到“免费”5次，“金钱”10次，“会议”100次（总共115个词）
-   40% 的电子邮件是垃圾邮件，60% 是非垃圾邮件

使用拉普拉斯平滑（alpha=1）：

```
P(免费 | 垃圾邮件)    = (80 + 1) / (150 + 3) = 81/153 = 0.529
P(金钱 | 垃圾邮件)   = (60 + 1) / (150 + 3) = 61/153 = 0.399
P(会议 | 垃圾邮件) = (10 + 1) / (150 + 3) = 11/153 = 0.072

P(免费 | 非垃圾邮件)    = (5 + 1) / (115 + 3) = 6/118 = 0.051
P(金钱 | 非垃圾邮件)   = (10 + 1) / (115 + 3) = 11/118 = 0.093
P(会议 | 非垃圾邮件) = (100 + 1) / (115 + 3) = 101/118 = 0.856
```

新邮件包含：“免费”（2次），“金钱”（1次），“会议”（0次）。

```
log P(垃圾邮件 | 邮件) = log(0.4) + 2*log(0.529) + 1*log(0.399) + 0*log(0.072)
                    = -0.916 + 2*(-0.637) + (-0.919) + 0
                    = -3.109

log P(非垃圾邮件 | 邮件) = log(0.6) + 2*log(0.051) + 1*log(0.093) + 0*log(0.856)
                        = -0.511 + 2*(-2.976) + (-2.375) + 0
                        = -8.838
```

垃圾邮件以巨大优势胜出。“免费”出现两次是垃圾邮件的强有力证据。注意“会议”没有出现对两个对数求和（log P 乘以 0）都没有贡献 —— 在多项式朴素贝叶斯（Multinomial NB）中，不存在的单词没有影响。显式建模单词缺失的是伯努利朴素贝叶斯（Bernoulli NB）。

### 三种变体

朴素贝叶斯有三种形式。每种都以不同方式建模 `P(特征 | 类别)`。

#### 多项式朴素贝叶斯（Multinomial Naive Bayes）

将每个特征建模为计数。最适合特征是词频（word frequencies）或 TF-IDF 值的文本数据。

```
P(word_i | 类别) = (类别中 word_i 的计数 + alpha) / (类别中总词数 + alpha * 词汇表大小)
```

`alpha` 是拉普拉斯平滑（将在下面解释）。此变体是文本分类的主力。

#### 高斯朴素贝叶斯（Gaussian Naive Bayes）

将每个特征建模为正态分布。最适合连续特征。

```
P(x_i | 类别) = (1 / sqrt(2 * pi * var)) * exp(-(x_i - mean)^2 / (2 * var))
```

每个类别在每个特征上都有自己的均值和方差。当特征在每个类别内确实遵循钟形曲线时，此方法效果很好。

#### 伯努利朴素贝叶斯（Bernoulli Naive Bayes）

将每个特征建模为二值（存在或不存在）。最适合短文本或二值特征向量。

```
P(word_i | 类别) = (类别中包含 word_i 的文档数 + alpha) / (类别中文档总数 + 2 * alpha)
```

与多项式不同，伯努利明确地惩罚单词的缺失。如果“免费”通常出现在垃圾邮件中，但此邮件中没有，伯努利会将其视为反对垃圾邮件的证据。

### 何时使用每种变体

| 变体 | 特征类型 | 最适合 | 示例 |
|------|----------|--------|------|
| 多项式（Multinomial） | 计数或频率 | 文本分类、词袋模型（bag-of-words） | 电子邮件垃圾邮件、主题分类 |
| 高斯（Gaussian） | 连续值 | 具有近似正态特征的表格数据 | 鸢尾花分类、传感器数据 |
| 伯努利（Bernoulli） | 二值（0/1） | 短文本、二值特征向量 | 短信垃圾邮件、存在/不存在特征 |

### 拉普拉斯平滑（Laplace Smoothing）

如果测试数据中出现一个单词，而该单词在训练数据中从未出现在某个类别中，会发生什么？

无平滑时：`P(word | 类别) = 0/N = 0`。一个零乘以整个乘积会使 `P(类别 | 特征) = 0`，无论所有其他证据如何。一个未见过的单词会破坏整个预测，无论有多少其他证据支持它。

拉普拉斯平滑为每个特征计数添加一个小计数 `alpha`（通常为 1）：

```
P(word_i | 类别) = (count(word_i, 类别) + alpha) / (类别中总词数 + alpha * 词汇表大小)
```

当 alpha=1 时，每个词至少获得一个极小的概率。在测试邮件中出现的单词“混乱”（discombobulate）不会再扼杀垃圾邮件的概率。平滑具有贝叶斯解释：它等价于在词分布上放置一个均匀的狄利克雷先验（Dirichlet prior）。

更高的 alpha 意味着更强的平滑（更均匀的分布）。更低的 alpha 意味着模型更信任数据。Alpha 是一个需要调节的超参数。

Alpha 的效果：

| Alpha | 效果 | 何时使用 |
|-------|------|----------|
| 0.001 | 几乎无平滑，信任数据 | 非常大的训练集，预计没有未见特征 |
| 0.1 | 轻度平滑 | 大型训练集 |
| 1.0 | 标准拉普拉斯平滑 | 默认起点 |
| 10.0 | 重度平滑，使分布扁平化 | 非常小的训练集，预计有许多未见特征 |

### 对数空间计算

将数百个概率（每个都小于 1）相乘会导致浮点数下溢（floating-point underflow）。即使真实值是一个非常小的正数，在浮点数中乘积也会变为零。

解决方法：在对数空间中工作。不再相乘概率，而是相加它们的对数：

```
log P(类别 | x1, x2, ..., xn) = log P(类别) + sum_i log P(xi | 类别)
```

这将预测转化为点积：

```
log_scores = X @ log_feature_probs.T + log_class_priors
prediction = argmax(log_scores)
```

矩阵乘法。这就是为什么朴素贝叶斯预测如此之快——它与单层线性模型的操作相同。

### 朴素贝叶斯 vs 逻辑回归

两者都是用于文本的线性分类器。区别在于它们建模的内容。

| 方面 | 朴素贝叶斯 | 逻辑回归 |
|--------|------------|-------------------|
| 类型 | 生成式（Generative，建模 P(X\|Y)） | 判别式（Discriminative，建模 P(Y\|X)） |
| 训练 | 计数频率 | 优化损失函数 |
| 小数据 | 更好（强先验有帮助） | 更差（样本不足以估计权重） |
| 大数据 | 更差（错误假设有害） | 更好（灵活的决策边界） |
| 特征 | 假设独立 | 处理相关性 |
| 速度 | 单次遍历，非常快 | 迭代优化 |
| 校准 | 概率较差 | 概率更好 |

经验法则：从朴素贝叶斯开始。如果你有足够的数据且朴素贝叶斯达到瓶颈，则切换到逻辑回归。

## 构建它

`code/naive_bayes.py` 中的代码从头实现了 MultinomialNB（多项式朴素贝叶斯）和 GaussianNB（高斯朴素贝叶斯）。

### MultinomialNB

从头实现：

1.  **fit(X, y)**：对于每个类别，统计每个特征的频率。添加拉普拉斯平滑。计算对数概率。存储类别先验（类别频率的对数）。

2.  **predict_log_proba(X)**：对于每个样本，计算 log P(类别) + 所有特征的 log P(特征_i | 类别) 之和。这是一个矩阵乘法：X @ log_probs.T + log_priors。

3.  **predict(X)**：返回具有最高对数概率的类别。

```python
class MultinomialNB:
    def __init__(self, alpha=1.0):
        self.alpha = alpha

    def fit(self, X, y):
        classes = np.unique(y)
        n_classes = len(classes)
        n_features = X.shape[1]

        self.classes_ = classes
        self.class_log_prior_ = np.zeros(n_classes)
        self.feature_log_prob_ = np.zeros((n_classes, n_features))

        for i, c in enumerate(classes):
            X_c = X[y == c]
            self.class_log_prior_[i] = np.log(X_c.shape[0] / X.shape[0])
            counts = X_c.sum(axis=0) + self.alpha
            self.feature_log_prob_[i] = np.log(counts / counts.sum())

        return self
```

关键见解：拟合后，预测只是矩阵乘法加上偏置。这就是朴素贝叶斯如此之快的原因。

### GaussianNB

对于连续特征，我们估计每个类别每个特征的均值和方差：

```python
class GaussianNB:
    def __init__(self):
        pass

    def fit(self, X, y):
        classes = np.unique(y)
        self.classes_ = classes
        self.means_ = np.zeros((len(classes), X.shape[1]))
        self.vars_ = np.zeros((len(classes), X.shape[1]))
        self.priors_ = np.zeros(len(classes))

        for i, c in enumerate(classes):
            X_c = X[y == c]
            self.means_[i] = X_c.mean(axis=0)
            self.vars_[i] = X_c.var(axis=0) + 1e-9
            self.priors_[i] = X_c.shape[0] / X.shape[0]

        return self
```

预测使用每个特征的高斯概率密度函数（Gaussian PDF），然后在特征间相乘（在对数空间中相加）。

### 演示：文本分类

代码生成合成词袋数据，模拟两个类别（科技文章 vs 体育文章）。每个类别有不同的词频分布。MultinomialNB 使用词计数对它们进行分类。
