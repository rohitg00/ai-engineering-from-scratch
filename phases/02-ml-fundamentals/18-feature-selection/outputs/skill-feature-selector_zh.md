# 特征选择策略

选择和应用正确特征选择方法的快速参考。

## 步骤 1：从清理开始

在应用任何方法前，移除明显无用的特征：

- **常数特征**：方差 = 0。移除它们。
- **近常数特征**：方差 < 0.01（或你的阈值）。移除它们。
- **重复特征**：相同的列。保留一个，删除其余的。
- **ID 列**：每行唯一，不携带可泛化信息。移除它们。

这只需几秒钟，可以在混乱的真实世界数据集中消除 10-30% 的特征。

## 步骤 2：根据你的情况选择方法

### 快速决策树

1. **< 50 个特征？** 从互信息排序开始。保留前 K 个。
2. **50 - 500 个特征？** 先用方差阈值，然后如果使用线性模型用 L1（Lasso），如果使用树用树重要性。
3. **> 500 个特征？** 链式方法：方差阈值 -> 互信息过滤（前 50%）-> 在幸存者上使用 RFE。
4. **需要可解释性？** L1 正则化给你精确的零/非零。树重要性给出排序分数。
5. **需要捕捉非线性关系？** 互信息或基于树的重要性。避免 L1（仅线性）。
6. **需要特征交互？** RFE 或基于树的重要性。过滤方法错过交互。

### 方法参考

| 方法 | 何时使用 | 何时避免 |
|------|----------|----------|
| 方差阈值 | 始终，作为第一步 | 永远不要跳过这步 |
| 互信息 | 快速排序，非线性关系 | 当你需要特征交互检测时 |
| RFE | 彻底选择，中等特征数 | 非常昂贵的模型，> 1000 特征 |
| L1 / Lasso | 线性模型，快速嵌入式选择 | 非线性问题，高度相关特征 |
| 树重要性 | 非线性关系，特征交互 | 被高基数特征偏差 |
| 排列重要性 | 模型无关验证，最终检查 | 初始筛选太慢 |

## 步骤 3：验证你的选择

- 比较选定特征与所有特征的模型性能
- 使用交叉验证，不要只用单次训练/测试分割
- 如果性能下降超过 1-2%，你可能移除了有用的特征
- 如果性能提高，你成功移除了噪声

## 步骤 4：处理常见陷阱

### 相关特征
- L1 从相关组中任意挑选一个并将其余归零
- 先计算相关矩阵并决定保留哪些相关特征
- 树重要性在相关特征间分散重要性

### 数据泄露
- 仅在训练数据上拟合特征选择
- 将相同选择应用于测试数据
- 在交叉验证中，特征选择必须在每折内部发生

### 过拟合特征选择
- 迭代次数太多的 RFE 可能过拟合到训练集
- 在保留数据上验证，不是用于选择的数据
- 对更稳健的结果使用稳定性选择（在子样本上重复）

## 步骤 5：生产清单

- [ ] 方差阈值作为第一过滤器应用
- [ ] 特征选择仅在训练数据上拟合
- [ ] 选定的特征已记录（名称、使用的方法、分数）
- [ ] 性能比较：选定特征 vs 所有特征
- [ ] 交叉验证，不是单分割评估
- [ ] 特征选择集成到训练流水线中（不是手动完成）
- [ ] 监控特征漂移（选定的特征可能变得陈旧）
