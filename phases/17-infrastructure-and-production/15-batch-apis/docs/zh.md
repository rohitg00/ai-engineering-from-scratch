# 批处理API —— 50%折扣作为行业标准

> 每个主要提供商都提供异步批处理API，带50%折扣和约24小时周转。OpenAI、Anthropic、Google和大多数推理平台（Fireworks批处理层、Together批处理）实现相同模式。堆叠批处理与提示缓存，夜间管道降至同步未缓存成本的约10%。规则极其简单：如果不是交互式的，它属于批处理。内容生成管道、文档分类、数据提取、报告生成、批量标记、目录标记 —— 任何容忍24小时延迟的东西都是留在桌上的钱，直到它移动到批处理。2026年生产模式是将每个新LLM工作负载分类到三个通道：交互式（带缓存的同步）、半交互式（带回退的异步队列）、批处理（夜间、堆叠缓存输入）。假装交互式但容忍分钟延迟的工作负载浪费最多。

## 学习目标

- 命名三个提供商批处理API（OpenAI、Anthropic、Google）和常见的50%折扣 + 24小时周转保证。
- 计算在夜间分类工作负载上堆叠批处理 + 缓存输入的成本，并与同步未缓存基线比较。
- 将工作负载分类为交互式/半交互式/批处理并证明通道。
- 命名两个陷阱：部分交互性（用户期望比24小时更快）和输出模式漂移（批处理文件格式因提供商而异）。

## 问题

你的团队发布夜间报告生成管道。50,000文档，总结每个，聚类摘要，起草执行简报。同步运行每晚需要4小时，$2,000。你听说批处理API。

批处理给你50%折扣。你还在系统提示（跨所有50k调用共享）上启用提示缓存。堆叠后，账单降至每晚$180 —— 约基线的9%。相同管道，三个配置更改。

批处理是LLM成本工具包中没人拉的最便宜杠杆。原因主要是组织性的：团队在想"实时"时，SLA实际上是"到早上"。本课程是关于不把90%的账单留在桌上。

## 概念

### 三个批处理API

**OpenAI批处理API**：JSONL文件上传，带请求列表。承诺24小时周转（实践中通常约2-8小时）。输入和输出token 50%折扣。`/v1/batches`端点。缓存合格输入也在顶部获得缓存输入定价。

**Anthropic消息批处理**：JSONL上传。24小时周转。50%折扣。支持`cache_control` —— 缓存写入是显式的，读取在批处理内自动发生。

**Google Vertex AI批处理预测**：BigQuery或GCS输入。Gemini类似50%折扣。与Vertex管道集成。

### 语义：异步，非慢

批处理是"我承诺在24小时内返回" —— 不是"这将需要24小时"。典型P50是2-6小时。提供商在GPU库存未充分利用的非高峰窗口期间调度你的批处理。

### 与缓存堆叠

具有相同4K token系统提示的50k文档摘要：

- 同步未缓存：50000 × ($input × 4000 + $output × 200) 全价。
- 同步缓存：首次写入后系统提示缓存；剩余49999获得10倍更便宜输入。
- 批处理缓存：所有上述加读写50%折扣。

堆叠：批处理 + 缓存 = 同步未缓存账单的约10%。任何夜间运行且具有共享系统提示的工作负载都应使用此。

### 工作负载分类

**交互式** —— 用户等待响应。TTFT重要。带提示缓存的同步调用。不能批处理。

**半交互式** —— 用户提交任务，几分钟后回来检查。带回退到同步的异步队列（如果批处理不可用）。考虑中等量RAG索引。

**批处理** —— 用户期望结果"到早上"或"下个小时"。内容管道、大规模分类、离线分析。始终批处理，始终堆叠缓存。

常见错误：因为管道是生产性的，将所有内容分类为交互式。生产性不是延迟规范 —— SLA是。

### 部分交互性陷阱

某些功能看起来交互式但容忍5-10分钟。示例：带"刷新"按钮的夜间客户健康报告。用户点击刷新；等待10分钟可以。团队将其作为同步发布。50个并发刷新成本是批处理并通过电子邮件交付的10倍。

要问的问题："24小时对这个用户意味着什么？"如果答案是"他们不会注意到"，批处理它。

### 输出模式陷阱

批处理文件格式因提供商而异：

- OpenAI：JSONL，每行一个请求。
- Anthropic：JSONL，每行一条消息；响应格式嵌入。
- Vertex：BigQuery表或带TFRecord的GCS前缀。

跨提供商编写"一个批处理客户端"意味着每个提供商的适配器代码。宣传多提供商批处理的网关（Portkey、LiteLLM某些层）仍然薄包装原始格式。

### 你应该记住的数字

- 跨提供商批处理折扣：输入 + 输出50%固定折扣。
- 周转SLA：24小时保证，2-6小时典型P50。
- 堆叠批处理 + 缓存输入：同步未缓存成本的约10%。
- 工作负载分类规则：如果24小时延迟可接受，始终批处理。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| 批处理API | "异步折扣" | 24小时周转50%折扣 |
| JSONL | "批处理格式" | 每行一个JSON请求；OpenAI/Anthropic标准 |
| 消息批处理 | "Anthropic批处理" | Anthropic的批处理API产品名称 |
| 批处理预测 | "Vertex批处理" | Vertex AI的批处理API产品 |
| 周转SLA | "24小时承诺" | 保证，非典型；典型是2-6小时 |
| 工作负载分类 | "交互性决策" | 交互式/半交互式/批处理路由决策 |
| 输出模式 | "响应格式" | 每提供商JSONL布局；不可移植 |
| 堆叠折扣 | "批处理 + 缓存" | 两者都适用时约同步未缓存账单的10% |
