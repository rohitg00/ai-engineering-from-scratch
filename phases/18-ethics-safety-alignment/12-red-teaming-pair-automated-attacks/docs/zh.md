# 红队测试：PAIR 与自动化攻击

> Chao, Robey, Dobriban, Hassani, Pappas, Wong (NeurIPS 2023, arXiv:2310.08419)。PAIR——提示自动迭代优化——是标准的自动化黑盒越狱方法。一个带有红队系统提示的攻击 LLM 迭代地为目标 LLM 提出越狱方案，在其自身的聊天历史中积累尝试和响应作为上下文反馈。PAIR 通常在 20 次查询内成功，比 GCG（Zou 等人的令牌级梯度搜索）效率高几个数量级，且不需要白盒访问。PAIR 现在与 GCG、AutoDAN、TAP 和 Persuasive Adversarial Prompt 一起，成为 JailbreakBench (arXiv:2404.01318) 和 HarmBench 中的标准基线。

**类型：** 构建
**语言：** Python（标准库，针对玩具目标的模拟 PAIR 循环）
**先决条件：** 阶段 18 · 01（指令遵循），阶段 14（代理工程）
**时间：** 约 75 分钟

## 学习目标

- 描述 PAIR 算法：攻击者系统提示、迭代优化、上下文反馈。
- 解释为什么当目标是黑盒时，PAIR 严格来说比 GCG 更高效。
- 说出四种其他自动化攻击基线（GCG、AutoDAN、TAP、PAP）并说明每种的一个显著特征。
- 描述 JailbreakBench 和 HarmBench 评估协议以及每种情况下的"攻击成功率"含义。

## 问题

红队测试过去是一种手动活动。少数专家测试人员构建对抗性提示并跟踪哪些有效。这无法扩展：攻击成功率需要统计样本，且目标随着每次模型发布而移动。PAIR 将红队测试操作化为针对黑盒目标的优化问题。

## 概念

### PAIR 算法

输入：
- 目标 LLM T（我们要攻击的模型）。
- 判断 LLM J（评分响应是否是越狱）。
- 攻击者 LLM A（红队优化器）。
- 目标字符串 G："响应 [有害指令]"。
- 预算 K（通常 20 次查询）。

循环，对于 k 在 1..K 中：
1. A 被提示目标 G 和迄今为止的（提示，响应）对历史。
2. A 发出新提示 p_k。
3. 提交 p_k 到 T；接收响应 r_k。
4. J 对（p_k, r_k）按目标评分。
5. 如果分数 >= 阈值，停止——找到越狱。
6. 否则，将（p_k, r_k）附加到 A 的历史；继续。

经验结果（NeurIPS 2023）：针对 GPT-3.5-turbo、Llama-2-7B-chat 的成功率 >50%；平均成功查询次数在 10-20 范围内。

### 为什么 PAIR 高效

GCG（Zou 等人 2023）通过梯度搜索对抗性令牌后缀；它需要白盒模型访问并产生不可读的后缀。PAIR 是黑盒的，并产生可跨模型迁移的自然语言攻击。PAIR 的上下文反馈让攻击者从每次拒绝中学习；GCG 没有等效项（每个新令牌更新必须重新发现先前的进展）。

### 相关自动化攻击

- **GCG（Zou 等人 2023, arXiv:2307.15043）。** 针对对抗性后缀的令牌级梯度搜索。白盒，可迁移，产生不可读字符串。
- **AutoDAN（Liu 等人 2023）。** 在提示上的进化搜索，由分层目标引导。
- **TAP（Mehrotra 等人 2024）。** 带剪枝的攻击树——分支多个 PAIR 风格的 rollout。
- **PAP（Zeng 等人 2024）。** 说服性对抗性提示——将人类说服技术编码为提示模板。

### JailbreakBench 和 HarmBench

两者（2024）标准化评估：

- **JailbreakBench（arXiv:2404.01318）。** 跨越 10 个 OpenAI 策略类别的 100 种有害行为。攻击成功率（ASR）作为主要指标。需要判断器（GPT-4-turbo、Llama Guard 或 StrongREJECT）。
- **HarmBench（Mazeika 等人 2024）。** 跨越 7 个类别的 510 种行为，带有语义和功能危害测试。比较 18 种攻击对抗 33 个模型。

ASR 通常在固定查询预算下报告。比较攻击需要匹配预算；200 次查询时的 90% ASR 与 20 次查询时的 85% ASR 不可比。

### 为什么它对 2026 年部署重要

现在每个前沿实验室在发布前都对生产模型运行 PAIR 和 TAP。ASR 轨迹出现在模型卡（课程 26）和安全案例附录（课程 18）中。攻击并不奇特——它是标准基础设施。

## 使用它

`code/main.py` 构建一个玩具 PAIR 循环。目标是一个模拟分类器，拒绝"明显"的有害提示（关键词过滤器）。攻击者是基于规则的优化器，尝试释义、角色扮演框架和编码。判断器对响应评分。你观察攻击者在约 5-15 次迭代中对抗关键词过滤器成功，而对抗语义过滤器失败。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|----------|----------|
| PAIR | "自动化越狱" | 提示自动迭代优化；攻击者 LLM + 判断器 LLM 循环 |
| GCG | "梯度越狱" | 针对对抗性后缀的白盒令牌级梯度搜索 |
| 攻击成功率（ASR） | "k 次查询时的越狱%" | 主要指标；必须与查询预算和判断器身份一起报告 |
| 判断器 LLM | "评分者" | 对响应是否满足有害目标评分的 LLM |
| JailbreakBench | "评估" | 带有标记类别的标准化有害行为集 |
| HarmBench | "更广泛的基准" | 510 种行为，功能 + 语义危害测试 |
| TAP | "攻击树" | 带分支 + 剪枝的 PAIR；更高计算下更好的 ASR |
