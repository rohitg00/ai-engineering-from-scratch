# 22 · 大语言模型的差分隐私

> DP-SGD 仍是标准方案——通过注入噪声的梯度更新提供形式化的 (ε, δ) 保证。其在计算、内存和效用方面的开销相当可观；参数高效的 DP 微调（LoRA + DP-SGD）是 2025 年的主流配置（ACM 2025）。两组证据之间存在矛盾：基于金丝雀的成员推断攻击（Duan et al., 2024）显示对语言模型的攻击效果有限；而训练数据提取攻击（Carlini et al., 2021; Nasr et al., 2025）则恢复出了大量逐字记忆内容。2025 年 3 月的解决思路（arXiv:2503.06808）：分歧在于测量对象的不同——插入的金丝雀与"最可提取"数据的差异。新的金丝雀设计使得无需影子模型的基于损失的 MIA 成为可能，并首次在真实数据上对具有实际 DP 保证的 LLM 进行了有意义的 DP 审计。替代方案：PMixED（arXiv:2403.15638）——在推理时通过专家混合对下一 token 分布进行私有预测；DP 合成数据生成（Google Research 2024）。新兴攻击：通过大语言模型反馈反转差分隐私——利用置信度分数泄露。

## 学习目标

- 定义 (ε, δ)-差分隐私（Differential Privacy），并陈述 DP-SGD 的标准流程。
- 解释 2024-2025 年间证据的矛盾：金丝雀成员推断攻击（Membership Inference Attack, MIA）与训练数据提取给出了截然不同的结论。
- 描述 PMixED，以及为什么推理时私有预测可以作为 DP 训练的替代方案。
- 描述通过大语言模型（LLM）反馈反转差分隐私的攻击方式。

## 问题所在

大语言模型会记忆训练数据。Carlini 等人（2021 年）证明，生产环境中的语言模型可以在被诱导时逐字复现训练文本。差分隐私（Differential Privacy, DP）是形式化防御手段：训练模型时，使输出在概率上可证明地对任何单个训练样本不敏感。2024-2025 年的证据表明，DP-SGD 是必要的，但实际部署中使用的 ε 值可能并不足以应对真实的威胁模型。

## 核心概念

### (ε, δ)-差分隐私

设 M 为一个随机化算法，若对于任意两个仅相差一条样本的数据集 D 和 D'，以及任意输出事件 S，满足：
P(M(D) ∈ S) ≤ e^ε · P(M(D') ∈ S) + δ

则称 M 满足 (ε, δ)-DP。

解读：输出分布的接近程度由 ε 参数化，使得任何单个个体的贡献无法被可靠推断，除非以概率 δ 允许例外。

### DP-SGD

Abadi 等人（2016 年）提出。标准流程如下：
1. 采样一个小批量（mini-batch）。
2. 计算每个样本的梯度。
3. 将每个样本的梯度裁剪到阈值 C。
4. 对裁剪后的梯度求和，并添加标准差为 σ · C 的高斯噪声（Gaussian noise）。
5. 使用加噪后的和来更新模型参数。

隐私开销由隐私审计器（accountant）跟踪，如矩审计器（Moments Accountant）或 Renyi DP 审计器。LLM 文献中报告的 ε 值因威胁模型、数据敏感度和效用目标不同而有很大差异；不存在通用的"安全"默认 ε。已发表的案例中，某些 LLM 训练场景的 ε 大约在 1 到 10 之间，但这仅作示例参考，并非推荐默认值。更低的 ε 通常需要更多噪声，并可能导致更严重的效用损失。

### LoRA + DP-SGD

对前沿模型进行完整 DP-SGD 训练的成本极高。LoRA（Hu et al., 2022）将梯度更新限制在一个小型适配器（adapter）上，从而减少了每个样本梯度的存储开销。LoRA + DP-SGD 是 2025 年的常见配置。DP 保证仅适用于适配器；基础模型保持冻结不变。

### 2024-2025 年的证据矛盾

两方面的证据：

- **金丝雀 MIA（Duan et al., 2024）。** 在训练数据中插入独特的金丝雀样本，然后衡量成员推断攻击是否能识别它们。报告显示对语言模型的攻击效果有限，表明 MIA 实施困难。
- **训练数据提取（Carlini 2021, Nasr et al., 2025）。** 用一段前缀提示模型，衡量模型是否能恢复出训练数据中的逐字文本。报告显示大量记忆内容，表明从隐私角度 MIA 实际很容易成功。

2025 年 3 月的解决思路（arXiv:2503.06808）：二者测量的是不同的东西。MIA 问的是"样本 e 是否在数据集 D 中？"——针对插入的金丝雀。而提取攻击问的是"我能从 D 中恢复出什么？"对隐私而言，真正重要的是"最可提取"的样本；金丝雀低估了这一风险，因为金丝雀并未被优化为易被提取的形式。

新的金丝雀设计使得无需影子模型（shadow model）的基于损失的 MIA 成为可能。这是首次在真实数据上对 LLM 进行具有实际 DP 保证的、有意义的 DP 审计。

### DP 训练的替代方案

- **PMixED（arXiv:2403.15638）。** 推理时的私有预测。对下一 token 分布使用专家混合（mixture of experts）；每个专家仅访问训练数据的一个分片（shard）；聚合时添加噪声以实现 DP。完全规避了 DP 训练。
- **DP 合成数据生成（Google Research 2024）。** 用 DP-SGD 对 LoRA 进行微调，生成合成数据，然后在合成数据上训练下游分类器。

两种方案都以不同的威胁模型为代价，规避了完整 DP 训练的效用损失。

### 通过大语言模型反馈反转差分隐私

这是 2025 年新兴的攻击方式。利用 DP 训练模型的置信度分数作为神谕（oracle）来重新识别个体。即使输出本身不泄露信息，置信度分布也可能泄露。

防御方案：不暴露置信度信息，或在暴露之前对其进行截断或量化。这是在 (ε, δ)-DP 训练之外的一项额外要求。

## 实战

`code/main.py` 在一个玩具二分类数据集上模拟 DP-SGD。你可以调整噪声乘数 σ 和裁剪范数 C，并跟踪 (ε, δ) 隐私预算以及准确率损失。一个"金丝雀攻击"会插入一条唯一训练样本，并衡量在 DP 前后基于对数损失的检测是否能发现它。

## 交付

本课产出 `outputs/skill-dp-audit.md`。针对某个语言模型部署中声明的 DP 保证，审计内容包括：(ε, δ) 值、所使用的审计器、MIA 评估协议，以及是否已评估置信度暴露向量。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| DP | "(ε, δ)-差分隐私" | 形式化隐私保证：在相邻数据集变化下输出分布足够接近 |
| DP-SGD | "注入噪声的 SGD" | 梯度裁剪 + 高斯噪声添加；标准的 DP 训练方法 |
| LoRA + DP-SGD | "高效的私有微调" | 在低秩适配器上使用 DP-SGD；2025 年标准配置 |
| MIA | "成员推断" | 判断某个样本是否在训练数据中的攻击 |
| 金丝雀（Canary） | "插入的水印样本" | 用于衡量 DP 泄露的唯一训练样本 |
| PMixED | "私有推理混合" | 通过对下一 token 分布使用专家混合实现的推理时 DP |
| DP 反转 | "置信度泄露攻击" | 利用模型置信度作为神谕进行重新识别的攻击 |
