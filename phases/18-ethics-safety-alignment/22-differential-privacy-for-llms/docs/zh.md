# LLM 的差分隐私

> DP-SGD 仍然是标准——注入噪声的梯度更新提供正式的 (ε, δ) 保证。在计算、内存和效用方面的开销很大；参数高效的 DP 微调（LoRA + DP-SGD）是常见的 2025 配置（ACM 2025）。两个证据体存在张力：基于金丝雀的成员推断（Duan 等人，2024）报告对语言模型的成功有限；训练数据提取（Carlini 等人，2021；Nasr 等人，2025）恢复了大量逐字记忆。解决方案（arXiv:2503.06808，2025 年 3 月）：差距在于测量什么——插入的金丝雀 vs "最易提取的"数据。新的金丝雀设计实现无需影子模型的基于损失的 MIA，并产生第一个在真实数据上训练且具有现实 DP 保证的 LLM 的非平凡 DP 审计。替代方案：PMixED（arXiv:2403.15638）—— 通过下一 token 分布上的专家混合进行推理时的私有预测；DP 合成数据生成（Google Research 2024）。新兴攻击：通过 LLM 反馈的差分隐私逆转——置信度分数泄露。

## 学习目标

- 定义 (ε, δ)-差分隐私并说明 DP-SGD 配方。
- 解释 2024-2025 年张力：金丝雀 MIA 与训练数据提取给出不同图景。
- 描述 PMixED 以及为什么推理时私有预测是 DP 训练的替代方案。
- 描述通过 LLM 反馈的差分隐私逆转攻击。

## 问题

LLM 记忆。Carlini 等人 2021 表明生产语言模型按需复现逐字训练文本。DP 是正式防御：训练使得输出可证明地对任何单个训练示例不敏感。2024-2025 年证据表明 DP-SGD 是必要的，但部署的 ε 值可能与威胁模型不匹配。

## 概念

### (ε, δ)-差分隐私

随机算法 M 是 (ε, δ)-DP 的，如果对于任何相差一个示例的两个数据集和任何事件 S：
P(M(D) in S) <= e^ε * P(M(D') in S) + δ。

解释：输出分布足够接近（由 ε 参数化），以至于任何单个个体的贡献不能被可靠推断，除了概率 δ。

### DP-SGD

Abadi 等人 2016。标准配方：
1. 采样一个小批量。
2. 计算每个示例的梯度。
3. 将每个每个示例的梯度裁剪到阈值 C。
4. 对裁剪的梯度求和并添加标准差为 σ * C 的高斯噪声。
5. 使用噪声和更新参数。

隐私成本由会计跟踪（矩会计、Rényi DP 会计）。LLM 文献中报告的 ε 值因威胁模型、数据敏感性和效用目标而有很大差异；没有普遍"安全"的默认 ε。发表的例子在某些 LLM 训练设置中大约 ε ≈ 1–10，但这些是说明性的——不是推荐的默认值。较低的 ε 通常需要更多噪声，可能增加效用损失。

### LoRA + DP-SGD

前沿模型的完整 DP-SGD 是禁止的。LoRA（Hu 等人 2022）将梯度更新限制在小型适配器，减少每个示例的梯度存储。LoRA + DP-SGD 是常见的 2025 配置。DP 保证适用于适配器；基础模型保持固定。

### 2024-2025 年张力

两条证据线：

- **金丝雀 MIA（Duan 等人 2024）。** 将唯一金丝雀插入训练数据，测量成员推断攻击者是否能识别它们。报告对语言模型的成功有限。表明 MIA 很难。
- **训练数据提取（Carlini 2021，Nasr 等人 2025）。** 用前缀提示模型；测量它是否从训练中恢复逐字文本。报告大量记忆。表明 MIA 在相关意义上很容易。

2025 年 3 月解决方案（arXiv:2503.06808）：两者测量不同的事物。MIA 问"示例 e 在 D 中吗？"在插入的金丝雀上。提取问"我能恢复 D 的什么？""最易提取的"示例对隐私重要；金丝雀低报这一点，因为它们没有被优化为可提取。

新的金丝雀设计。无需影子模型的基于损失的 MIA。第一个在真实数据上训练且具有现实 DP 保证的 LLM 的非平凡 DP 审计。

### DP 训练的替代方案

- **PMixED（arXiv:2403.15638）。** 推理时的私有预测。下一 token 分布上的专家混合；每个专家看到训练数据的一个分片；聚合添加噪声以实现 DP。完全避免 DP 训练。
- **DP 合成数据生成（Google Research 2024）。** 使用 DP-SGD 进行 LoRA 微调，采样合成数据，在合成数据上训练下游分类器。

两者以不同威胁模型的代价规避完整 DP 训练的效用成本。

### 通过 LLM 反馈的差分隐私逆转

2025 年新兴攻击。使用 DP 训练模型的置信度分数作为预言来重新识别个体。即使输出不泄露，置信度分布也可以。

防御：不要暴露置信度，或在暴露前截断/量化它们。这是 (ε, δ)-DP 训练之外的额外要求。

`code/main.py` 在模拟二分类数据集上模拟 DP-SGD。你可以扫描噪声乘数 σ 和裁剪范数 C，并跟踪 (ε, δ) 预算和准确率成本。"金丝雀攻击"插入唯一训练示例，并测量在 DP 前后对数损失测试是否能检测它。

本课产生 `outputs/skill-dp-audit.md`。给定语言模型部署上的 DP 声明，它审计：(ε, δ) 值、使用的会计、MIA 评估协议，以及是否已评估置信度暴露向量。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|----------|----------|
| DP | "(ε, δ)-差分隐私" | 正式隐私：在相邻数据集更改下输出分布接近 |
| DP-SGD | "注入噪声的 SGD" | 梯度裁剪 + 高斯噪声添加；标准 DP 训练 |
| LoRA + DP-SGD | "高效私有微调" | 低秩适配器上的 DP-SGD；标准 2025 配置 |
| MIA | "成员推断" | 确定示例是否在训练数据中的攻击 |
| 金丝雀 | "插入的水印示例" | 用于测量 DP 泄露的唯一训练示例 |
| PMixED | "私有推理混合" | 通过下一 token 分布上的专家混合进行推理时 DP |
| DP 逆转 | "置信度泄露攻击" | 使用模型置信度作为重新识别预言的攻击 |
