# 神经网络调试检查清单

训练出错时的系统性调试协议。按顺序执行这些——大多数bug在前3步被捕获。

## 训练前（预防bug）

1. 打印模型架构和参数数量。大小对你的数据合理吗？
2. 用随机输入运行单次前向传播。输出形状匹配目标形状吗？
3. 检查标签是正确的dtype（CrossEntropyLoss需要Long，BCELoss需要Float）
4. 验证数据归一化：输入均值应接近0，标准差接近1
5. 打印5个随机（输入、标签）对。标签符合你的期望吗？
6. 确认训练/测试分割没有重复样本

## 单批次过拟合测试（60秒，捕获80%的bug）

1. 从训练集中取8-32个样本
2. 用合理的学习率训练200步
3. 损失应接近0。训练准确率应达到100%
4. 如果失败：bug在你的模型、损失函数或训练循环中——不是数据或超参数
5. 如果通过：继续进行完整训练

## 损失不下降

1. 检查学习率。尝试3个值：当前/10、当前、当前*10
2. 打印每层梯度范数。全零意味着死亡网络或分离图
3. 检查参数上的`requires_grad=True`。检查`loss.backward()`被调用
4. 检查`loss.backward()`前调用`optimizer.zero_grad()`
5. 检查`loss.backward()`后调用`optimizer.step()`
6. 验证模型参数传递给优化器：`optimizer = Adam(model.parameters())`

## 损失是NaN或Inf

1. 降低学习率10倍
2. 所有log()调用添加epsilon：`torch.log(x + 1e-7)`
3. 所有除法添加epsilon：`x / (y + 1e-8)`
4. 钳制预测：`torch.clamp(pred, 1e-7, 1 - 1e-7)`在BCE损失前
5. 使用`torch.autograd.detect_anomaly()`找到确切操作
6. 检查输入数据中的NaN：`assert not torch.isnan(x).any()`

## 损失振荡

1. 降低学习率3-10倍
2. 增加批量大小（减少梯度噪声）
3. 添加梯度裁剪：`torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)`
4. 从SGD切换到Adam（每参数自适应LR）
5. 训练前5-10%添加学习率预热

## 过拟合（训练准确率高，测试准确率低）

1. 添加dropout（从p=0.1开始，增加到0.5）
2. 优化器添加权重衰减：`Adam(params, weight_decay=1e-4)`
3. 减少模型大小（更少层或更窄层）
4. 添加数据增强
5. 使用早停：验证损失增加5+轮时停止
6. 检查训练集和测试集间的数据泄露

## 欠拟合（训练测试准确率都低）

1. 增加模型容量（更多层、更宽层）
2. 训练更多轮
3. 增加学习率（小心）
4. 暂时移除正则化以验证模型能学习
5. 检查你的模型对任务是否足够表达

## 死亡ReLU神经元

1. 检查每层零激活的比例。>50%是问题
2. 切换到LeakyReLU(0.01)或GELU
3. 权重使用Kaiming初始化
4. 降低学习率（大更新可能将神经元推入死亡区）
5. 激活函数前添加批归一化

## 快速参考：学习率起始点

| 优化器 | 任务 | 起始LR |
|--------|------|--------|
| Adam | 从头训练 | 1e-3 |
| Adam | 微调预训练 | 1e-5 |
| SGD + momentum | 从头训练 | 1e-1 |
| SGD + momentum | 微调预训练 | 1e-3 |
| AdamW | Transformer训练 | 3e-4 |

## 快速参考：批大小影响

| 批大小 | 梯度噪声 | 内存 | 泛化 |
|--------|---------|------|------|
| 8-16 | 高（噪声） | 低 | 通常更好 |
| 32-64 | 中等 | 中等 | 好默认 |
| 128-256 | 低（平滑） | 高 | 可能需要预热 |
| 512+ | 很低 | 很高 | 需要LR缩放 |

## 当什么都不奏效时

1. 简化模型到1个隐藏层。它能学习吗？
2. 简化数据到100个样本。它能过拟合吗？
3. 用MSE替换你的损失。它收敛吗？
4. 用SGD(lr=0.01)替换你的优化器。它有进展吗？
5. 用合成数据替换你的数据（例如y = x[0] > 0）。它能学习吗？
6. 如果这些都不行：bug在你没看的代码中（数据加载、预处理、张量形状）
