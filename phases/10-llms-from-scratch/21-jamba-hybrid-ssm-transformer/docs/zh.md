# Jamba — 混合SSM-Transformer模型

> 状态空间模型（SSM）和Transformer追求不同的目标。Transformer通过注意力机制以二次成本换取质量。SSM通过循环实现线性时间推理和恒定内存，但质量稍逊。AI21的Jamba（2024年3月）和Jamba 1.5（2024年8月）将两者置于同一模型中：每7个Mamba层配1个Transformer层，隔块使用混合专家（MoE），并拥有256k上下文窗口，可单卡运行在80GB GPU上。Mamba-3（ICLR 2026）通过复数值状态空间和多输入多输出（MIMO）投影强化了SSM侧。本课程从头到尾解读这两种架构，并解释为何这种混合配方在纯SSM和纯Transformer的长上下文尝试均未成功的情况下，能够经受住三年规模化的考验。

**先修知识：** 阶段10·14（开源模型架构），阶段10·17（原生稀疏注意力）

## 学习目标

- 解释Jamba模块中的三种基本单元——Transformer层、Mamba层、MoE——以及1:7:交错的混合配方。

- 说出Mamba-3的三项创新（指数梯形离散化、复数值状态更新、MIMO）以及它们各自针对的问题。

## 存在的问题

注意力的计算复杂度随序列长度呈二次增长。状态空间模型是线性的。这种差异会急剧放大：在256k token处，一个Transformer的注意力图每头有650亿个条目；而SSM的循环状态大小固定，与序列长度无关。

纯SSM模型（Mamba、Mamba-2）在小规模下能匹配Transformer的困惑度，但在状态跟踪任务上落后，并且在某些上下文检索类别上失败。直觉上：SSM将历史压缩为固定状态，当历史很长时信息会泄露。Transformer能完美记住所有内容，但付出二次成本。

显而易见的解决方案是两者都用。在需要精确回忆的地方放置Transformer层，其他地方使用SSM层。调整比例。Jamba是首个规模化部署这种混合配方（520亿总参、120亿活跃、256k上下文、单张80GB GPU）的生产级模型。Jamba 1.5将家族扩展到3980亿总参/940亿活跃。Mamba-3（ICLR 2026）是目前最好的纯SSM基线，混合模型可以围绕它重建。

本课程阅读这三篇论文，并形成“选择正确比例”的心智模型。

## 概念

### 一页读懂SSM

状态空间模型通过固定大小的状态 `h` 处理序列 `x_1, ..., x_N`：

```
h_t = A h_{t-1} + B x_t
y_t = C h_t
```

每一步状态通过线性动力学 `A` 演化，接收输入 `B x_t`，并输出 `C h_t`。`A, B, C` 可学习。注意到关键性质：计算 `y_t` 只需 `h_{t-1}` 和 `x_t`，不需要任何更早的 `x`。内存恒定。推理每个token的复杂度为O(1)。

建模质量的关键在于 `A` 的结构。S4（Gu 2021）使用高度结构化的矩阵，可以在训练期间作为长卷积高效计算。Mamba（Gu, Dao 2023）将固定的 `A, B, C` 替换为数据相关的（即“选择性”部分）。Mamba-2（2024）进一步简化了结构。Mamba-3（2026）在特定位置重新增加了复杂性。

关键性质：对于解码器LLM而言，SSM层可以直接替代注意力层，每层状态大小固定，而非不断增长的KV缓存。

### Jamba模块

Jamba模块根据两个数字交错排列层：

- `l`：注意力与Mamba的比例。Jamba使用 `l = 8`，即每7个Mamba层配1个Transformer层（7个Mamba + 1个注意力 = 每组8层）。
- `e`：MoE频率。Jamba使用 `e = 2`，表示隔层应用MoE。

一个模块内的层序列：

```
M  M  M  M  M  M  M  A    (7个Mamba + 1个注意力)
|  M  |  M  |  M  |  M    (| 标记应用了MoE的层)
```

每个Jamba模块为8层。在4个模块深度（共32层）时，得到28个Mamba层和4个注意力层。其中16层使用了MoE。

### 为什么是1:7比例

AI21进行了消融实验：注意力与Mamba的什么比例能在长上下文评估中给出最佳的每参数困惑度和上下文检索性能？

- 注意力过多（1:1）：质量上升但内存和速度下降。
- 注意力过少（1:15）：内存极佳但上下文检索失败。
- 最佳点：1:7或1:8。

直觉：Transformer层处理精确回忆和状态跟踪，Mamba层处理廉价的大批量处理。

### 位置编码

Mamba层本身具有位置感知能力（通过循环）。原始基于Mamba的混合模型中的注意力层未使用RoPE——SSM层提供了位置信息。Jamba 1.5为注意力层添加了RoPE以实现更长的上下文泛化，这是基于经验性长上下文评估的事后改进。

### 内存预算

对于Jamba-1的配置（32层：28个Mamba + 4个注意力，隐藏层4096，32个注意力头）：

- KV缓存（仅注意力层）：`2 * 4 * 32 * 128 * 256k * 2 = 8.4 GB`（BF16，256k上下文）。只有4个注意力层贡献。
- SSM状态：每个token前缀 `28 * hidden * state_size`，但这是每层的固定大小，不随序列长度缩放。典型Mamba状态为每个特征16，隐藏层4096：`28 * 4096 * 16 * 2 = 3.7 MB` 总计。

与纯Transformer对比：32层、相同隐藏层、全MHA 32头：`2 * 32 * 32 * 128 * 256k * 2 = 128 GB`（BF16，256k上下文）。KV缓存减少了8倍。即使与大多数2024年模型使用的GQA(8)基线（`2 * 32 * 8 * 128 * 256k * 2 = 32 GB`）相比，Jamba的1:7混合模型在16 GB下仍然小2倍。

这就是AI21所说的“256k上下文运行在单张80GB GPU上”的含义。全MHA纯Transformer的KV缓存放不下；即使是GQA基线也没有空间容纳权重和激活值；而Jamba可以。

### Mamba-3：2026年的纯SSM基线

Mamba-3（ICLR 2026，arXiv:2603.15569）在纯SSM侧引入了三项创新：

1. **指数梯形离散化（Exponential-trapezoidal discretization）**。将Mamba-2中的欧拉方法离散化替换为更具表现力的循环。在核心循环内部对状态-输入应用类似卷积的操作，而非作为 `x_t` 的外部卷积。
2. **复数值状态更新（Complex-valued state update）**。之前的Mamba将状态矩阵从复数（S4）降维到实对角（Mamba），再到缩放单位矩阵（Mamba-2）。Mamba-3重新引入复数值——相当于在状态上施加数据依赖的旋转位置编码。这恢复了先前实数值简化所损失的状态跟踪能力。
3. **多输入多输出投影（Multi-input multi-output, MIMO）**。使用矩阵值投影替代每特征的标量投影。在不增加解码延迟的情况下，提升了建模能力和推理时的硬件利用率。

在1.5B参数规模下，Mamba-3的平均下游准确率比Gated DeltaNet提高0.6点；MIMO变体额外增加1.2点，总计1.8点提升。在相同状态大小下，Mamba-3以一半的状态大小匹配Mamba-2。

Mamba-3尚未大规模部署于生产混合模型——但它是下一个Jamba级模型SSM侧的首选候选。

### 何时采用混合模型

混合模型在以下情况下胜出：

- 上下文足够长，使得纯Transformer的KV缓存变得痛苦（64k以上）。
- 任务混合了短程结构（适合SSM）和长程回忆（需要Transformer）。
- 你希望在单GPU内存预算下部署，而纯Transformer的KV缓存本身放不下。

混合模型在以下情况下失败：

- 上下文较短（16k以下）。SSM开销浪费，纯Transformer即可。
- 任务需要全局注意力（深度推理、多文档交叉引用）。混合模型中注意力层的稀疏性会带来损失。
- 你正扩展到万亿参数前沿模型。纯Transformer + MLA + MoE（DeepSeek-V3风格）目前正在能力竞赛中胜出。

### 竞争格局

| 模型 | 家族 | 规模 | 独特宣称 |
|-------|--------|------|-------------|
| Mamba-2 | 纯SSM | 3B | 线性时间，恒定内存 |
| Jamba | 混合 | 52B/12B | 256k在80GB GPU上 |
| Jamba 1.5 Large | 混合 | 398B/94B | 企业级长上下文 |
| Mamba-3 | 纯SSM | 1.5B（论文） | 状态跟踪恢复 |
| DeepSeek-V3 | 纯Transformer + MoE | 671B/37B | 前沿能力 |

2026年格局：纯Transformer MoE主导前沿，但混合模型拥有256k以上上下文这一利基市场。Mamba-3在状态跟踪上的胜出可能推动下一代混合比例降低（更多SSM，更少注意力）。

## 使用它

`code/main.py` 是一个混合架构的内存计算器。给定SSM-Transformer比例和隐藏层大小/层数配置，它计算：

- 目标上下文下的KV缓存。
- SSM状态内存。
- 在不同模型形状下上下文N处的总内存。

计算器支持：

- 纯Transformer基线（KV缓存随N增长）。
- Jamba风格的1:7混合。
- 纯SSM（完全没有KV缓存）。

这些数字直接来自Jamba-1和Jamba-1.5论文的发布形状，并对假设变体进行外推。

实际部署的集成考虑：

- 大多数生产推理服务器（vLLM、SGLang）支持Jamba和Mamba。检查具体版本。
- 在256k上下文下，Jamba的内存优势体现在并发请求吞吐量上。在相同VRAM上，你可以容纳比Transformer序列更多的Jamba序列。
- Mamba-3作为独立模型尚未在生产中发布——研究预览仅1.5B。

## 关键术语

| 术语 | 人们常说的 | 实际含义 |
|------|----------------|------------------------|
| 状态空间模型（SSM） | “具有固定状态的循环” | 一个带有学习到的循环 `h_t = A h_{t-1} + B x_t` 的层；每个token内存恒定 |
| 选择性SSM | “Mamba的窍门” | 数据相关的A、B、C参数，使模型在线性时间内具有类似门控的选择性 |
| 注意力与Mamba比例 | “多少个注意力层” | 在Jamba中，`l = 8` 表示每7个Mamba层有1个注意力层 |
| Jamba模块 | “那8层一组” | 一个注意力层加七个Mamba层，并隔层应用MoE |
| SSM状态 | “隐藏缓冲区” | 每层固定大小的状态，替代Mamba层的KV缓存 |
| 256k上下文 | “Jamba的旗舰数字” | Jamba-1在单张80GB GPU上能容纳的序列长度；纯Transformer在这个大小下无法做到 |
| Mamba-3 | “2026年纯SSM” | 当前最好的纯SSM架构，具有复数状态 + MIMO；混合模型重建的基线 |
| MIMO | “多输入多输出” | Mamba-3创新，使用矩阵值投影替代每特征 scalar 投影 |
| 指数梯形离散化 | “Mamba-3的循环” | 比Mamba-2的欧拉方法离散化更具表现力的循环 |
| 混合架构 | “混合注意力与SSM” | 任何交错放置Transformer和SSM层的模型；Jamba是生产级原型 |
