# 差分注意力机制（Differential Attention）（V2版）

> Softmax注意力会在每个不匹配的词元上分配少量概率。当词元数量超过10万时，这种噪声累积起来会淹没信号。差分Transformer（Ye等人，ICLR 2025）通过将注意力计算为两个softmax的差值来消除共享的噪声基底，从而解决了这个问题。DIFF V2（微软，2026年1月）是基于生产环境的重写版本：解码延迟与基线Transformer持平，无需定制内核，兼容FlashAttention。本教程将从V1到V2进行端到端讲解，并提供一个可在stdlib Python中运行的简易差值操作实现。

**类型：** 构建
**语言：** Python（stdlib）
**前置知识：** 阶段7·02（自注意力机制）、阶段7·15（注意力变体）、阶段10·14（架构详解）
**时长：** 约60分钟

## 学习目标

- 精确阐述为什么softmax注意力存在噪声基底，以及该基底如何随上下文长度增长。
- 推导差分注意力公式，并解释为什么减法能在保留信号的同时抵消共享噪声成分。
- 对比V1到V2的差异：哪些部分更快、更简单、更稳定，以及为什么每个更改对生产环境预训练都是必要的。
- 用纯Python从头实现差分注意力，并在合成信号加噪声的查询上实证验证噪声抵消特性。

## 问题所在

标准softmax注意力在数学上有一个特性，当规模扩大时会给操作带来麻烦。对于查询`q`，注意力权重为`softmax(qK^T / sqrt(d))`。Softmax永远无法产生精确的零——每个不匹配的词元都会获得一些正质量。这些残余质量就是噪声，并且它随上下文长度缩放。在128k个词元下，即使每个不匹配词元只获得0.001%的概率，127,999个词元的合计贡献也占到总量的约12%。模型必须学会绕过这个随上下文增长的噪声基底。

在实践中，这表现为注意力头干扰：长上下文RAG中的幻觉引用、100k词元检索任务中的中间位置丢失（lost-in-the-middle）失败，以及超过32k词元的大海捞针基准测试中的细微精度下降。差分Transformer论文（arXiv:2410.05258，ICLR 2025）量化了这一差距：DIFF Transformer在相同大小的基线上实现了更低的困惑度、更高的长上下文准确率和更少的幻觉。

DIFF V1存在三个问题，使其无法进入前沿预训练流程。它的值缓存每次解码步骤需要加载两次，需要破坏FlashAttention兼容性的定制CUDA内核，并且每个头的RMSNorm在70B以上规模的长时间训练中不稳定。DIFF V2（微软unilm博客，2026年1月20日）修复了所有三个问题。本教程将讲解两个版本，构建差分算子，并在一个玩具查询上基准测试噪声抵消效果。

## 概念

### softmax的噪声基底

对于查询`q`和键`K = [k_1, ..., k_N]`，注意力权重为：

```
w_i = exp(q . k_i / sqrt(d)) / sum_j exp(q . k_j / sqrt(d))
```

没有任何`w_i`为零。如果`k_i`与`q`完全无关，则分数`q . k_i`不是0——它围绕零波动，方差为`||q||^2 / d`。经过softmax归一化后，每个无关词元仍然贡献`O(1/N)`给加权和。无关词元的总贡献为`O((N-1)/N) = O(1)`——这不是一个小量。

模型想要的是某种硬top-k：匹配词元上的高权重，其他地方接近零权重。Softmax太平滑，无法直接做到这一点。

### 差分思想

将每个头的Q和K投影分成两部分：Q = (Q_1, Q_2) 和 K = (K_1, K_2)。计算两个注意力图：

```
A_1 = softmax(Q_1 K_1^T / sqrt(d))
A_2 = softmax(Q_2 K_2^T / sqrt(d))
```

输出：

```
DiffAttn = (A_1 - lambda * A_2) V
```

减法会抵消两个图共享的任何噪声分布。如果两个图在127k个无关词元上都有大致均匀的权重（在随机初始化时确会如此），这些权重就会抵消。信号——在少数真正相关词元上的尖峰权重——只有以相同幅度出现在两个图中时才会被抵消，而在模型训练后这种情况不会发生。

`lambda`是一个每个头可学习的标量，参数化为`lambda = exp(lambda_q1 dot lambda_k1) - exp(lambda_q2 dot lambda_k2) + lambda_init`。它可以为负值。`lambda_init`默认值为像0.8这样的小正数。

### 为什么这与带头噪消除匹配

想象两个噪声麦克风录制同一声音。两者都拾取说话者以及相关的背景噪声。将一个从另一个中减去，共享的噪声就会消失。声音幸存下来，因为两个信号在相位或幅度上存在足够差异，从而防止完全抵消。每个头的`lambda`正是学习这种平衡。

### V1 vs V2：差异

V1保持参数量与基线Transformer相等。为了每个头获得两个查询，它将头维度减半。这牺牲了头的表达能力，并且——更糟糕的是——每个头的值缓存减半。解码时必须每步加载两次值缓存（每个softmax分支一次）。结果是尽管参数量匹配，但解码速度比基线慢。

V2将查询头数量加倍，同时保持KV头不变（从上投影借用参数）。头维度与基线保持一致。减法之后，额外的维度被投影回原始大小，以匹配基线Transformer的O_W投影。这样一来同时实现了三件事：

1. 解码速度与基线持平（KV缓存只加载一次）。
2. FlashAttention无需修改即可运行（无需定制内核）。
3. 解码时的算术强度提高（每从HBM加载一个字节执行的运算更多）。

V2还移除了V1用于稳定减法的每个头RMSNorm。在70B级别的预训练规模下，该RMSNorm会在后期训练中引发不稳定。V2用更简单的初始化方案替代了它，无需额外模块即可保持训练稳定。

### 何时使用

| 工作负载 | 优势 |
|----------|------|
| 长上下文RAG（64k以上） | 更干净的注意力图，更少的幻觉引用 |
| 大海捞针基准测试 | 超过32k后准确率显著提升 |
| 多文档问答 | 减少跨文档干扰 |
| 8k下的代码补全 | 效果微乎其微，不值得架构改动 |
| 短聊天（< 4k） | 与基线基本无区别 |

价值随上下文长度增长。在4k词元时，噪声基底足够小，标准注意力已足够。在128k时，它会损害性能。

## 动手构建

`code/main.py`用纯Python实现了差分注意力。通过一个已知信号加噪声结构的玩具查询，可以直接测量噪声抵消比率。

### 第一步：标准softmax注意力

stdlib矩阵运算：列表的列表，手动矩阵乘法，带数值稳定性的softmax（减去最大值）。

```python
def softmax(row):
    m = max(row)
    exps = [math.exp(x - m) for x in row]
    s = sum(exps)
    return [e / s for e in exps]
```

### 第二步：将Q、K分成两半

V1风格：将头维度减半。V2风格：保持头维度不变，将头数加倍。玩具实现采用V1风格以便教学清晰——数学完全相同，仅记账方式不同。

### 第三步：两条softmax分支 + 减法

```python
A1 = [softmax([dot(q1, k) / scale for k in K1]) for q1 in Q1]
A2 = [softmax([dot(q2, k) / scale for k in K2]) for q2 in Q2]
diff_weights = [[a1 - lam * a2 for a1, a2 in zip(r1, r2)] for r1, r2 in zip(A1, A2)]
out = [[sum(w * v[j] for w, v in zip(row, V)) for j in range(d_v)] for row in diff_weights]
```

注意：输出权重可能为负值。这没问题——值缓存仍然处理有符号贡献。后续的V投影会吸收符号。

### 第四步：噪声抵消测量

构建一个长度为1024的合成序列。将信号词元放在已知位置，其余部分填入噪声。计算（a）标准softmax注意力在信号位置上的权重，以及（b）差分注意力权重。测量每个位置的信号噪声比。根据两个分支训练后的差异程度，DIFF注意力可靠地将信噪比提高3-10倍。

### 第五步：V1和V2的参数量核算

给定一个配置（hidden=4096, heads=32, d_head=128），输出：

- 基线Transformer：Q、K、V各为`hidden * hidden`，MLP为4 * hidden。
- DIFF V1：Q、K各为`hidden * hidden`，V为`hidden * hidden`（不变），头维度内部减半。增加了每个头的`lambda`参数（O(heads * d_head)）。
- DIFF V2：Q为`2 * hidden * hidden`，K为`hidden * hidden`，V为`hidden * hidden`。额外维度在O_W之前被投影回原始大小。增加了相同的`lambda`参数。

玩具实现会测量V2的额外参数成本（每个注意力块大约`hidden * hidden`额外参数）并输出。

## 使用它

截至2026年4月，DIFF V2尚未在每个生产推理服务器中部署，但集成工作正在vLLM和SGLang中进行。同时，该模式已在以下场景中出现：

- 微软内部长上下文生产模型。
- 几项针对256k以上上下文的开源模型训练研究的复现。
- 混合架构，将DIFF注意力与滑动窗口注意力在交替层中结合。

在2026年何时会使用它：

- 从头训练一个针对64k以上有效上下文的新模型。从一开始就加入差分注意力；后续重新训练代价高昂。
- 微调一个长上下文模型，其中间位置丢失失败占主导地位。对Q投影进行LoRA可以近似DIFF结构。

何时不会使用它：

- 你正在部署一个预训练的密集模型，其长上下文性能已经稳定。重新训练的成本很少能在现有权重上收回。
- 你的上下文始终低于16k。噪声基底可忽略不计。

## 关键术语

| 术语 | 通常说法 | 实际含义 |
|------|----------|----------|
| 差分注意力 | “两个softmax相减” | 将Q、K分成两半，计算两个softmax图，将第二个图（按lambda缩放）从第一个中减去，然后乘以V |
| 噪声基底 | “softmax的非零尾部” | softmax在每个无关词元上分配的O(1/N)权重，在长上下文中总和为O(1) |
| lambda | “减法缩放因子” | 每头可学习的标量参数，参数化为`exp(lq1.lk1) - exp(lq2.lk2) + lambda_init`；可为负值 |
| DIFF V1 | “ICLR 2025版本” | 原始差分Transformer；为保持参数数量将头维度减半，需要定制内核，解码较慢 |
| DIFF V2 | “2026年1月的修复” | 保持KV头不变，将Q头加倍；解码速度与基线持平，兼容FlashAttention |
| 每头RMSNorm | “V1的稳定器” | V1在差值后应用的额外归一化；V2移除了它以阻止后期训练不稳定 |
| 信噪比 | “注意力浪费了多少” | 注意力权重在真实信号位置上的值与无关位置平均值的比率 |
| 中间位置丢失 | “长上下文失败模式” | 经验现象：检索准确率在长上下文中间位置的文档下降——DIFF注意力可减轻此问题 |
| 算术强度 | “每加载一个字节执行的FLOPs” | V2在解码时通过每次KV加载执行双倍查询来提高的比率；对内存受限的解码很重要 |
