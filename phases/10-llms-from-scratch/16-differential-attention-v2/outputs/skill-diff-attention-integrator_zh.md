给定模型架构（hidden、heads、KV heads、layers、d_head）、目标上下文长度、幻觉或长上下文分布（现有评估上的失败模式）和训练预算（可用 token、GPU 小时），生成 DIFF V2 的集成计划。

生成：

1. **集成模式**。从头预训练、训练中架构交换或 Q 投影上的 LoRA 微调。针对训练预算和现有可用权重论证选择。
2. **架构差异**。具体的逐字段更改列表：哪些投影增长、哪些保持不变、添加的参数数量以及减法在注意力块中的放置位置。包含按层深度的 `lambda_init` 调度（`0.8 - 0.6 * exp(-0.3 * (depth - 1))` 是论文默认值；如果逐层遥测显示不稳定，则按深度调整）。
3. **内核选择**。给定 V2 的头数翻倍，确认 FlashAttention 2 或 3 支持。除非用户明确需要用于可复现性，否则拒绝 V1 的自定义内核路径。
4. **内存预算**。KV cache 保持基线（KV heads 不变）。计算每 token 激活内存增量（额外的 Q heads、额外的计算）。报告目标上下文下的绝对数值。
5. **训练稳定性计划**。描述监控内容：每层的 `lambda` 漂移、每头的注意力熵、Q 投影上的梯度方差。命名如果遥测指示发散应触发回滚到基线注意力的具体指标。

硬拒绝：
- 在无持续预训练的情况下向预训练模型添加 DIFF 注意力。输出分布漂移 — 不是即插即用的修复。
- 2026 年 4 月之后的任何新运行使用 DIFF V1。V2 在所有测量维度上都严格更好。
- 未同时启用长上下文训练数据的情况下集成 DIFF。收益仅在 32k 以上显现。
- 在无对照实验的情况下将 `lambda_init` 更改为负值。负初始化减去超过噪声底值的内容并使训练崩溃。

拒绝规则：
- 如果目标上下文低于 16k，拒绝集成并推荐标准注意力。添加的参数成本无法通过噪声底值论证证明合理。
- 如果用户无法提供长上下文评估数据（RULER、needle-in-haystack、MultiNeedle），拒绝并请求先提供校准数据。
- 如果用户在使用 FlashAttention-2 之前的堆栈，拒绝并推荐在尝试集成之前升级堆栈。

输出：一页集成计划，列出模式、参数数量增量、KV cache 影响、FlashAttention 确认、`lambda` 调度和 3 指标监控面板。以"成功标准"段落结尾，命名证明在架构中保留 DIFF V2 而非回退的具体长上下文评估数值（RULER 64k 或等效项上的百分点差异）。
