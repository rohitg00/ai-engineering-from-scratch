给定一个拟议的训练流水线清单（描述分词器、数据、预训练、SFT、对齐、评估、量化和 serving 阶段的 YAML 或 JSON），生成一份审查报告，涵盖：

1. **阶段图**。确认每个阶段都有类型化的输入和输出。指出缺失的依赖项、隐式状态，或任何消耗裸目录而非命名工件哈希的阶段。
2. **哈希链**。验证阶段 N 的 output_hash 等于每个下游阶段的 input_hashes 之一。任何不匹配意味着清单不连贯，流水线不得启动。
3. **评估门**。门列表中的每个指标必须是数字的，有运算符、阈值和测量来源。拒绝任何主观的（"看起来不错"）、无界的（无阈值）或在训练数据上测量的门。
4. **回归保护**。新模型的核心基准（MMLU、MATH、HumanEval+、GPQA 或领域特定等效项）必须附加基线数字。没有基线的运行就是没有回归检测的运行。
5. **KL 预算**。对齐阶段（RLHF、DPO、CAI、GRPO）必须声明对参考模型的累积 KL 上限。无界 KL 就是无界漂移。
6. **污染检查**。训练数据分片和评估集必须有记录的重叠检查（精确匹配或 13-gram）。要求的通过阈值：<0.1%。
7. **成本估算**。每个阶段的预运行估算加上总计，与预算门比较。如果估算 > 预算，流水线拒绝启动。
8. **回滚计划**。每个阶段在失败时的命名操作：重新运行、回退到上一个工件、修订输入并重新运行下游。昂贵阶段（预训练）必须有热检查点策略。
9. **工件存储**。检查点、数据集、分词器、评估报告必须是内容寻址的（SHA-256）。文件名寻址的工件（"latest.pt"）是硬拒绝。
10. **可观测性**。每个阶段必须发出结构化日志，包含跟踪 ID、阶段名称、输入哈希、输出哈希、实际耗时和成本。缺失跟踪 ID 意味着运行事后无法调试。

停止审查的危险信号：
- 门缺失测量来源（对没有阶段计算的指标设门）
- 与下游阶段共享检查点的阶段（无关注点分离）
- 没有参考模型的对齐阶段（KL 无锚点）
- LLM-as-judge 评估中，评判与策略模型同一家族（污染）
- 成本估算超过预算 20% 以上
- 回滚计划仅由"从头重新运行"组成

输出：两页审查报告，每道门通过/保留，产生每个裁决的确切清单字段或缺失字段，以及将保留翻转为通过所需的最小更改。
