# 分词器选择与实现

启动 LLM 项目时，使用以下决策框架来选择分词器。

## 何时使用每种分词器

**字节级 BPE (tiktoken)：** 你正在基于 GPT 系列模型进行构建或微调。你需要保证处理任何输入字节序列。你不希望出现未知 token。

**WordPiece (Hugging Face)：** 你正在使用 BERT 系列模型进行分类、NER 或嵌入任务。你需要 "##" 延续前缀，用于依赖词边界信号的下游任务。

**SentencePiece (BPE 或 Unigram)：** 你正在从头训练。你需要语言无关的分词。你的数据包含 CJK 语言、泰语或其他没有空格分词边界的文字。LLaMA、T5 和大多数多语言模型使用此方案。

## 词表大小指南

- 32K token：单语言模型的良好默认值，保持嵌入层较小
- 50K-64K token：更适合多语言或代码密集型模型
- 100K+ token：仅当你拥有海量训练数据且希望序列更短时使用

更大的词表意味着更短的序列（推理更便宜），但嵌入矩阵中的参数更多。对于 100K 词表和 4096 维嵌入，仅嵌入层就有 400M 参数。

## 重要的预分词规则

1. 在 BPE 之前按空格拆分，防止跨词合并
2. 如果你希望模型学习算术，则将数字单独拆分
3. 分词前进行 Unicode 规范化 (NFC)，确保行为一致
4. 为你的用例添加特殊 token：`<pad>`、`<eos>`、`<bos>`、`<unk>`，以及任何任务特定标记

## 分词器行为中的危险信号

- 目标语言的 fertility 高于 2.0：模型浪费上下文窗口
- 常见领域词被拆分为 3 个以上 token：用领域数据重新训练
- 数字分词不一致：检查数字拆分规则
- 词表很大但有很多仅用一次的 token：减小词表大小

## 构建自定义分词器 - 检查清单

1. 收集代表性训练数据（目标领域至少 1GB 文本）
2. 选择算法：通用场景用 BPE，多语言用 Unigram
3. 根据上述指南设置词表大小
4. 配置预分词：空格拆分、数字处理、标点符号
5. 添加特殊 token
6. 使用 Hugging Face tokenizers 库训练（Rust 后端，速度快）
7. 验证：在所有目标语言的保留文本上检查 fertility
8. 测试边界情况：空字符串、超长输入、二进制数据、emoji、RTL 文本
9. 保存并对分词器进行版本管理，与模型检查点一起存放
