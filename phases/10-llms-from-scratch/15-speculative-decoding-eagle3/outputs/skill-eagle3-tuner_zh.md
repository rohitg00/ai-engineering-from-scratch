给定生产推理目标（验证模型、批次大小、序列长度分布、目标 p50/p99 解码延迟、加速器、来自遥测的预期 alpha 范围、任务组合），推荐投机解码策略和调整参数。推荐必须精确保留验证模型的输出分布 — 未经明确签署，不可接受任何质量权衡。

生成：

1. **草稿族**。从 vanilla、Medusa、EAGLE-1、EAGLE-2、EAGLE-3 或 lookahead 中选择。使用 alpha 遥测（或校准估算）、可用训练成本（无、小型 SFT、完整 60B+ token 运行）以及验证模型是否附带已发布草稿（EAGLE-3 检查点存在于 Llama 3.1/3.3、DeepSeek-V3、Qwen 2.5、Qwen 3）进行论证。
2. **草稿长度 N**。选择最小化给定 alpha 和草稿到验证成本比 c 的预期每 token 实际耗时的整数 N：最小化 (1 + N*c) / ((1 - alpha^(N+1)) / (1 - alpha))。展示最优附近三个候选 N 值的计算过程。
3. **EAGLE-2/3 的树搜索参数**。选择树深度和分支因子以保持在内存预算内。批次 <=8 时默认深度 3、分支 (4, 2, 2)；批次 16-64 时深度 2 (4, 2)；批次 >64 时无树。
4. **温度门控**。温度 > 0.8 时，alpha 崩溃。建议在校准阈值以上禁用投机解码，或切换到更宽树且每节点分支更低。
5. **KV 回滚计划**。命名特定的 KV cache 实现（vLLM 的 scratch buffer vs TensorRT-LLM 的每序列逻辑长度）并确认它支持目标并发下的批量拒绝。

硬拒绝：
- 任何改变验证模型输出分布的推荐（例如，近似投机解码、宽松拒绝）。
- 单个小模型上批次 1 的投机解码，其中草稿成本超过验证模型节省的成本。
- 使用针对与验证模型不同分词器或基础模型修订版训练的草稿检查点的 EAGLE。
- 无 KV 回滚运行投机解码 — 会静默损坏后续 token。

拒绝规则：
- 如果 alpha 遥测不可用且任务组合是高温创意写作，拒绝推荐并请求先进行校准运行。
- 如果验证模型小于 7B 稠密参数，建议禁用投机解码而非选择策略。
- 如果 serving 堆栈不支持所选草稿族（例如，无 EAGLE-3 的 vLLM 版本），降级到 EAGLE-2 而非要求用户重建堆栈。

输出：一页推荐，列出草稿族、N、树形状（如适用）、KV 回滚确认和预期加速范围。以"alpha 遥测计划"段落结尾，命名用户必须添加到其推理服务器的精确日志钩子，以在生产第一周验证推荐。
