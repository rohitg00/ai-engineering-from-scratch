给定长上下文预训练运行规范（目标上下文、基础架构、可用训练 token、GPU 拓扑、部署目标），生成 NSA 集成计划。

生成：

1. **压缩块大小 `l`**。选择 32、64 或 128。针对目标上下文论证：`l = 32` 用于 16k-32k，`l = 64` 用于 64k-128k，`l = 128` 用于 256k 以上。更大的 `l` 意味着更少的压缩键但更粗的路由信号。
2. **Top-k 选择数量**。在 8 到 32 之间选择。论文默认值为 16。针对目标任务组合论证：推理密集型任务（数学、代码）受益于更高的 `k`，因为选择精度更重要。检索密集型任务在较低的 `k` 下工作。
3. **滑动窗口 `W`**。选择 256、512 或 1024。默认 512。对于本地上下文足够的重度结构化内容（代码）更短；对于散文更长。
4. **门控 MLP**。指定宽度和初始化。默认：从 `hidden` 到 3 的线性层，使用 `sigmoid` 或 `softplus` 激活。警告如果门控权重崩溃到偏向一个分支 — 这表明 `l`、`k` 或 `W` 调谐错误。
5. **内核选择**。确认目标加速器的 Triton 或 CUDA 内核可用性。拒绝在推理时回退到稠密注意力（NSA 的全部意义在于节省解码计算）。如果只有前向内核存在而没有反向内核，拒绝预训练并推荐在现有稠密检查点上继续训练。

硬拒绝：
- 在未持续预训练的情况下，对使用稠密注意力预训练的模型使用 NSA。无法在推理时附加。
- 目标上下文低于 16k。三分支开销占主导。
- 在没有 NSA 内核支持的堆栈上进行仅推理部署。推荐改用 MLA 或滑动窗口注意力。

拒绝规则：
- 如果没有长上下文评估数据（RULER、LongBench、needle-in-haystack），拒绝并请求先提供校准数据。
- 如果训练数据上下文分布以短序列为主，拒绝并推荐在集成 NSA 之前进行数据重新加权。
- 如果加速器早于 A100，拒绝 — NSA 的内核优势假设 H100/H200/MI300 内存层次结构。

输出：一页集成计划，列出 `l`、`k`、`W`、门控配置、内核路径和目标上下文下的预期计算节省。以"成功标准"段落结尾：证明保留 NSA 的具体 RULER 或 LongBench 数值（与匹配的稠密注意力基线相比的百分点）。包含回滚触发器 — 架构应回退到 MLA 或稠密 GQA 的指标阈值。
