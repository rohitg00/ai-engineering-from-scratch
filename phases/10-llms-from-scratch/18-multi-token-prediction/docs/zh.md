# 多令牌预测 (MTP)

> 从GPT-2到Llama 3，每个自回归LLM都在每个位置使用一个损失函数进行训练：预测下一个令牌。DeepSeek-V3在每个位置添加了第二个损失函数：预测那个令牌之后的令牌。额外的140亿参数（在671B模型上）通过梯度流蒸馏回主模型，训练好的MTP头在推理时被重新用作推测解码的草稿生成器，接受率超过80%。1.8倍的生成吞吐量是免费获得的。本课程根据DeepSeek技术报告构建顺序MTP模块，计算损失和共享头参数布局，并解释为什么MTP保持了因果链，而Gloeckle等人的原始并行MTP破坏了它。

**类型：** 构建
**语言：** Python (stdlib)
**先决条件：** 第10阶段·04（预训练一个小型GPT），第10阶段·15（推测解码）
**时间：** 约60分钟

## 学习目标

- 阐述MTP训练目标并推导跨预测深度的联合损失。
- 解释Gloeckle等人的并行MTP头（2024）与DeepSeek-V3的顺序MTP模块之间的区别，以及为什么顺序设计保留了因果链。
- 计算在预训练运行中添加MTP模块的参数和内存开销。
- 从零开始实现一个MTP模块：共享嵌入、每深度transformer块、投影和共享输出头。

## 问题

下一个令牌预测是标准的LLM训练目标。每个隐藏状态都被监督来预测一件事：紧随其后的令牌。这是一个出奇弱的信号。序列中的大部分信息超出了一个令牌的范围——结构、连贯性、事实性、算术流程。模型必须通过积累数万亿个单令牌信号来学习这些。

MTP提出：如果每个隐藏状态都被监督来同时预测多个未来令牌会怎样？Gloeckle等人（Meta，2024）表明这有所帮助。他们的实现是在骨干模型上放置了几个独立的输出头，每个头预测不同的偏移量。并行、简单，但这些头看到了相同的隐藏状态，没有任何层次化的改进——并且预测没有因果链，因此它们不能用于推测解码。

DeepSeek-V3（2024年12月）重新设计了MTP作为顺序模块，在每个预测深度保持因果链。模型从`h_i^(0)`预测`t+1`，然后从结合了`h_i^(0)`和`E(t+1)`嵌入的新隐藏状态`h_i^(1)`预测`t+2`，依此类推。每个深度都有自己的小型transformer块。共享嵌入和共享输出头保持适度的参数开销。在DeepSeek-V3的规模下，在671B主模型权重之上，MTP模块共有140亿额外参数。这2%的开销获得了更密集的训练信号以及在推理时现成的推测解码草稿。

本课程从头开始构建单个MTP模块和D深度损失。数学很简洁。实现代码为150行。

## 概念

### 顺序MTP配方

DeepSeek-V3在主模型之上添加了`D`个MTP模块。每个模块`k`（对于`k = 1..D`）预测深度`k`处的令牌——即，给定通过位置`i`的前缀，预测`t_{i+k}`。

模块`k`由以下部分组成：

- 一个具有自己注意力机制和MLP的transformer块`T_k`。
- 一个投影矩阵`M_k`，它结合前一深度的隐藏状态和下一深度真实令牌的嵌入。
- 共享嵌入`E`（与主模型相同）。
- 共享输出头`Out`（与主模型相同）。

在训练时，对于通过位置`i`的前缀，每深度隐藏状态为：

```
h_i^(0) = 主模型在位置i的骨干
h_i^(k) = T_k( M_k * concat(RMSNorm(h_i^(k-1)), RMSNorm(E(t_{i+k}))) )   for k >= 1
```

每深度预测为：

```
logits_{i+k} = Out(h_i^(k-1))   for k = 1..D
```

每深度损失是针对真实令牌`t_{i+k}`的交叉熵：

```
L_k = CE(logits_{i+k}, t_{i+k})
```

跨深度的联合损失：

```
L_MTP = (lambda / D) * sum_{k=1..D} L_k
```

`lambda`是一个小的加权因子——DeepSeek-V3在训练的前10%使用0.3，之后使用0.1。总训练损失是`L_main + L_MTP`。

### 为什么是顺序的，而不是并行的

Gloeckle的原始并行MTP有D个输出头，每个都直接应用于`h_i^(0)`。每个头从相同的骨干隐藏状态预测`t_{i+k}`。这训练得很好，但预测不是相互依赖的。你不能使用`head_1`的输出来帮助`head_2`——这些头是并行触发的。

DeepSeek-V3的顺序设计从`h_i^(k-1)`加上实际的下一个令牌嵌入`E(t_{i+k})`构建`h_i^(k)`。这保留了因果链：为了预测`t_{i+k+1}`，深度`k+1`处的模块看到了`t_{i+k}`处的内容。这在结构上与自回归解码器如何消耗自身输出相同——使MTP模块可以直接用作推测解码的草稿生成器。

在推理时：将`h_i^(k-1)`和草稿化的`t_{i+k}`输入模块`k+1`，得到`t_{i+k+1}`的预测。重复。这完全是一种EAGLE风格的草稿，使用训练好的MTP模块作为草稿网络。DeepSeek-V3报告第一个MTP模块的接受率超过80%和约1.8倍的加速。

### 参数计算

对于具有隐藏层`h`和词汇表`V`的模型：

- 主模型：数十亿参数，加上一个大小为`V * h`的输出头。
- 共享输出头：重用主模型的头。没有额外参数。
- 共享嵌入：重用主模型的嵌入。没有额外参数。
- 每个MTP模块：
  - 投影`M_k`：`(2h) * h = 2h^2`。
  - Transformer块`T_k`：注意力（MHA为`4h^2`）加上MLP（通常对于比例为8/3的SwiGLU为`8h^2`）。每块约`12h^2`。

每个模块的总额外参数：`~14h^2`。对于DeepSeek-V3的`h = 7168`，D = 1个模块：理论上`~14 * 7168^2 = ~720M`参数。DeepSeek-V3报告140亿——差异主要是MTP模块中的专家层也是MoE。

### 推测解码的回报

在预训练期间，MTP模块使训练速度减慢约10%（更多的前向计算，额外损失）。回报是双重的：

1. 更密集的训练信号。每个隐藏状态看到D+1个监督目标。在MMLU、GSM8K、MATH、HumanEval上测量的效果：在DeepSeek-V3的消融实验中，有持续几个百分点的改进。

2. 在推理时免费获得推测解码草稿。MTP模块已经训练用来预测接下来的几个令牌。重新用作草稿网络时，它提供超过80%的接受率。在这个水平上，N=3或N=5的推测解码提供1.8倍的吞吐量。10%的训练时间成本在你第一次运行推理时就得到了回报。

### 与EAGLE的关系

EAGLE在预训练之后单独训练一个小型草稿模型。MTP将草稿烘焙到预训练中。两种方法在相似的接受率上收敛，但通过不同的管道：

| 维度 | EAGLE-3 | MTP (DeepSeek-V3) |
|-----------|---------|------------------|
| 训练时间 | 预训练后 | 预训练期间 |
| 与现有权重的向后兼容性 | 是 | 否（需要重新训练） |
| 草稿参数 | 1-2个transformer层 | 1个transformer块+投影 |
| 接受率 | 0.88-0.92 | 深度1处为0.80+ |
| 超越加速的收益 | 仅推测解码 | 更密集的训练信号+加速 |

## 构建它

`code/main.py`从头到尾构建单个MTP模块：共享嵌入、投影、transformer块、共享输出头。然后它在短合成序列上计算每深度交叉熵损失，并按组件打印参数计数。一个32个令牌的玩具词汇表使数字可读。

### 步骤1：共享嵌入表

单个`vocab_size x hidden`表被主模型和每个MTP模块在每个深度使用。不是第二个副本——实际上是相同的张量。

### 步骤2：每深度组合

```python
def combine(prev_hidden, next_token_embed, M_k):
    # 沿特征维度连接，然后投影到隐藏层大小
    concat = rms_norm(prev_hidden) + rms_norm(next_token_embed)  # 向量加法的替代
    projected = matvec(M_k, concat)
    return projected
```

真实的DeepSeek-V3将两个RMSNorm向量连接到`[2h]`并使用`h x 2h`矩阵进行投影。玩具版本使用向量加法是为了stdlib的简洁。

### 步骤3：深度k处的transformer块

自注意力加上MLP。在玩具版本中，单层线性注意力块和SwiGLU MLP在没有numpy的情况下保持结构可见。

### 步骤4：共享输出头

重用主模型的输出投影。词汇表上的logits。

### 步骤5：每深度损失

softmax(logits)针对偏移量`k`处真实令牌的交叉熵。使用`lambda / D`缩放因子跨深度聚合。

### 步骤6：参数计算

打印总参数计数、共享（嵌入、头）计数和每模块额外计数。显示MTP额外参数与主模型大小的比率。

## 使用它

MTP已集成到DeepSeek-V3（2024年12月）和DeepSeek-R1系列中。在推理时：

- DeepSeek自己的服务栈开箱即用地将MTP模块作为推测解码器使用。
- 截至2026年4月，vLLM和SGLang有DeepSeek-V3 MTP的集成路径。
- AMD的ROCm SGLang教程展示了一个特定的MTP推测解码配置，在V3检查点上测得1.8倍的加速。

何时在新预训练运行中使用MTP：

- 你控制完整的预训练管道并希望获得更密集的训练信号。
- 你知道你将大规模部署模型并希望免费获得推测解码。
- 你的隐藏层大小至少为4096。在1B规模时，开销的伤害大于增益的帮助。

何时不使用：

- 微调现有的预训练密集模型。MTP模块没有经过训练。
- 研究模型中，你想要一个干净的基线进行比较。MTP改变了架构。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|----------------|------------------------|
| MTP模块 | "额外损失块" | 一个小型transformer块加上投影，预测主模型前方`k`位置的令牌 |
| 预测深度 | "哪个偏移量" | 整数`k`，使得模块`k`从通过位置`i`的前缀预测`t_{i+k}` |
| 并行MTP | "Gloeckle风格" | 在相同骨干隐藏状态上的D个独立头，没有条件链 |
| 顺序MTP | "DeepSeek-V3风格" | 每个模块依赖于前一深度的隐藏状态加上下一个令牌的嵌入；保留因果链 |
| 共享输出头 | "重用主头" | MTP模块调用主模型的LM头，不是单独的输出投影 |
| 共享嵌入 | "重用主表" | 在各处使用相同的词汇嵌入表；没有重复参数 |
| 投影矩阵M_k | "组合隐藏+下一个令牌" | 一个`h x 2h`的线性层，将前一隐藏状态和目标令牌嵌入折叠到下一深度的输入 |
| 联合损失L_MTP | "平均的额外损失" | 每深度交叉熵损失的算术平均值，按`lambda`缩放 |
| 深度1处的接受率 | "MTP草牌正确的频率" | D=1 MTP模块的top-1预测与主模型的top-1预测相等的比率；在DeepSeek-V3上为80%+ |
| Lambda加权 | "额外损失的重要性" | 每深度缩放因子；在训练开始时为0.3，在DeepSeek-V3上后期为0.1 |
