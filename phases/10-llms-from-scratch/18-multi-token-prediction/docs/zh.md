# 18 · 多 Token 预测（Multi-Token Prediction, MTP）

> 从 GPT-2 到 Llama 3，每一个自回归 LLM 在每个位置上只训练一个损失：预测下一个 token。DeepSeek-V3 为每个位置加上了第二个损失：预测再下一个 token。这额外的 14B 参数（叠加在 671B 模型之上）通过梯度流回流、蒸馏进了主模型，而训练好的 MTP 头在推理时被重新利用为推测解码（speculative decoding）的草稿器，接受率（acceptance）超过 80%。1.8 倍的生成吞吐量是白赚的。本课依据 DeepSeek 技术报告构建顺序式 MTP 模块，计算损失与共享头的参数布局，并解释为什么 MTP 保留了因果链，而 Gloeckle 等人最初的并行式 MTP 却打破了它。

**类型：** 构建
**语言：** Python（标准库）
**前置：** 阶段 10 · 04（预训练一个迷你 GPT）、阶段 10 · 15（推测解码）
**时长：** 约 60 分钟

## 学习目标

- 陈述 MTP 的训练目标，并推导跨预测深度的联合损失。
- 解释 Gloeckle 等人的并行式 MTP 头（2024）与 DeepSeek-V3 的顺序式 MTP 模块之间的区别，以及为什么顺序式设计能保留因果链。
- 计算在预训练任务中加入 MTP 模块所带来的参数与显存开销。
- 从零实现一个 MTP 模块：共享嵌入、按深度划分的 transformer 块、投影，以及共享输出头。

## 问题所在

下一 token 预测是标准的 LLM 训练目标。每个隐藏状态都被监督去预测恰好一件事：紧随其后的那个 token。这是一个出乎意料地弱的信号。序列中的大部分信息都延伸到不止一个 token 之外——结构、连贯性、事实性、算术流。模型必须通过在数万亿 token 上累积许许多多单 token 信号来学会这些。

MTP 提出一个问题：如果每个隐藏状态都被监督去一次性预测多个未来 token 会怎样？Gloeckle 等人（Meta，2024）证明了这有帮助。他们的实现在主干之上放置了若干个独立的输出头，每个头预测一个不同的偏移。并行、简单，但这些头看到的是同一个隐藏状态，没有任何层级化的细化——而且这些预测在因果上并不形成链条，因此无法用于推测解码。

DeepSeek-V3（2024 年 12 月）将 MTP 重新设计为顺序式模块，在每个预测深度上保留因果链。模型先从 `h_i^(0)` 预测 `t+1`，然后从一个新的隐藏状态 `h_i^(1)`（它把 `h_i^(0)` 与 `E(t+1)` 嵌入结合在一起）预测 `t+2`，以此类推。每个深度都是它自己的小型 transformer 块。共享嵌入与共享输出头让参数开销保持适中。在 DeepSeek-V3 的规模下，MTP 模块在 671B 主模型权重之上额外引入了 14B 参数。这 2% 的开销换来了更密集的训练信号，并且在推理时还附赠了一个现成的推测解码草稿。

本课从零构建单个 MTP 模块以及 D 深度损失。数学很简洁，实现只有 150 行。

## 核心概念

### 顺序式 MTP 配方

DeepSeek-V3 在主模型之上加入 `D` 个 MTP 模块。每个模块 `k`（`k = 1..D`）预测深度 `k` 处的 token——也就是说，在给定截至位置 `i` 的前缀时，预测 `t_{i+k}`。

模块 `k` 由以下部分组成：

- 一个带有自身注意力和 MLP 的 transformer 块 `T_k`。
- 一个投影矩阵 `M_k`，它把上一深度的隐藏状态与下一深度的真值（ground-truth）token 的嵌入结合起来。
- 共享嵌入 `E`（与主模型相同）。
- 共享输出头 `Out`（与主模型相同）。

在训练时，对于截至位置 `i` 的前缀，按深度划分的隐藏状态为：

```
h_i^(0) = main model backbone at position i
h_i^(k) = T_k( M_k * concat(RMSNorm(h_i^(k-1)), RMSNorm(E(t_{i+k}))) )   for k >= 1
```

按深度划分的预测为：

```
logits_{i+k} = Out(h_i^(k-1))   for k = 1..D
```

按深度划分的损失是针对真值 `t_{i+k}` 的交叉熵：

```
L_k = CE(logits_{i+k}, t_{i+k})
```

跨深度的联合损失：

```
L_MTP = (lambda / D) * sum_{k=1..D} L_k
```

`lambda` 是一个较小的加权因子——DeepSeek-V3 在训练前 10% 使用 0.3，之后使用 0.1。总训练损失为 `L_main + L_MTP`。

### 为什么是顺序式，而非并行式

Gloeckle 最初的并行式 MTP 有 D 个输出头，每个都直接作用于 `h_i^(0)`。每个头都从相同的主干隐藏状态预测 `t_{i+k}`。这样训练起来没问题，但这些预测彼此之间并不相互条件化。你无法用 `head_1` 的输出来帮助 `head_2`——这些头是并行触发的。

DeepSeek-V3 的顺序式设计从 `h_i^(k-1)` 加上实际的下一 token 嵌入 `E(t_{i+k})` 来构建 `h_i^(k)`。这保留了因果链：要预测 `t_{i+k+1}`，深度 `k+1` 处的模块能看到 `t_{i+k}` 处的内容。这在结构上与自回归解码器消费自身输出的方式完全相同——使得 MTP 模块可以直接用作推测解码的草稿器。

在推理时：把 `h_i^(k-1)` 和草稿出的 `t_{i+k}` 喂入模块 `k+1`，得到对 `t_{i+k+1}` 的预测。重复这一过程。这正是一种 EAGLE 风格的草稿，使用训练好的 MTP 模块作为草稿网络。DeepSeek-V3 报告第一个 MTP 模块的接受率超过 80%，加速约 1.8 倍。

### 参数核算

对于一个隐藏维度为 `h`、词表大小为 `V` 的模型：

- 主模型：数十亿参数，外加一个大小为 `V * h` 的输出头。
- 共享输出头：复用主模型的头，无额外参数。
- 共享嵌入：复用主模型的嵌入，无额外参数。
- 每个 MTP 模块：
  - 投影 `M_k`：`(2h) * h = 2h^2`。
  - transformer 块 `T_k`：注意力（MHA 为 `4h^2`）加 MLP（SwiGLU 在比率 8/3 下通常为 `8h^2`）。每块约 `12h^2`。

每个模块的额外参数总计：约 `14h^2`。对于 DeepSeek-V3 的 `h = 7168`、D = 1 个模块：理论上约 `14 * 7168^2 = 约 720M` 参数。DeepSeek-V3 报告的是 14B——差异主要在于 MTP 模块中的专家层同样是 MoE。

### 推测解码带来的回报

在预训练期间，MTP 模块使训练放慢约 10%（更多前向计算、额外损失）。回报是双重的：

1. 更密集的训练信号。每个隐藏状态看到 D+1 个监督目标。在 MMLU、GSM8K、MATH、HumanEval 上测得的效果：在 DeepSeek-V3 的消融实验中，稳定地带来几个百分点的提升。

2. 推理时免费的推测解码草稿。MTP 模块已经被训练去预测接下来的几个 token。把它重新用作草稿网络，可以交付 80%+ 的接受率。在这个水平上，N=3 或 N=5 的推测解码能带来 1.8 倍吞吐量。10% 的训练时开销在你第一次运行推理时就回本了。

### 与 EAGLE 的关系

EAGLE 在预训练之后**单独**训练一个小型草稿模型。MTP 则把草稿烘焙进了预训练。这两种方法在相近的接受率上殊途同归，但走的是不同的管线：

| 维度 | EAGLE-3 | MTP（DeepSeek-V3） |
|-----------|---------|------------------|
| 何时训练 | 预训练之后 | 预训练期间 |
| 与既有权重向后兼容 | 是 | 否（需重新训练） |
| 草稿参数 | 1-2 个 transformer 层 | 1 个 transformer 块 + 投影 |
| 接受率 | 0.88-0.92 | 深度 1 处 0.80+ |
| 加速之外的收益 | 仅推测解码 | 更密集的训练信号 + 加速 |

## 动手构建

`code/main.py` 端到端地构建一个 MTP 模块：共享嵌入、投影、transformer 块、共享输出头。然后它在一个短的合成序列上计算按深度划分的交叉熵损失，并按组件打印参数量。一个仅含 32 个 token 的玩具词表让这些数字保持可读。

### 第 1 步：共享嵌入表

一个 `vocab_size x hidden` 的表被主模型**以及**每个 MTP 模块在每个深度共同使用。不是第二份副本——字面意义上的同一个张量。

### 第 2 步：按深度的组合

```python
def combine(prev_hidden, next_token_embed, M_k):
    # 沿特征维拼接，然后投影回 hidden 维
    concat = rms_norm(prev_hidden) + rms_norm(next_token_embed)  # 用向量加法作为替代
    projected = matvec(M_k, concat)
    return projected
```

真实的 DeepSeek-V3 把两个经 RMSNorm 的向量拼接为 `[2h]`，并用一个 `h x 2h` 的矩阵投影。这个玩具为了标准库的简洁性改用向量加法。

### 第 3 步：深度 k 处的 transformer 块

自注意力加 MLP。在玩具中，一个单层线性注意力块和一个 SwiGLU MLP 让结构清晰可见，且无需 numpy。

### 第 4 步：共享输出头

复用主模型的输出投影。在词表上得到 logits。

### 第 5 步：按深度的损失

softmax(logits) 针对偏移 `k` 处真值 token 的交叉熵。用 `lambda / D` 缩放因子在各深度上汇总。

### 第 6 步：参数核算

打印参数总量、共享部分（嵌入、头）的数量，以及每个模块的额外数量。展示 MTP 额外开销相对于主模型规模的比例。

## 实际运用

MTP 已集成进 DeepSeek-V3（2024 年 12 月）和 DeepSeek-R1 系列。在推理时：

- DeepSeek 自家的服务栈开箱即用地把 MTP 模块当作推测解码器消费。
- 截至 2026 年 4 月，vLLM 和 SGLang 都有针对 DeepSeek-V3 MTP 的集成路径。
- AMD 的 ROCm SGLang 教程展示了一份具体的 MTP 推测解码配置，在 V3 检查点上测得 1.8 倍加速。

何时在一次新的预训练任务中使用 MTP：

- 你掌控整条预训练管线，并想攒下更密集的训练信号。
- 你确定将大规模部署该模型，并想白嫖推测解码。
- 你的隐藏维度至少为 4096。在 1B 规模下，开销带来的损伤比收益带来的帮助更大。

何时不使用：

- 微调一个既有的、预训练好的稠密模型。此时 MTP 模块并未经过训练。
- 想要一个干净基线以供对比的研究型模型。MTP 会改变架构。

## 练习

1. 运行 `code/main.py`。展示当合成信号增强时，按深度划分的损失单调下降。修改合成数据以使用固定模式，并验证深度 1 和深度 2 的损失都收敛。

2. 计算一个稠密 70B 模型（hidden 8192，80 层）在 D=1 个 MTP 模块下的参数开销。与 DeepSeek-V3 报告的 14B 开销作对比。解释为什么 DeepSeek 的数字更高：MTP transformer 块继承了相同的 MoE 结构，从而抬高了每模块的参数量。

3. 在玩具中实现 D=2：加入第二个 MTP 模块，它接收 h^(1) 并预测 `t_{i+2}`。验证联合损失与参数核算是否与 DeepSeek 论文的公式 19-21 相符。

4. 把玩具切换为并行式 MTP（Gloeckle 风格）：在主隐藏状态之上加入 D 个输出头，每个预测一个不同的偏移。测量在相同合成信号下，各深度的损失与顺序式版本相比如何。顺序式版本对于 k > 1 应产生更低的深度 k 损失，因为它以中间预测为条件。

5. 把训练好的 MTP 模块用作 EAGLE 风格的草稿：在推理时调用模块 k 来提议 `t_{i+k}`。在一个留出序列上，针对主模型的预测测量这些草稿 token 的接受率。如果你在玩具上达到 50%+，那你就复现了「MTP 作为草稿」这一经验性质。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|----------------|------------------------|
| MTP 模块 | “额外损失块” | 一个小型 transformer 块加投影，预测比主模型超前 `k` 个位置的 token |
| 预测深度（prediction depth） | “哪个偏移” | 整数 `k`，使得模块 `k` 从截至位置 `i` 的前缀预测 `t_{i+k}` |
| 并行式 MTP | “Gloeckle 风格” | 在同一主干隐藏状态上的 D 个独立头，无条件链 |
| 顺序式 MTP | “DeepSeek-V3 风格” | 每个模块以上一深度的隐藏状态加下一 token 的嵌入为条件；保留因果链 |
| 共享输出头 | “复用主头” | MTP 模块调用主模型的 LM 头，而非单独的输出投影 |
| 共享嵌入 | “复用主表” | 处处使用同一个词表嵌入表；无重复参数 |
| 投影矩阵 M_k | “组合 hidden + 下一 token” | 一个 `h x 2h` 的线性层，把上一隐藏状态与目标 token 的嵌入折叠成下一深度的输入 |
| 联合损失 L_MTP | “平均后的额外损失” | 按深度划分的交叉熵损失的算术平均，并由 `lambda` 缩放 |
| 深度 1 处的接受率 | “MTP 草稿对的频率” | D=1 的 MTP 模块的 top-1 预测等于主模型 top-1 预测的比率；在 DeepSeek-V3 上为 80%+ |
| Lambda 加权 | “额外损失的重要性” | 按深度的缩放因子；DeepSeek-V3 上训练开始时为 0.3，之后为 0.1 |
