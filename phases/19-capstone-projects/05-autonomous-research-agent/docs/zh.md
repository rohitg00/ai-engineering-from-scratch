# 综合项目 05 — 自主研究智能体（AI-Scientist 类别）

> Sakana 的 AI-Scientist-v2 发表了完整论文。Agent Laboratory 运行了实验。Allen AI 分享了追踪记录。2026 年的形态是在实验上进行规划-执行-验证树搜索，有预算代价、沙箱化代码执行、视觉反馈 LaTeX 编写器，以及自动的 NeurIPS 风格评审员集成。本综合项目是构建一个，在每篇论文 $30 内端到端运行，并在 Sakana 记录的沙箱逃逸红队测试中存活。

**类型：** 综合项目
**语言：** Python（智能体 + 沙箱）、LaTeX（输出）
**前置条件：** 第 2 阶段（ML）、第 3 阶段（深度学习）、第 7 阶段（Transformer）、第 10 阶段（从零构建 LLM）、第 14 阶段（智能体）、第 15 阶段（自主）、第 16 阶段（多智能体）、第 18 阶段（安全）
**涉及阶段：** P0 · P2 · P3 · P7 · P10 · P14 · P15 · P16 · P18
**时间：** 40 小时

## 问题描述

自主研究智能体在 2026 年跨越了一个门槛。Sakana AI 的 AI-Scientist-v2 在《自然》杂志上发表，生成的论文通过了研讨会同行评审。ShinkaEvolve（ICLR 2026）将这条线扩展到了演化假设。AMD 的 Agent Laboratory 发布了可重现的追踪记录。这些智能体不是魔法——它们是在候选实验树上运行的规划-执行-验证循环，带有代价上限、种子绑定的沙箱和自动评审。工艺在于循环、预算和安全故事。

你通过在狭窄领域（例如，在 1 亿参数 Transformer 上进行注意力稀疏性消融实验）中针对种子想法实现循环来学习循环。价值不在于首次运行就发现新东西。价值在于基础设施：树搜索、实验沙箱、编写器-评审器循环、红队报告。Sakana 团队记录了沙箱逃逸失败；你的智能体必须通过相同的红队测试。

## 核心概念

智能体是一种最佳优先树搜索。节点是实验规范：（假设、配置、代码、预期结果）。扩展步骤提出带有小编辑的子节点（交换优化器、移动批次大小、消融一个组件）。每个子节点在带有硬性资源上限的全新沙箱中运行。结果反馈到评分函数，按（新颖性 × 质量 × 剩余预算）对节点排序。树不断增长，直到预算耗尽，然后撰写最佳分支。

编写器是多模态的。它生成 LaTeX 草稿、编译、渲染图表，并将渲染的 PDF 反馈到 Claude Opus 4.7 的视觉模式，以对布局、图表清晰度和声明-证据对齐进行批评。由五个 LLM 裁判组成的评审员集成发出 NeurIPS 风格的评分（新颖性、严谨性、清晰度、可重现性、影响力）；如果平均值低于阈值，论文将带着批评返回给编写器。

安全性是承重墙。每个实验在 E2B 或 Daytona 沙箱中运行，没有网络出口、有界时钟时间，以及固定的资源限制。智能体的代码生成步骤通过策略层，阻止逃逸沙箱的系统调用。红队报告重现了 Sakana 记录的攻击面（fork 炸弹、文件系统逃逸、LLM 编写的网络调用）。

## 技术栈

- 编排：带检查点和人工批准门的 LangGraph
- 树搜索：实验节点上的自定义最佳优先（来自 Sakana v2 的 AB-MCTS 风格）
- 沙箱：每个实验 E2B，Docker-in-Docker 后备；通过 cgroups 的资源上限
- 文献：Semantic Scholar Graph API + OpenAlex + 摘要的本地 FAISS 缓存
- 编写器：LaTeX 模板 + Claude Opus 4.7（视觉模式），用于图表批评和布局
- 评审员：5 个裁判的集成（Opus 4.7、GPT-5.4、Gemini 3 Pro、DeepSeek R1、Qwen3-Max），带加权聚合
- 实验框架：PyTorch 2.5 用于物理实验，W&B 用于记录
- 可观测性：用于智能体追踪的 Langfuse，每篇论文硬性预算 $30

## 构建步骤

1. **种子和领域范围界定。** 接受一个种子想法（例如，"调查 10 亿参数以下 Transformer 中注意力图的稀疏性模式"）。定义搜索空间：模型、数据集、计算预算。

2. **文献传递。** 查询 Semantic Scholar + OpenAlex 中 50 篇被引用最多的相关论文；本地缓存摘要；生成 1 页领域摘要。

3. **树脚手架。** 用种子假设初始化根节点。实现 `expand(node) -> children`，带有小编辑建议（每个子节点一次配置变更）。实现 `score(node)` 作为加权的新颖性 × 质量 × 预算项。

4. **沙箱封装。** 每个实验运行 `docker run --network=none --memory=8g --cpus=2 --pids_limit=256 --read-only`（或等效的 E2B 策略）。种子被写入沙箱；输出以只读方式挂载出来。

5. **规划-执行-验证循环。** `plan` 提出子节点。`execute` 运行沙箱，捕获日志和指标。`verify` 对指标运行单元测试检查（损失是否下降？消融是否隔离了效果？）。失败节点在树上存储失败原因。

6. **编写器。** 预算用完后，选择最佳分支。用 matplotlib 渲染图表。通过上下文中带有分支追踪的 Claude Opus 4.7 生成 LaTeX 草稿。编译。将编译好的 PDF 反馈给 Opus 4.7 视觉模式进行批评。迭代。

7. **评审员集成。** 五个裁判用 NeurIPS 风格的评分标准对草稿进行评分（新颖性、严谨性、清晰度、可重现性、影响力）。如果平均值 < 4.0/5，带着批评返回给编写器。3 次重写后硬性停止。

8. **红队。** 构建或集成一组针对沙箱的对抗任务：fork 炸弹、网络渗透尝试、文件系统逃逸、LLM 编写的 shell 元字符。确认所有任务都被阻止。撰写发现。

9. **可重现性。** 每篇论文都附带其树搜索追踪 JSON、种子、W&B 运行链接、沙箱配置，以及端到端重现它的 README。

## 练习

1. 在同一领域针对三个不同的种子想法运行管道。比较树搜索的哪些部分重叠。识别重复浪费的计算。

2. 在估计超过 $5 的节点执行实验之前，添加一个人在回路中的门。测量总代价下降了多少。

3. 将评审员集成换为单个裁判。在已知不良论文的留出集上测量误接受率。

4. 引入网络渗透红队测试：智能体编写尝试 `curl` 外部地址的代码。确认 `--network=none` 策略阻止了它。记录尝试。

5. 将你的树搜索与平坦随机基线（相同预算，无扩展策略）进行比较。报告新颖性 × 质量收益。

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|------|-----------------|------------------------|
| 树搜索 | "AB-MCTS 风格扩展" | 在实验节点上的最佳优先探索，带有新颖性×质量×预算评分 |
| 沙箱 | "实验隔离" | 无网络、有界 CPU/内存、固定种子、只读输入的容器 |
| 视觉批评 | "渲染后读取" | 将论文编译为 PDF，将 PDF 反馈给 VLM 以进行布局和声明-证据批评 |
| 评审员集成 | "自动同行评审" | 多个 LLM 裁判用 NeurIPS 评分标准对论文评分；加权聚合控制管道 |
| 新颖性评分 | "这是新的吗？" | 惩罚与 50 篇论文文献缓存接近度的启发式方法 |
| 代价上限 | "$ 预算" | 每篇论文总支出的硬性上限；Langfuse 计数器 + 运行前估计 |
| 红队 | "沙箱逃逸审计" | 如果策略错误将逃逸沙箱的对抗任务 |
