# 顶点项目 05 —— 自主研究智能体（AI-科学家级别）

> Sakana 的 AI-Scientist-v2 发表了完整论文。Agent Laboratory 运行了实验。Allen AI 分享了追踪记录。2026 年的形态是计划-执行-验证树搜索，覆盖实验、预算成本、沙盒化代码执行、视觉反馈 LaTeX 撰写器，以及自动化 NeurIPS 风格的评审团。顶点项目是构建一个，在每张论文 30 美元内端到端运行，并存活 Sakana 记录的沙盒逃逸红队测试。

**类型：** 顶点项目
**语言：** Python（智能体 + 沙盒）、LaTeX（输出）
**先决条件：** Phase 2（ML）、Phase 3（深度学习）、Phase 7（transformers）、Phase 10（从头开始 LLM）、Phase 14（智能体）、Phase 15（自主系统）、Phase 16（多智能体）、Phase 18（安全）
**涉及阶段：** P0 · P2 · P3 · P7 · P10 · P14 · P15 · P16 · P18
**时间：** 40 小时

## 问题

自主研究智能体在 2026 年跨越了一个阈值。Sakana AI 的 AI-Scientist-v2 发表在 Nature 上，生成的论文通过了研讨会同行评审。ShinkaEvolve（ICLR 2026）将该系列扩展到进化假设。AMD 的 Agent Laboratory 交付了可复现的追踪记录。这些智能体不是魔法——它们是运行在候选实验树上的计划-执行-验证循环，带有成本上限、种子绑定沙盒和自动化评审。技艺在于循环、预算和安全故事。

你通过在一个狭窄领域针对种子想法实现一个来学习循环（例如，1 亿参数 transformer 上的注意力稀疏性消融）。价值不在于首次运行中发现新东西。价值在于基础设施：树搜索、实验沙盒、撰写器-评审器循环、红队报告。Sakana 团队记录了沙盒逃逸失败；你的智能体必须通过相同的红队测试。

## 概念

智能体是最佳优先树搜索。节点是实验规范：（假设、配置、代码、预期结果）。扩展步骤提出带有小编辑的子节点（交换优化器、调整批量大小、消融组件）。每个子节点在带有硬资源上限的全新沙盒中运行。结果反馈到评分函数，按（新颖性 × 质量 × 剩余预算）对节点排序。树增长直到预算耗尽，然后最佳分支被撰写。

撰写器是多模态的。它生成 LaTeX 草稿，编译它，渲染图表，并将渲染的 PDF 反馈回 Claude Opus 4.7 的视觉模式，以批评布局、图表清晰度和声明-证据对齐。五个 LLM 评审员的评审团发出 NeurIPS 风格分数（新颖性、严谨性、清晰度、可复现性、影响力）；如果平均分低于阈值，论文带着批评返回撰写器。

安全性是承重性的。每个实验在 E2B 或 Daytona 沙盒中运行，没有网络出口，受限的挂钟时间，和固定的资源限制。智能体的代码生成步骤通过策略层，阻止逃逸沙盒的系统调用。红队报告复现 Sakana 记录的攻击面（fork 炸弹、文件系统逃逸、LLM 编写的网络调用）。

  |                    |      (E2B / Daytona)

  child_1           child_k   无网络出口
  |                    |      确定性种子

## 技术栈

- 编排：LangGraph，带检查点和人工审批门
- 树搜索：实验节点上的自定义最佳优先（来自 Sakana v2 的 AB-MCTS 风格）
- 沙盒：每个实验 E2B，Docker-in-Docker 后备；通过 cgroups 的资源上限
- 文献：Semantic Scholar Graph API + OpenAlex + 摘要的本地 FAISS 缓存
- 撰写器：LaTeX 模板 + Claude Opus 4.7（视觉模式）用于图表批评和布局
- 评审员：5 个评审员的评审团（Opus 4.7、GPT-5.4、Gemini 3 Pro、DeepSeek R1、Qwen3-Max），带加权聚合
- 实验框架：物理实验用 PyTorch 2.5，日志用 W&B
- 可观察性：智能体追踪用 Langfuse，每张论文硬预算 30 美元

## 构建它

1. **种子和领域范围。** 取一个种子想法（例如，"调查子 1B transformer 注意力图中的稀疏性模式"）。定义搜索空间：模型、数据集、计算预算。

2. **文献通过。** 查询 Semantic Scholar + OpenAlex 获取 50 篇最相关的引用论文；本地缓存摘要；生成 1 页领域摘要。

3. **树脚手架。** 用种子假设初始化根节点。实现 `expand(node) -> children`，带小编辑提议（每个子节点一个配置更改）。实现 `score(node)` 为加权新颖性 × 质量 × 预算项。

4. **沙盒包装。** 每个实验运行 `docker run --network=none --memory=8g --cpus=2 --pids-limit=256 --read-only`（或等效的 E2B 策略）。种子写入沙盒；输出以只读方式挂载回传。

5. **计划-执行-验证循环。** `计划` 提出子节点。`执行` 运行沙盒，捕获日志和指标。`验证` 对指标运行单元检查（损失是否减少？消融是否隔离了效果？）。失败节点在树上存储失败原因。

6. **撰写器。** 预算后，选择最佳分支。用 matplotlib 渲染图表。通过 Claude Opus 4.7 生成 LaTeX 草稿，分支追踪在上下文中。编译。将编译的 PDF 反馈给 Opus 4.7 视觉进行批评。迭代。

7. **评审团。** 五个评审员在（新颖性、严谨性、清晰度、可复现性、影响力）上评分草稿，使用 NeurIPS 风格评分标准。如果均值 < 4.0/5，带着批评返回撰写器。3 次重写后硬停止。

8. **红队。** 构建或集成一组针对沙盒的对抗性任务：fork 炸弹、网络泄露尝试、文件系统逃逸、LLM 编写的 shell 元字符。确认所有都被阻止。撰写发现。

9. **可复现性。** 每张论文附带其树搜索追踪 JSON、种子、W&B 运行链接、沙盒配置和端到端复现它的 README。

## 交付它

`outputs/skill-ai-scientist.md` 是可交付成果。给定种子想法 + 领域 + 30 美元预算，它运行完整管道并发出可评审的论文和可复现性包。

| 权重 | 标准 | 测量方式 |
|:-:|---|---|
| 25 | 论文质量 | 与已发布研讨会论文的盲评评分标准对比 |
| 20 | 实验严谨性 | 基线、种子、消融；每个声明由结果表中的单元格支持 |

| **100** | | |

## 练习

1. 在同一领域针对三个不同的种子想法运行管道。比较树搜索的哪些部分重叠。识别重复的浪费计算。

2. 在执行前为估计高于 5 美元的节点添加人工审批门。测量总成本下降多少。

3. 将评审团换成单个评审员。在已知坏论文的保留集上测量误接受率。

4. 引入网络泄露红队测试：智能体编写尝试 `curl` 外部地址的代码。确认 `--network=none` 策略阻止它。记录尝试。

5. 将树搜索与平坦随机基线（相同预算，无扩展策略）进行比较。报告新颖性 × 质量增益。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|----------|----------|
| 树搜索 | "AB-MCTS 风格扩展" | 实验节点上的最佳优先探索，带新颖性×质量×预算分数 |
| 沙盒 | "实验隔离" | 无网络、受限 CPU/内存、固定种子、只读输入的容器 |
| 视觉批评 | "渲染-然后-阅读" | 将论文编译为 PDF，将 PDF 反馈给 VLM 进行布局和声明-证据批评 |
| 评审团 | "自动化同行评审" | 多个 LLM 评审员用 NeurIPS 评分标准评分论文；加权聚合门控管道 |
| 新颖性分数 | "这是新的吗？" | 惩罚与 50 篇论文文献缓存接近度的启发式 |
| 成本上限 | "$ 预算" | 每张论文总花费的硬上限；Langfuse 计数器 + 预运行估计 |
| 红队 | "沙盒逃逸审计" | 如果策略错误将逃逸沙盒的对抗性任务 |
