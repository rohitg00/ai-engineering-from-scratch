给定种子想法、狭窄领域和30美元计算预算，构建一个运行实验树搜索、撰写可审查LaTeX论文并发出可复现性包的代理。

构建计划：

1. 文献遍历：Semantic Scholar Graph API + OpenAlex；在FAISS中缓存摘要；生成1页领域摘要。
2. 树搜索：实现实验节点上的最佳优先扩展，`expand(node) -> children`（每个子节点一次配置编辑）和`score(node) = novelty*0.4 + quality*0.5 + budget*0.1`。
3. 每节点沙箱：每个实验运行`docker run --network=none --memory=8g --cpus=2 --pids-limit=256 --read-only`或E2B等效；确定性种子；强制执行资源上限。
4. 规划-执行-验证：验证步骤检查损失是否收敛、基线是否运行、消融是否隔离声明。
5. 撰写者：生成LaTeX，编译为PDF，将PDF输入Claude Opus 4.7视觉模式以批评布局和声明-证据对齐，最多迭代3次。
6. 评审者集合：五位评委（Opus 4.7、GPT-5.4、Gemini 3 Pro、DeepSeek R1、Qwen3-Max）按NeurIPS评分标准（新颖性、严谨性、清晰度、可复现性、影响力）评分；均值<4.0返回撰写者。
7. 红队：集成对抗任务（fork炸弹、文件系统逃逸、LLM编写的网络调用）。确认全部阻止。发出`red_team.md`。
8. 可复现性包：paper.pdf + review.md + 树搜索trace JSON + 种子 + W&B运行链接 + 沙箱配置 + 一行重跑命令。

评估标准：

| 权重 | 标准 | 测量 |
|:-:|---|---|
| 25 | 论文质量 | 与同一种子主题的已发表研讨会论文进行盲评 |
| 20 | 实验严谨性 | 基线、种子、消融；每个声明由结果表中的单元格支持 |
| 20 | 成本和计算纪律 | 每篇论文30美元上限强制执行，Langfuse跟踪 |
| 20 | 安全性 | 沙箱红队通过；网络策略和终止开关用记录尝试验证 |
| 15 | 可复现性 | 一行重跑命令用相同种子复现论文 |

硬性拒绝：
- 在沙箱外运行的实验。该毕业设计的整个论点是执行是受控的。
- 不重新读取编译PDF的撰写步骤（视觉批评是承重性的）。
- 没有基线、种子或消融部分的论文。
- 成本预算仅作为事后警告而非硬上限强制执行。

拒绝规则：
- 拒绝发布评审者均值低于4.0/5的论文，除非有明确的人工覆盖。
- 拒绝在种子想法需要沙箱内网络访问的情况下运行。改为添加单独的只读数据集卷。
- 拒绝重跑未执行和记录红队的论文。

输出：包含树搜索引擎、沙箱策略、撰写者/评审者循环、三个带可复现性包的示例运行、红队报告、成本分类账csv的仓库，以及一份说明复现了哪些Sakana v2失败模式及缓解措施如何运作的撰写。
