# 推理技术选择指南

当你需要 LLM 推理问题时，先选择技术再编写提示词。技术决定推理架构，提示词填充细节。

## 快速决策树

1. 任务是简单的事实查找还是单步分类？
   - 是：使用**零样本**。CoT 增加成本但无准确率提升。
   - 否：继续。

2. 任务是否需要多步推理（数学、逻辑、规划）？
   - 是：使用**思维链**。继续到步骤 3。
   - 否：如果格式重要使用**少样本**，否则使用零样本。

3. 单个推理错误是否可接受？
   - 是：使用**少样本 CoT**（单样本，温度 0.0）。
   - 否：使用**自一致性**（N=5，温度 0.7）。继续到步骤 4。

4. 问题是具有多条可能路径的搜索/规划问题吗？
   - 是：使用**思维树**。
   - 否：自一致性已足够。

5. 任务是否需要外部信息或计算？
   - 是：使用**ReAct**（推理 + 工具调用）。
   - 否：纯推理技术已足够。

## 技术矩阵

| 技术 | 准确率提升 | 成本倍数 | 延迟 | 最适合 |
|-----------|--------------|-----------------|---------|----------|
| 零样本 | 基线 | 1x | ~1s | 简单任务、事实问答 |
| 少样本 | +5-15% | 1.2x | ~1s | 格式匹配、分类 |
| 零样本 CoT | +10-20% | 1.3x | ~1.5s | 快速推理提升 |
| 少样本 CoT | +15-25% | 1.5x | ~2s | 数学、逻辑、多步 |
| 自一致性 (N=5) | 比 CoT 高 +2-5% | 5x | ~5s | 高风险推理 |
| 自一致性 (N=10) | 比 N=5 高 +1-2% | 10x | ~10s | 仅关键决策 |
| 思维树 | 任务相关 | 10-40x | ~30s+ | 搜索、规划、谜题 |
| ReAct | 任务相关 | 3-10x | ~5-15s | 基于知识的任务 |
| 提示链 | 比单遍高 +5-10% | 2-5x | ~5-10s | 复杂多部分任务 |

## 模型特定指导

### GPT-4o / GPT-4.1
- 强大的基线推理能力。零样本 CoT 通常已足够。
- 带 3 个示例的少样本 CoT 在 GSM8K 上达到 95%。
- 自一致性带来的提升有限（95% 到 97%）——仅对关键任务值得。
- 原生支持结构化输出以提取答案。

### Claude 3.5 Sonnet / Claude 3.7 Sonnet
- 擅长遵循结构化提示词格式（XML 标签）。
- 带 XML 分隔示例的少样本 CoT 效果最佳。
- 扩展思考（Claude 3.7）是原生 CoT——无需提示。
- 自一致性有效，因为 Claude 在温度 0.7 时的推理变化良好。

### Llama 3.1/3.3 70B
- 最受益于少样本 CoT（与零样本相比准确率差距更大）。
- 推理任务推荐使用 N=5 的自一致性。
- 比商业模型需要更明确的格式指令。
- ToT 在本地推理上成本高昂——仅考虑用于批处理。

### Gemini 2.5 Pro
- 开箱即用的强大多步推理能力。
- 思考模式提供内置 CoT，无需提示工程。
- 少样本示例对格式一致性的帮助大于准确率。
- 大上下文窗口（1M）使重示例的少样本变得实用。

## 反模式

**对简单任务使用 CoT**：问"2+2 等于多少？让我们一步一步思考"浪费 token。模型无需推理痕迹就能正确完成简单算术。CoT 在有 3+ 步骤时才有帮助。

**在温度 0.0 时使用自一致性**：所有 N 个样本将完全相同。必须使用温度 > 0（推荐 0.5-0.8）以获得多样化的推理路径。

**对所有内容使用 ToT**：ToT 需要 O(b^d) 次 LLM 调用，其中 b=分支因子，d=深度。一个 b=3, d=3 的树最多需要 39 次调用。仅在更便宜的技术失败时保留使用。

**使用错误的少样本示例**：带有推理错误的示例会教模型犯那些错误。每个示例都必须验证。一个错误的示例可能比零个示例更能降低准确率。

**没有一致格式就提取答案**：自一致性需要跨样本比较答案。如果答案格式变化（"$18"、"18 dollars"、"eighteen"），投票会失败。始终强制执行："答案是 [数字]。"

## 成本优化

对于每天处理 10,000 次查询的生产系统，使用 GPT-4o 定价（输入 $2.50/1M，输出 $10/1M）：

| 技术 | 平均 Token/查询 | 每日成本 | 准确率 |
|-----------|-----------------|------------|----------|
| 零样本 | ~200 | ~$5 | 78% |
| 少样本 CoT | ~600 | ~$15 | 95% |
| 自一致性 (N=5) | ~3,000 | ~$75 | 97% |
| ToT (b=3, d=2) | ~6,000 | ~$150 | 任务相关 |

大多数应用的成本最优策略：从少样本 CoT 开始。仅在置信度低的查询上添加自一致性（来自构建部分的升级模式）。

## 与提示链集成

推理技术与提示链组合：

**链步骤 1**（提取）：零样本，温度 0.0
**链步骤 2**（推理）：少样本 CoT，温度 0.0
**链步骤 3**（验证）：N=3 的自一致性，温度 0.7

这个三步链的成本约为单次 CoT 调用的 3 倍，但能捕获提取错误、推理错误，并从验证步骤提供置信度分数。

## 何时超越提示词工程

如果你在提示词工程上花费的时间超过了编写应用程序代码，考虑：

1. **微调**：如果你有 500+ 标注示例且任务范围狭窄
2. **DSPy 编译**：如果你想要自动化的提示词优化
3. **Agent 框架**：如果任务需要多轮工具使用（阶段 14）
4. **RAG**：如果模型需要访问私有/当前知识（课程 06-07）

提示词技术是基础。它们适用于任何模型、任何提供商，且不需要训练数据。但它们有极限。知道何时升级到下一个层次，与掌握技术本身同样重要。
