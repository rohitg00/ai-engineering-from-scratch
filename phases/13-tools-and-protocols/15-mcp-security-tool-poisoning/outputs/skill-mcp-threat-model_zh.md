给定 MCP 部署（服务器列表、工具列表、权限列表），生成威胁模型。

生成：

1. 攻击适用性。对于七个攻击类别中的每一个（工具投毒、rug pull、shadowing、MPMA、寄生工具链、采样攻击、供应链伪装），将适用性评为高/中/低并附一句理由。
2. 防御清单。列出已部署的防御（哈希固定、静态检测器、网关、签名注册表、MELON、双规则则执行）。
3. 双规则则审计。对于每个工具，分类为不受信任/敏感/后果性，并标记任何单轮次中三者的组合。
4. 缺失防御。根据威胁概况命名尚未应用的最高杠杆防御。
5. 运行手册。团队下周应采取的改进安全态势的三项行动。

硬性拒绝：
- 任何说"攻击类别 X 不适用，因为我们信任此服务器"的威胁模型。假设一个服务器将被入侵。
- 任何使用静默覆盖命名空间解析的部署。
- 任何启用采样但没有每会话速率限制器的部署。

拒绝规则：
- 如果部署没有已批准工具描述的文档，拒绝并首先强制哈希固定。
- 如果部署使用公共未签名 MCP 注册表，标记供应链风险并推荐迁移到已验证注册表。
- 如果任何工具组合不受信任输入、敏感数据和后果性行动，拒绝批准并要求拆分。

输出：一页威胁模型，包含攻击适用性表、防御清单、双规则则标志列表和三行动运行手册。以此部署的单一最高价值安全补充结尾。
