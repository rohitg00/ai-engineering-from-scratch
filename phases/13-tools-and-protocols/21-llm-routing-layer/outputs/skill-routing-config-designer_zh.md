给定工作负载概况（延迟要求、合规约束、团队规模、支出预算），生成路由网关选择和配置。

生成：

1. 网关选择。LiteLLM（自托管）、OpenRouter（托管 SaaS）或 Portkey（生产级 w/ 护栏）。一段理由。
2. 别名列表。应用程序使用的逻辑模型名称。示例：`smart`、`fast`、`coding`、`long_context`。
3. 回退链。每个别名，带重试预算的优先级排序具体模型列表。
4. 护栏。PII 编辑规则、策略违规列表、输出过滤规则。
5. 成本预算。每团队/每项目支出上限，执行粒度。

硬性拒绝：
- 任何将提示词发送到违反合规约束区域的配置。
- 任何仅有一个提供商的回退链。单一故障域违背目的。
- 任何工作负载直接处理用户输入却无护栏的设置。

拒绝规则：
- 如果工作负载是单模型原型且预期保持如此，拒绝推荐网关；直接 API 调用更简单。
- 如果团队没有 SRE 且选择自托管，标记运营风险。
- 如果用户要求特定模型而无替代方案，拒绝并要求至少一个回退。

输出：一页路由配置，包含网关选择、别名、回退链、护栏、成本计划。以部署后首个告警指标结尾（通常是回退使用率）。
