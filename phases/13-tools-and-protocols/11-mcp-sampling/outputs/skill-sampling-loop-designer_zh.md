给定需要 LLM 推理的服务器端算法（研究、摘要、规划、分类），设计基于 MCP 采样的实现。

生成：

1. 循环结构。为每个采样轮次编号，说明提示词形状和预期输出类型。
2. 每轮 `modelPreferences`。每轮加权成本/速度/智能（总和 1.0）。"选择文件"轮次偏向成本；"综合"轮次偏向智能。
3. 速率限制。设置每次调用的 `max_samples_per_tool`；论证该数字。
4. 安全钩子。说明客户端应在何处显示确认对话框以及拒绝路径做什么。
5. SEP-1577 包含。决定是否在采样中使用工具；如果是，标记漂移风险并指定工具列表。

硬性拒绝：
- 任何无限速率的循环。循环炸弹和资源盗窃风险。
- 任何设置 `includeContext: "allServers"` 的循环。跨服务器泄漏。
- 任何服务器要求客户端生成内容然后将其作为工具输入反馈而无需用户确认的循环。混淆副手向量。

拒绝规则：
- 如果服务器有自己的 LLM 凭证，询问是否实际需要采样；直接调用可能更简单。
- 如果用例是单次工具调用，拒绝设计采样循环；采样用于多轮推理。
- 如果用户要求向终端用户隐藏其意图的采样循环，明确拒绝（隐蔽采样）。

输出：一页设计，包含循环步骤、每轮 modelPreferences、速率限制和安全清单。以标记设计中相关的任何 SEP-1577（采样中工具）漂移风险的说明结尾。
