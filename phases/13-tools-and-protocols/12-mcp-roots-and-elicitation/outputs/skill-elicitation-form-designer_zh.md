给定行为可能需要调用中用户输入的工具，设计引导模式和消息。

生成：

1. 触发条件。说明应导致工具调用 `elicitation/create` 的确切输入或歧义。
2. 消息模板。主机向用户显示的一句话。 plain、具体、无行话。
3. 模式。带类型属性和 `enum` 列表（用于消歧）或 `boolean`（用于确认）的扁平 JSON Schema。不要嵌套。
4. 分支处理。将 `accept` / `decline` / `cancel` 映射到工具行为。
5. 速率限制规则。限制每次工具调用的引导次数；绝不在循环中引导。

硬性拒绝：
- 任何嵌套对象的模式。引导 v1 是扁平的。
- 任何用于填充 LLM 本可以用散文询问的缺失参数的引导。
- 任何高频引导（每次工具调用超过一次）。

拒绝规则：
- 如果工具是只读且低风险的，拒绝引导并直接返回结果。
- 如果工具是破坏性的且主机支持 `destructiveHint` 注释，建议使用注释并让客户端原生处理确认。
- 如果需要是 OAuth 登录，推荐 URL 模式引导并标记 SEP-1036 漂移风险。

输出：一页设计，包含触发条件、消息模板、模式、分支处理、速率限制规则和关于表单模式或 URL 模式更适合的说明。
