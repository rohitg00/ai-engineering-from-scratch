给定一个生成任务（代码、创意写作、推理、对话、结构化输出）和延迟/质量目标，输出：

1. 采样方法。其中之一：贪心、仅温度、top-k、top-p、min-p、beam-k、推测解码。一句话说明原因。
2. 参数值。温度、top-k、top-p、min-p、重复惩罚——与任务类型相关的具体数字。（例如，代码的温度 0.2 + top-p 1.0；聊天的 min-p 0.1 + 温度 0.7。）
3. 停止条件。`max_new_tokens`、停止 token 列表、基于模式的停止（例如，闭合的 `</tool_call>`）。
4. 确定性切换。固定种子以确保可重复性；标记用例（评估、法律）是否需要它。
5. 质量检查。针对任务目标的一行测试（编译/通过单元测试、事实性、格式有效性等）。

拒绝为结构化输出或代码补全推荐温度 > 1.0——幻觉风险急剧上升。拒绝为开放式对话推荐纯贪心——模型会循环。拒绝在模型可以生成模板/工具时没有指定停止 token 列表的情况下交付采样配置。
