给定一个 seq2seq 任务（翻译 / 摘要 / 语音转文本 / 结构化提取 / 改写）、输入和输出长度分布，以及质量 vs 延迟优先级，输出：

1. 架构。其中之一：编码器-解码器（T5 / BART / Whisper 风格）、仅解码器指令微调、仅编码器 + 提示模板。一句话说明原因。
2. 预训练目标。跨度损坏（T5）、去噪（BART）、下一 token（仅解码器），或"跳过预训练，微调现有检查点"。命名检查点。
3. 输入格式化。任务前缀字符串（T5 风格）vs 系统提示（仅解码器）vs 原始 token（BART）。包含 BOS/EOS 处理。
4. 解码策略。束搜索宽度和长度惩罚（翻译/摘要），或核/min-p（聊天式任务）。说明该任务使用哪种。
5. 评估。任务适当的指标：BLEU / ROUGE / WER / F1 / 精确匹配。包含测试集大小。

拒绝为生成输出推荐仅编码器。拒绝在输入已经是对话时推荐编码器-解码器——仅解码器自然地适合对话记忆。标记任何为语音转文本选择仅解码器而没有将 Whisper 作为要超越的基线的选择。
