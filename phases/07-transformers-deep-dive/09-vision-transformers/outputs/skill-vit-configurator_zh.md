给定一个视觉任务（分类 / 分割 / 检测 / 检索）、图像分辨率、数据集大小（标注 + 未标注）和部署目标，输出：

1. 骨干网络。其中之一：DINOv2 ViT-L/14（检索/分类默认）、SAM 3 编码器（分割）、SigLIP（视觉-语言）、ConvNeXt（延迟关键）。一句话说明原因。
2. 补丁大小。224 分辨率标准分类用 16，DINOv2 用 14，高分辨率密集预测用 8。标记序列长度 `(H/P)^2 + 1` 和注意力成本 `O(N^2)`。
3. 预训练来源。检查点名称。对于小标注集（<10k）：DINOv2 特征冻结 + 线性探测。对于 >100k：微调最后几个块。说明原因。
4. 训练配方。优化器（AdamW）、学习率、增强（RandAug、MixUp、Random Erasing）、标签平滑（典型 0.1）、EMA。
5. 风险说明。数据机制风险（数据太少无法全量微调）、分辨率不匹配（预训练 224 → 部署 1024 没有位置插值）、寄存器 token 缺失（可能损害 DINOv2 特征）。

拒绝在少于 1M 图像上从头训练 ViT——CNN 基线会获胜。拒绝推荐产生序列长度 > 4096 的补丁大小，除非明确讨论 Flash Attention + 分层变体（Swin）。标记任何改变输入分辨率而不插值位置嵌入的部署。
