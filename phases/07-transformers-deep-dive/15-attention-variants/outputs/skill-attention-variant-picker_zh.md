# 注意力变体选择器

帮助开发者为新的 transformer 或正在扩展到更长上下文的现有模型选择和证明注意力拓扑。

## 需要收集的输入

1. **训练和推理时的目标上下文长度**（通常不同——许多模型在 16K 训练并在推理时扩展）。
2. **检索需求**，1-5 级：1 = 纯聊天，5 = 大海捞针 / RAG / 带长仓库上下文的代码。
3. **推理内存预算**——每请求 KV 缓存容忍度（每 token 每层字节是正确的单位）。
4. **训练成本容忍度**——从头训练 SWA 很便宜；将差分注意力改造到预训练模型中很昂贵。
5. **硬件目标**——Hopper+ 有完整的 FlashAttention-3，Ada 有 FA2，旧 GPU 受掩码限制。

## 决策规则

- **上下文 ≤ 16K 且检索 ≤ 3**：使用 FlashAttention 的全注意力。不要过早优化。
- **上下文 16–128K 且检索 ≤ 3**：混合 SWA + 全局 5:1，窗口 1024（Gemma 3 形状）。在压缩 KV 的同时保持检索可用。
- **上下文 > 128K**：全 SWA，每 4-6 层一个全局层，加上位置插值 / YaRN 缩放（第 04 课）。
- **检索 = 5 且训练预算允许**：仅在前 4 层考虑差分注意力（一半的 KV 加倍，大部分 sink 抵消收益）。
- **你正在发布公共 API**：优先选择稳定模式（全注意力、SWA、Gemma-3 混合）。除非你有内核工程师，否则跳过原生稀疏 / DIFF。
- **你无法更改基础模型**：SWA 可以通过掩码在推理时改造；差分和稀疏不能。

## 始终标记

- 低于 7B 的纯 SWA 模型通常在推理基准上明显损失。建议反对。
- 窗口大小 < 512 几乎从不对。选择更大的或使用不同的拓扑。
- 差分注意力的论文报告是在小模型（3-7B）上。截至 2026 年初，扩展证据很少。
- 每个变体都与 RoPE / YaRN 缩放交互（第 04 课）。明确说明位置方案。

## 输出格式

返回：

1. **推荐**——单一命名拓扑（例如"Gemma-3 混合，W=1024，5:1 SWA:全局"）。
2. **理由**——将每个输入映射到上面的决策规则。
3. **KV 缓存估计**——在目标上下文下，每 token 每层字节数和批次 1 的 GB 数。
4. **迁移路径**——如果基础模型已经训练好，如何改造。
5. **已知风险**——哪些基准/工作负载可能回退。
