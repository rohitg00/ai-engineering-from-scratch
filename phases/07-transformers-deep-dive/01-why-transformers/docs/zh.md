# 为什么Transformer——RNN的问题

> RNN逐个处理令牌。Transformer一次性处理所有令牌。这一架构决策改变了2017年后深度学习的所有规模曲线。

**类型：** 学习  
**语言：** Python  
**前置知识：** 阶段3（深度学习核心），阶段5·09（序列到序列），阶段5·10（注意力机制）  
**时间：** 约45分钟

## 问题

在2017年之前，地球上每一个最先进的序列模型——语言、翻译、语音——都是循环神经网络（Recurrent Neural Network, RNN）。LSTM和GRU主导了翻译基准测试长达五年。它们是当时唯一可用的工具。

它们有三个致命弱点。顺序计算意味着无法沿着时间轴并行化：令牌`t+1`需要来自令牌`t`的隐藏状态。一个1024令牌的序列意味着GPU上需要1024个串行步骤，而GPU每个周期可以执行1,000,000次浮点运算。在专为并行设计的硬件上，训练墙钟时间与序列长度呈线性增长。

梯度消失意味着50个令牌之前的信息已经通过了50个非线性变换被压缩。门控循环单元（LSTM、GRU）减轻了压缩，但从未消除。长距离依赖——"我去年夏天在飞往京都的飞机上读的那本书是……"——经常失败。

固定宽度的隐藏状态意味着编码器在解码器看到任何东西之前，将整个源序列压缩成一个单一向量。无论源序列是5个令牌还是500个，瓶颈的形状相同。

2017年的论文《Attention Is All You Need》提出了一个激进的方案：彻底放弃循环。让每个位置并行地关注其他所有位置。在一个大的矩阵乘法中训练，而不是1024个顺序的矩阵乘法。

到2026年，这个结果主导了每一种模态。语言（GPT-5、Claude 4、Llama 4）、视觉（ViT、DINOv2、SAM 3）、音频（Whisper）、生物学（AlphaFold 3）、机器人（RT-2）。相同的模块，不同的输入。

## 概念

![RNN顺序计算 vs Transformer并行注意力](../assets/rnn-vs-transformer.svg)

**循环作为瓶颈。** RNN计算`h_t = f(h_{t-1}, x_t)`。每一步依赖于前一步。在计算`h_4`之前无法计算`h_5`。在拥有10,000多个并行核心的现代GPU上，对于长序列，这浪费了99%的硅片。

**注意力作为广播。** 自注意力（Self-attention）同时为每一对`(i, j)`计算`output_i = sum_j(a_ij * v_j)`。整个N×N注意力矩阵在一次批处理矩阵乘法中填充。没有步骤依赖于另一个步骤。GPU喜欢它。

**加速不是一个常数。** 这是`O(N)`串行深度与`O(1)`串行深度之间的差异。在实践中，在N=512时，Transformer每轮训练比匹配硬件上的RNN快5-10倍，并且随着序列长度的增长，差距会扩大，直到你遇到注意力的`O(N²)`内存墙（Flash Attention后来解决了这个问题——见第12课）。

**Transformer的代价。** 注意力的内存缩放为`O(N²)`。对于2K上下文，没问题。对于128K上下文，你需要滑动窗口、RoPE外推、Flash Attention分块或线性注意力变体。循环在时间和内存上都是`O(N)`；Transformer用时间换取内存，然后通过并行性赢回时间。

**归纳偏置的转变。** RNN假设局部性和近因性。Transformer不假设任何东西——每一对都是注意力的候选。这就是为什么Transformer需要更多数据才能训练良好，但一旦拥有足够数据，它们能扩展得更远。Chinchilla（2022）将其形式化：给定足够的令牌，一个Transformer总是能击败同等参数数量的RNN。

## 动手构建

这里没有神经网络——我们用数值模拟核心瓶颈，让你在笔记本上感受到差距。

### 第1步：测量串行深度

参见`code/main.py`。我们构建两个函数。一个将序列编码为加法链（串行，如RNN）。一个将其编码为并行规约（广播，如注意力）。相同的数学，不同的依赖图。

```python
def rnn_style(xs):
    h = 0.0
    for x in xs:
        h = 0.9 * h + x   # 无法并行化：h依赖于前一个h
    return h

def attention_style(xs):
    return sum(xs) / len(xs)  # 每个x是独立的
```

我们计时这两个函数，序列长度最高100,000个元素。RNN版本是O(N)且为单CPU流水线。即使在纯Python中，注意力风格的规约在长度≥1,000时也能击败它，因为Python的`sum()`是用C实现的，并且每次迭代没有解释器开销。

### 第2步：计算理论操作

两种算法都进行N次加法。区别在于*依赖深度*：在下一步开始之前必须顺序执行的操作数量。RNN深度 = N。注意力深度 = log(N)（使用树规约）或1（使用并行扫描）。决定GPU时间的是深度，而不是操作数量。

### 第3步：长序列上的经验缩放

我们打印一个时间表，使O(N)差距可见。在2026年的Mac笔记本上，长度小于1,000个元素的序列快得无法测量。长度为100,000的序列显示出清晰的线性扫描。将其扩展到具有12层LSTM等效的16,384令牌Transformer，你就会明白为什么在2016年训练墙钟时间是一个阻碍。

## 应用

2026年何时仍然选择RNN：

| 场景 | 选择 |
|-----------|------|
| 流式推理，一次一个令牌，恒定内存 | RNN或状态空间模型（Mamba, RWKV） |
| 极长序列（>100万令牌），注意力内存爆炸 | 线性注意力、Mamba 2、Hyena |
| 边缘设备，无矩阵乘法加速器 | 深度可分离RNN在每瓦特FLOPs上仍然胜出 |
| 其他一切（训练、批处理推理、上下文最高128K） | Transformer |

状态空间模型（State-Space Model, SSM）如Mamba本质上是具有结构化参数化的RNN，兼具两者优点：`O(N)`扫描内存，通过选择性扫描实现并行训练。它们恢复了Transformer质量的90%，同时具有更好的长上下文缩放。在2026年，大多数前沿实验室训练混合SSM+Transformer模型（例如Jamba、Samba）——循环并未消亡，它成为了一个组件。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|-----------------|-----------------------|
| 循环（Recurrence） | "RNN是顺序的" | 步骤`t`依赖于步骤`t-1`的计算，强制沿时间轴串行执行。 |
| 串行深度（Serial depth） | "图有多深" | 依赖操作的最长链；即使在无限硬件上也限制了墙钟时间。 |
| 注意力（Attention） | "让令牌互相看" | 加权和`sum_j a_ij v_j`，其中`a_ij`来自位置i和j之间的相似性得分。 |
| 上下文窗口（Context window） | "模型能看多少" | 注意力层可以作为输入的位置数量；二次方内存成本在此缩放。 |
| 归纳偏置（Inductive bias） | "架构中内置的假设" | 关于数据看起来像什么的先验；CNN假设平移不变性，RNN假设近因性。 |
| 状态空间模型（State-space model） | "带有代数背景的RNN" | 通过结构化状态空间矩阵参数化以实现并行训练的循环。 |
| 二次方瓶颈（Quadratic bottleneck） | "为什么上下文这么贵" | 注意力内存 = 序列长度的`O(N²)`；Flash Attention隐藏了常量，而不是缩放。 |
