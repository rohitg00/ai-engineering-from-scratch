# 01 · 为什么需要 Transformer——RNN 的种种问题

> RNN 一次只处理一个 token，Transformer 则一次处理全部 token。正是这一个架构上的押注，改变了 2017 年之后深度学习的每一条扩展曲线。

**类型：** 学习
**语言：** Python
**前置：** 阶段 3（深度学习核心）、阶段 5 · 09（序列到序列）、阶段 5 · 10（注意力机制）
**时长：** 约 45 分钟

## 问题所在

在 2017 年之前，地球上所有最先进的序列模型——无论是语言、翻译还是语音——都是「循环神经网络（recurrent neural network，RNN）」。在长达半个十年的时间里，「长短期记忆网络（LSTM）」和「门控循环单元（GRU）」赢下了堪比 ImageNet 级别的翻译基准测试。它们是当时所有人手中唯一的工具。

它们有三个致命弱点。串行计算意味着你无法沿时间轴并行：token `t+1` 需要 token `t` 的隐藏状态。一个 1,024 token 的序列，意味着在一块每周期可执行 1,000,000 次浮点运算的 GPU 上要跑 1,024 个串行步骤。在专为并行而设计的硬件上，训练的实际耗时（wall-clock time）却随序列长度线性增长。

「梯度消失（vanishing gradients）」意味着 50 个 token 之前的信息，早已被 50 个非线性变换层层压缩。门控循环单元（LSTM、GRU）缓和了这种碾压，却从未将其根除。长程依赖——比如「我去年夏天在飞往京都的飞机上读的那本书是……」——常常会失效。

固定宽度的隐藏状态意味着编码器要把整个源序列挤压成一个单一向量，解码器才能开始接触到任何信息。无论源序列是 5 个 token 还是 500 个 token 都无所谓——瓶颈的形状是一样的。

2017 年的论文《Attention Is All You Need》提出了一个激进的想法：彻底抛弃循环。让每个位置都能并行地关注到其他所有位置。用一次大型矩阵乘法来训练，而不是 1,024 次串行运算。

到 2026 年，其成果已主导了每一种模态。语言（GPT-5、Claude 4、Llama 4）、视觉（ViT、DINOv2、SAM 3）、音频（Whisper）、生物学（AlphaFold 3）、机器人（RT-2）。同一个模块，不同的输入。

## 核心概念

〔图：RNN 串行计算与 Transformer 并行注意力对比〕

**循环作为瓶颈。** RNN 计算 `h_t = f(h_{t-1}, x_t)`。每一步都依赖前一步。你无法在算出 `h_4` 之前就算出 `h_5`。在拥有 10,000 个以上并行核心的现代 GPU 上，对于长序列而言，这会浪费掉 99% 的硅片算力。

**注意力作为广播。** 「自注意力（self-attention）」为每一对 `(i, j)` 同时计算 `output_i = sum_j(a_ij * v_j)`。整个 N×N 的注意力矩阵在一次批量矩阵乘法（matmul）中一次性填满。没有任何一步依赖另一步。GPU 对此求之不得。

**加速并非一个常数倍率。** 它是 `O(N)` 串行深度与 `O(1)` 串行深度之间的差距。实践中，在 N=512、硬件相当的条件下，Transformer 每个 epoch 的训练速度要快 5–10 倍，而且随着序列长度增加，这一差距会不断拉大，直到你撞上注意力 `O(N²)` 的内存墙（后来由 Flash Attention 解决——参见第 12 课）。

**Transformer 的代价。** 注意力内存以 `O(N²)` 增长。对于 2K 上下文，没问题；对于 128K 上下文，你就需要滑动窗口、RoPE 外推、Flash Attention 分块（tiling），或者线性注意力的各种变体。循环在时间和内存上都是 `O(N)`；Transformer 用内存换时间，再通过并行把时间赢回来。

**归纳偏置的转变。** RNN 假设了局部性和近因性。Transformer 什么都不假设——每一对位置都是注意力的候选对象。这正是为什么 Transformer 需要更多数据才能训练得好，但一旦数据充足，它的扩展能力也更强。Chinchilla（2022）将这一点形式化了：只要 token 足够多，参数量相同的情况下 Transformer 总能击败 RNN。

## 动手构建

这里没有神经网络——我们用数值方式模拟核心瓶颈，让你在自己的笔记本上就能感受到这种差距。

### 步骤 1：测量串行深度

参见 `code/main.py`。我们构建两个函数。一个把序列编码为一连串加法（串行，类似 RNN）。另一个把它编码为并行归约（broadcast，类似注意力）。数学相同，依赖图不同。

```python
def rnn_style(xs):
    h = 0.0
    for x in xs:
        h = 0.9 * h + x   # 无法并行：h 依赖于前一个 h
    return h

def attention_style(xs):
    return sum(xs) / len(xs)  # 每个 x 都是独立的
```

我们对最长达 100,000 个元素的序列分别计时。RNN 版本是 O(N)，且是单条 CPU 流水线。即便在纯 Python 中，当长度 ≥ 1,000 时，注意力风格的归约也会胜出，因为 Python 的 `sum()` 是用 C 实现的，迭代时不会在每一步都带来解释器开销。

### 步骤 2：计算理论运算量

两种算法都做 N 次加法。区别在于*依赖深度*：在下一步能够开始之前，必须串行发生多少次运算。RNN 深度 = N。采用树形归约时注意力深度 = log(N)，采用并行扫描时则为 1。决定 GPU 时间的是深度，而非运算次数。

### 步骤 3：长序列上的实证扩展

我们打印一张计时表，让 O(N) 的差距清晰可见。在一台 2026 年的 Mac 笔记本上，小于 1,000 个元素的序列太快了，根本测不出来。100,000 个元素的序列则呈现出干净的线性扫描。把这个规模放大到一个 16,384 token 的 Transformer，与一个 12 层 LSTM 的等价物相比，你就明白了为什么在 2016 年训练的实际耗时会成为一道障碍。

## 实际运用

2026 年，何时仍应选择 RNN：

| 场景 | 选择 |
|-----------|------|
| 流式推理，一次一个 token，内存恒定 | RNN 或状态空间模型（Mamba、RWKV） |
| 极长序列（>1M token），注意力内存会爆炸 | 线性注意力、Mamba 2、Hyena |
| 没有矩阵乘加速器的边缘设备 | 深度可分离 RNN 在 FLOPs/watt 上仍占优 |
| 其他一切（训练、批量推理、上下文最长 128K） | Transformer |

像 Mamba 这样的「状态空间模型（state-space model，SSM）」本质上是带有结构化参数化的 RNN，从而兼得两者之长：`O(N)` 的扫描内存，以及通过选择性扫描（selective scan）实现的并行训练。它们能恢复 Transformer 90% 的质量，同时拥有更好的长上下文扩展性。在 2026 年，大多数前沿实验室都在训练 SSM+Transformer 的混合模型（例如 Jamba、Samba）——循环并未消亡，它只是成了一个组件。

## 练习

1. **简单。** 取 `code/main.py` 中的 `rnn_style`，把标量隐藏状态替换为一个长度为 64 的隐藏状态向量。重新测量。串行开销随隐藏状态维度增长了多少？
2. **中等。** 用纯 Python 实现一个并行前缀和（Hillis-Steele 扫描）。验证它在长度 1024 上产生与串行扫描相同的数值输出。统计其深度。
3. **困难。** 把注意力风格的归约移植到 GPU 上的 PyTorch。在序列长度从 64 扫到 65,536 的过程中分别计时。绘制曲线并解释其形状。

## 关键术语

| 术语 | 人们常说的 | 它实际的含义 |
|------|-----------------|-----------------------|
| 循环（Recurrence） | "RNN 是串行的" | 步骤 `t` 依赖于步骤 `t-1` 的计算，迫使沿时间轴串行执行。 |
| 串行深度（Serial depth） | "图有多深" | 最长的依赖运算链；即便硬件无限，它也限定了实际耗时的下界。 |
| 注意力（Attention） | "让 token 互相看见彼此" | 加权和 `sum_j a_ij v_j`，其中 `a_ij` 来自位置 i 与 j 之间的相似度分数。 |
| 上下文窗口（Context window） | "模型能看到多少" | 一个注意力层能作为输入接收的位置数量；二次方的内存开销在此处扩展。 |
| 归纳偏置（Inductive bias） | "烘焙进架构里的假设" | 关于数据长什么样的先验；CNN 假设平移不变性，RNN 假设近因性。 |
| 状态空间模型（State-space model） | "背后有代数支撑的 RNN" | 经过参数化、可通过结构化状态空间矩阵进行并行训练的循环。 |
| 二次方瓶颈（Quadratic bottleneck） | "为什么上下文这么贵" | 注意力内存 = 序列长度的 `O(N²)`；Flash Attention 隐藏的是常数项，而非扩展规律。 |
