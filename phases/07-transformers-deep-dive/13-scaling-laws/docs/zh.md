# 13 · 缩放定律

> 2020 年的 Kaplan 论文说：模型越大，损失越低。2022 年的 Hoffmann 论文说：你们训练得不够。算力会流向两个桶——参数与 token——而二者的分配方式并不显而易见。

**类型：** 学习
**语言：** Python
**前置：** 第 7 阶段 · 05（完整 Transformer），第 7 阶段 · 07（GPT）
**时长：** 约 45 分钟

## 问题所在

当你拥有 C FLOPs 的训练算力，并希望训出最好的模型时，你面对两个旋钮：

1. **多少参数（N）？** 模型越大，容量越高。
2. **多少训练 token（D）？** 数据越多，对容量的利用越充分。

FLOPs 近似按 `6 × N × D` 缩放。你可以把 N 调大、D 调小，也可以把 D 调大、N 调小。哪种更好？

在 2022 年之前，答案是「使劲把 N 推大」。GPT-3（2020）有 175B 参数，在约 300B token 上训练，token 与参数之比约为 1.7。Kaplan 缩放定律（Kaplan scaling laws）支持了这一做法。

Hoffmann 等人（2022）训练了一个名为 Chinchilla 的小型模型家族，发现了不同的结论：最优比例更接近 **每个参数 20 个 token**。GPT-3 的训练量少了 10 倍。Chinchilla（70B 参数，1.4T token）在每一项基准测试上都击败了 GPT-3（175B，300B token），而推理成本却低了 2.5 倍。

2026 年是 Chinchilla 的世界——但有一个重要的转折。Llama 3 8B 在 15 万亿 token 上训练，比例为每个参数 1,875 个 token，是 Chinchilla 最优值的 94 倍。对于会被大规模使用的模型而言，推理成本比训练成本更重要，因此为了获得更小的可部署体量而进行「过训练（over-training）」（超出 Chinchilla 最优）成了 2026 年的默认做法。

## 概念

〔图：不同 N/D 比例下损失随算力变化的 Chinchilla 曲线〕

### Hoffmann 定律

根据 Chinchilla 论文，损失遵循：

```
L(N, D) = A / N^α + B / D^β + E
```

- `N` = 参数量（不含 embedding）。
- `D` = 训练 token 数。
- `α ≈ 0.34`，`β ≈ 0.28`（大致对称）。
- `E ≈ 1.69`，不可约的损失下限。
- `A ≈ 406`，`B ≈ 411`。

随着规模扩大，两项相互权衡。在固定算力（C = 6ND）下对 `N` 求导并求解：

```
N_opt ≈ 0.6 × (C/6)^0.5
D_opt ≈ 0.6 × (C/6)^0.5
D_opt / N_opt ≈ 20
```

算力最优（compute-optimal）：每个参数 20 个 token。

### 为什么还是要过训练

Chinchilla 最优使「每个训练 FLOP 的训练损失」最小化。但训练成本你只付一次；推理成本却要永远付下去。

对于每月服务一万亿 token 的聊天机器人，推理在总成本中占主导。Llama 的做法是：训得更小、更久。8B 在 15T token 上训练是深度面向推理优化的：

- 能在消费级 GPU 上运行。
- 延迟只是 70B Chinchilla 最优模型的一小部分。
- 对大多数任务来说，质量已足够接近。

DeepMind 2024 年的论文（《过训练是新的最优解》）将此形式化。对于推理主导的工作负载，正确的比例更接近每个参数 100–500 个 token，具体取决于服务量。

### 涌现 vs 平滑

观点：某些能力（算术、多步推理、遵循思维链）会在某个规模上突然「涌现（emerge）」。

Schaeffer 等人（2023）认为这是一种测量假象：涌现指标采用了不连续的评分方式（精确匹配、阈值准确率），从而掩盖了底层 logits 中平滑的改进。连续指标（交叉熵）则呈现平滑曲线。

2026 年的共识是：基于连续损失的预测是可靠的。基准测试上的跳变往往是评分器假象。请用连续指标来规划预算。

### 2026 年的图景

缩放定律依然有效，但是：

| 因素 | 改变方式 |
|--------|-------------|
| 数据质量 | 精选「优质」token（Phi 风格）可使曲线移动 >2 倍的有效算力 |
| MoE | 总参数量与活跃 FLOPs 解耦；缩放定律按每活跃 FLOP 计 |
| 后训练 | 某些能力（指令遵循、代码）受 SFT+RLHF 的影响大于预训练 |
| 多模态 | 图像 + 文本 token 一起缩放；每种模态有各自的曲线 |
| 合成数据 | 模型自己生成训练数据；有效算力可以复利式增长 |

Muon 优化器（Kimi Moonlight，2024）在数据相同的条件下相比 AdamW 实现了约 2 倍的有效算力增益。一些 2026 年的训练任务默认使用 Muon。它改变的是缩放定律中的绝对常数，而非其形状。

## 动手实现

参见 `code/main.py`。我们实现 Chinchilla 损失方程，并在若干算力预算下分别求解算力最优的 `(N, D)`。

### 第 1 步：Chinchilla 损失

```python
def chinchilla_loss(N, D, A=406.4, B=410.7, alpha=0.34, beta=0.28, E=1.69):
    return A / N ** alpha + B / D ** beta + E
```

在固定 `C = 6ND` 下，将 `L` 作为 `(N, D)` 上的等高线绘制，找到最小值。

### 第 2 步：算力最优前沿

对于从 `1e17` 到 `1e25` FLOPs 的算力预算，在约束 `6ND = C` 下找到使损失最小的 `(N, D)`。验证比例 `D/N ≈ 20`。

### 第 3 步：过训练成本

计算为了训练一个小 10 倍的模型（N 取最优值的 1/10，D 取最优值的 10 倍）所需多付出的损失。报告作为交换所节省的推理 FLOP（与 N 成正比）。

代入 GPT-3、Chinchilla、Llama 3 8B、DeepSeek-V3（活跃参数）的已知 `(N, D)` 对，比较预测损失与已报告损失。

## 实际应用

你不太可能自己去训练一个前沿模型。但缩放定律能告诉你：

1. **你的微调是否有足够的数据。** 如果你的任务特定数据低于基座模型每个参数 20 个 token，预计会在某个损失下限处饱和。
2. **是否该选一个更大的基座模型。** 如果你的预算全花在推理上，那就偏向选更小、训练更久的模型。
3. **收益何时递减。** 超过 Chinchilla 最优值 1000 倍后，对数损失的变化就成了噪声。

**2026 年的研究走向：**

- **数据受限的局面。** 网络上高质量 token 数量有限（过滤后约 5–10 万亿英文 token）。前沿预训练正在逼近这一上限。合成数据、多语言、多模态以及 RLHF 规模化微调是下一批杠杆。
- **算力乘数技巧。** Muon 优化器、MoE、更好的数据精选——每一项都改变绝对常数，而非渐近线。
- **强化学习的缩放定律。** 仍是开放问题。早期证据表明 RL 样本上存在幂律关系，但其指数与预训练截然不同。

## 关键术语

| 术语 | 人们常说 | 实际含义 |
|------|-----------------|-----------------------|
| 参数（N） | 「模型大小」 | 不含 embedding 的权重数量；决定容量。 |
| Token（D） | 「训练数据」 | 已见过的训练 token 数；决定参数被利用得有多好。 |
| 算力（C） | 「花掉的 FLOPs」 | 对标准 transformer 约为 `6 × N × D`。 |
| Chinchilla 最优 | 「D/N ≈ 20」 | 使每个预训练 FLOP 损失最小的比例。 |
| 过训练 | 「超出 Chinchilla」 | 多花训练 FLOPs 以节省推理 FLOPs；D/N >> 20。 |
| 不可约损失 | 「下限」 | 缩放定律中的 `E` 项；数据本身的熵。 |
| 涌现能力 | 「规模上的突然跳变」 | 往往是评分器假象；连续损失是平滑的。 |
| 有效算力 | 「训练效率乘数」 | 更好的数据 / 优化器 / 架构会放大一个 FLOP 能走多远。 |
