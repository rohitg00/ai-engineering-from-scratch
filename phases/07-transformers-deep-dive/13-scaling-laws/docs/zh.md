# 缩放定律（Scaling Laws）

> 2020年的Kaplan论文称：模型越大，损失越低。2022年的Hoffmann论文则指出：你训练不足。计算资源被分配到两个篮子——参数（parameters）和词元（tokens）——如何分配并不显而易见。

**类型：** 学习
**语言：** Python
**前置知识：** Phase 7 · 05（完整Transformer），Phase 7 · 07（GPT）
**时间：** 约45分钟

## 问题

当你拥有C FLOPs的训练计算预算，并希望获得最佳模型时，面临两个旋钮：

1. **参数数量（N）**？模型越大，容量越高。
2. **训练词元数量（D）**？数据越多，容量利用越好。

FLOPs大致按 `6 × N × D` 缩放。你可以增加N减少D，或者增加D减少N。哪种更优？

在2022年之前，答案是“大力增加N”。GPT-3（2020年）是175B参数，在约300B词元上训练。每个参数大约对应1.7个词元。Kaplan缩放定律支持这一结论。

Hoffmann等人（2022年）训练了一个名为Chinchilla的小模型系列，发现了不同结果：最优比例接近**每个参数20个词元**。GPT-3的训练不足达10倍。Chinchilla（70B参数，1.4T词元）在以2.5倍更低的推理成本下，在每一项基准测试中都击败了GPT-3（175B参数，300B词元）。

2026年是Chinchilla的世界——但有一个重要转折。Llama 3 8B在15万亿词元上训练，每个参数对应1875个词元，是Chinchilla最优值的94倍。对于大规模使用的模型，推理成本比训练成本更重要，因此过度训练（超出Chinchilla最优值）以获得更小的可部署规模，已成为2026年的默认做法。

## 概念

![Chinchilla曲线：不同N/D比下损失 vs 计算量](../assets/scaling-laws.svg)

### Hoffmann定律

根据Chinchilla论文，损失遵循：

```
L(N, D) = A / N^α + B / D^β + E
```

- `N` = 参数（非嵌入层）。
- `D` = 训练词元。
- `α ≈ 0.34`，`β ≈ 0.28`（大致对称）。
- `E ≈ 1.69`，不可约损失上限。
- `A ≈ 406`，`B ≈ 411`。

随着规模扩大，两个项相互权衡。在固定计算量（C = 6ND）下对`N`求导并求解：

```
N_opt ≈ 0.6 × (C/6)^0.5
D_opt ≈ 0.6 × (C/6)^0.5
D_opt / N_opt ≈ 20
```

计算最优：每个参数20个词元。

### 为什么要过度训练

Chinchilla最优值最小化了单位训练FLOP的训练损失。但训练成本只付一次，推理成本却是持续的。

对于一个每月服务万亿词元的聊天机器人，推理成本占主导地位。Llama的方法：训练更久的小模型。8B模型在15T词元上训练，深度优化了推理：

- 可装入消费级GPU。
- 延迟仅为70B Chinchilla最优模型的一小部分。
- 质量在大多数任务上足够接近。

DeepMind 2024年的论文（“Over-training is the new optimal”）正式化了这一观点。对于推理主导的工作负载，正确的比例更接近每个参数100–500个词元，具体取决于服务量。

### 涌现（Emergence） vs 平滑性

有人声称：某些能力（算术、多步推理、思维链遵循）会在某个规模上“突然涌现”。

Schaeffer等人（2023年）认为这是测量假象：涌现指标使用不连续评分（精确匹配、阈值准确率）掩盖了底层logits的平滑改进。连续指标（交叉熵）则显示平滑曲线。

2026年的共识是：通过连续损失进行的预测是可靠的。基准测试的跳跃往往是评分器假象。应依据连续指标规划预算。

### 2026年的图景

缩放定律仍然有效，但：

| 因素 | 变化方式 |
|------|----------|
| 数据质量 | 精选“优质”词元（Phi风格）可将曲线移动超过2倍有效计算量 |
| 混合专家（MoE） | 总参数与活跃FLOPs解耦；按活跃FLOPs建立缩放定律 |
| 后训练 | 某些能力（指令遵循、代码）通过SFT+RLHF的提升大于预训练 |
| 多模态 | 图像+文本词元共同缩放；每种模态有独立曲线 |
| 合成数据 | 模型生成训练数据；有效计算量可以复合增长 |

Muon优化器（Kimi Moonlight，2024）在相同数据量下显示比AdamW有约2倍有效计算增益。2026年的一些训练运行默认使用Muon。它改变了缩放定律中的绝对常数，而非其形状。

## 动手实现

参见 `code/main.py`。我们实现Chinchilla损失方程，并在多个计算预算下求解计算最优的`(N, D)`。

### 步骤1：Chinchilla损失

```python
def chinchilla_loss(N, D, A=406.4, B=410.7, alpha=0.34, beta=0.28, E=1.69):
    return A / N ** alpha + B / D ** beta + E
```

在固定`C = 6ND`下绘制`(N, D)`上的`L`等高线。找到最小值。

### 步骤2：计算最优前沿

对于从 `1e17` 到 `1e25` FLOPs的计算预算，找到在约束`6ND = C`下最小化损失的`(N, D)`。验证比值`D/N ≈ 20`。

### 步骤3：过度训练的成本

计算训练一个规模小10倍（N为最优值的1/10，D为最优值的10倍）的模型所付出的额外损失。报告节省的推理FLOPs（与N成正比）。

### 步骤4：与实际模型对比

填入已知的GPT-3、Chinchilla、Llama 3 8B、DeepSeek-V3（活跃参数）的`(N, D)`对，比较预测损失与报告损失。

## 实际应用

你不太可能亲自训练一个前沿模型，但缩放定律告诉你：

1. **你的微调是否有足够数据。** 如果任务特定数据低于基础模型每个参数20个词元，预计损失会在某个下限饱和。
2. **是否应该选择更大的基础模型。** 如果你的全部预算都花在推理上，偏好更小但训练更久的模型。
3. **收益在何处递减。** 超过Chinchilla最优值1000倍后，对数损失变化会变成噪声。

**2026年的研究方向：**

- **数据受限 regime。** 网页上高质量词元有限（过滤后英语约5–10万亿）。前沿预训练正在接近这一上限。合成数据、多语言、多模态和RLHF规模微调是下一个杠杆。
- **计算倍增技巧。** Muon优化器、MoE、更好的数据筛选——每个都改变绝对常数，而非渐近线。
- **强化学习的缩放定律。** 未决问题。早期证据表明RL样本呈现幂律关系，但指数与预训练大不相同。

## 关键术语

| 术语 | 人们常说的意思 | 实际含义 |
|------|---------------|----------|
| 参数（N） | "模型大小" | 非嵌入层权重数量；决定容量。 |
| 词元（D） | "训练数据" | 看到的训练词元数量；决定参数利用程度。 |
| 计算量（C） | "消耗的FLOPs" | 对于标准Transformer，大约为 `6 × N × D`。 |
| Chinchilla最优 | "D/N ≈ 20" | 最小化单位预训练FLOP损失的比例。 |
| 过度训练 | "超出Chinchilla" | 花费额外训练FLOPs以节省推理FLOPs；D/N >> 20。 |
| 不可约损失 | "下限" | 缩放定律中的`E`项；数据自身的熵。 |
| 涌现能力 | "规模上的突然跳跃" | 通常是评分器假象；连续损失是平滑的。 |
| 有效计算量 | "训练效率倍增器" | 更好的数据/优化器/架构使每个FLOP走得更远。 |
