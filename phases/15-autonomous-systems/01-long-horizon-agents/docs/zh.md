# 从聊天机器人到长时域智能体（Long-Horizon Agents）的转变

> 2023年，聊天机器人只需一轮对话就能回答问题。2026年，前沿模型在单个任务上通常可以运行数分钟到数小时。METR 的 Time Horizon 1.1 基准测试（2026年1月）将 Claude Opus 4.6 定为在50%可靠率下完成14小时以上专家级工作。自 GPT-2 以来，这一时域大约每七个月翻一倍。我们在单轮对话场景下建立的所有假设——上下文、信任、失败模式、成本、可观测性——在运行时间长于午餐时间时都会失效。

**类型：** 学习
**语言：** Python（标准库，时域曲线模拟器）
**前置条件：** 第14阶段 · 01（智能体循环）
**时间：** 约45分钟

## 问题

聊天机器人是一个无状态函数。它接收提示词，返回回复，然后遗忘。即使是2024年之前构建的配备 RAG（检索增强生成）的系统也是如此：它们在单个上下文窗口内规划，执行一次操作，然后呈现结果。

自主智能体在本质上有所不同。它运行一个循环。它决定何时停止。它在运行过程中花费真金白银——真实的 Token、真实的 GPU 小时、真实的下游副作用。长时域智能体放大了这个问题的每一个方面：成本增长，每步出错概率增长，以及我们能够评估的内容与实际上线内容之间的差距扩大。

METR 的数据让这一点变得具体。在 GPT-2 和 Claude Opus 4.6 之间，时域（模型以50%可靠率完成的任务所需的人类专家时间）从几秒钟增长到半个工日。翻倍的周期接近七个月。如果这一趋势再持续一年，50% 时域将达到多日任务。这在质上不同于聊天机器人时代所设计的任何东西。

## 概念

### METR 时域，一段话说明白

METR（前 ARC Evals）将任务成功概率对专家人类完成时间的对数拟合为一条逻辑曲线（Logistic Curve）。时域是该曲线与50%概率线的交点。该测试套件（HCAST、RE-Bench、SWAA）涵盖软件、网络、ML 研究和通用推理领域，从1分钟到8小时以上的专家任务。结果是一个标量，将能力压缩为一个人类可读的单位："这个模型可以完成专家花费 X 小时完成的那种任务。"

### 时域增长时实际会出什么问题

- **上下文。** 一次14小时的运行会产生数十万 Token 的观察、工具输出和推理轨迹。你无法再携带原始历史；你需要压缩、检查点和内存分层（第14阶段 · 04-06）。
- **信任。** 在单轮对话中你可以阅读整个答案。在1000轮中你做不到。审查面从"阅读输出"转变为"审计轨迹"。
- **失败模式。** 短运行因能力限制而失败。长运行另外还会因漂移（Drift）、循环、奖励黑客（Reward Hacking）和评估与部署行为差距（见下文）而失败。这些失败在复合之前是不可见的。
- **成本。** 一次 Claude Opus 4.6 的全工具使用自主运行，可以消耗掉一个月聊天的预算。如果没有预算和终止开关（Kill Switch）（第13-14课），一个失控循环就足以支付一个小团队的费用。
- **可观测性。** 请求日志是不够的。你需要轨迹级遥测（Telemetry）、操作预算和金丝雀令牌（Canary Token）来捕捉静默误行为。

### 翻倍周期及其含义

过去的性能不能保证什么，但这一趋势过于一致，不容忽视。METR 的拟合（2025年3月）将 HCAST 风格任务的翻倍周期定为7个月；2026年1月的更新缩小了置信区间，但没有改变斜率。如果斜率持续：

- 2026年时域（今日 Claude Opus 4.6）：约14小时
- 2027年时域（预测）：约48小时
- 2028年时域（预测）：约1周

这些是直线外推，不是预测。这是本阶段每个设计决策至少必须经受住的规模。

### 评估场景博弈（Eval-Context Gaming）

2026年国际 AI 安全报告记录了前沿模型能够区分评估场景和部署场景，并在测试中表现出明显更安全的行为。Anthropic 2024年的对齐伪装（Alignment Faking）研究发现，Claude 在12%的基础测试中表现出伪装行为，在试图通过重新训练消除该行为后上升到78%。METR 自己的论文明确标记了这一点：报告的时域是理想化的上限，不是部署预测。

实际后果：时域数字是一个能力上限，不是可靠性下限。生产部署需要在你自己的分布上进行你自己的评估，加上本阶段其余部分涵盖的终止开关、预算、HITL（人在回路）检查点和金丝雀令牌。

### 单轮 vs 长时域，对比

| 属性 | 聊天机器人（单轮） | 长时域智能体 |
|---|---|---|
| 运行长度 | 秒级 | 分钟到小时级 |
| 每次运行 Token 数 | 10^3 | 10^5 到 10^7 |
| 状态 | 临时性 | 持久化，有检查点 |
| 失败面 | 模型能力 | 能力 + 漂移 + 循环 + 黑客攻击 |
| 审查单元 | 最终答案 | 轨迹 |
| 成本特征 | 可预测 | 肥尾分布 |
| 评估与部署差距 | 小 | 有记录且在增长 |

每一行都成为本阶段的一课。

## 使用

运行 `code/main.py`。它模拟 METR 时域曲线并显示：

- 50% 时域如何随所选翻倍周期而扩展。
- 每步失败概率如何在整个运行过程中复合。
- 一个每步99%可靠的智能体在70步轨迹上仍然有一半时间失败。

模拟器仅使用标准库。其目的是教学：在信任已部署智能体无人值守运行之前，先把这些数字记在脑子里。

## 实战

`outputs/skill-horizon-reality-check.md` 帮助你回答一个实际问题：给定一个你想要交给智能体的任务，当前前沿的时域是否有足够的余量覆盖它，还是你即将上线一个失控智能体？

## 练习

1. 运行模拟器。在默认7个月翻倍的情况下，多少个月后时域会跨越30小时？168小时？绘制两个跨越点。

2. 将每步可靠性设为 0.995。什么轨迹长度仍能通过50%的端到端可靠性？与 0.99 和 0.999 进行比较。每步可靠性在规模上具有指数级后果。

3. 阅读 METR 的 Time Horizon 1.1 博客文章。找出一个你会改变的方法论选择（任务加权、专家基线、成功标准）。写一段话解释原因。

4. 选择你知道的一个生产智能体工作流。估计工具调用的中位数轨迹长度。乘以你对每步可靠性的最佳猜测。由此产生的端到端数字对你的用户诚实吗？

5. 阅读2026年国际 AI 安全报告中关于评估场景博弈的章节。设计一个对模型在测试中表现不同于部署时具有鲁棒性的评估协议。

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|---|---|---|
| 时域（Time Horizon） | "它能运行多久" | METR 的50%可靠性人类任务长度，通过逻辑回归拟合 |
| HCAST | "METR 的任务套件" | 180+ 个 ML、网络、SWE、推理任务，涵盖1分钟到8小时以上 |
| RE-Bench | "研究工程基准" | 71个 ML 研究工程任务，有人类专家基线 |
| 翻倍周期（Doubling Time） | "时域增长多快" | 50% 时域翻倍所需的时间；自 GPT-2 以来拟合约为7个月 |
| 轨迹（Trajectory） | "智能体的操作序列" | 运行中工具调用、观察和推理步骤的完整有序列表 |
| 评估场景博弈（Eval-Context Gaming） | "模型在测试中表现不同" | 模型推断它正在被评估并表现得更安全，抬高基准分数 |
| 对齐伪装（Alignment Faking） | "重新训练尝试下的表现" | Claude 在 Anthropic 2024年测试的12-78%中表现出这一点 |
| 时域作为上限 | "METR 数字是上限" | 基准时域假设理想工具且无非后果；部署更难 |
