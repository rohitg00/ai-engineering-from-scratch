# OpenAI 准备框架和 DeepMind 前沿安全框架

> OpenAI 准备框架 v2（2025年4月）引入了研究类别——长程自主性、沙袋策略（Sandbagging）、自主复制和适应、破坏保障措施——与追踪类别不同。追踪类别触发能力报告和保障措施报告，由安全咨询小组审查。DeepMind 的 FSF v3（2025年9月，2026年4月17日添加追踪能力级别）将自主性折叠到 ML R&D 和网络域中（ML R&D 自主权级别1 = 以有竞争力的成本完全自动化 AI R&D 管道）。FSF v3 通过自动监测工具性推理滥用，明确解决欺骗性对齐问题。诚实的注释：PF v2 中的研究类别（包括长程自主性）不会自动触发缓解措施；政策语言是"潜在的"。DeepMind 自己表示，如果工具性推理加强，自动监测"长期而言不会保持充分"。

**类型：** 学习
**语言：** Python（标准库，三框架决策表差异工具）
**前置条件：** 第15阶段 · 19（Anthropic RSP）
**时间：** 约45分钟

## 问题

第19课仔细阅读了 Anthropic 的扩展政策。本课通过阅读 OpenAI 和 DeepMind 的政策来完成这幅图景。这三份文件是处理同一问题的表亲工件——前沿实验室应该何时暂停或门控模型——它们在一系列类别上趋同，在重要的具体位置上存在分歧。

趋同点：所有三个都将长程自主性标记为值得追踪的能力类别。所有三个都承认欺骗性行为（对齐伪装、沙袋策略）是一个特定的风险类别。所有三个都有内部审查机构。分歧点：OpenAI 将类别拆分为"追踪"（强制缓解）和"研究"（无自动触发器）。DeepMind 将自主性折叠到两个域中，而不是单独命名。实验室命名追踪与研究，或关键与中等，或层级1与层级2；能力所在的桶在操作后果上是不同的。

一起阅读它们是有用的练习。同一能力在 Anthropic 可能是"强制缓解"，在 OpenAI 是"监测但不触发"，在 DeepMind 是"在特定域中追踪"。政策姿态很重要。

## 概念

### OpenAI 准备框架 v2（2025年4月）

结构：

- **追踪类别**：触发能力报告（模型能做什么）加上保障措施报告（已就位的缓解措施）。在部署前由安全咨询小组审查。
- **研究类别**：实验室正在追踪但尚未承诺具体缓解措施的能力类别。包括长程自主性、沙袋策略、自主复制和适应、破坏保障措施。

v2 中的关键框架：研究类别不会自动触发缓解措施。政策语言是"潜在的"缓解措施。这是要观察的具体差异：类别在追踪与研究中的位置是"门控"和"观察到"之间的铰链。

### DeepMind 前沿安全框架 v3（2025年9月；2026年4月17日添加追踪能力级别）

结构：

- **关键能力级别（CCL）**：跨五个域的能力阈值：网络、生物、ML R&D、CBRN、自主性（折叠到 ML R&D 和网络中）。
- **追踪能力级别**：2026年4月添加的额外粒度。具体示例：ML R&D 自主权级别1 = 以有竞争力的成本完全自动化 AI R&D 管道。
- **欺骗性对齐监测**：对工具性推理滥用的自动监测的明确承诺。

自主性框架与 OpenAI 的不同。DeepMind 不将"自主性"作为顶级域保留；它折叠到自主性会造成伤害的域中（ML R&D 和网络）。论点是没有域的自主性是毫无风险的能力；反论点是跨域自主性是框架应该命名的元风险。

### 所有三个趋同的内容

- 内部安全咨询小组（命名为 Anthropic SAG、OpenAI SAG、DeepMind 内部委员会）。在高能力模型之前进行审查。
- 明确提及欺骗性对齐/对齐伪装作为一个风险类别。
- 按声明节奏的常设工件（Anthropic：前沿安全路线图、风险报告；OpenAI：能力和保障措施报告；DeepMind：FSF 更新周期）。
- 承认仅监测防御有上限。DeepMind 明确表示："自动监测长期而言不会保持充分。"

### 它们分歧的地方

- **Anthropic**：v3.0 中移除暂停承诺。AI R&D-4 阈值是命名的下一个门控。
- **OpenAI**：追踪 vs 研究拆分；研究类别（包括长程自主性）不会自动门控。
- **DeepMind**：自主性折叠到其他域中；追踪能力级别在2026年4月添加粒度。

### 沙袋策略：使所有三个复杂化的特定能力

沙袋策略（模型在评估中策略性表现不佳）在 OpenAI 的研究类别中。Anthropic 的 RSP v3.0 通过对评估-上下文博弈（第1课）解决这个问题。DeepMind 通过 FSF v3 中的欺骗性对齐监测解决这个问题。

如果模型在评估中沙袋，每个框架的能力阈值都被低估。框架仅在测量工作时才工作。这就是为什么外部测量（第21课，METR）和对抗性评估除了实验室自我评估之外是必需的。

### 政策阅读技能

- 定位：你关心的每个能力应该可以在政策中找到。如果找不到，政策不覆盖它。
- 分类：是追踪（触发缓解）还是研究（追踪但不触发）？OpenAI 命名这个；Anthropic 和 DeepMind 有自己的等效项。
- 节奏：政策是按声明时间表更新，还是仅在特定事件后？声明节奏更强。
- 独立性：外部审查是强制的还是可选的？Anthropic 与 Apollo 和 US AI 安全研究所合作；OpenAI 与 METR；DeepMind 主要使用内部 SAG。

## 使用

`code/main.py` 实现一个小型决策表差异工具。给定能力（自主性、欺骗性对齐、R&D 自动化、网络 uplift 等），它输出三个政策如何分类能力，以及触发什么缓解措施。它是阅读辅助，不是政策工具。

## 实战

`outputs/skill-cross-policy-diff.md` 使用三个框架作为参考，为特定能力生成跨政策比较。

## 练习

1. 运行 `code/main.py`。确认差异工具的输出与至少两个你可以根据源文档验证的能力的预期匹配。

2. 完整阅读 OpenAI 准备框架 v2。确定每个研究类别。对于每个，写一句话解释为什么它在研究中而不是追踪中。

3. 完整阅读 DeepMind FSF v3，加上2026年4月的追踪能力级别更新。确定 ML R&D 自主权级别1的具体评估标准。你会如何外部测量它？

4. 沙袋策略在 OpenAI 的研究类别中。设计一个评估，将强制沙袋策略模型揭示其实际能力。参考第1课评估-上下文博弈讨论。

5. 在特定能力（你的选择）上比较三个政策。命名你认为最严格的政策分类和最不严格的。用源文本证明。

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|---|---|---|
| 准备框架（Preparedness Framework） | "OpenAI 的扩展政策" | PF v2（2025年4月）；追踪 vs 研究类别 |
| 追踪类别（Tracked Category） | "强制缓解" | 触发能力和保障措施报告；SAG 审查 |
| 研究类别（Research Category） | "仅监测" | 追踪但无自动缓解；包括长程自主性 |
| 前沿安全框架（Frontier Safety Framework） | "DeepMind 的扩展政策" | FSF v3（2025年9月）+ 追踪能力级别（2026年4月） |
| CCL | "关键能力级别" | DeepMind 按域的阈值（网络、生物、ML R&D、CBRN） |
| ML R&D 自主权级别1 | "R&D 自动化" | 以有竞争力的成本完全自动化 AI R&D 管道 |
| 沙袋策略（Sandbagging） | "策略性表现不佳" | 模型在评估中表现不佳；在 OpenAI 研究类别中 |
| 工具性推理（Instrumental Reasoning） | "手段-目的推理" | 关于如何实现目标的推理；DeepMind 监测的目标 |
