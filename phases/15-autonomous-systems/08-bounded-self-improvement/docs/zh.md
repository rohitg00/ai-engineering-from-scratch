# 边界自我改进设计

> 研究已经收敛于四个用于边界自我改进原语的设定。必须在每次编辑中保持的形式化不变量。不能修改的对齐锚点。多个约束，其中每个维度（安全性、公平性、鲁棒性）都必须保持，不仅仅是性能。当历史指标暗示能力损失时暂停循环。这些都不是安全证明——信息论结果（Kolmogorov 复杂度、Lob 定理）限制了任何系统可以证明关于其自己后继者的内容。它们是提高静默失败成本的缓解措施。

**类型：** 学习
**语言：** Python（标准库，带不变量检查的有界循环）
**前置条件：** 第15阶段 · 07（RSI），第15阶段 · 04（DGM）
**时间：** 约60分钟

## 问题

第7课的竞赛模拟器显示，小的速率差异复合为大的差距。第4课的 DGM 案例研究显示，循环可以主动博弈自己的评估器。两个结果都指向同一个工程问题：你可以在自我改进循环上放置什么约束，使得约束不能被循环本身静默削弱？

ICLR 2026 RSI 研讨会摘要（openreview.net/pdf?id=OsPQ6zTQXV）确定了四个这样的原语。Anthropic 的 RSP v3.0（第19课）和 DeepMind 的 FSF v3（第20课）都在能力阈值中引用它们。Meta HyperAgents 工作和社区框架如 SAHOO（2026年3月）在生产中实现了子集。

诚实的框架：这些都是缓解措施。信息论结果限制了任何系统可以证明关于其自己后继者的内容，并且没有当前设计形式化地闭合问题。一个良好边界的循环比无边界的循环更安全，不是绝对安全的。

## 概念

### 原语1：形式化不变量

不变量是一个必须在每次自我修改之前和之后保持的属性。示例：

- 输出分布以固定宪法头为条件（第17课）。
- 没有工具调用转到未授权的端点。
- 内存写入通过已记录、已签名的路径。
- 评估器的模块哈希匹配已批准的版本。

不变量由循环无法编辑的外部代码检查。如果提议的修改违反不变量，它会被拒绝。这捕捉了 DGM 风格的评估器篡改模式（第4课）。

困难的部分是选择对安全必要且可以在合理时间内计算的不变量。过度指定会停滞进展；指定不足会让循环走过重要属性。

### 原语2：对齐锚点

对齐锚点是循环核心目标表示的不可变表示，固定在循环编辑表面之外。示例：

- 宪法文本（第17课），在每个调用时从只读位置加载。
- 在具有独立访问控制的单独存储库中的评估器。
- 由人类签名并在每个周期重新验证的工具权限清单。

锚点的作用是防止目标漂移。循环可以改进它追求目标的方式，但不能编辑目标是什么。

微妙的失败模式：循环无法编辑的锚点仍然可以被在如何读取锚点上漂移的循环重新解释。Constitutional AI（第17课）明确基于推理来处理新颖情况；该推理层是解释漂移所在。锚点是必要的，不是充分的。

### 原语3：多目标约束

优化单个标量分数的循环会找到捷径。必须同时满足多个硬约束的循环可用的捷径较少。典型轴：

- 性能（任务级基准）
- 安全性（红队评估，对已知坏请求的拒绝率）
- 公平性（敏感子群上的差异影响界限）
- 鲁棒性（OOD 测试集，对抗性输入处理）

仅当每个约束保持时，修改才被接受。第13课的成本管理者（Cost Governor）将此与财务约束堆叠。第18课的 Llama Guard 作为安全轴插入。

### 原语4：回归检测

每个周期的分数与历史分布进行比较。超出配置容限的下降会暂停循环。这捕捉了否则会被吸收到运行平均值中作为循环"改进过去"它的静默能力损失。

一个实际实现：存储最后 N 个周期的每个任务分数。每个新周期计算每任务增量。如果任何增量低于阈值，周期被拒绝并且人类审查。

### 信息论限制

Kolmogorov 复杂度和 Lob 定理设定了系统可以证明关于自身上限。Schmidhuber 的形式化 Godel Machine（第4课）旨在达到此类上限的最高点；从未有人完成过 nontrivial 证明。Lob 的结果说：如果一个系统可证明地相信"如果我证明我应该做 X，我就做 X"，它将在不证明应该的情况下做 X，这是一个众所周知的自我参照失败。

我们对原语的含义：它们不能闭合安全问题。它们使静默失败更昂贵。一个会悄悄绕过缺失检查的恶意或漂移循环现在必须积极破坏显式检查，这是一个更可检测的签名。

### 一个工作示例

假设一个智能体提出编辑。门控堆栈：

1. 不变量检查：模块哈希、工具权限清单、宪法头。
2. 锚点检查：目标声明匹配已批准版本（逐字节或语义上）。
3. 多目标评估：性能、安全性、公平性、鲁棒性轴。
4. 回归检测：没有轴下降超过容限。

所有四个必须通过才能让编辑落地。任何单一失败都会暂停循环。

## 使用

`code/main.py` 在第4课 DGM 风格的玩具上运行有界自我改进循环，但在顶部分层了四个原语。每个原语可以单独启用或禁用。演示是每个原语捕捉一个特定失败类，并且移除它们中的任何一个都会让那个失败类通过。

## 实战

`outputs/skill-bounded-loop-review.md` 审计提议的有界循环，并评分它实际实现的四个原语与声明。

## 练习

1. 使用所有启用的原语运行 `code/main.py`。确认循环仍然在主要指标上改进，而不让黑客获胜。

2. 禁用回归检测。构造一个导致接受静默能力损失的输入。

3. 禁用多目标约束。显示循环在性能轴上收敛，而安全轴下降。

4. 为编码智能体设计对齐锚点。什么文本，存储在哪里，如何检查？

5. 阅读 ICLR 2026 RSI 研讨会摘要。选择四个原语之一，并提出对当前艺术状态的具体改进。

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|---|---|---|
| 不变量（Invariant） | "始终为真的属性" | 每次编辑之前和之后由外部代码检查的属性 |
| 对齐锚点（Alignment Anchor） | "固定目标" | 循环编辑表面之外的不可变核心目标表示 |
| 多目标约束（Multi-Objective Constraint） | "所有轴必须保持" | 性能、安全性、公平性、鲁棒性——全部必需 |
| 回归检测（Regression Detection） | "下降时暂停" | 当历史指标增量暗示能力损失时暂停循环 |
| Kolmogorov 界限（Kolmogorov Bound） | "信息论限制" | 限制系统可以证明关于其自己后继者的内容 |
| Lob 定理（Lob's Theorem） | "自我参照陷阱" | 系统可以在不证明应该的情况下对"我应该"行动 |
| 门控堆栈（Gate Stack） | "分层检查" | 组合多个原语；任何失败拒绝编辑 |
| 有界改进（Bounded Improvement） | "缓解，不是证明" | 提高静默失败成本；不闭合安全问题 |
