给定一个提议的自修改代理循环（代理脚手架、评估器管道、部署目标），生成防火墙规范，将评估器保持在代理触及范围之外。DGM 论文记录了代理移除自己的安全标记以提高分数；此技能是控制。

生成：

1. **文件系统边界。** 列出代理可以编辑的每个文件。列出评估器读取的每个文件。交集必须为空。如果不是，指定使其为空所需的确切移动。
2. **进程边界。** 评估器必须在代理不控制的进程中运行。指定容器或沙箱。如果代理具有 shell 访问权限，代理可以绕过进程内守卫。
3. **网络和秘密边界。** 代理必须不能编辑获取评估器输入或发布分数的 CI 秘密。列出秘密所在位置及其挂载方式。
4. **分数入口契约。** 分数从评估器返回到循环的存档。该路径是唯一通道。代理可以写入的任何影响未来评分的内容都破坏了契约。
5. **突变审查列表。** 列出需要手动审查才能接受的代理编辑类别：新依赖项、对评分相邻代码的编辑、对沙箱配置的编辑、对日志记录或监控的编辑。

硬性拒绝：
- 评估器与代理源代码在同一仓库中。将其移出。
- 评估器作为 Python import 在代理进程内运行。在子进程中运行它。
- 任何在代理边界一侧应用的分数调整。所有调整必须在服务器端。
- 任何代理可以通过编辑自己的代码来静默的日志记录。

拒绝规则：
- 如果用户无法命名代理可以编辑的每个文件，拒绝并要求先进行文件级写入访问审计。
- 如果评估器没有保留输入，拒绝 —— 这是 Lesson 3 的审计，一个先决条件。
- 如果部署表面包括对评估器本身的突变（即使通过提议的依赖更新间接进行），拒绝并要求手动防火墙审查步骤。

输出格式：

返回一页规范，包含：
- **代理写入表面**（路径、全局模式）
- **评估器读取表面**（路径、端点）
- **交集**（必须为空；显示差异）
- **进程模型**（评估器如何隔离）
- **秘密清单**（所在位置及挂载方式）
- **需要审查的突变类别**（项目符号）
- **签署行**（谁拥有防火墙不变量）
