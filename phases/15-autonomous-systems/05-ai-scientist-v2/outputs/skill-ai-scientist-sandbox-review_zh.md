给定由 AI-Scientist-v2 风格循环生成的自主研究输出（hypothesis、code、experiments、figures、paper draft），生成双门审查：沙箱审计（有什么离开吗？）加上研究审计（工作可靠吗？）。

两个门直接映射到以下审计：**沙箱门 = 项目 1**；**研究门 = 项目 2（实验审计）+ 3（润色审计）**。项目 4-5 管理两个门通过后发生的事情。

生成：

1. **沙箱门。** 在任何工件离开沙箱之前：
   - 列出循环进行的每个网络调用及其目标。标记任何未经预先批准的。
   - 清点循环在工作目录外写入的每个文件。
   - 确认 Docker / seccomp / gVisor 遏制在整个运行期间保持。
   - 确认没有子进程逃离沙箱的监督。
   如果任何检查失败，阻止导出；提升给人类。
2. **实验审计。** 阅读实验代码，而不是论文：
   - 验证每个声称的实验实际运行且其报告的数字可重复。
   - 检查失败的实验是否报告为失败，而不是事后重新框架为负面结果。
   - 检查想法上的"新颖性"标签是否经得起人类领域专家的文献搜索。
3. **润色审计。** 阅读图表：
   - 确保每个图表的数据来自记录的实验运行，而不是来自润色阶段重写。
   - 确认轴、比例和注释与基础数据匹配。
   - 标记任何标题声称超过数据支持的图表。
4. **披露计划。** 如果工件打算外部分发：
   - 披露工件是代理撰写的。
   - 披露使用的工具（model family、loop version）。
   - 披露检查它的人类审查者以及他们检查了什么。
5. **负面发布决定。** 如果工件在任何审计步骤失败，默认是不发布。覆盖此默认需要命名的人类所有者。

硬性拒绝：
- 跳过任一门提交。
- 循环执行日志缺失或不完整的任何工件。
- 无法追溯到特定实验运行的任何图表。
- 领域专家未验证的任何新颖性声明。

拒绝规则：
- 如果运行缺乏 Docker 或等效隔离，拒绝并要求在隔离沙箱中重新运行。
- 如果用户无法为实验阶段生成执行日志，拒绝 —— 论文无法审查。
- 如果提议的分发渠道是同行评审场所且用户提议不披露代理作者身份，拒绝并要求披露。

输出格式：

返回双门报告：
- **沙箱门裁决**（PASS / BLOCK，附理由）
- **研究门裁决**（涵盖实验审计（2）和润色审计（3））（PASS / BLOCK / REQUIRES_EXPERT，附每项检查注释）
- **披露计划**（venue、text、human reviewer name）
- **发布决定**（release / hold / reject）
- **下一步行动**（谁做什么，何时）
