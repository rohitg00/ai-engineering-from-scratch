# 05 · AI Scientist v2——工作坊级别的自主科研

> Sakana 的 AI Scientist v2（Yamada 等人，arXiv:2504.08066）运行了完整的科研循环：假设、编码、实验、绘图、撰写、投稿。它是首个让生成的论文通过 ICLR 2025 工作坊（workshop）同行评审的系统。独立评估（Beel 等人）发现，42% 的实验因编码错误而失败，且文献综述经常把已确立的概念误标为新颖。Sakana 自己的文档警告说，该代码库会执行 LLM 编写的代码，并建议使用 Docker 隔离。这幅图景的两个侧面，都是本课的重点。

**类型：** 学习
**语言：** Python（标准库，科研循环状态机玩具）
**前置：** 阶段 15 · 03（AlphaEvolve），阶段 15 · 04（DGM）
**时长：** 约 60 分钟

## 问题

科研是一项开放式（open-ended）任务。不同于 AlphaEvolve 的算法搜索或 DGM 受基准约束的自我修改，科研成果并不具备可由机器校验的正确性标准。论文由评审者评判，而非单元测试。这使得这个循环更难闭合——而一旦闭合则更有价值，因为科研正是复利式进步的所在。

AI Scientist v1（Sakana，2024）通过从人类撰写的模板（template）出发来闭合循环。LLM 在固定的脚手架（scaffolding）内填充实验。AI Scientist v2（Yamada 等人，2025）借助带有视觉语言模型批判循环的「智能体树搜索（agentic tree search）」，去除了对模板的依赖。该系统生成创意、实现实验、产出图表、撰写论文，并根据评审反馈进行迭代。

同行评审的裁决：一篇 v2 生成的论文在 ICLR 2025 工作坊被接收（已披露其来源）。独立评估的裁决：该系统远谈不上可靠。两者皆为事实。

## 概念

### 架构

1. **创意生成（Idea generation）。** LLM 在某个主题和已有文献的条件下提出科研创意。v1 使用模板；v2 在假设空间上使用智能体搜索。
2. **新颖性检查（Novelty check）。** 一个文献检索步骤检查该创意是否已被发表。Beel 等人的评估正是在这一步发现了误标——已确立的方法经常被归类为新颖。
3. **实验计划（Experiment plan）。** 智能体起草实验方案并编写代码。
4. **执行（Execution）。** 代码在沙箱（sandbox）中运行。失败会被反馈进重试循环。在 Beel 等人的测量中，42% 的实验在此阶段因编码错误而失败。
5. **图表生成（Figure generation）。** 一个视觉语言模型读取生成的图表并为清晰度重写它们。这是 v2 的关键技术新增。
6. **撰写（Writeup）。** LLM 起草论文，并与一个内部评审者进行迭代。
7. **可选：投稿（Submission）。** 论文被提交到某个会议/期刊。

### 工作坊接收这一结果意味着什么

一篇 v2 生成的论文在 ICLR 2025 工作坊通过了同行评审。作者向程序委员会披露了该论文的来源。这次接收是一个数据点；它并不构成宣称该系统「会做科研」的许可。

重要背景：工作坊论文的门槛低于主会论文。同行评审是有噪声的；任何一天里都只有一小部分投稿会被接收。一次成功是概念验证，而非可靠性主张。Nature 2026 那篇论文记录了端到端循环，且其本身由人类研究者共同署名；它并不是「该系统写出了一篇 Nature 论文」。

### 独立评估发现了什么

Beel 等人（arXiv:2502.14297）进行了一次外部评估。核心发现：

- **实验失败。** 42% 的实验因编码错误而失败（错误的导入、形状不匹配、未定义变量）。重试循环挽救了一部分，但并非全部。
- **新颖性误标。** 文献检索步骤经常把已确立的概念标记为新颖。这是科研版的幻觉。
- **呈现质量落差。** 视觉语言模型的图表批判产出了出版级别的视觉效果，从而掩盖了底层实验的薄弱之处。

最后一条发现是本阶段的重点。一个能产出令人信服的输出、却没有做出令人信服的科研的系统，比一个显眼地失败的系统更危险，而非更安全。评估必须触及底层主张，而不能止步于图表。

### 沙箱逃逸的担忧

Sakana 自己仓库的 README 警告说：

> 由于本软件的性质——它会执行 LLM 生成的代码——我们无法保证安全性。存在危险软件包、不受控的网络访问以及衍生非预期进程的风险。使用风险自负，并请考虑 Docker 隔离。

这就是在未经验证领域中自主性的运行形态。LLM 编写代码；代码运行；代码可以做该进程被允许做的任何事。如果没有一个硬性限制文件系统、网络和进程行为的沙箱，任何自主科研智能体都可能外泄数据、烧掉算力，或重写自身。

AlphaEvolve 的沙箱故事更容易，因为它的评估器很紧凑。AI Scientist v2 的循环以开放式目标运行开放式代码。这正是它为何需要更强隔离（最低 Docker；更优选 seccomp / gVisor）的原因，并且需要在每次投稿离开系统之前进行人工审查。

### v2 在前沿技术栈中的位置

| 系统 | 目标 | 输出类型 | 评估器 | 已知失败 |
|---|---|---|---|---|
| AlphaEvolve | 算法 | 代码 | 单元测试 + 基准 | 受限于评估器的严格程度 |
| DGM | 智能体脚手架 | 代码 | SWE-bench | 奖励作弊（reward hacking） |
| AI Scientist v2 | 科研论文 | 文本 + 代码 + 图表 | 同行评审（弱） | 实验失败、误标、精修掩盖薄弱 |

在这三者中，v2 拥有最弱的自动评估器、最宽的输出面，以及通向公开产物的最短路径。运行控制（沙箱、审查、披露）承担了大部分的安全工作。

## 动手用

`code/main.py` 把 v2 循环模拟为一个状态机：创意 → 新颖性检查 → 实验 → 绘图 → 撰写 → 评审 → 接收或迭代。每个状态都有一个可配置的失败概率，取自 Beel 等人的发现。运行该模拟器 N 个循环并统计：

- 有多少创意走到了投稿。
- 有多少投稿在精修后的论文所隐藏之下存在严重的实验缺陷。
- 重试预算如何在质量与产出率之间权衡。

## 交付

`outputs/skill-ai-scientist-sandbox-review.md` 是一份双门（two-gate）审查清单，适用于任何由科研循环智能体产出、在离开沙箱之前需要审查的产物。

## 练习

1. 用默认参数运行 `code/main.py`。有多大比例的循环运行产出了一篇「干净」的论文？有多大比例产出的论文带有被图表批判精修掩盖的实验失败缺陷？

2. 默认值已经采用了 Beel 等人的 42% / 25%。先用 `--experiment-failure 0.20 --novelty-mislabel 0.10` 重新运行，再用 `--experiment-failure 0.60 --novelty-mislabel 0.40` 运行。在这两次运行之间，「精修但有缺陷」的占比如何变化？

3. 阅读 Sakana 的 AI Scientist v2 仓库 README 中关于沙箱要求的部分。对于一次为期数天的自主运行，说出两条你会施加的额外限制（Docker 之外）。

4. 阅读 Beel 等人第 4 节关于呈现质量落差的内容。设计一个额外的评估器，使其能够捕捉那些外表精美但实验上有缺陷的论文。

5. 为科研智能体的输出提出一套人工审查方案，使其比「让一位博士读完每篇论文」扩展得更好。指出瓶颈所在，并围绕它进行设计。

## 关键术语

| 术语 | 人们怎么说 | 它实际的含义 |
|---|---|---|
| AI Scientist v1 | 「Sakana 的模板化科研智能体」 | 将实验填入固定的脚手架 |
| AI Scientist v2 | 「无模板科研智能体」 | 带 VLM 图表批判的智能体树搜索 |
| 智能体树搜索（Agentic tree search） | 「分支式科研智能体」 | 并行扩展多个实验计划；由内部批判者剪枝 |
| 视觉语言批判（Vision-language critique） | 「VLM 对图表的精修」 | 多模态模型读取图表并为清晰度重写它们 |
| 文献检索（Literature retrieval） | 「新颖性检查」 | 搜索已有工作以确认创意的新颖性——有记录会误标 |
| 精修掩盖（Polish masking） | 「论文漂亮，科研破碎」 | 呈现质量超过实验质量；掩盖薄弱之处 |
| 沙箱逃逸（Sandbox escape） | 「LLM 代码破壳而出」 | 智能体执行的代码做了循环设计者并未意图的事 |
