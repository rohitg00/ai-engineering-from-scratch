# AI Scientist v2 — 研讨会级自主研究

> Sakana 的 AI Scientist v2（Yamada 等人，arXiv:2504.08066）运行完整的研究循环：假设、代码、实验、图表、撰写、提交。这是第一个让生成的论文通过 ICLR 2025 研讨会同行评审的系统。独立评估（Beel 等人）发现42%的实验因编码错误而失败，文献综述经常将既定概念错误标记为新奇。Sakana 自己的文档警告代码库执行 LLM 编写的代码，并建议 Docker 隔离。图片的两个部分都是重点。

**类型：** 学习
**语言：** Python（标准库，研究循环状态机玩具）
**前置条件：** 第15阶段 · 03（AlphaEvolve），第15阶段 · 04（DGM）
**时间：** 约60分钟

## 问题

研究是一项开放式任务。与 AlphaEvolve 的算法搜索或 DGM 的基准绑定自我修改不同，研究结果没有可机器检查的正确性标准。论文由审稿人评判，而不是单元测试。这使得循环更难闭合——如果闭合则更有价值，因为研究是复合进展所在。

AI Scientist v1（Sakana，2024）通过从人类编写的模板开始来闭合循环。LLM 在固定脚手架内填充实验。AI Scientist v2（Yamada 等人，2025）通过使用带有视觉语言模型评论循环的智能体树搜索，消除了模板要求。系统生成想法、实施实验、生成图表、撰写论文，并根据审稿人反馈进行迭代。

同行评审结论：一篇 v2 生成的论文被 ICLR 2025 研讨会接受（已披露）。独立评估结论：系统远非可靠。两者都是真实的。

## 概念

### 架构

1. **想法生成。** LLM 根据主题和先前文献提出研究想法。v1 使用模板；v2 在假设空间上使用智能体搜索。
2. **新颖性检查。** 文献检索步骤检查该想法是否已发表。这是 Beel 等人评估发现错误标记的步骤——既定方法经常被分类为新奇。
3. **实验计划。** 智能体起草实验协议并编写代码。
4. **执行。** 代码在沙箱中运行。失败被反馈到重试循环。在 Beel 等人的测量中，42%的实验在此阶段因编码错误而失败。
5. **图表生成。** 视觉语言模型读取生成的图表并为其重写以提高清晰度。这是 v2 的关键技术补充。
6. **撰写。** LLM 起草论文，与内部审稿人迭代。
7. **可选：提交。** 论文提交到某个场所。

### 研讨会接受结果意味着什么

一篇 v2 生成的论文通过了 ICLR 2025 研讨会的同行评审。作者向程序委员会披露了论文的来源。接受是一个数据点；它不是声称系统"做研究"的许可证。

重要背景：研讨会论文的门槛低于主会议论文。同行评审是有噪声的；在任何给定日期，都有一小部分提交被接受。一次成功是概念验证，不是可靠性声明。Nature 2026 论文记录了端到端循环，并且本身由人类研究人员共同撰写；它不是"系统写了一篇 Nature 论文"。

### 独立评估发现了什么

Beel 等人（arXiv:2502.14297）进行了外部评估。主要发现：

- **实验失败。** 42%的实验因编码错误（错误导入、形状不匹配、未定义变量）而失败。重试循环捕获了一些，但不是全部。
- **新颖性错误标记。** 文献检索步骤经常将既定概念标记为新奇。这是研究的幻觉等价物。
- **呈现质量差距。** 视觉语言图表评论产生了出版级视觉效果，掩盖了底层实验弱点。

最后一个发现是这个阶段的重要发现。一个产生令人信服的输出而没有做令人信服的研究的系统比一个明显失败的系统更危险，而不是更安全。评估必须触及底层声明，而不是停留在图表上。

### 沙箱逃逸（Sandbox-Escape）顾虑

Sakana 自己的存储库 README 警告：

> 由于此软件的性质，它执行 LLM 生成的代码，我们无法保证安全。存在危险包、不受控制的 Web 访问和生成意外进程的风险。使用风险自负，并考虑 Docker 隔离。

这是未验证领域中自主性的操作形态。LLM 编写代码；代码运行；代码可以做进程被允许做的任何事情。如果没有硬限制文件系统、网络和进程操作的沙箱，任何自定向研究智能体都可以外泄数据、消耗计算或重写自己。

AlphaEvolve 的沙箱故事更容易，因为它的评估器是严格的。AI Scientist v2 的循环运行具有开放式目标和开放式代码。这就是为什么它需要更强的隔离（Docker 最低；seccomp / gVisor 首选）以及在系统离开之前对每次提交进行手动审查。

### v2 在前沿技术栈中的位置

| 系统 | 目标 | 输出种类 | 评估器 | 已知失败 |
|---|---|---|---|---|
| AlphaEvolve | 算法 | 代码 | 单元测试 + 基准 | 受评估器严谨性限制 |
| DGM | 智能体脚手架 | 代码 | SWE-bench | 奖励黑客 |
| AI Scientist v2 | 研究论文 | 文本 + 代码 + 图表 | 同行评审（弱） | 实验失败、错误标记、润色掩盖弱点 |

v2 在三者中具有最弱的自动评估器、最宽的输表面和到公共工件的最短路径。操作控制（沙箱、审查、披露）正在做大部分安全工作。

## 使用

`code/main.py` 将 v2 循环模拟为状态机：想法 → 新颖性检查 → 实验 → 图表 → 撰写 → 评审 → 接受或迭代。每个状态都有从 Beel 等人发现中提取的可配置失败概率。运行模拟器 N 个循环并计数：

- 多少想法达到提交。
- 多少提交会有润色论文隐藏的实验性缺陷。
- 重试预算如何权衡质量与产量。

## 实战

`outputs/skill-ai-scientist-sandbox-review.md` 是研究循环智能体产生的任何东西在离开沙箱之前的双门审查清单。

## 练习

1. 使用默认参数运行 `code/main.py`。循环运行产生"干净"论文的分数是多少？产生图表评论润色过的实验失败缺陷论文的分数是多少？

2. 默认值已经使用 Beel 等人的 42% / 25%。用 `--experiment-failure 0.20 --novelty-mislabel 0.10` 重新运行，然后用 `--experiment-failure 0.60 --novelty-mislabel 0.40` 重新运行。两次运行之间润色但有缺陷的份额如何变化？

3. 阅读 Sakana 的 AI Scientist v2 存储库 README 关于沙箱要求。说出你将针对多日自主运行应用的另外两个限制（除了 Docker）。

4. 阅读 Beel 等人第4节关于呈现质量差距。设计一个额外的评估器，用于捕捉看起来润色但实验有缺陷的论文。

5. 提出一个研究智能体输出的、比"博士阅读每篇论文"扩展性更好的、人类审查协议。确定瓶颈并围绕它进行设计。

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|---|---|---|
| AI Scientist v1 | "Sakana 的模板化研究智能体" | 将实验填充到固定脚手架中 |
| AI Scientist v2 | "无模板研究智能体" | 带有 VLM 图表评论的智能体树搜索 |
| 智能体树搜索（Agentic Tree Search） | "分支研究智能体" | 并行扩展多个实验计划；由内部评论家修剪 |
| 视觉语言评论（Vision-Language Critique） | "图表上的 VLM 润色" | 多模态模型读取图表并为其重写以提高清晰度 |
| 文献检索（Literature Retrieval） | "新颖性检查" | 搜索先前工作以确认想法新颖性——有记录显示错误标记 |
| 润色掩盖（Polish Masking） | "漂亮论文，破碎研究" | 呈现质量超过实验质量；隐藏弱点 |
| 沙箱逃逸（Sandbox Escape） | "LLM 代码突破" | 智能体执行的代码做了循环设计者不打算做的事情 |
