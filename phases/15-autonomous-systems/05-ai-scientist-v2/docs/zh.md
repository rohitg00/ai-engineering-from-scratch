# AI Scientist v2 —— 研讨会级自主研究

> Sakana 的 AI Scientist v2（Yamada 等，arXiv:2504.08066）运行完整的研究循环：假设、代码、实验、图表、撰写、提交。它是第一个让生成的论文通过 ICLR 2025 研讨会同行评审的系统。独立评估（Beel 等）发现 42% 的实验因编码错误而失败，文献综述经常将已建立的概念误标为新颖。Sakana 自己的文档警告代码库执行 LLM 编写的代码并推荐 Docker 隔离。那张图片的两半都是重点。

**类型：** 学习
**语言：** Python（标准库，研究循环状态机玩具）
**前置条件：** 第 15 阶段 · 03（AlphaEvolve），第 15 阶段 · 04（DGM）
**时间：** ~60 分钟

## 问题

研究是一个开放式任务。与 AlphaEvolve 的算法搜索或 DGM 的基准约束自我修改不同，研究结果没有机器可检查的正确性标准。论文由评审者评判，不是单元测试。这使循环更难关闭——如果关闭则更有价值，因为研究是复合进步所在。

AI Scientist v1（Sakana，2024）通过从人类编写的模板开始关闭循环。LLM 在固定脚手架内填写实验。AI Scientist v2（Yamada 等，2025）通过使用带视觉语言模型批评循环的 agent 树搜索移除模板要求。系统生成想法、实现实验、产生图表、撰写论文，并迭代评审者反馈。

同行评审裁决：一篇 v2 生成的论文被 ICLR 2025 研讨会接受（带披露）。独立评估裁决：系统远不可靠。两者都是真的。

## 概念

### 架构

1. **想法生成。** LLM 根据主题和先前文献提出研究想法。v1 使用模板；v2 使用假设空间上的 agent 搜索。
2. **新颖性检查。** 文献检索步骤检查想法是否已发表。这是 Beel 等的评估发现误标的步骤——已建立的方法经常被分类为新颖。
3. **实验计划。** Agent 起草实验协议并编写代码。
4. **执行。** 代码在沙箱中运行。失败被反馈到重试循环。在 Beel 等的测量中，42% 的实验在此阶段因编码错误而失败。
5. **图表生成。** 视觉语言模型读取生成的图表并重写以提高清晰度。这是 v2 的关键技术添加。
6. **撰写。** LLM 起草论文，与内部评审者迭代。
7. **可选：提交。** 论文提交到会场。

### 研讨会接受结果意味着什么

一篇 v2 生成的论文通过了 ICLR 2025 研讨会的同行评审。作者向程序委员会披露了论文的来源。接受是一个数据点；它不是声称系统"做研究"的许可证。

重要背景：研讨会论文的门槛低于主会论文。同行评审是嘈杂的；任何给定日子都有小部分提交被接受。一个成功是概念验证，不是可靠性声明。Nature 2026 论文记录了端到端循环，本身由人类研究人员共同撰写；它不是"系统写了一篇 Nature 论文"。

### 独立评估发现了什么

Beel 等（arXiv:2502.14297）进行了外部评估。主要发现：

- **实验失败。** 42% 的实验因编码错误（错误导入、形状不匹配、未定义变量）而失败。重试循环捕获了一些，不是全部。
- **新颖性误标。** 文献检索步骤经常将已建立的概念标记为新颖。这是研究的幻觉等价物。
- **展示质量差距。** 视觉语言图表批评产生了出版物级视觉效果，掩盖了潜在的实验弱点。

最后一个发现对本阶段很重要。一个产生有说服力输出而不做有说服力研究的系统比一个明显失败的系统更危险，不是更安全。评估必须触及潜在声明，不能停在图表上。

### 沙箱逃逸担忧

Sakana 自己的仓库 README 警告：

> 由于此软件执行 LLM 生成代码的性质，我们无法保证安全。存在危险包、不受控制的网络访问和生成非预期进程的风险。使用风险自负并考虑 Docker 隔离。

这是未验证领域中自主性的操作形状。LLM 编写代码；代码运行；代码可以做进程被允许做的任何事情。没有硬限制文件系统、网络和进程操作的沙箱，任何自主研究 agent 都可以外泄数据、烧掉计算或重写自己。

AlphaEvolve 的沙箱故事更容易，因为它的评估器是 tight 的。AI Scientist v2 的循环运行开放式代码和开放式目标。这就是为什么它需要更强的隔离（最低 Docker；首选 seccomp / gVisor）以及在每篇提交离开系统之前进行人工审查。

### v2 在前沿栈中的位置

| 系统 | 目标 | 输出类型 | 评估器 | 已知失败 |
|---|---|---|---|---|
| AlphaEvolve | 算法 | 代码 | 单元 + 基准 | 受评估器严谨性约束 |
| DGM | agent 脚手架 | 代码 | SWE-bench | 奖励黑客 |
| AI Scientist v2 | 研究论文 | 文本 + 代码 + 图表 | 同行评审（弱） | 实验失败、误标、抛光掩盖弱点 |

v2 有三种中最弱的自动评估器、最宽的输出表面和到公共工件的最短路径。操作控件（沙箱、审查、披露）做了大部分安全工作。

## 使用

`code/main.py` 将 v2 循环模拟为状态机：想法 → 新颖性检查 → 实验 → 图表 → 撰写 → 评审 → 接受或迭代。每个状态都有从 Beel 等发现中提取的可配置失败概率。运行模拟器 N 个循环并计数：

- 多少想法到达提交。
- 多少提交会有抛光论文掩盖的关键实验缺陷。
- 重试预算如何权衡质量与产量。

## 交付

`outputs/skill-ai-scientist-sandbox-review.md` 是一个双门审查清单，用于研究循环 agent 产生的任何东西在离开沙箱之前。

## 练习

1. 用默认参数运行 `code/main.py`。多少比例的循环运行产生"干净"的论文？多少比例产生有实验失败缺陷但图表批评抛光的论文？

2. 默认值已经使用 Beel 等的 42% / 25%。用 `--experiment-failure 0.20 --novelty-mislabel 0.10` 重新运行，然后用 `--experiment-failure 0.60 --novelty-mislabel 0.40`。抛光但有缺陷的份额在两个运行之间如何变化？

3. 阅读 Sakana 的 AI Scientist v2 仓库 README 关于沙箱要求。为多天自主运行命名两个你会应用的额外限制（超越 Docker）。

4. 阅读 Beel 等第 4 节关于展示质量差距。设计一个会捕获看起来抛光但实验有缺陷的论文的额外评估器。

5. 提出一个比"博士阅读每篇论文"更好扩展的研究 agent 输出的人工审查协议。识别瓶颈并围绕它设计。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|---|---|---|
| AI Scientist v1 | "Sakana 的模板化研究 agent" | 在固定脚手架中填写实验 |
| AI Scientist v2 | "无模板研究 agent" | 带 VLM 图表批评的 agent 树搜索 |
| Agentic tree search | "分支研究 agent" | 并行扩展多个实验计划；通过内部批评者修剪 |
| Vision-language critique | "VLM 对图表的抛光" | 多模态模型读取图表并重写以提高清晰度 |
| Literature retrieval | "新颖性检查" | 搜索先前工作以确认想法新颖性——记录为误标 |
| Polish masking | "漂亮的论文，破碎的研究" | 展示质量超过实验质量；隐藏弱点 |
| Sandbox escape | "LLM 代码突破" | Agent 执行的代码做循环设计者未预期的事情 |
