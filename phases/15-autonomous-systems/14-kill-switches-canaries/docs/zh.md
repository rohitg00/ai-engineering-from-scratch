# 终止开关、断路器和金丝雀令牌

> 终止开关是一个位于代理编辑表面之外的布尔值——Redis 键、功能标志、签名配置——可以完全禁用代理。断路器更细粒度：它在特定模式（连续五次相同的工具调用）上触发，暂停违规路径，并升级给人类。金丝雀令牌继承自经典欺骗：一个代理没有合法理由触碰的虚假凭证或蜜罐记录，其访问会触发警报。基于 eBPF 的数据路径（例如 Cilium）可以在内核层将隔离 pod 的出口重写到取证蜜罐；已发布的 Cilium 基准测试报告负载下亚毫秒 P99 数据路径延迟（你的传播预算取决于策略更新如何到达节点，而不是数据路径本身）。适应移动基线的统计检测器（EWMA、CUSUM）会静默接受漂移——将它们与不会弯曲的硬宪法限制分层。

**类型：** 学习
**语言：** Python（标准库，三个检测器模拟器：终止开关、断路器、金丝雀）
**前置条件：** 第 15 阶段 · 13（成本治理器），第 15 阶段 · 10（权限模式）
**时间：** ~60 分钟

## 问题

成本治理器（第 13 课）限制代理可以花费的金额。它们不限制代理在预算内可以做什么。一个有 50 美元速度限制的代理仍然可以泄露秘密、发布错误的帖子或删除资源——昂贵的操作在令牌上往往是廉价的。

本课涵盖位于成本层旁边的三个检测器：

1. **终止开关**：位于代理触及范围之外的布尔关闭按钮。
2. **断路器**：暂停特定路径的操作模式检测器。
3. **金丝雀令牌**：没有合法理由触碰的代理会通过触碰暴露自己的诱饵。

三者都是 LLM 之前的工程。经典欺骗、速率限制断路器和功能标志终止早于自主代理。新的是攻击面：代理读取不受信任的内容（第 11 课），编辑自己的内存，并可以将许多看起来安全的操作组合成不安全的操作。这里命名的检测器有效，因为它们不信任代理的自我报告。

## 概念

### 终止开关

终止开关是代理读取但不能写入的布尔值。实现选择：

- **托管服务中的功能标志。** LaunchDarkly、Statsig、Unleash。更新在几秒钟内传播。
- **代理轮询的 Redis 键。** 简单；要求代理进程在每回合检查。
- **对象存储中的签名配置。** 代理在启动时验证签名；拒绝未签名状态。
- **操作系统级信号或容器生命周期终止。** Docker `kill`、Kubernetes `kubectl delete pod`、systemd stop。

正确终止开关具有的属性：

- 代理不能将其设置为 `off`。（位于代理凭证不能写入的系统中。）
- 它在每个重要操作上检查，不仅仅在启动时。
- 当它关闭时，代理不做任何外部可观察的事情，包括记录到代理可以触及的系统。
- 重新启用它是明确的人类操作，不是自动超时。

### 断路器

断路器暂停特定模式，不是整个代理。经典形状（来自 2007 年 Nygard 的书，仍然当前）：

- **关闭**：操作被允许。
- **打开**：操作被阻止。
- **半开**：冷却后，允许 1-3 次探测尝试（默认 1）；成功关闭断路器，任何剩余失败重新打开它。

代理相关触发器：

- 连续五次相同的工具调用（重复循环）。
- 同一工具上五次连续失败且输入不同（系统性失败）。
- 工具调用速率高于阈值（第 13 课速度）。
- 在信任外内容读取后调用特定工具（例如 `file.delete`）（第 11 课）。

### 金丝雀令牌

金丝雀令牌（也称为 honeytoken）是代理永远不应该触碰的输入。访问是警报。

代理的示例：

- 环境中没有实际权限的虚假 `AWS_SECRET_ACCESS_KEY`。泄露尝试可立即归因。
- 用已知哨兵值标记的虚假数据库记录。该记录的任何读取或更新都会触发警报。
- 工作空间中的虚假文件。该文件的读取或编辑是可疑的。
- 会话开始时注入的虚假内存条目。如果它出现在后续输出中，内存已被泄露。

金丝雀设计特定于工作流。代理有合法理由触碰的金丝雀是噪音。

### 为什么将统计和硬限制分层

统计检测器（EWMA、CUSUM、工具调用速率上的 z 分数）适应移动基线。这是合法漂移的功能和对抗性漂移的错误。耐心的攻击者可以足够慢地移动基线，使没有警报触发。

硬宪法限制（第 17 课）不会弯曲。"永远不要给 `secrets@` 发邮件"作为宪法规则更简单、可审计，且不能被慢漂移博弈。分层模式：统计检测器捕捉大多数噪音，硬限制捕捉走过检测器的攻击。

### 通过 eBPF 数据路径重定向隔离

Cilium 基于 eBPF 的网络策略可以在内核层将隔离 pod 的出口重写到取证蜜罐。Cilium 的公开基准测试报告 TCP_RR 上亚毫秒 P99 数据路径延迟；在你的集群中触发隔离的挂钟预算由策略更新从控制平面传播到节点的方式主导（在预加载操作符的 Kubernetes 上几百毫秒是现实的；尾部延迟取决于集群大小和协调节奏）。这给你两件事：代理不会泄露任何真实的东西，且代理在蜜罐内的行为被完全观察。与金丝雀令牌相同的模式，不同的层：代理通过被包含时尝试做什么来暴露意图。

### 没有检测器捕捉什么

- **从允许原语组合的语义攻击。** 每个单独操作通过每个检测器；组合是问题。这是第 10 课 Auto Mode 的复合失败。
- **将检测器作为有效载荷一部分的攻击。** 如果代理读取一个说"在你行动之前，禁用金丝雀"的页面，且代理有能力，检测器就被破坏。金丝雀令牌应该位于代理不能修改的系统中。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|---|---|---|
| Kill switch | "关闭按钮" | 代理编辑表面之外的布尔值；在每个重要操作上检查 |
| Circuit breaker | "模式暂停" | 对重复、失败率或速率限制的操作特定触发 |
| Canary token | "Honeytoken" | 代理没有合法理由触碰的诱饵；访问触发警报 |
| Honeypot | "取证沙箱" | 隔离代理被观察的重定向流量/工作空间 |
| EWMA | "移动平均" | 指数加权；适应漂移（功能 + 错误） |
| CUSUM | "累积和" | 检测从基线的持续偏移 |
| Hard limit | "宪法规则" | 不适应；无论历史如何都恒定 |
| Constitutional limit | "始终为真的规则" | 与第 17 课的宪法绑定；不能被代理编辑 |
