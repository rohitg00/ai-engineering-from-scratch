# 生成模型——分类与历史

> 每个图像模型、文本模型、视频模型和3D模型都归属于五类之一。选错类别，你将与数学苦战数周；选对类别，过去十二年的进展就会清晰地堆叠在你脑海中。

**类型：** 学习
**语言：** Python
**前置要求：** 阶段2（机器学习基础）、阶段3（深度学习核心）、阶段7 · 14（Transformer）
**时长：** 约45分钟

## 问题

生成模型只做一件事：给定从某个未知分布 `p_data(x)` 中抽取的训练样本，输出看起来像来自同一分布的新样本。人脸、句子、MIDI文件、蛋白质结构——如果你粗略一看，都是同一个问题。

关键在于 `p_data` 存在于一个具有数百万维度的空间中（一张512x512的RGB图像约有786k个维度），样本位于该空间内部的一个薄流形上，而你只有大约1000万个样本。暴力求解密度是毫无希望的。每种生成模型都是一种妥协，用一个稍微不那么困难的问题来交换一个困难问题。

在过去十二年中有五个家族存活了下来。了解每个家族所做的妥协，就能明白为什么它们在某些任务上表现出色，而在其他任务上则崩溃。

## 概念

![生成模型的五个家族——按建模对象分类](../assets/taxonomy.svg)

**1. 显式密度，可计算。** 将 `log p(x)` 写成你可以实际求和的表达式。自回归（Autoregressive）模型（PixelCNN、WaveNet、GPT）将 `p(x) = ∏ p(x_i | x_<i)` 进行分解。归一化流（Normalizing flows）（RealNVP、Glow）将 `p(x)` 构建为一个简单基的逆变换。优点：精确似然，清晰的训练损失。缺点：自回归推理是顺序的（对长序列很慢），流需要可逆架构（架构受限）。

**2. 显式密度，近似。** 从下界（ELBO）限制 `log p(x)` 并优化该界。VAE（Kingma 2013）使用带有变分后验的编码器-解码器。扩散模型（DDPM，Ho 2020）训练一个去噪器，它隐式地优化一个加权的ELBO。扩散是2026年图像、视频和3D领域占主导地位的主干。

**3. 隐式密度。** 完全跳过密度；学习一个生成器 `G(z)` 产生样本，和一个判别器 `D(x)` 区分真假。GAN（Goodfellow 2014）。推理快（一次前向传播），但训练时臭名昭著地不稳定。即使在2026年，StyleGAN 1/2/3在固定域照片级真实感（人脸、卧室）方面仍然是最先进的。

**4. 基于分数 / 连续时间。** 直接学习对数密度 `∇_x log p(x)` 的梯度（分数）。Song & Ermon (2019) 表明分数匹配将扩散推广到随机微分方程（SDE）。流匹配（Flow matching）（Lipman 2023）是2024-2026年的热门：无需模拟的训练，更直的路径，比DDPM快4-10倍采样。Stable Diffusion 3、Flux、AudioCraft 2 都使用流匹配。

**5. 基于离散编码的标记自回归。** 使用向量量化变分自编码器（VQ-VAE）或残差量化器将高维数据压缩为短序列的离散标记，然后用Transformer对标记序列进行建模。Parti、MuseNet、AudioLM、VALL-E、Sora的patch分词器都使用这种方法。这是第一类加上一个学习到的分词器。

## 五个问题的分类法

当一篇新的生成模型论文出现时，在阅读方法部分之前，先回答这五个问题。

1. **建模对象是什么？** 像素、潜变量、离散标记、3D高斯、网格、波形？
2. **密度是显式还是隐式？** 他们写下了 `log p(x)` 吗？
3. **采样：一次性还是迭代？** 迭代意味着推理更慢；一次性通常意味着对抗性或蒸馏。
4. **条件：无条件、类别、文本、图像、姿态？** 这决定了损失函数和架构支撑。
5. **评估：FID、CLIP分数、IS、人类偏好、任务准确率？** 每种都有已知的失败模式（见第14课）。

在本阶段的每节课中，你都会重新回答这五个问题。到最后，它们将成为一种直觉。

## 动手实现

本课的代码是一个轻量级可视化：使用三种玩具方法（核密度估计、离散直方图和最近样本的"类GAN"生成器）从样本中拟合一维高斯混合模型，这样你就可以在一个屏幕上打印的问题中看到显式密度与隐式密度的区别。

运行 `code/main.py`。它从双峰高斯混合中抽取2000个样本，然后输出：

```
显式密度（直方图）：p(x in [-0.5, 0.5]) ≈ 0.38
近似密度（KDE）：   p(x in [-0.5, 0.5]) ≈ 0.41
隐式密度（最近样本生成器）：打印了20个新样本，无 p(x)
```

注意：前两个让你可以问"这个点的可能性有多大？"第三个不行。这就是*显式与隐式*的区别，对于未来的每一节课都很重要。

## 实际应用

在2026年，对于哪个任务，使用哪个家族？

| 任务 | 最佳家族 | 原因 |
|------|---------|------|
| 照片级真实感人脸，狭窄领域 | StyleGAN 2/3 | 仍然最清晰，推理最快。 |
| 通用文本到图像 | 潜扩散 + 流匹配 | SD3, Flux.1, DALL-E 3. |
| 快速文本到图像 | 整流流 + 蒸馏 | SDXL-Turbo, SD3-Turbo, LCM. |
| 文本到视频 | 扩散Transformer + 流匹配 | Sora, Veo 2, Kling. |
| 语音 + 音乐 | 基于标记的自回归（AudioLM, VALL-E, MusicGen）或流匹配（AudioCraft 2） | 离散标记扩展成本低。 |
| 3D场景 | 高斯泼溅拟合，扩散先验 | 重建用3D-GS，新视角用扩散。 |
| 密度估计（无需采样） | 流 | 唯一能提供精确 `log p(x)` 的家族。 |
| 模拟 / 物理 | 流匹配，分数SDE | 直线路径，平滑向量场。 |

## 关键术语

| 术语 | 通俗说法 | 实际含义 |
|------|---------|---------|
| 生成模型 | "它生成新东西" | 学习 `p_data(x)` 的采样器，可选地暴露 `log p(x)`。 |
| 显式密度 | "你可以对它求值" | 模型提供封闭形式或可计算的 `log p(x)`。 |
| 隐式密度 | "GAN风格" | 只有采样器——无法求给定点的 `p(x)`。 |
| ELBO | "证据下界" | `log p(x)` 的可计算下界；VAE和扩散优化它。 |
| 分数 | "对数密度的梯度" | `∇_x log p(x)`；扩散和SDE模型学习这个场。 |
| 流形假设 | "数据存在于一个曲面上" | 高维数据集中在低维流形上；这是降维有效的原因。 |
| 自回归 | "预测下一个" | 将联合分布分解为条件分布的乘积。 |
| 潜变量 | "压缩编码" | 低维表示，解码器可以从中重建输入。 |

## 生产实践注记：五个家族，五种推理形状

每个家族对应不同的推理服务器成本曲线。生产推理文献将LLM推理框架化为预填充+解码；同样的分解在这里也适用：

- **自回归（第1和第5类）。** 顺序解码主导延迟；KV缓存、连续批处理和推测解码都直接适用。
- **VAE / 扩散 / 流匹配（第2和第4类）。** 没有LLM意义上的解码。成本 = `步数 × 每步成本`，而`每步成本`是在完整潜分辨率上的Transformer或U-Net前向传播。生产中的调节旋钮是步数（DDIM / DPM-Solver / 蒸馏）、批次大小和精度（bf16 / fp8 / int4）。
- **GAN（第3类）。** 一次前向传播。没有调度，没有KV缓存。TTFT ≈ 总延迟。这就是为什么StyleGAN在狭窄领域用户体验上仍然胜出的原因。

当你在论文摘要中看到"比扩散更快"时，将其翻译为"更少的步数 × 相同的每步成本"或"相同的步数 × 更低的每步成本"。其他一切都是营销。
