# 潜在扩散与稳定扩散

> 512 x 512图像上的像素空间扩散是计算战争罪。Rombach等人（2022）注意到，生成图像并不需要所有786 k个维度--您需要足够的维度来捕捉语义结构，并需要一个单独的解码器来完成其余的任务。在VAE的潜在空间内运行扩散。这个想法就是稳定扩散。

** 类型：** 构建
** 语言：** Python
** 先决条件：** 第8期· 02期（VAE）、第8期· 06期（DDPM）、第7期· 09期（ViT）
** 时间：** ~75分钟

## 问题

512²处的像素空间扩散意味着U-Net以形状“[B，3，512，512]'的张量运行。对于500兆参数U-Net，每个采样步骤约为100 GFLOPS。五十步相当于每张图像5 TFLOPS。训练十亿张图像，计算费用是荒谬的。

大多数FLOP都是为了通过网络推送感知上不重要的细节--有损VAE可以压缩掉的高频纹理。Rombach的想法：训练一次VAE（* 第一阶段 *），将其冷冻，然后完全在4通道64 x 64潜伏空间中进行扩散（* 第二阶段 *）。同样的U-Net。像素的1/16。FLOP减少约64倍，质量相当。

这是稳定扩散食谱。SD 1.x / 2.x在“64 x 64 x 4”上使用860 M U-Net，SDXL在“128 x 128 x 4”上使用2.6B U-Net，SD 3将U-Net换成具有流量匹配的扩散Transformer（DiT）。Flux.1-dev（Black Forest Labs，2024）发布了12 B参数DiT-MMDiT。所有这些都在相同的两级基片上运行。

## 概念

![Latent diffusion: VAE compression + diffusion in latent space](../assets/latent-diffusion.svg)

** 两个阶段，分别训练。**

1. ** 第1阶段- VAE。**编码器' E（x）-z '，解码器' D（z）-x '。目标压缩：在每个空间轴上进行8倍下采样+调整通道，使总潜在大小约为像素计数的1/16。损失=重建（L1 + LPIPS感知）+ KL（权重小，因此“z”不会被强迫过于高斯化，因为我们不需要从“z”进行精确采样）。通常会进行对抗性损失的训练，因此解码的图像非常清晰。

2. ** 第二阶段-在“z”上扩散。**将“z = E（x_real）”视为数据。训练U-Net（或DiT）来降噪“z_t”。推断时：通过扩散采样“z_0”，然后“x = D（z_0）”。

** 文本条件反射。**两个额外的组件。冻结文本编码器（CLIP-L用于SD 1.x，CLIP-L+OpenCLIP-G用于SD 2/XL，T5-XXL用于SD 3和Flux）。交叉注意力注入：每个U-Net块都接受“[Q =图像特征，K = V =文本标记]”并将它们混合在一起。标记是文本影响图像的唯一方式。

** 损失函数与第06课相同。**相同的DDPM /流量匹配噪声MSE。您只需交换数据域。

## 架构变体

| 模型 | 年 | 骨干 | 潜在形状 | 文本编码器 | Params |
|-------|------|----------|--------------|--------------|--------|
| SD 1.5 | 2022 | U-Net | 64×64×4 | CLIP-L（77个代币） | 860M |
| SD 2.1 | 2022 | U-Net | 64×64×4 | OpenCLIP-H | 865 M |
| SDXL | 2023 | U-Net +精炼机 | 128×128×4 | CLIP-L + OpenCLIP-G | 2.6B + 6.6B |
| SDXL-涡轮增压 | 2023 | 蒸馏 | 128×128×4 | 相同 | 1-4分步抽样 |
| SD3 | 2024 | MMDiT（多模式DiT） | 128×128×16 | T5-XXL + CLIP-L + CLIP-G | 2B / 8B |
| Flux.1-dev | 2024 | MMDiT | 128×128×16 | T5-XXL + CLIP-L | 12B |
| Flux.1-schnell | 2024 | MMDiT蒸馏 | 128×128×16 | T5-XXL + CLIP-L | 12 B，1-4步 |

趋势：用DiT（潜在补丁上的Transformer）取代U-Net，扩展文本编码器（T5击败CLIP以快速遵守），增加潜在通道（4 - 16提供更多细节裕度）。

## 建设党

' code/main.py '在06课的DDPM之上堆叠了一个玩具1-D“VAE”（身份编码器+解码器，用于演示;真正的VAE将是一个conv net），并添加了具有无分类器指导的课堂条件反射。它表明，无论您在原始1-D值上还是在编码值上运行，相同的扩散损失都有效-这是关键见解。

### 第1步：编码器/解码器

```python
def encode(x):    return x * 0.5          # toy "compression" to smaller scale
def decode(z):    return z * 2.0
```

真正的VAE已经训练了体重。对于教育学来说，这个线性地图足以表明扩散在“z”上运行，而不关心原始数据空间。

### 第2步：“z”-空间中的扩散

与第06课相同的DDPM。网络看到的数据是“z = E（x）”。采样“z_0”后，用“D（z_0）”解码。

### 第3步：无分类器指导

在培训期间，有10%的时间删除班级标签（替换为空令牌）。在推理时，计算“e_cond”和“e_uncond”，然后：

```python
eps_cfg = (1 + w) * eps_cond - w * eps_uncond
```

' w = 0 '=无指导（完全多样性），' w = 3 '=默认，' w = 7+'=饱和/过锐。

### 第4步：文本条件反射（概念，而不是代码）

用冻结的文本编码器输出替换类标签。通过交叉注意将文本嵌入反馈到U-Net：

```python
h = h + CrossAttention(Q=h, K=text_embed, V=text_embed)
```

这是类别条件扩散模型和稳定扩散之间唯一的实质性区别。

## 陷阱

- ** VAE规模不匹配。** SD 1.x VAE在编码后应用了一个缩放常数（“scaling_factor ð 0.18215”）。忘记这一点会让U-Net的潜在差异变得非常错误。每个检查站都会运送一个。
- ** 文本编码器悄然错误。** SD 3需要具有>=128个令牌的T5-XXL，并且回退到仅CLIP是有损失的。始终检查“use_t5=True”或提示保真度陨石坑。
- ** 混合潜在空间。** SDXL、SD 3、Flux均使用不同的VAE。接受SDXL潜伏训练的LoRA无法在SD3上工作。拥抱脸扩散器0.30+拒绝加载不匹配的检查点。
- ** CGM太高。** ' w > 10 '会产生饱和、油性的图像，并以牺牲多样性为代价过度契合提示。最佳点是“w = 3-7”。
- ** 阴性提示泄漏。**空的否定提示变成空标记;已填充的否定提示变成' s_uncond '。这些是不一样的;有些管道默认为空。

## 生产说明：在8 GB消费级图形处理器上运行Flux-12 B

Flux集成的参考是典型的“我有一个消费级图形处理器，我可以运送这个吗？“食谱。技巧在于应用于扩散DiT的相同三旋钮配方生产推理文献列表：

1. ** 错开加载。** Flux有三个不需要在VRAM中共存的网络：T5-XXL文本编码器（fp 32中约10 GB）、CLIP-L（小型）、12 B MMDiT和VAE。首先编码提示，* 删除 * 编码器，加载DiT，去噪，* 删除 * DiT，加载VAE，解码。消费级8 GB图形处理器一次仅适用一级。
2. ** 通过bitsandbytes进行4位量化。** T5编码器和DiT上的“BitsAndBytesConfig（load_in_4bit=True，bnb_4bit_compute_dtype=torch.bfloat16）”。削减内存8倍，根据Aritra的基准测试（链接在笔记本电脑中），文本到图像的质量下降是察觉不到的。
3. ** 处理器卸载。** ' pipe. connect_mode_pu_off（）'随着每次前向传递的进行，都会自动交换中央处理器和图形处理器之间的模块。增加10-20%的延迟，但使管道完全运行。

内存核算为：“10 GB T5 / 8 = 1.25 GB”量化，“12 B参数x 0.5字节= ~6 GB”量化DiT，加上激活。用stas 00的话来说，这是TP=1推理的极端-没有模型并行性，最大量化。对于生产，您需要在H100上运行TP=2或TP=4;对于单个开发笔记本电脑，这就是食谱。
