# StyleGAN

> 大多数发生器会同时将“z”搅拌到每一层中。StyleGAN将其拆分：首先将“z”映射到中间“w”，然后通过AdaIN在每个分辨率级别 * 注入 *' w '。这一变化解开了潜在的空间，并使逼真的面孔连续七年得到了解决。

** 类型：** 构建
** 语言：** Python
** 先决条件：** 阶段8 · 03（GAN）、阶段4 · 08（规范化）、阶段3 · 07（CNN）
** 时间：** ~45分钟

## The Problem

DCGAN通过一堆转置卷积将“z”映射到图像。问题是：“z”控制着纠缠在一起的一切--姿势、灯光、身份、背景。沿着“z”的一个轴移动，四个轴都会发生变化。您不能要求模型“同一个人，不同的姿势”，因为表示并不以这种方式考虑。

Karras等人（2019年，NVIDIA）建议：停止将“z”直接输入conv层。输入一个恒定的“4 x 4 x 512”张量作为网络输入。学习映射' z ei Z ' w ei W '的8层MLP。通过 * 自适应实例规范化 *（AdaIN）以每个分辨率注入“w”：规范化每个conv特征地图，然后通过“w”的仿射投影进行缩放和移动。为随机细节（皮肤毛孔、发绺）添加每层噪音。

结果是：“W”的“高级风格”（姿势、身份）与“精细风格”（灯光、颜色）的轴大致正交。您可以通过使用图像A的`w`表示低分辨率级别，图像B的`w`表示高分辨率级别，在两个图像之间交换样式。这解锁了编辑，跨域风格化，以及整个“StyleGAN-inversion”研究路线。

## The Concept

![StyleGAN: mapping network + AdaIN + per-layer noise](../assets/stylegan.svg)

** 映射网络。** ' f：Z ' W '，8层MLP。' Z = N（0，I）' 512 '。“W”并不被迫是高斯型的--它会学习适应数据的形状。

** 综合网络。**从习得的常数“4 x 4 x 512”开始。每个分辨率块：'上采样' conv ' AdaIN（w_i）'噪音'。决议双倍：4、8、16、32、64、128、256、512、1024。

** 亚当。**

```
AdaIN(x, y) = y_scale · (x - mean(x)) / std(x) + y_bias
```

其中“y_scale”和“y_biasis”来自“w”的仿射投影。对每个要素地图进行规范化，然后重新设计样式。这里的“样式”是特征地图的一阶和二阶统计数据。

** 每层噪音。**将单通道高斯噪音添加到每个特征地图中，并通过学习的每通道因子进行缩放。控制随机细节，而不影响全球结构。

** 截断技巧。**在推断时，采样“z”，计算“w =映射（z）”，然后“w”=+·（w -）'其中“”是许多样本的平均值“w”。''以多样性换取质量。几乎每个StyleGAN演示都使用“ðŸ 0.7”。

## StyleGAN 1 → 2 → 3

| 版本 | 年 | 创新 |
|---------|------|------------|
| StyleGAN | 2019 | 地图网络+ AdaIN +噪音+渐进增长。 |
| StyleGAN2 | 2020 | 权重解调取代AdaIN（修复液滴伪影）;跳过/剩余架构;路径长度正规化。 |
| StyleGAN 3 | 2021 | 无别名卷积+等变核;消除了纹理粘附在像素网格上。 |
| 风格GAN-XL | 2022 | 类别条件，1024²，ImageNet。 |
| R3 GAN | 2024 | 以更强的注册表进行重塑;以20倍的参数缩小了FFHQ-1024上的扩散差距。 |

2026年，StyleGAN 3仍然是（a）高FPS下的窄域真实感，（b）少镜头域自适应（在具有100张图像的新数据集上训练，冻结映射），（c）基于倒置的编辑（找到重建真实照片的“w”，然后编辑该“w”）的默认选项。对于开放领域的文本到图像来说，它不是工具，而是扩散。

## Build It

' code/main.py '以1-D形式实现了一个玩具“style-GAN Lite”：映射MLP，这是一个合成函数，它采用学习到的常数载体，并用'-派生的尺度/偏差和每层的噪音对其进行调制。它表明，通过仿射调制注入“w”与将“z”级联到生成器的输入相匹配或优于。

### Step 1: mapping network

```python
def mapping(z, M):
    h = z
    for i in range(num_layers):
        h = leaky_relu(add(matmul(M[f"W{i}"], h), M[f"b{i}"]))
    return h
```

### Step 2: adaptive instance normalization

```python
def adain(x, w_scale, w_bias):
    mu = mean(x)
    sd = std(x)
    x_norm = [(xi - mu) / (sd + 1e-8) for xi in x]
    return [w_scale * xi + w_bias for xi in x_norm]
```

每个要素地图的比例和偏差通过线性投影来自“w”。

### Step 3: per-layer noise

```python
def add_noise(x, sigma, rng):
    return [xi + sigma * rng.gauss(0, 1) for xi in x]
```

每个渠道的西格玛是可以学习的。

## Pitfalls

- ** 水滴文物。** StyleGAN 1在特征地图中产生了一个斑点小滴，因为AdaIN将平均值归零。StyleGAN 2的权重解调通过缩放卷积权重来修复它。
- ** 纹理粘着。** StyleGAN 1和2纹理遵循像素坐标，而不是对象坐标（内插时可见）。StyleGAN 3的无别名卷积通过窗口sinc过滤器解决了这个问题。
- ** 模式覆盖。**截断“RST < 0.7”看起来干净，但样本来自狭窄的圆锥体;如果您需要多样性，请使用“RST = 1.0”。
- ** 反转是有损失的。**将真实照片倒置为“W”通常通过优化或编码器（e4 e、ReStyle、HyperStyle）完成。结果在多次迭代中漂移。

## Use It

| 用例 | 方法 |
|----------|----------|
| 真实的人脸（动漫、产品、狭隘） | StyleGAN 3 FFHQ /自定义微调 |
| 从照片中编辑面部 | e4 e倒置+ StyleSpace / InterFaceGAN方向 |
| 换脸/重演 | StyleGAN +编码器+混合 |
| 阿凡达管道 | StyleGAN 3 w/ ADA，用于低数据微调 |
| 根据一些图像改编的领域 | 冻结映射网络，微调合成 |
| 多模式或文本条件生成 | 不要-使用扩散 |

对于答案是“一个人的面部照片”的产品级演示，StyleGAN在推理成本（单次向前传递，4090上<10 ms）和相同质量条的清晰度上击败了扩散。

## Production note: why StyleGAN still ships in 2026

4090上的StyleGAN 3在10 ms内生成1024² FFHQ面部-' num_steps = 1 '，无VAE解码，无交叉注意传递。用生产术语来说，这是任何图像生成器的下限延迟。相同分辨率下的50步SDXL + VAE解码流水线约为3秒。这是一个 **300 x差距 **，对于窄域产品（化身服务、ID文档管道、股票面生成），它在总体拥有成本上获胜。

两个运营后果：

- ** 没有调度器，没有批处理器。**目标占用率下的静态批次是最佳的。连续收件箱（对于LLM和扩散至关重要）提供零收益，因为每个请求都需要相同的FLOP。
- ** 截断“”是安全旋钮。**“RST & 0.7”来自映射网络范围的窄圆锥体的样本。这是服务层具有超过样本方差的唯一杠杆。在高峰负载时降低“负载”，为高级用户提高它。
