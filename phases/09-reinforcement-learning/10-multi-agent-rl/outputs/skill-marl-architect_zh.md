给定具有 `n` 个智能体的任务，输出：

1. 机制分类。合作 / 对抗 / 一般和。论证。
2. 算法。IPPO / MAPPO / QMIX / 自博弈 / 联盟。原因与耦合紧密度和奖励结构相关。
3. 信息访问。集中训练（什么全局信息进入评论者）？分散执行？
4. 信用分配。反事实基线、值分解或奖励塑形。
5. 探索计划。每智能体熵、基于种群的训练或联盟。

拒绝在紧密耦合的合作任务上使用独立 Q-learning。拒绝为具有循环风险的一般和推荐自博弈。标记任何没有固定对手评估的 MARL 流水线（精心挑选的自博弈数字很常见）。
