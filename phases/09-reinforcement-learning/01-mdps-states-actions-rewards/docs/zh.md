# 马尔可夫决策过程（MDP）、状态、动作与奖励

> 一个马尔可夫决策过程由五个要素构成：状态、动作、转移、奖励、折扣因子。强化学习中的所有内容——Q-learning、PPO、DPO、GRPO——都是在这个框架上进行优化的。掌握好这个基础，你就能轻松理解和阅读后续的强化学习内容。

**类型：** 学习
**语言：** Python
**前置知识：** 阶段1·06（概率与分布），阶段2·01（机器学习分类法）
**时长：** 约45分钟

## 问题场景

你正在编写一个国际象棋机器人、一个库存规划器、一个交易智能体，或者一个用于训练推理模型的PPO循环。这四个不同的领域，却有一个共同点：它们都可以归结为同一个数学对象。

监督学习提供的是 `(x, y)` 对，要求你拟合一个函数。而强化学习不提供标签——只提供一串状态、你采取的动作以及一个标量奖励。这一步棋是否赢得了对局？这个补货决策是否节省了成本？这笔交易是否盈利？大语言模型刚刚生成的这个token是否让评判模型给出了更高的奖励？

在将这一过程形式化之前，你无法从中学习。"我看到了什么"、"我做了什么"、"接下来发生了什么"、"那有多好"——这些都必须成为你可以推理的对象。这种形式化就是马尔可夫决策过程（MDP）。本阶段中的所有强化学习算法，包括最后的RLHF和GRPO循环，都在这个框架上进行优化。

## 核心概念

![马尔可夫决策过程：状态、动作、转移、奖励、折扣因子](../assets/mdp.svg)

**五个组成部分。**

- **状态（States）** `S`。智能体进行决策所需的全部信息。在网格世界中是单元格；在国际象棋中是棋盘；在大语言模型中是上下文窗口加上任何记忆。
- **动作（Actions）** `A`。可供选择的行为。上/下/左/右移动、走一步棋、生成一个token。
- **转移（Transitions）** `P(s' | s, a)`。在给定状态`s`和动作`a`下，下一个状态的分布。国际象棋中是确定性的，库存问题中是随机的，大语言模型解码中几乎是确定性的。
- **奖励（Rewards）** `R(s, a, s')`。标量信号。胜利=+1，失败=-1。收入减去成本。GRPO中的对数似然比项。
- **折扣因子（Discount）** `γ ∈ [0, 1)`。未来奖励相对于当前奖励的重要程度。`γ = 0.99` 对应约100步的有效视野；`γ = 0.9` 对应约10步。

**马尔可夫性质（Markov property）** `P(s_{t+1} | s_t, a_t) = P(s_{t+1} | s_0, a_0, …, s_t, a_t)`。未来仅依赖于当前状态。如果不满足，说明状态表示不完整——这不是方法的失败，而是状态的失败。

**策略与回报。** 策略（policy）`π(a | s)` 将状态映射到动作分布。回报（return）`G_t = r_t + γ r_{t+1} + γ² r_{t+2} + …` 是未来奖励的折扣和。价值函数（value）`V^π(s) = E[G_t | s_t = s]` 是在策略`π`下从状态`s`开始的期望回报。Q值（Q-value）`Q^π(s, a) = E[G_t | s_t = s, a_t = a]` 是从指定动作开始的期望回报。每个强化学习算法都会估计这两个量中的一个，然后相应地改进`π`。

**贝尔曼方程（Bellman equations）。** 本阶段所有内容都使用的固定点方程：

`V^π(s) = Σ_a π(a|s) Σ_{s', r} P(s', r | s, a) [r + γ V^π(s')]`
`Q^π(s, a) = Σ_{s', r} P(s', r | s, a) [r + γ Σ_{a'} π(a'|s') Q^π(s', a')]`

这些方程将期望回报分解为"这一步的奖励"加上"到达状态的折扣价值"。递归结构。阶段9中的每个算法要么迭代该方程直至收敛（动态规划），要么从中采样（蒙特卡洛），或者进行一步自助法（时间差分）。

## 动手实现

### 步骤1：一个微小的确定性MDP

一个4×4的网格世界。智能体从左上角开始，终点在右下角，每步奖励为-1，动作集合为`{上, 下, 左, 右}`。见 `code/main.py`。

```python
GRID = 4
TERMINAL = (3, 3)
ACTIONS = {"up": (-1, 0), "down": (1, 0), "left": (0, -1), "right": (0, 1)}

def step(state, action):
    if state == TERMINAL:
        return state, 0.0, True
    dr, dc = ACTIONS[action]
    r, c = state
    nr = min(max(r + dr, 0), GRID - 1)
    nc = min(max(c + dc, 0), GRID - 1)
    return (nr, nc), -1.0, (nr, nc) == TERMINAL
```

五行代码。这就是整个环境。确定性转移，恒定的步数惩罚，吸收终止状态。

### 步骤2：展开一个策略

策略是从状态到动作分布的映射。最简单的：均匀随机。

```python
def uniform_policy(state):
    return {a: 0.25 for a in ACTIONS}

def rollout(policy, max_steps=200):
    s, total, steps = (0, 0), 0.0, 0
    for _ in range(max_steps):
        a = sample(policy(s))
        s, r, done = step(s, a)
        total += r
        steps += 1
        if done:
            break
    return total, steps
```

运行随机策略1000次。对于这个4×4棋盘，平均回报大约在-60到-80之间。最优回报是-6（直线路径向右下）。缩小这个差距就是阶段9的全部内容。

### 步骤3：通过贝尔曼方程精确计算 `V^π`

对于小型MDP，贝尔曼方程是一个线性系统。枚举所有状态，应用期望，迭代直到价值不再变化。

```python
def policy_evaluation(policy, gamma=0.99, tol=1e-6):
    V = {s: 0.0 for s in all_states()}
    while True:
        delta = 0.0
        for s in all_states():
            if s == TERMINAL:
                continue
            v = 0.0
            for a, pi_a in policy(s).items():
                s_next, r, _ = step(s, a)
                v += pi_a * (r + gamma * V[s_next])
            delta = max(delta, abs(v - V[s]))
            V[s] = v
        if delta < tol:
            return V
```

这是迭代策略评估。它是Sutton & Barto书中的第一个算法，也是后续所有强化学习方法的理论基础。

### 步骤4：`γ` 是一个具有物理含义的超参数

有效视野大致为 `1 / (1 - γ)`。`γ = 0.9` → 10步。`γ = 0.99` → 100步。`γ = 0.999` → 1000步。

设置过低时，智能体行为短视。设置过高时，信用分配变得嘈杂，因为许多早期步骤共同为遥远的未来奖励负责。大语言模型的RLHF通常使用`γ = 1`，因为回合短且有界。控制任务使用 `0.95–0.99`。长视野策略游戏使用 `0.999`。

## 常见陷阱

- **非马尔可夫状态。** 如果需要最近三个观测才能决策，那么"状态"就不仅仅是当前观测。解决方法：堆叠帧（Atari上的DQN堆叠4帧）或使用循环状态（对观测序列使用LSTM/GRU）。
- **稀疏奖励。** 唯胜利奖励在大状态空间中使得学习几乎不可能。可以塑造奖励（中间信号）或通过模仿进行引导（阶段9 · 09）。
- **奖励欺骗（Reward hacking）。** 优化代理奖励常常导致病态行为。OpenAI的赛艇智能体为了收集强化道具而不断旋转，永远不完成比赛。一定要基于目标结果定义奖励，而不是代理指标。
- **折扣设置错误。** 在无限视野任务中设置`γ = 1`会使每个价值都无限大。必须通过有限视野或`γ < 1`来限制。
- **奖励尺度。** 奖励为{+100, -100}与{+1, -1}会得到相同的最优策略，但梯度幅度差异巨大。在接入PPO/DQN之前，归一化到`[-1, 1]`左右的范围内。

## 实际应用

2026年的技术栈在接触代码之前，会将每个强化学习流程简化为一个MDP：

| 场景 | 状态 | 动作 | 奖励 | γ |
|-----------|-------|--------|--------|---|
| 控制（运动、操作） | 关节角度+速度 | 连续力矩 | 任务特定塑造 | 0.99 |
| 游戏（国际象棋、围棋、扑克） | 棋盘+历史 | 合法动作 | 赢=+1 / 输=-1 | 1.0（有限） |
| 库存/定价 | 库存量+需求 | 订货数量 | 收入-成本 | 0.95 |
| 大语言模型的RLHF | 上下文token | 下一个token | 奖励模型最终得分 | 1.0（回合~200个token） |
| 推理的GRPO | 提示+部分回复 | 下一个token | 验证器 0/1 最终结果 | 1.0 |

在编写任何训练循环之前，先写出这五个元组。大多数"强化学习不工作"的错误报告都可以追溯到纸面上就有问题的MDP公式。

## 练习

1. **简单。** 在 `code/main.py` 中实现4×4网格世界和随机策略回滚。运行10,000个回合。报告回报的均值和标准差。与最优回报(-6)进行比较。
2. **中等。** 对均匀随机策略运行 `policy_evaluation`，参数 `γ ∈ {0.5, 0.9, 0.99}`。对每个γ，将`V`以4×4网格形式打印出来。解释为什么终端附近的状态价值随γ增大而增长更快。
3. **困难。** 使网格世界变为随机：每个动作以概率 `p = 0.1` 滑向相邻方向。重新评估均匀策略。`V[start]` 变好了还是变坏了？为什么？

## 关键术语

| 术语 | 人们说的意思 | 实际含义 |
|------|-----------------|-----------------------|
| MDP | "强化学习设置" | 满足马尔可夫性质的元组 `(S, A, P, R, γ)`。 |
| 状态 | "智能体看到的内容" | 在所选策略类别下，未来动态的充分统计量。 |
| 策略 | "智能体的行为" | 条件分布 `π(a | s)` 或确定性映射 `s → a`。 |
| 回报 | "总奖励" | 从当前步开始的折扣和 `Σ γ^t r_t`。 |
| 价值 | "一个状态有多好" | 在`π`下从`s`开始的期望回报。 |
| Q值 | "一个动作有多好" | 在`π`下从`s`开始并采取第一步动作`a`的期望回报。 |
| 贝尔曼方程 | "动态规划递归" | 将价值/Q值分解为一步奖励加上折扣后继价值的固定点分解。 |
| 折扣因子 `γ` | "未来 vs 现在" | 对未来奖励的几何权重；有效视野 `~1/(1-γ)`。 |
