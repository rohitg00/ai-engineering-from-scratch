# 多 Agent 辩论与协作

> Du 等（ICML 2024，"Society of Minds"）运行 N 个模型实例独立提出答案，然后在 R 轮中迭代地相互批评以收敛。提高事实性、规则遵循、推理能力。稀疏拓扑在 token 成本上击败全网格。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置条件：** 第 14 阶段 · 12（工作流模式），第 14 阶段 · 05（Self-Refine 和 CRITIC）
**时间：** ~60 分钟

## 学习目标

- 解释辩论协议：N 个提议者，R 轮，收敛到共享答案。
- 描述为什么辩论提高事实性、规则遵循和推理能力。
- 解释稀疏拓扑：不是每个辩论者都需要看到其他所有人。
- 实现标准库辩论，覆盖脚本化 LLM，包含全网格和稀疏变体；测量 token 成本 vs 准确率。

## 问题

Self-Refine（第 05 课）是一个模型批评自己 —— 存在群体思维风险。CRITIC（第 05 课）将批评基于外部工具 —— 并非总是可用。辩论引入第三种模式：多个实例、交叉批评、通过分歧收敛。

## 概念

### Society of Minds（Du 等，ICML 2024）

- N 个模型实例独立对同一问题提出答案。
- 在 R 轮中，每个模型阅读其他模型的提议并批评它们。
- 模型根据批评更新答案。
- R 轮后，返回收敛的答案。

原始实验使用 N=3，R=2 由于成本。在难题上（MMLU、GSM8K、国际象棋移动有效性、传记生成），准确率随更多 agent 和更多轮次提高。

跨模型组合击败单模型辩论：ChatGPT + Bard 一起 > 各自单独。

### 稀疏拓扑

"通过稀疏通信拓扑改进多 Agent 辩论"（arXiv:2406.11776，2024-2025）表明全网格辩论并非总是最优。稀疏拓扑（星型、环形、轮辐）可以在更低 token 成本下匹配准确率。每个辩论者只看到一部分同伴。

含义：

- 全网格 N=5，R=3 = 5 × 3 = 15 个提议，每个阅读 4 个同伴 = 60 次批评操作。
- 星型 N=5，R=3（一个中心 + 4 个辐条）= 15 个提议，辐条只阅读中心 = 12 次批评操作。

### 辩论何时有帮助

- **事实性。** N 个独立提议，交叉检查减少幻觉。
- **规则遵循。** 国际象棋移动有效性 —— 一个模型错过规则，其他模型捕获。
- **开放式推理。** 多种框架缩小到正确答案。

### 辩论何时有害

- **延迟敏感 UX。** N × R 串行轮次是你可能没有延迟。
- **成本敏感规模。** N × R 每个问题的 token。
- **简单事实查找。** 一次查找比五次辩论更便宜。

### 2026 年实际实例化

- **Anthropic 编排器-工作者**（第 12 课）—— 辩论的一种变体，带有综合步骤。
- **LangGraph 监督者**（第 13 课）—— 中央路由器 + 专家 agent 可以将辩论实现为节点。
- **OpenAI Agents SDK**（第 16 课）—— agent 来回交接进行迭代批评。
- **多 agent 评估** —— 配对辩论 + 评估器-优化器用于评估信号。

### 此模式出错的地方

- **收敛崩溃。** 所有 agent 收敛到第一个错误答案。通过要求分歧轮次缓解。
- **中心故障。** 在星型拓扑中，坏的中心腐蚀所有人。轮换或使用多个中心。
- **提示同质化。** 所有 agent 使用相同提示；它们产生相同答案。使用多样化提示和/或模型。

## 构建

`code/main.py` 实现标准库辩论：

- `Debater` 类（带有每个辩论者意见漂移的脚本化 LLM）。
- `FullMeshDebate` 和 `SparseDebate` 运行器。
- 三个问题：一个事实性、一个基于规则、一个推理。
- 指标：收敛答案、收敛轮次、总批评操作。

运行：

```
python3 code/main.py
```

输出：每种协议的准确率和成本；稀疏在 2/3 问题上以更低成本匹配全网格。

## 使用

- **Anthropic 编排器-工作者** 用于简单 2-3 工作者辩论。
- **LangGraph** 用于带检查点的有状态多轮辩论。
- **自定义** 用于研究或专门正确性保证。

## 练习

1. 实现"强制分歧"规则：在第 1 轮，每个辩论者必须产生不同的提议。测量对收敛速度的影响。
2. 添加置信度加权聚合：辩论者返回（答案，置信度）；聚合器按置信度加权。有帮助吗？
3. 将一个"agent"替换为具有不同意见的不同脚本化 LLM。异质性提高准确率吗？
4. 测量你的 3 个问题上全网格 vs 稀疏的 token 成本。绘制成本 vs 准确率。
5. 阅读 Society of Minds 论文。将你的玩具移植到 N=5，R=3。什么坏了？什么变好了？

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| Debate | "多 agent 批评" | N 个提议者，R 轮交叉批评，收敛 |
| Full mesh | "每个人阅读每个人" | 每个辩论者每轮阅读每个同伴 |
| Sparse topology | "有限同伴视图" | 辩论者只阅读一部分同伴 |
| Hub-and-spoke | "星型拓扑" | 一个中央辩论者，N-1 个辐条只阅读中心 |
| Convergence | "一致" | 辩论者收敛到共享答案 |
| Society of Minds | "Du 等辩论论文" | ICML 2024 多 agent 辩论方法 |
