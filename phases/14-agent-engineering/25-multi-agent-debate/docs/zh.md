# 25 · 多智能体辩论与协作

> Du 等人（ICML 2024，「心智社会（Society of Minds）」）运行 N 个模型实例，让它们各自独立提出答案，然后在 R 轮中相互批评、迭代收敛。该方法提升了事实性、规则遵循能力和推理能力。在 token 成本上，稀疏拓扑优于全连接网状（full mesh）。

**类型：** 学习 + 实践
**语言：** Python（标准库）
**前置：** 阶段 14 · 12（工作流模式），阶段 14 · 05（Self-Refine 与 CRITIC）
**时长：** 约 60 分钟

## 学习目标

- 解释辩论协议：N 个提议者、R 轮、收敛到一个共享答案。
- 描述为什么辩论能提升事实性、规则遵循能力和推理能力。
- 解释稀疏拓扑（sparse topology）：并非每个辩论者都需要看到其他所有人。
- 用标准库在一个脚本化的 LLM 之上实现辩论，包含全连接网状与稀疏两种变体；测量 token 成本与准确率的关系。

## 问题所在

Self-Refine（第 05 课）是一个模型对自己进行批评——存在「群体思维（groupthink）」的风险。CRITIC（第 05 课）把批评建立在外部工具之上——但工具并非总是可用。辩论引入了第三种模式：多个实例、相互批评、通过分歧达成收敛。

## 核心概念

### 心智社会（Du et al., ICML 2024）

- N 个模型实例针对同一个问题各自独立提出答案。
- 在 R 轮中，每个模型阅读其他模型的提议并加以批评。
- 模型根据这些批评更新自己的答案。
- R 轮之后，返回收敛后的答案。

最初的实验出于成本考虑采用 N=3、R=2。在困难问题上（MMLU、GSM8K、国际象棋走子合法性、传记生成），增加智能体数量和轮数都能提升准确率。

跨模型组合优于单模型辩论：ChatGPT + Bard 一起使用 > 任一单独使用。

### 稀疏拓扑

《Improving Multi-Agent Debate with Sparse Communication Topology》（arXiv:2406.11776，2024-2025）表明，全连接网状辩论并非总是最优。稀疏拓扑（星形、环形、中心辐射式）可以在更低的 token 成本下达到相当的准确率。每个辩论者只看到一部分同伴。

由此带来的影响：

- 全连接网状 N=5、R=3 = 5 × 3 = 15 个提议，每个读取 4 个同伴 = 60 次批评操作。
- 星形 N=5、R=3（一个中心 + 4 个辐条）= 15 个提议，辐条只读取中心 = 12 次批评操作。

### 辩论何时有帮助

- **事实性。** N 个独立提议，交叉核对可减少幻觉。
- **规则遵循。** 国际象棋走子合法性——一个模型漏掉某条规则，其他模型能发现。
- **开放式推理。** 多种表述方式逐步收敛到正确答案。

### 辩论何时有害

- **延迟敏感的用户体验。** N × R 轮串行执行带来的延迟，你可能承受不起。
- **成本敏感的规模化。** 每个问题需要 N × R 倍的 token。
- **简单的事实查找。** 一次查找比五次辩论便宜。

### 2026 年的实际落地形态

- **Anthropic 编排者-工作者（orchestrator-workers）**（第 12 课）——带有一个综合步骤的辩论变体。
- **LangGraph supervisor**（第 13 课）——中央路由器 + 专家智能体，可以把辩论实现为一个节点。
- **OpenAI Agents SDK**（第 16 课）——智能体来回交接（handoff），进行迭代式批评。
- **多智能体评估** ——将辩论与评估器-优化器（evaluator-optimizer）配对，产生评估信号。

### 这个模式在哪里会出问题

- **收敛坍塌。** 所有智能体都收敛到第一个错误答案上。可通过强制设置分歧轮次来缓解。
- **中心失效。** 在星形拓扑中，一个糟糕的中心会污染所有人。可轮换中心或使用多个中心。
- **提示同质化。** 所有智能体使用相同的提示词，于是产生相同的答案。应使用多样化的提示词和/或模型。

## 动手构建

`code/main.py` 用标准库实现了辩论：

- `Debater` 类（脚本化 LLM，带有每个辩论者各自的观点漂移）。
- `FullMeshDebate` 与 `SparseDebate` 运行器。
- 三个问题：一个事实类、一个规则类、一个推理类。
- 指标：收敛答案、收敛所需轮数、批评操作总数。

运行它：

```
python3 code/main.py
```

输出：各协议的准确率与成本；稀疏拓扑在 2/3 的问题上以更低成本达到了与全连接网状相当的效果。

## 如何使用

- **Anthropic 编排者-工作者** 用于简单的 2-3 个工作者的辩论。
- **LangGraph** 用于带检查点（checkpointing）的有状态多轮辩论。
- **自定义实现** 用于研究或专门的正确性保证。

## 练习

1. 实现一条「强制分歧」规则：在第 1 轮，每个辩论者必须产生一个各不相同的提议。测量其对收敛速度的影响。
2. 加入按置信度加权的聚合：辩论者返回 (answer, confidence)；聚合器按置信度加权。这有帮助吗？
3. 把其中一个「智能体」换成持有不同观点的另一个脚本化 LLM。异质性能提升准确率吗？
4. 在你的 3 个问题上测量全连接网状与稀疏的 token 成本。绘制成本与准确率的关系图。
5. 阅读心智社会论文。把你的玩具示例移植到 N=5、R=3。什么会崩坏？什么会变好？

## 关键术语

| 术语 | 人们怎么说 | 它实际指什么 |
|------|----------------|------------------------|
| Debate（辩论） | 「多智能体批评」 | N 个提议者，R 轮相互批评，收敛 |
| Full mesh（全连接网状） | 「每个人读每个人」 | 每轮每个辩论者都读取每个同伴 |
| Sparse topology（稀疏拓扑） | 「有限的同伴视野」 | 辩论者只读取一部分同伴 |
| Hub-and-spoke（中心辐射式） | 「星形拓扑」 | 一个中心辩论者，N-1 个辐条只读取中心 |
| Convergence（收敛） | 「达成一致」 | 辩论者收敛到一个共享答案 |
| Society of Minds（心智社会） | 「Du 等人的辩论论文」 | ICML 2024 的多智能体辩论方法 |
