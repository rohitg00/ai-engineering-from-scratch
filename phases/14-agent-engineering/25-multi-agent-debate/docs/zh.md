# 多智能体辩论与协作

> Du 等人（ICML 2024，"Society of Minds"）运行 N 个模型实例独立提出答案，然后在 R 轮中迭代地相互批评以收敛。提高事实性、规则遵循、推理能力。稀疏拓扑在 token 成本上优于全网格。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置条件：** 阶段 14 · 12（工作流模式），阶段 14 · 05（Self-Refine 和 CRITIC）
**时间：** ~60 分钟

## 学习目标

- 解释辩论协议：N 个提议者，R 轮，收敛到共享答案。
- 描述为什么辩论提高事实性、规则遵循和推理能力。
- 解释稀疏拓扑：不是每个辩论者都需要看到其他所有人。
- 使用标准库实现脚本化 LLM 的辩论，包含全网格和稀疏变体；测量 token 成本与准确性。

## 问题

Self-Refine（课程 05）是一个模型自我批评——存在群体思维风险。CRITIC（课程 05）基于外部工具进行批评——并非总是可用。辩论引入第三种模式：多个实例，交叉批评，通过分歧收敛。

## 概念

### Society of Minds（Du 等人，ICML 2024）

- N 个模型实例独立提出相同问题的答案。
- 在 R 轮中，每个模型读取其他模型的提议并批评它们。
- 模型根据批评更新其答案。
- R 轮后，返回收敛的答案。

由于成本，原始实验使用 N=3，R=2。在困难问题（MMLU、GSM8K、Chess Move Validity、传记生成）上，更多智能体和更多轮次提高准确性。

跨模型组合击败单模型辩论：ChatGPT + Bard 一起 > 单独任一。

### 稀疏拓扑

"使用稀疏通信拓扑改进多智能体辩论"（arXiv:2406.11776，2024-2025）表明全网格辩论并不总是最优的。稀疏拓扑（星型、环形、中心辐射型）可以以更低的 token 成本匹配准确性。每个辩论者仅看到同行的一个子集。

影响：

- 全网格 N=5，R=3 = 5 × 3 = 15 个提议，每个读取 4 个同行 = 60 个批评操作。
- 星型 N=5，R=3（一个中心 + 4 个辐条）= 15 个提议，辐条仅读取中心 = 12 个批评操作。

### 辩论有帮助的情况

- **事实性。** N 个独立提议，交叉检查减少幻觉。
- **规则遵循。** 国际象棋走子有效性——一个模型错过规则，其他捕获它。
- **开放式推理。** 多种框架缩小到正确答案。

### 辩论有损的情况

- **延迟敏感 UX。** N × R 串行轮次是您可能没有的延迟。
- **成本敏感规模。** 每个问题 N × R 个 token。
- **简单事实查找。** 一次查找比五次辩论更便宜。

### 2026 年实际实例化

- **Anthropic orchestrator-workers**（课程 12）— 带有合成步骤的辩论变体。
- **LangGraph supervisor**（课程 13）— 中央路由器 + 专家智能体可以将辩论实现为节点。
- **OpenAI Agents SDK**（课程 16）— 智能体来回切换以进行迭代批评。
- **多智能体评估** — 配对辩论 + 评估器-优化器以获取评估信号。

### 这种模式出错的地方

- **收敛崩溃。** 所有智能体收敛到第一个错误答案。通过要求的分歧轮次缓解。
- **中心故障。** 在星型拓扑中，坏中心破坏每个人。轮换或使用多个中心。
- **提示同质化。** 所有智能体使用相同提示；它们产生相同答案。使用多样化提示和/或模型。

## 构建

`code/main.py` 实现标准库辩论：

- `Debater` 类（带每个辩论者观点漂移的脚本化 LLM）。
- `FullMeshDebate` 和 `SparseDebate` 运行器。
- 三个问题：一个事实性的，一个基于规则的，一个推理的。
- 指标：收敛答案，到收敛的轮次，总批评操作。

运行它：

```
python3 code/main.py
```

输出：每协议准确性和成本；稀疏在 2/3 问题上以更低成本匹配全网格。

## 使用

- **Anthropic orchestrator-workers** 用于简单的 2-3 工作者辩论。
- **LangGraph** 用于带检查点的有状态多轮辩论。
- **自定义** 用于研究或专门的正确性保证。

## 交付

`outputs/skill-debate.md` 搭建具有可配置拓扑、N、R 和收敛规则的多智能体辩论。

## 练习

1. 实现"强制分歧"规则：在第 1 轮，每个辩论者必须产生不同的提议。测量对收敛速度的影响。
2. 添加置信度加权聚合：辩论者返回（答案，置信度）；聚合器按置信度加权。有帮助吗？
3. 将一个"智能体"换成为具有不同观点的不同脚本化 LLM。异质性提高准确性吗？
4. 测量全网格与稀疏在您的 3 个问题上的 token 成本。绘制成本与准确性。
5. 阅读 Society of Minds 论文。将您的玩具移植到 N=5，R=3。什么坏了？什么变好了？

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|------|----------------|------------------------|
| 辩论 | "多智能体批评" | N 个提议者，R 轮交叉批评，收敛 |
| 全网格 | "每个人读取每个人" | 每个辩论者每轮读取每个同行 |
| 稀疏拓扑 | "有限同行视图" | 辩论者仅读取同行的一个子集 |
| 中心辐射型 | "星型拓扑" | 一个中央辩论者，N-1 个辐条仅读取中心 |
| 收敛 | "一致" | 辩论者收敛到共享答案 |
| Society of Minds | "Du 等人辩论论文" | ICML 2024 多智能体辩论方法 |
