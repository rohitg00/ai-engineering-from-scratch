# 评估驱动的智能体开发

> Anthropic 的指导："从简单提示开始，用全面评估优化它们，并仅在需要时添加多步骤智能体系统。"评估不是最后一步。它是驱动阶段 14 中每个其他选择的外循环。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置条件：** 阶段 14 全部。
**时间：** ~60 分钟

## 学习目标

- 命名三个评估层——静态基准、自定义离线、在线生产——以及每个的用途。
- 解释评估器-优化器紧密循环。
- 描述 2026 年最佳实践：评估与代码共存，在 CI 中运行，门控 PR。
- 将阶段 14 的每个课程连接到它生成的评估案例。

## 问题

智能体通过演示。它们在生产中以演示无法预测的方式失败。基准回答问题"这个模型是否广泛有能力？"而不是"这个智能体是否为我的产品交付正确的补丁？"答案：三层评估，持续运行，每个防护栏和学习的规则映射到评估案例。

## 概念

### 三个评估层

1. **静态基准** — 用于代码的 SWE-bench Verified（课程 19）、用于浏览/桌面的 WebArena/OSWorld（课程 20）、用于通才的 GAIA（课程 19）、用于工具使用的 BFCL V4（课程 06）。用于跨模型比较和回归门控。污染是真实的：SWE-bench+ 发现 32.67% 解决方案泄漏。始终报告 Verified / +-audited 分数。

2. **自定义离线评估** — 您产品的形态：
   - LLM 作为判断（Langfuse、Phoenix、Opik — 课程 24）。
   - 基于执行的（运行补丁，检查测试）。
   - 基于轨迹的（将动作序列与黄金比较；OSWorld-Human 显示顶级智能体 1.4-2.7 倍超过黄金）。

3. **在线评估** — 生产：
   - 会话重放（Langfuse）。
   - 防护栏触发的警报（课程 16、21）。
   - 每步成本/延迟跟踪（课程 23 OTel span）。

### 评估器-优化器（Anthropic）

紧密循环：

1. 提议者生成输出。
2. 评估器判断。
3. 精炼直到评估器通过。

这是 Self-Refine（课程 05）的泛化。您关心的任何智能体流都可以包装在评估器-优化器中以提高可靠性。

### 2026 年最佳实践

- 评估与代码共存。
- 在每个 PR 中在 CI 中运行。
- 门控合并基于评估分数（例如，"无回归 > 5% vs main"）。
- 每个防护栏映射到评估案例。
- 每个学习的规则（Reflexion、pro-workflow learn-rule）映射到失败案例。

### 评估驱动开发失败的地方

- **无基线。** 没有最后已知良好的评估不可读。存储基线。
- **无依据的 LLM 判断。** 判断者也幻觉。CRITIC 模式（课程 05）— 判断者基于外部工具。
- **过度拟合评估。** 为评估优化偏离生产有用性。轮换案例。
- **不稳定评估。** 非确定性案例导致误报。固定种子，快照状态。

## 构建

`code/main.py` 是标准库评估框架：

- 带类别的案例注册表（基准、自定义、在线）。
- 受测脚本化智能体。
- 评估器-优化器循环：提议、判断、精炼直到通过或最大轮次。
- CI 门：聚合通过率 + 对基线的回归。

运行它：

```
python3 code/main.py
```

输出：每案例通过/失败、回归标志、CI 门判决。

## 使用

- 在与智能体代码相同的仓库中编写评估案例。
- 通过 CI 在每个 PR 上运行它们。
- 在回归时使构建失败。
- 随时间跟踪通过率。
- 将每个生产失败绑定到新案例。

## 交付

`outputs/skill-eval-suite.md` 为智能体产品构建带 CI 门和回归跟踪的三层评估套件。

## 练习

1. 取您的一个生产失败。编写重现它的评估案例。您的智能体现在通过它吗？
2. 为您的领域构建带三个维度（事实性、语气、范围）的 LLM 判断 rubric。评分 50 个会话。
3. 将评估套件连接到 CI。在 >=5% 回归时使构建失败。
4. 添加轨迹效率指标：智能体采取的步骤与黄金轨迹相比如何？
5. 将阶段 14 的每个课程映射到您套件中的评估案例。有任何缺失吗？那是需要关闭的差距。

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|------|----------------|------------------------|
| 静态基准 | "现成评估" | SWE-bench、GAIA、AgentBench、WebArena、OSWorld |
| 自定义离线评估 | "领域评估" | 在您产品形态上的 LLM 作为判断/执行/轨迹 |
| 在线评估 | "生产评估" | 会话重放、防护栏警报、成本/延迟跟踪 |
| 评估器-优化器 | "提议-判断-精炼" | 迭代直到判断通过 |
| CI 门 | "合并阻挡器" | 在评估回归时使构建失败 |
| 基线 | "最后已知良好" | 检测回归的参考分数 |
| 轨迹效率 | "步骤超过黄金" | 智能体步骤计数除以人类专家最小值 |
