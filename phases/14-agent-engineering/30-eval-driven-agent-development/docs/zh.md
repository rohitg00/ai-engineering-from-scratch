# 评估驱动的 Agent 开发

> Anthropic 的指导："从简单提示词开始，用全面评估优化它们，只在需要时添加多步 agent 系统。"评估不是最后一步。它是驱动第 14 阶段所有其他选择的外循环。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置条件：** 第 14 阶段全部。
**时间：** ~60 分钟

## 学习目标

- 说出三个评估层 —— 静态基准、自定义离线、在线生产 —— 以及各自的用途。
- 解释评估器-优化器紧密循环。
- 描述 2026 年最佳实践：评估与代码共存，在 CI 中运行，控制 PR。
- 将第 14 阶段的每节课连接到它生成的评估用例。

## 问题

Agent 通过演示。它们以演示无法预测的方式在生产中失败。基准回答"这个模型是否广泛有能力？"而不是"这个 agent 是否为我的产品发送了正确的补丁？"答案是：三个层的评估，持续运行，每个护栏和学习规则都映射到评估用例。

## 概念

### 三个评估层

1. **静态基准** —— SWE-bench Verified 用于代码（第 19 课），WebArena/OSWorld 用于浏览/桌面（第 20 课），GAIA 用于通才（第 19 课），BFCL V4 用于工具使用（第 06 课）。用于跨模型比较和回归控制。污染是真实的：SWE-bench+ 发现 32.67% 的解决方案泄漏。始终报告 Verified / +-审计分数。

2. **自定义离线评估** —— 你的产品形态：
   - LLM-as-judge（Langfuse、Phoenix、Opik —— 第 24 课）。
   - 基于执行（运行补丁，检查测试）。
   - 基于轨迹（将动作序列与黄金标准比较；OSWorld-Human 显示顶级 agent 超过黄金标准 1.4-2.7 倍）。

3. **在线评估** —— 生产：
   - 会话回放（Langfuse）。
   - 护栏触发警报（第 16、21 课）。
   - 每步成本/延迟跟踪（第 23 课 OTel span）。

### 评估器-优化器（Anthropic）

紧密循环：

1. 提议者生成输出。
2. 评估器评判。
3. 优化直到评估器通过。

这是 Self-Refine（第 05 课）的泛化。你关心的任何 agent 流程都可以包装在评估器-优化器中以获得可靠性。

### 2026 年最佳实践

- 评估与代码共存。
- 在每个 PR 上在 CI 中运行。
- 根据评估分数控制合并（例如"与主分支相比无 > 5% 的回归"）。
- 每个护栏映射到评估用例。
- 每个学习规则（Reflexion、专业工作流学习规则）映射到失败用例。

### 评估驱动开发失败的地方

- **无基线。** 没有最后已知良好的评估是不可读的。存储基线。
- **LLM-judge 无依据。** Judge 也会幻觉。CRITIC 模式（第 05 课）—— judge 基于外部工具。
- **过拟合到评估。** 为评估优化偏离生产实用性。轮换用例。
- **不稳定的评估。** 非确定性用例导致误报。固定种子，快照状态。

## 构建

`code/main.py` 是标准库评估工具：

- 带有类别（基准、自定义、在线）的用例注册表。
- 被测脚本化 agent。
- 评估器-优化器循环：提议、评判、优化直到通过或最大轮次。
- CI 门控：聚合通过率 + 与基线的回归。

运行：

```
python3 code/main.py
```

输出：每用例通过/失败、回归标志、CI 门控裁决。

## 使用

- 在与你的 agent 代码相同的仓库中编写评估用例。
- 通过 CI 在每个 PR 上运行它们。
- 回归时构建失败。
- 跟踪随时间的通过率。
- 将每个生产失败连接到新用例。

## 交付

`outputs/skill-eval-suite.md` 为带有 CI 门控和回归跟踪的 agent 产品构建三层评估套件。

## 练习

1. 取一个你的生产失败。编写一个重现它的评估用例。你的 agent 现在通过了吗？
2. 为你的领域构建一个具有三个维度（事实、语气、范围）的 LLM-judge 评分标准。评分 50 个会话。
3. 将评估套件接入 CI。>=5% 回归时构建失败。
4. 添加轨迹效率指标：agent 花费了多少步 vs 黄金轨迹？
5. 将第 14 阶段的每节课映射到你套件中的评估用例。有缺失吗？那是要弥补的差距。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| Static benchmark | "现成评估" | SWE-bench、GAIA、AgentBench、WebArena、OSWorld |
| Custom offline eval | "领域评估" | 针对你的产品形态的 LLM-as-judge / 执行 / 轨迹 |
| Online eval | "生产评估" | 会话回放、护栏警报、成本/延迟跟踪 |
| Evaluator-optimizer | "提议-评判-优化" | 迭代直到 judge 通过 |
| CI gate | "合并阻止器" | 评估回归时构建失败 |
| Baseline | "最后已知良好" | 检测回归的参考分数 |
| Trajectory efficiency | "超过黄金的步数" | Agent 步数除以人类专家最小值 |
