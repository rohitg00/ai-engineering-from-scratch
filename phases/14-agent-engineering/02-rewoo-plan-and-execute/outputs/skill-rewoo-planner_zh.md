给定用户请求和工具目录（名称、输入模式、描述），生成 ReWOO 计划：包含工具调用和证据引用（`#E1`、`#E2`、...）的步骤 DAG。在交给执行器之前验证计划。

生成：

1. 计划 DAG。每个节点有 id（`E1`、`E2`、...）、工具名称、参数字典（字符串可能包含 `#E<k>` 引用）和可选 `parallel_group` 标签。
2. 验证输出。通过拓扑排序进行无环性检查；引用解析检查（每个 `#E<k>` 有前面的生产者）；工具存在检查（每个工具名称在目录中）；参数模式检查（每个参数匹配工具的输入模式）。
3. 并行提示。对于每个拓扑级别，列出可并发执行的节点。
4. 规划器/求解器拆分建议。如果计划少于 3 步，推荐 ReAct。如果计划有无限循环需求（每步重新规划），推荐带重新规划器的 Plan-and-Execute。如果计划超过 30 步或目标为 web/移动，推荐带合成计划数据的 Plan-and-Act。

硬性拒绝：
- 有环计划。ReWOO 假设 DAG；环是 ReAct 或 LATS 的问题。
- 引用拓扑顺序中尚不存在的 `#E<k>` 的计划。发出失败的特定边。
- 调用目录中不存在的工具的计划。不要发明工具来使计划工作。
- 引用参数类型不匹配工具模式的计划（例如，`#E1` 替换字符串但工具期望 int）。

拒绝规则：
- 如果任务是开放式探索（需要未知工具、未知步骤），拒绝并推荐 ReAct 或 LATS（Lesson 04）。
- 如果工具目录包含无门控批准工具的破坏性工具，拒绝并指向 Lesson 09（权限、沙箱）。

输出：结构化计划（JSON 或 YAML）、验证报告、并行映射和指向执行器（ReWOO Worker）、重新规划器（Plan-and-Execute）或更大轨迹采样循环（Plan-and-Act）的后续操作。

以"接下来阅读什么"注释结尾，如果任务类别之前尝试过指向 Lesson 03（Reflexion），或如果计划将受益于搜索指向 Lesson 04（LATS）。
