# 思维树与 LATS：审慎搜索

> 单一的思维链轨迹没有回溯空间。ToT（Yao 等人，2023）将推理变成一棵带自评估的树。LATS（Zhou 等人，2024）在蒙特卡洛树搜索下统一了 ToT、ReAct 和 Reflexion。24 点游戏从 4%（CoT）提升到 74%（ToT）；LATS 在 HumanEval 上达到 92.7% pass@1。

**类型：** Build
**语言：** Python（stdlib）
**前置知识：** Phase 14 · 01（智能体循环），Phase 14 · 03（Reflexion）
**时间：** ~75 分钟

## 学习目标

- 将推理框架为搜索：节点是"思考"，边是"扩展"，值是"多 promising"。
- 实现一个 stdlib ToT 风格 BFS 树搜索，带自评估评分。
- 扩展到一个玩具 LATS MCTS 循环，包含选择 / 扩展 / 模拟 / 反向传播。
- 决定何时搜索值得令牌乘数（24 点游戏、代码生成）以及何时单轨迹足够（简单问答）。

## 问题所在

思维链是线性行走。如果第一步错了，每一步后续步骤都在错误前提下工作。在 24 点游戏（用四个数字和 + − × ÷ 组成 24）上，GPT-4 CoT 达到 4% 准确率。模型早期选择了错误的子表达式，无法恢复。

推理需要的是提出多个候选、评估它们、选择有前景的、并在死胡同出现时回溯的能力。这就是搜索。思维树和 LATS 是两种规范形式。

## 核心概念

### 思维树（Yao 等人，NeurIPS 2023）

每个节点是一个连贯的中间步骤（"一个思考"）。每个节点可以扩展为 K 个子思考。LLM 用评分提示自评估每个节点。搜索探索树 —— BFS、DFS 或束搜索。

```
                     (根: "从 4 6 4 1 找出 24")
                    /               |            \
           ("6 - 4 = 2")    ("4 + 1 = 5")    ("4 * 6 = 24")  <- 评分: 高
              /   \              |                  |
          ...    ...          ...                完成
```

自评估是承重部分。论文展示了三种变体：`sure / likely / impossible` 分类、`1..10` 数字评分、候选间投票。三种都在 24 点游戏上大幅击败 CoT（GPT-4 从 4% -> 74%）。

### LATS（Zhou 等人，ICML 2024）

LATS 在 MCTS 下统一了 ToT、ReAct 和 Reflexion。LLM 扮演三个角色：

- **策略**：提出候选下一步行动（ReAct 风格）。
- **价值函数**：评分部分轨迹（ToT 风格自评估）。
- **自反思器**：失败时，写下自然语言反思（Reflexion 风格）并用它重新播种未来 rollout。

环境反馈（观察）混合到价值函数中，因此搜索由真实工具结果通知，而不仅仅是模型意见。论文时的结果：HumanEval pass@1 92.7% 用 GPT-4（SOTA），WebShop 平均 75.9 用 GPT-3.5（接近基于梯度的微调）。

### MCTS，最小化

每次迭代四个阶段：

1. **选择** —— 使用 UCT（树的上置信界）从根走到叶。
2. **扩展** —— 通过策略生成 K 个子节点。
3. **模拟** —— 使用策略从子节点 rollout，用价值函数（或环境奖励）评分叶节点。
4. **反向传播** —— 更新路径上的访问计数和价值估计。

UCT 公式：`Q(s, a) + c * sqrt(ln N(s) / N(s, a))`。第一项是利用；第二项是探索。按任务调整 `c`。

### 成本现实

搜索爆炸令牌。ToT 在 24 点游戏上使用 CoT 的 100-1000 倍令牌。LATS 类似。这不是免费的；为以下情况保留搜索：

- 单轨迹明显不足的任务（24 点游戏、复杂代码）。
- 挂钟时间不如正确性重要的任务。
- 有廉价、可靠价值函数的任务（代码单元测试、数学显式目标）。

如果你的任务有单一正确答案和嘈杂评估器，搜索经常使事情更糟 —— 它找到一个"评分好"的错误答案。

### 2026 年定位

大多数生产智能体不运行 LATS。它们运行带工具基础验证的 ReAct（CRITIC，第 05 课）。搜索出现在专业细分领域：

- 将测试作为价值函数的编码智能体（HumanEval 风格）。
- 探索多个查询路径的深度研究智能体。
- LangGraph 子图内部的规划密集型工作流。

AlphaEvolve（第 11 课）是 2025 年的极端：代码的进化搜索，机器可检查的适应度，前沿增益（56 年来首次 4x4 matmul 改进）。

## 构建它

`code/main.py` 实现：

- 一个风格化"选择算术运算"任务上的微型 ToT BFS。
- 同一任务上的玩具 LATS MCTS 循环（选择 / 扩展 / 模拟 / 反向传播），带 UCT 选择。
- 一个组合符号评分加自评估评分的价值函数。

运行它：

```
python3 code/main.py
```

追踪显示 ToT 每节点扩展三个候选用 BFS，与 LATS 通过 MCTS 收敛到最佳 rollout 比较。打印两者的令牌计数。

## 使用它

LangGraph 将 ToT 风格探索作为子图模式提供；LangChain 团队关于 LATS 的博客（2024 年 5 月）是参考教程。LlamaIndex 提供一个 `TreeOfThoughts` 智能体。对于大多数 2026 年生产智能体，此模式位于 `if task_complexity > threshold: use_search()` 门后 —— 参见第 05 课的评估器-优化器模式。

## 交付它

`outputs/skill-search-policy.md` 给定任务形状、预算和评估器保真度，在线性 ReAct、ToT、LATS 和进化搜索之间选择。

## 练习

1. 用 UCT c=0.1 vs c=2.0 运行玩具 LATS。追踪中有什么变化？
2. 将价值函数换成更嘈杂的评分器（添加随机抖动）。MCTS 仍然找到最佳叶节点？它容忍的最小信噪比是多少？
3. 实现束搜索 ToT（每层保留 top-k）并与 BFS 比较。在紧张令牌预算上哪个更好？
4. 阅读 LATS 第 5.1 节。复现 HumanEval 轨迹计数：需要多少 rollout 才能达到报告的 pass@1？
5. 阅读 LATS 论文关于"LATS 帮助较少时"的讨论。写一段决策规则，将任务形状映射到搜索策略。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|----------------|------------------------|
| 思维树 | "分支 CoT" | Yao 等人 —— 带自评估的思维节点树 |
| LATS | "LLM 的 MCTS" | Zhou 等人 —— 在 MCTS 下统一 ToT + ReAct + Reflexion |
| UCT | "上置信界" | 选择公式平衡利用（Q）和探索（ln N / n） |
| 价值函数 | "这个状态有多好" | 提示的 LLM 分数或环境奖励；反馈反向传播 |
| 策略 | "行动提议者" | ReAct 风格生成器；发出候选下一步思考/行动 |
| Rollout | "模拟轨迹" | 使用策略从节点走到叶，用价值评分 |
| 反向传播 | "更新祖先" | 将叶节点的奖励推上路径，更新访问计数和 Q |
| 搜索成本 | "令牌爆炸" | 24 点游戏上是 CoT 的 100-1000 倍；采用前预算 |
