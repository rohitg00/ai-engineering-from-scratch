给定任务类别（重复的 agent 运行类型 — 例如"重构函数"、"关闭支持工单"），维护反射的情景记忆缓冲区。每个反射记录自然语言中的失败模式和纠正洞察。缓冲区被前置到相同任务类别的下一次试验。

生成：

1. 反射捕获。试验以评估器分数低于阈值结束时，发出一行形状的反射"我未能做 X 因为 Y；下次，Z。"丢弃外部失败（网络、上游 500）上的反射，除非它们可复现。
2. TTL 和去重。反射默认在 N 次试验后过期（建议 10）。精确重复折叠。近似重复（小型嵌入模型上 >0.9 余弦或共享子串 >= 80%）仅保留最近的。
3. 范围策略。三个范围：task-class（每任务名称）、user（同用户跨任务）、agent（跨所有用户）。默认是 task-class。仅当反射引用用户特定偏好时才升级到 user 范围；绝不自动升级到 agent 范围。
4. 压缩。当缓冲区超过预算时，运行睡眠时压缩：聚类近似重复、摘要、合并。压缩在热路径外运行 — 不要延迟主 agent 响应。
5. 提示词集成。发出标题为"从先前试验中学到的"的单个块及项目符号列表。提示词中限制 6 项；溢出到单独摘要项（"... 和关于超时的 4 个较旧反射"）。

硬性拒绝：
- 将反射写为"下次更小心"。这不可操作。使用强制具体下次指令的提示词重新运行反射器。
- 基于挂钟时间而非试验计数使反射过期。TTL 应是试验范围的，而非时间范围的，用于离线可重放运行。
- 存储引用秘密（API 密钥、令牌、PII）的反射。在提交到缓冲区前以特定"包含秘密"类错误拒绝。

拒绝规则：
- 如果未附加评估器，拒绝并推荐 Lesson 05（Self-Refine/CRITIC）— 反射需要信号，而非直觉。
- 如果任务类别是单次（从不重复），拒绝；情景记忆对从不重复的任务无用。

输出：结构化缓冲区文件（JSON 包含反射对象：trial id、task class、scope、text、created_at、ttl_remaining）、下一次试验的提示词块和列出即将过期条目的"陈旧反射"报告。

以"接下来阅读什么"注释结尾，如果缓冲区持续达到上限指向 Lesson 06（上下文压缩），或指向 Lesson 08（Letta 睡眠时计算）将压缩移出热路径。
