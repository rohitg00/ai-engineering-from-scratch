# 编排模式：监督者、群体、层次结构

> 四种编排模式在 2026 年框架中反复出现：监督者-工作者、群体/点对点、层次结构、辩论。Anthropic 的指导："关键在于为你的需求构建合适的系统。"从简单开始；只有当单个 agent 加五种工作流模式不足时，才添加拓扑。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置条件：** 第 14 阶段 · 12（工作流模式），第 14 阶段 · 25（多 Agent 辩论）
**时间：** ~60 分钟

## 学习目标

- 说出四种反复出现的编排模式以及各自的适用场景。
- 描述 2026 年 LangChain 推荐：基于工具调用的监督 vs 监督者库。
- 解释 Anthropic 的"构建合适系统"规则以及它如何控制拓扑选择。
- 针对共同的脚本化 LLM 在标准库中实现所有四种模式。

## 问题

团队在需要之前就选择"多 agent"。四种模式在框架中反复出现；一旦你能命名它们，你就能选择正确的 —— 或完全跳过拓扑。

## 概念

### 监督者-工作者

- 中央路由 LLM 分派给专家 agent。
- 决定：循环回自身、交接给专家、终止。
- 专家之间不相互交谈；所有路由通过监督者。

框架：LangGraph `create_supervisor`、Anthropic 编排器-工作者、CrewAI 层次流程。

**2026 年 LangChain 推荐：** 通过直接工具调用进行监督，而不是 `create_supervisor`。提供更精细的上下文工程控制 —— 你精确决定每个专家看到什么。

### 群体 / 点对点

- Agent 通过共享工具表面直接交接。
- 无中央路由器。
- 比监督者更低的延迟（更少的跳数）。
- 更难推理（无单一控制点）。

框架：LangGraph 群体拓扑、OpenAI Agents SDK 交接（当所有 agent 可以相互交接时）。

### 层次结构

- 监督者管理子监督者管理工作者。
- 在 LangGraph 中实现为嵌套子图；CrewAI 中的嵌套 crew。
- 以操作复杂性为代价扩展到大型 agent 群体。

何时需要：当单个监督者的上下文预算无法容纳所有专家的描述时。

### 辩论

- 并行提议者 + 迭代交叉批评（第 25 课）。
- 不完全是编排 —— 更多是验证 —— 但在框架中作为拓扑选择出现。

### CrewAI Crew vs Flow

CrewAI 将两种部署模式形式化：

- **Flow** 用于确定性事件驱动自动化（推荐的生产起点）。
- **Crew** 用于自主基于角色的协作。

这与上面的四种模式正交，但映射到拓扑：Flow 通常是监督者或层次结构；Crew 通常是带有 LLM 路由器的监督者。

### Anthropic 的指导

"LLM 领域的成功不在于构建最复杂的系统。关键在于为你的需求构建合适的系统。"

决策顺序：

1. 单个 agent + 工作流模式（第 12 课）—— 从这里开始。
2. 监督者-工作者 —— 当你有 2-4 个专家时。
3. 群体 —— 当延迟比推理清晰度更重要时。
4. 层次结构 —— 仅当监督者上下文预算失败时。
5. 辩论 —— 当准确率比成本更重要时。

### 此模式出错的地方

- **拓扑优先思维。** 在识别多 agent 解决什么问题之前"我们需要多 agent"。
- **群体中的弹跳交接。** A -> B -> A -> B。使用跳数计数器。
- **虚假层次结构。** 三层因为"企业"；两个实际团队。折叠。

## 构建

`code/main.py` 针对脚本化 LLM 在标准库中实现所有四种模式：

- `Supervisor` —— 中央路由器。
- `Swarm` —— 带有直接交接的点对点。
- `Hierarchical` —— 监督者的监督者。
- `Debate` —— 并行提议者 + 批评。

每种模式处理相同的三意图任务（退款 / bug / 销售）。跟踪形状不同。

运行：

```
python3 code/main.py
```

输出：每种模式的跟踪 + 操作计数。监督者最干净；群体最短；层次结构最深；辩论最昂贵。

## 使用

- **LangGraph** 用于监督者和层次结构（嵌套子图）。
- **OpenAI Agents SDK** 用于作为工具的交接（监督者形状）。
- **CrewAI Flow** 用于生产确定性。
- **自定义** 用于辩论或当你想要精确控制时。

## 练习

1. 通过移除路由器将监督者-工作者转换为群体。什么坏了？什么改善了？

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| Supervisor-worker | "路由器 + 专家" | 中央 LLM 分派给专家；他们不相互交谈 |
| Swarm | "点对点" | 通过共享工具直接交接；无中央路由器 |
| Hierarchical | "监督者的监督者" | 用于大型群体的嵌套子图 |
| Debate | "提议者 + 批评" | 并行提议者，交叉批评（第 25 课） |
| Tool-call-based supervision | "无库监督者" | 将监督者实现为直接工具调用以进行上下文控制 |
| Crew | "自主团队" | CrewAI 的基于角色的协作模式 |
| Flow | "确定性工作流" | CrewAI 的事件驱动生产模式 |
