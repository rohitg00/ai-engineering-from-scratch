# 失败模式：为什么 Agent 会崩溃

> MASFT（Berkeley，2025）在 3 个类别中编目了 14 种多 agent 失败模式。Microsoft 的分类法记录了现有 AI 失败如何在 agent 设置中放大。行业现场数据收敛于五种反复出现的模式：幻觉动作、范围蔓延、级联错误、上下文丢失、工具误用。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置条件：** 第 14 阶段 · 05（Self-Refine 和 CRITIC），第 14 阶段 · 24（可观察性）
**时间：** ~60 分钟

## 学习目标

- 说出 MASFT 的三个失败类别和每个类别中至少四种具体模式。
- 解释为什么 agent 失败放大现有 AI 失败模式（偏见、幻觉）。
- 描述五种行业反复出现的模式及其缓解措施。
- 实现一个标准库检测器，用失败模式标签标记 agent 跟踪。

## 问题

团队发布的 agent 在 90% 的跟踪上工作。10% 的失败不是随机噪声 —— 它们落入少数反复出现的类别。一旦你能命名它们，你就能监控它们并修复它们。

## 概念

### MASFT（Berkeley，arXiv:2503.13657）

多 Agent 系统失败分类法。14 种失败模式聚类为 3 个类别。注释者间 Cohen's Kappa 0.88 —— 类别可靠可区分。

核心主张：失败是多 agent 系统中的基本设计缺陷，不是可以通过更好的基础模型修复的 LLM 限制。

### Microsoft Agentic AI 系统失败模式分类法

- 现有 AI 失败（偏见、幻觉、数据泄漏）在 agent 设置中放大。
- 自主性产生新失败：大规模意外动作、工具误用、任务漂移。
- 白皮书是 agent 产品的风险登记册。

### Agentic AI 中的故障特征（arXiv:2603.06847）

- 失败源于编排、内部状态演化和环境交互。
- 不只是"坏代码"或"坏模型输出"。

### LLM Agent 幻觉调查（arXiv:2509.18970）

两种主要表现：

1. **指令遵循偏差** —— agent 不遵循系统提示词。
2. **长程上下文误用** —— agent 忘记或误用早期轮次的上下文。

子意图错误：遗漏（错过步骤）、冗余（重复步骤）、混乱（顺序错误步骤）。

### 五种行业反复出现的模式

Arize、Galileo、NimbleBrain 2024-2026 年现场分析收敛于：

1. **幻觉动作。** Agent 调用不存在的工具或编造参数。
2. **范围蔓延。** Agent 将任务扩展到用户请求之外（创建额外 PR、发送额外邮件）。
3. **级联错误。** 一个错误调用触发下游效应。一个幻影 SKU 幻觉触发四个 API 调用 —— 多系统事件。
4. **上下文丢失。** 长程任务忘记早期轮次约束。
5. **工具误用。** 用错误参数调用正确工具，或完全调用错误工具。

级联是杀手。Agent 无法区分"我失败了"和"任务不可能"，经常在 400 错误上幻觉成功消息以关闭循环。

### 缓解：每一步的门控

推理链每一步的自动验证门控，针对环境状态检查事实依据。具体：

- 每步安全分类器（第 21 课）。
- 工具调用参数验证（第 06 课）。
- 交叉检查检索到的内容与已知事实（第 05 课，CRITIC）。
- 通过重新探测状态检测成功幻觉（文件实际创建了吗？）。

### 失败监控出错的地方

- **只标记崩溃。** 大多数 agent 失败产生看起来有效的输出。需要内容级检查。
- **无基线。** 漂移检测需要最后已知良好；没有它你无法说"这正在变糟"。
- **过度告警。** 每个失败产生页面。聚类和速率限制。

## 练习

1. 添加"成功幻觉"检测器：agent 返回成功但目标状态未改变。
2. 标记你构建的产品的 100 个真实跟踪。哪种模式主导？修复它的成本是多少？
3. 实现"级联半径"指标：给定步骤 N 的失败，它影响了多少下游步骤？
4. 阅读 MASFT 的 14 种失败模式。选择三个适用于你产品的。编写检测器。
5. 将一个检测器接入 CI 作业：如果 >=5% 的跟踪标记某种模式，则构建失败。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| MASFT | "多 agent 失败分类法" | Berkeley 14 模式分类 |
| Cascading error | "涟漪失败" | 一个早期错误通过 N 步传播 |
| Context loss | "忘记约束" | 长程轮次丢失早期轮次事实 |
| Tool misuse | "错误工具 / 错误参数" | 有效调用，错误调用 |
| Success hallucination | "伪造完成" | Agent 在 400 上声称成功；状态未改变 |
| Scope creep | "过度延伸" | Agent 做得比要求的多 |
| Instruction-following deviation | "不服从" | 忽略系统提示词或用户约束 |
| Sub-intention errors | "计划错误" | 计划执行中的遗漏、冗余、混乱 |
