# 失败模式：为什么智能体会崩溃

> MASFT（伯克利，2025）在 3 个类别中编目了 14 种多智能体失败模式。Microsoft 的分类法记录了现有 AI 失败如何在智能体设置中放大。行业现场数据收敛于五个反复出现的模式：幻觉操作、范围蔓延、级联错误、上下文丢失、工具误用。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置条件：** 阶段 14 · 05（Self-Refine 和 CRITIC），阶段 14 · 24（可观测性）
**时间：** ~60 分钟

## 学习目标

- 命名 MASFT 的三个失败类别以及每个类别中至少四种特定模式。
- 解释为什么智能体失败会放大现有 AI 失败模式（偏见、幻觉）。
- 描述五个行业反复出现的模式及其缓解措施。
- 实现用失败模式标签标记智能体跟踪的标准库检测器。

## 问题

团队发布在 90% 的跟踪上工作的智能体。10% 的失败不是随机噪声——它们属于少数反复出现的类别。一旦您可以命名它们，您就可以监控它们并修复它们。

## 概念

### MASFT（伯克利，arXiv:2503.13657）

多智能体系统失败分类法。14 种失败模式聚集成 3 个类别。标注间 Cohen's Kappa 0.88——类别可可靠区分。

核心主张：失败是多智能体系统中的基本设计缺陷，不是要用更好的基础模型修复的 LLM 限制。

### Microsoft 智能体 AI 系统失败模式分类法

- 现有 AI 失败（偏见、幻觉、数据泄漏）在智能体设置中放大。
- 新的失败从自主性中出现：大规模意外操作、工具误用、任务漂移。
- 白皮书是智能体产品的风险寄存器。

### 表征智能体 AI 中的故障（arXiv:2603.06847）

- 失败来自编排、内部状态演变和环境交互。
- 不仅仅是"坏代码"或"坏模型输出"。

### LLM 智能体幻觉调查（arXiv:2509.18970）

两种主要表现：

1. **指令遵循偏离** — 智能体不遵循系统提示。
2. **长距离上下文误用** — 智能体忘记或误用早期轮次的上下文。

子意图错误：遗漏（错过步骤）、冗余（重复步骤）、无序（乱序步骤）。

### 五个行业反复出现的模式

Arize、Galileo、NimbleBrain 2024-2026 年现场分析收敛于：

1. **幻觉操作。** 智能体调用不存在的工具或捏造参数。
2. **范围蔓延。** 智能体将任务扩展到用户要求之外（创建额外的 PR、发送额外的邮件）。
3. **级联错误。** 一个错误的调用触发下游影响。幻影 SKU 幻觉触发四次 API 调用——多系统事件。
4. **上下文丢失。** 长周期任务忘记早期轮次约束。
5. **工具误用。** 用错误参数调用正确的工具，或完全调用错误的工具。

级联是杀手。智能体无法区分"我失败了"和"任务不可能"，并且经常在 400 错误上幻觉成功消息以闭合循环。

### 缓解：每一步的门控

推理链每一步的自动验证门控，根据环境状态检查事实依据。具体地：

- 每步安全分类器（课程 21）。
- 工具调用参数验证（课程 06）。
- 根据已知事实交叉检查检索的内容（课程 05，CRITIC）。
- 通过重新探测状态检测成功幻觉（文件实际创建了吗？）。

### 失败监控出错的地方

- **仅标记崩溃。** 大多数智能体失败产生看起来有效的输出。需要内容级检查。
- **无基线。** 漂移检测需要最后已知良好；没有它您无法说"这变得更糟"。
- **过度警报。** 每个失败产生一个页面。聚类和速率限制。

## 练习

1. 为"成功幻觉"添加检测器：智能体返回成功但目标状态未更改。
2. 从您构建的产品中标记 100 个真实跟踪。哪种模式占主导？修复它的成本是多少？
3. 实现"级联半径"指标：给定步骤 N 的失败，它影响了多少下游步骤？
4. 阅读 MASFT 的 14 种失败模式。选择适用于您产品的三种。编写检测器。
5. 将一个检测器连接到 CI 作业：如果 >=5% 的跟踪标记模式，则使构建失败。

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|------|----------------|------------------------|
| MASFT | "多智能体失败分类法" | 伯克利 14 模式分类 |
| 级联错误 | "涟漪失败" | 一个早期错误通过 N 个步骤传播 |
| 上下文丢失 | "忘记约束" | 长周期轮次丢弃早期轮次事实 |
| 工具误用 | "错误工具/错误参数" | 有效调用，错误调用 |
| 成功幻觉 | "伪造完成" | 智能体在 400 上声称成功；状态未更改 |
| 范围蔓延 | "过度延伸" | 智能体做得比要求的多 |
| 指令遵循偏离 | "不服从" | 忽略系统提示或用户约束 |
| 子意图错误 | "计划错误" | 计划执行中的遗漏、冗余、无序 |
