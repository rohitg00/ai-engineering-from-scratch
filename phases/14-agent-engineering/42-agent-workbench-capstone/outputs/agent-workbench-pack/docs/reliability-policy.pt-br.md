# Política de Confiabilidade

O ambiente de trabalho absorve os cinco modos de falha recorrentes no setor:

1. Ação alucinada – capturada pelo conjunto de regras + portão de verificação.
2. Aumento do escopo – detectado pela verificação de diferenças do contrato de escopo.
3. Erros em cascata – detectados por registros de feedback + recusa na saída nula.
4. Perda de contexto – absorvida pela memória repo; chat não é a fonte da verdade.
5. Uso indevido de ferramentas — detectado pela dimensão de verificação da rubrica do revisor.

A política é aplicada pelo portão de verificação. O caminho de substituição está assinado
e auditado; os agentes não podem se autosubstituir.
