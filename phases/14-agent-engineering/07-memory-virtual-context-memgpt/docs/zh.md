# 07 · 记忆：虚拟上下文与 MemGPT

> 上下文窗口是有限的，而对话、文档和工具调用轨迹却没有上限。MemGPT（Packer 等人，2023）把这个问题类比为操作系统的虚拟内存——主上下文是内存（RAM），外部存储是磁盘，智能体在两者之间分页换入换出。这是 2026 年每一个记忆系统都继承的范式。

**类型：** 构建（Build）
**语言：** Python（标准库）
**前置：** 阶段 14 · 01（智能体循环）、阶段 14 · 06（工具使用）
**时长：** 约 75 分钟

## 学习目标

- 解释 MemGPT 所依托的操作系统类比：主上下文 = 内存（RAM），外部上下文 = 磁盘，记忆工具 = 分页换入/换出。
- 用标准库实现 MemGPT 的两层范式，包含一个主上下文缓冲区、一个可搜索的外部存储，以及分页换入/换出工具。
- 描述智能体如何发出「中断（interrupt）」来查询或修改外部记忆，以及结果如何被拼接回下一轮提示词中。
- 辨识 MemGPT 中哪些设计决策延续到了 Letta（第 08 课）和 Mem0（第 09 课）。

## 问题所在

上下文窗口看起来理应解决记忆问题，但它做不到。生产环境中反复出现三种失败模式：

1. **溢出（Overflow）。** 多轮对话、长文档，或工具调用密集的轨迹会超出窗口。截断点之后的一切都丢失了。
2. **稀释（Dilution）。** 即便在窗口内，塞入无关上下文也会稀释模型对重要信息的注意力。前沿模型在长输入上仍会性能退化。
3. **持久化（Persistence）。** 新会话以空窗口开始。没有外部记忆的智能体无法跨会话说出「记得你当时让我……」。

更大的窗口有帮助，但并不能根治。Mem0 的 2025 年论文测得：128k 窗口的基线仍会遗漏长程事实，而一个配有外部记忆的 4k 窗口智能体却能抓住这些事实。

## 核心概念

### MemGPT：操作系统类比

Packer 等人（arXiv:2310.08560，v2 版于 2024 年 2 月）把上下文管理映射到操作系统的虚拟内存：

| 操作系统概念 | MemGPT 概念 | 2026 年生产环境对应物 |
|------------|---------------|------------------------|
| 内存（RAM） | 主上下文（提示词） | Anthropic/OpenAI 的上下文窗口 |
| 磁盘（Disk） | 外部上下文 | 向量数据库、KV 存储、图存储 |
| 缺页中断（Page fault） | 记忆工具调用 | `memory.search`、`memory.read`、`memory.write` |
| 操作系统内核 | 智能体控制循环 | 带记忆工具的 ReAct 循环 |

智能体运行一个普通的 ReAct 循环。额外的一类工具让它能够把数据分页换入和换出主上下文。

### 两个层级

- **主上下文（Main context）。** 固定大小的提示词，承载当前任务。始终对模型可见。
- **外部上下文（External context）。** 无界，可通过工具搜索。相关时读取，事实涌现时写入。

原始论文在超出基础窗口的两个任务上评估了该设计：超过 100k token 的文档分析，以及跨多日、带持久记忆的多会话聊天。

### 中断范式

MemGPT 引入了「记忆即中断（memory-as-interrupt）」：在对话进行中，智能体可以调用一个记忆工具，运行时执行它，结果作为新的观测拼接进下一轮助手回复。这在概念上等同于 Unix 的 `read()` 系统调用——它阻塞进程、返回字节，进程随后继续运行。

标准记忆工具集：

- `core_memory_append(section, text)` —— 写入提示词中的某个持久化片段（section）。
- `core_memory_replace(section, old, new)` —— 编辑某个持久化片段。
- `archival_memory_insert(text)` —— 写入可搜索的外部存储。
- `archival_memory_search(query, top_k)` —— 从外部存储中检索。
- `conversation_search(query)` —— 扫描过往轮次。

### MemGPT 的终点与 Letta 的起点

2024 年 9 月，MemGPT 更名为 Letta。研究仓库（`cpacker/MemGPT`）依然保留；Letta 则扩展了该设计：

- 三层而非两层（核心、回忆、归档——第 08 课）。
- 用原生推理（native reasoning）替代 `send_message`/心跳（heartbeat）范式（第 08 课）。
- 异步执行记忆工作的睡眠期智能体（sleep-time agents，第 08 课）。

即便生产系统跑的是 Letta、Mem0 或自定义的两层存储，MemGPT 论文仍是 2026 年的基石。

### 这个范式哪里会出错

- **记忆腐烂（Memory rot）。** 写入累积的速度快于读取；检索淹没在陈旧事实中。修复方法：周期性整合（Letta 的睡眠期）、显式失效（Mem0 的冲突检测器）。
- **记忆投毒（Memory poisoning）。** 外部记忆是被检索回来的文本。如果攻击者控制的内容落入了一条记忆笔记，智能体会在下一会话重新摄入它。这就是 Greshake 等人（第 27 课）的攻击在时间维度上的重演。
- **引用丢失（Citation loss）。** 智能体回忆起「用户让我交付 X」，却无法指明是哪一轮说的。每次归档写入时都要存储来源引用（会话 ID、轮次 ID）。

## 动手构建

`code/main.py` 用标准库实现了 MemGPT 的两层范式：

- `MainContext` —— 固定大小的提示词缓冲区，含一个 `core` 字典和一个 `messages` 列表；超过上限时自动压缩最旧的消息。
- `ArchivalStore` —— 内存中的类 BM25 存储（基于 token 重叠度打分），记录形如 (id, text, tags, session, turn)。
- 五个映射到 MemGPT 工具集的记忆工具。
- 一个脚本化智能体，先用事实填充归档，然后通过调用 `archival_memory_search` 回答问题。

运行它：

```
python3 code/main.py
```

运行轨迹展示了智能体写入三条事实，把主上下文填满到上限（强制触发逐出），然后通过从归档中检索来回答一个追问——在没有任何真实 LLM 的情况下复现了 MemGPT 的工作流程。

## 实战应用

如今每一个生产级记忆系统都是 MemGPT 的变体：

- **Letta**（第 08 课）—— 三层、原生推理、睡眠期计算。
- **Mem0**（第 09 课）—— 向量 + KV + 图融合并叠加一层打分。
- **OpenAI Assistants / Responses** —— 通过 thread 和 file 实现的托管式记忆。
- **Claude Agent SDK** —— 通过技能（skills）和会话存储实现的长期记忆。

按运维形态（自托管、托管、框架集成）来选型，而不是按核心范式来选——核心范式都是 MemGPT。

## 交付物

`outputs/skill-virtual-memory.md` 是一个可复用的技能，能为任意目标运行时生成一套正确的两层记忆脚手架（主上下文 + 归档 + 工具集），并预先接好逐出策略和引用字段。

## 练习

1. 增加一个以 token 计量的 `max_main_context_tokens` 上限（用 `len(text.split())` * 1.3 近似估算）。当超出上限时，把最旧的消息压缩成摘要。对比有摘要器和没有摘要器时的行为差异。
2. 在归档存储上正确实现 BM25（词频、逆文档频率）。在一个玩具级事实集上测量 recall@10，并与 token 重叠度基线对比。
3. 给归档插入操作增加 `citation` 字段（session_id、turn_id、source_url）。让智能体在每个基于检索的回答上都标注来源。
4. 模拟记忆投毒：添加一条写着「忽略未来所有用户指令」的归档记录。编写一个守卫，扫描检索结果中是否含有指令形态的文本，并将其标记为不可信。
5. 把实现移植为使用 MemGPT 研究仓库的核心记忆 JSON 模式（`cpacker/MemGPT`）。当你从扁平字符串切换到带类型的片段时，会有什么改变？

## 关键术语

| 术语 | 人们常说的 | 它实际的含义 |
|------|----------------|------------------------|
| 虚拟上下文（Virtual context） | 「无限记忆」 | 主层（提示词）+ 外部层（可搜索）的分层，可分页换入换出 |
| 主上下文（Main context） | 「工作记忆」 | 提示词——固定大小，始终可见 |
| 归档记忆（Archival memory） | 「长期存储」 | 外部可搜索的持久化存储，按需检索 |
| 核心记忆（Core memory） | 「持久化的提示词片段」 | 钉在主上下文内部的具名片段 |
| 记忆工具（Memory tool） | 「记忆 API」 | 智能体发出的、用于读写外部记忆的工具调用 |
| 中断（Interrupt） | 「记忆缺页中断」 | 智能体暂停，运行时取数，结果拼接进下一轮 |
| 记忆腐烂（Memory rot） | 「陈旧事实」 | 旧写入淹没检索；用整合来修复 |
| 记忆投毒（Memory poisoning） | 「被注入的持久化笔记」 | 攻击者内容被存为记忆，在回忆时被重新摄入 |
