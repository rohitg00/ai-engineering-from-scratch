# LLaVA-OneVision：单图、多图、视频统一模型

> 在 LLaVA-OneVision（Li 等人，2024 年 8 月）之前，开源 VLM 世界有各自独立的谱系：LLaVA-1.5 用于单图，Mantis 和 VILA 等多图模型，Video-LLaVA 和 Video-LLaMA 等视频模型。每个都在自己的基准上获胜，但在其他基准上失败。LLaVA-OneVision 认为单一课程可以训练一个模型来主导所有三种场景，并且涌现的任务迁移效应（单图技能迁移到视频，多图推理迁移到单图）超过了专家模型的总和。配方出奇地简单：一个在三种场景下保持恒定的视觉 token 预算，加上一个从单图到 OneVision（多图）再到视频的显式课程。本课解读了预算、课程和涌现行为。

**类型：** Build
**语言：** Python（stdlib，token 预算求解器 + 课程规划器）
**前置知识：** Phase 12 · 05（LLaVA），Phase 12 · 06（任意分辨率）
**时间：** ~180 分钟

## 学习目标

- 设计一个在单图、多图和视频输入之间保持恒定的视觉 token 预算。
- 安排一个训练课程，将技能从单图迁移到视频而不发生灾难性遗忘。
- 解释为什么在课程正确的情况下，单一模型在相同参数数量下优于专家模型。
- 命名 LLaVA-OneVision 报告的三个涌现能力：多摄像头推理、标记集提示、iPhone 截图代理。

## 问题所在

图像、多图和视频各自以不同方式考验模型。

单图需要高分辨率 token（AnyRes，约 2880 视觉 token）来捕捉 OCR 和精细细节。每样本预算：一张图像，2880 token。

多图需要几张中等分辨率的图像（每张约 576 token），以便跨图像推理能放入上下文。每样本预算：4-8 张图像，每张 576，共 2300-4600 token。

视频需要许多低分辨率的帧（池化后每帧约 196 token）来捕捉时间动态。每样本预算：8-32 帧，每帧 196，共 1600-6200 token。

如果你训练单独的模型，你选择一个预算。如果你训练一个模型，你需要预算在场景间合理扩展而不爆炸上下文。

OneVision 之前，默认答案是"训练一个场景，忽略其他"。Video-LLaVA 通过额外训练阶段将视频改造到图像模型上。LLaVA-NeXT 通过平铺添加多图支持。没有一个能干净地处理所有三种。

## 核心概念

### OneVision token 预算

LLaVA-OneVision 选择统一的每样本视觉 token 预算约 3000-4000 token，每种场景分配不同：

- 单图：AnyRes-9（3x3 瓦片 + 缩略图），每个瓦片在 384 分辨率下有 729 个 patch，激进的双线性池化 2x2 → 每瓦片 182。总计：9 * 182 + 182 = 1820 token。或 AnyRes-4 每瓦片 729 = 2916 + 729。
- 多图：每张图像中等分辨率（384，无平铺），无池化 729 token。预算 6 张图像 → 4374 token。
- 视频：32 帧在 384 分辨率下，激进 3x3 双线性池化 → 每帧 81 token。总计：32 * 81 = 2592 token。

分配保持大致恒定的总 token。LLM 永远不会看到上下文爆炸的 batch。编码器每种场景产生不同几何，但 LLM 消耗相同预算。

### 三阶段课程

LLaVA-OneVision 分三阶段训练：

1. 单图 SFT（SI 阶段）。所有数据是单图加文本。在高分辨率 AnyRes 输入上训练。这教授感知、OCR 和细粒度理解。使用 LLaVA-NeXT 数据加 OneVision 特定单图数据。
2. OneVision SFT（OV 阶段）。混合单图 + 多图 + 视频（均匀采样帧）。在统一 token 预算上训练。这教授模型处理异构 batch 形状。无权重重置——从 SI 阶段继续。
3. 任务迁移（TT 阶段）。继续目标任务混合，通常根据产品侧重多图或视频。可选部署微调。

关键：课程顺序很重要。先训练视频或多图会比先训练单图产生更差的图像性能，即使数据相同。论文明确消融了这一点。

### 为什么课程有效

单图训练建立感知基础。Patch token 携带细粒度视觉特征；LLM 学习将它们与文本整合。多图和视频引入结构性挑战（哪张图是哪张，先发生了什么），没有强大的感知基础很难学习。

如果你从一开始就一起训练所有场景，模型对感知欠拟合（每 batch 单图数据有限）并对结构过拟合（大量多图/视频数据）。结果：一个遵循跨图像推理模式但视觉浅薄的模型。

课程排序从 SI 阶段给你感知强度，然后从 OV 阶段给你组合/时间推理，而不丢失任何一方。

### 涌现跨场景技能

LLaVA-OneVision 论文报告了三个涌现能力：

1. 多摄像头推理。分别在多图和视频上训练；推理时，要求推理多摄像头驾驶场景。模型正确整合视图，尽管训练中从未见过该确切格式。
2. 标记集提示。用户在图像中用数字标记标注物体；模型推理"标记 3 相对于标记 7 在做什么"。训练中没有标记或标注；从空间定位 + 多图参考的组合中学习。
3. iPhone 截图代理。用户提供 iPhone 屏幕截图并要求规划下一次点击。在 UI 截图、用户工作流视频和多图前后对上训练。泛化到代理用例。

这些不是训练任务；它们从课程的组合结构中涌现。

### 视觉 token 池化

Token 预算需要池化。OneVision 在 2D patch 网格上使用双线性插值：24x24 = 576 patch 变成 12x12 = 144（2 倍因子）或 8x8 = 64（3 倍因子）。池化在 patch 网格空间进行，而非 token 空间，以保留局部性。

每种场景的池化因子选择本身就是一个超参数。更少池化 = 更多 token = 更丰富的表示。更多池化 = 更少 token = 容纳更多帧/图像。

### LLaVA-OneVision-1.5

2025 年的后续（LLaVA-OneVision-1.5，arXiv 2509.23661）在训练数据、模型权重和代码上"完全开源"。在某些基准上匹配专有差距并民主化配方。相同课程，更多数据，更好的基础 LLM。无架构变化。

### 与 Qwen2.5-VL 的对比

Qwen2.5-VL（第 12.09 课）做出不同选择。它使用 M-RoPE 和动态 FPS 而非固定池化。其预算随输入缩放——1 分钟视频比 5 秒视频使用更多 token。LLaVA-OneVision 固定预算并缩放池化。两者都有效；它们在可配置性与可预测性之间权衡。

## 使用它

`code/main.py` 是 OneVision 风格 VLM 的课程和预算规划器。给定每样本 token 预算和目标场景混合（比如 40% 单图、30% 多图、30% 视频），它：

- 分配每种场景的分辨率、池化因子和帧数。
- 检查每种场景是否适合共享预算。
- 报告预期 token 数量、LLM FLOPs 以及哪些场景 token 不足。
- 打印逐阶段训练计划。

用它来规划 OneVision 微调或验证 VLM 部署的每请求成本。

## 交付它

本课产出 `outputs/skill-onevision-budget-planner.md`。给定目标任务分布和每样本预算，它发出 AnyRes 因子、每帧池化、视频帧数和课程阶段权重。每当你训练或微调统一场景 VLM 时使用它。

## 练习

1. 你的产品支持 80% 单图、10% 多图（2-4 张）、10% 视频（8-16 帧）。设计 token 预算。你会把不用于重度多图的额外预算放在哪里？

2. 阅读 LLaVA-OneVision 第 4.3 节（涌现能力）。提出课程可能解锁但论文未报告的第四个涌现技能。

3. 交换课程顺序——先训练多图，然后单图，然后视频。预测哪些基准会下降以及为什么。

4. 论文报告视频基准仅在每样本 8 帧上训练。这泛化到推理时的 30 秒视频吗？什么先崩溃——token 预算还是时间推理？

5. 24x24 patch 双线性池化到 12x12 是每维 4 倍缩减。在 stdlib Python 中实现池化，并验证每个 2x2 块的均值与双线性输出匹配。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------------|------------------------|
| OneVision 场景 | "单图、多图或视频" | 统一 VLM 处理的三种输入形状之一；预算在它们之间保持恒定 |
| Token 预算 | "每样本多少 token" | LLM 每训练/推理样本看到的总视觉 token，通常 3000-4000 |
| 课程 | "训练顺序" | 阶段排序（单图 → 多图 → 视频），为涌现迁移而选择 |
| 双线性池化 | "Token 缩减" | 对 patch 网格（2D）应用双线性插值以减少 token 数量同时保留局部性 |
| 涌现技能 | "没训练过，仍然有效" | 推理时出现的能力，没有匹配的训练数据，由于课程组合而产生 |
| AnyRes-k | "k 瓦片设置" | k 个固定分辨率子瓦片加一个缩略图，典型 k ∈ {4, 9} |
| 任务迁移 | "跨场景泛化" | 通过共享骨干在单图上学习的技能应用到视频（反之亦然） |
