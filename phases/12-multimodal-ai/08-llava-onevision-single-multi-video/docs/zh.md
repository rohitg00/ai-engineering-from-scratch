# LLaVA-OneVision：单图、多图、视频于一体的模型

> 在LLaVA-OneVision（Li等人，2024年8月）之前，开源视觉语言模型（VLM）领域存在不同的分支：用于单图的LLaVA-1.5、用于多图的Mantis和VILA类模型、用于视频的Video-LLaVA和Video-LLaMA。每个模型都在自己的基准测试中获胜，但在其他任务中失败。LLaVA-OneVision认为，通过单一课程（Curriculum）可以训练一个模型在所有三种场景中占据主导地位，并且涌现出的任务迁移效果（单图能力迁移到视频，多图推理迁移到单图）超越了各个专家的总和。其方法出奇地简单：一个跨场景保持恒定的视觉令牌预算（Visual-Token Budget），加上一个从单图到OneVision（多图）再到视频的显式课程。本课程将解读预算、课程和涌现行为。

**类型：** 构建
**语言：** Python（标准库，令牌预算求解器 + 课程规划器）
**前置知识：** 阶段12 · 05（LLaVA），阶段12 · 06（任意分辨率）
**时间：** 约180分钟

## 学习目标

- 设计一个在单图、多图和视频输入中保持恒定的视觉令牌预算。
- 安排一个训练课程，使技能从单图迁移到视频而不发生灾难性遗忘。
- 解释为什么当课程设计正确时，单个模型能在相同参数数量下击败各个专家。
- 列举LLaVA-OneVision报告的三种涌现能力：多摄像头推理、标记集提示（Set-of-Mark Prompting）、iPhone截图代理（iPhone-Screenshot Agent）。

## 问题

图像、多图和视频对模型的压力各不相同。

单图需要高分辨率令牌（AnyRes，约2880个视觉令牌）以捕捉OCR和细节。每个样本的预算：一张图像，2880个令牌。

多图需要中等分辨率的多张图像（每张约576个令牌），以便跨图像推理适合上下文。每个样本的预算：4-8张图像，每张576个令牌，共2300-4600个令牌。

视频需要许多低分辨率帧（池化后每帧约196个令牌）以捕捉时间动态。每个样本的预算：8-32帧，每帧196个令牌，共1600-6200个令牌。

如果训练单独的模型，你会选择一个预算。如果训练一个模型，你需要预算在不同场景中合理缩放，而不超出上下文限制。

在OneVision之前，默认答案是“只训练一个场景，忽略其他”。Video-LLaVA通过额外的训练阶段将视频改造到图像模型上。LLaVA-NeXT通过分块（Tiling）增加了多图支持。但没有一个模型能干净地处理所有三种场景。

## 概念

### OneVision令牌预算

LLaVA-OneVision为每个样本选择约3000-4000个令牌的统一视觉令牌预算，根据场景不同分配不同：

- 单图：AnyRes-9（3x3分块+缩略图），每个分块384像素，729个补丁，激进的双线性池化（2x2）→每块182个令牌。总计：9 * 182 + 182 = 1820个令牌。或者AnyRes-4，每块729个令牌 = 2916 + 729。
- 多图：每张图像中等分辨率（384，无分块），729个令牌，无池化。预算6张图像→4374个令牌。
- 视频：32帧，384分辨率，激进地3x3双线性池化→每帧81个令牌。总计：32 * 81 = 2592个令牌。

这种分配大致保持总令牌数恒定。LLM永远不会看到超出其上下文的批次。编码器为不同场景生成不同几何形状，但LLM消耗相同的预算。

### 三阶段课程

LLaVA-OneVision分三个阶段训练：

1. **单图监督微调（SFT）（阶段SI）**。所有数据都是单图加文本。在高分辨率AnyRes输入上训练。这教会感知、OCR和细粒度理解。使用LLaVA-NeXT数据以及OneVision特定的单图数据。
2. **OneVision监督微调（SFT）（阶段OV）**。混合单图+多图+视频（均匀采样帧）。在统一令牌预算上训练。这教会模型处理异构批次形状。不重置权重——从阶段SI继续。
3. **任务迁移（阶段TT）**。继续使用目标任务混合，通常根据产品需求加重多图或视频。可选的部署微调。

关键：课程顺序很重要。先训练视频或多图会得到比先训练单图更差的图像性能，即使使用相同的数据。论文明确消融了这一点。

### 课程为何有效

单图训练建立了感知基础。补丁令牌携带细粒度视觉特征；LLM学会将它们与文本集成。多图和视频引入了结构挑战（哪张图像是哪个，发生了什么顺序），这些在没有强大感知基础的情况下很难学习。

如果从零开始一起训练所有场景，模型会欠拟合感知（每个批次中单图数据有限）而过拟合结构（大量多图/视频数据）。结果：一个遵循跨图像推理模式但在视觉上浅薄的模型。

课程顺序让你从阶段SI获得感知强度，然后从阶段OV获得组合/时间推理，而不丢失任何一个。

### 涌现的跨场景技能

LLaVA-OneVision论文报告了三种涌现能力：

1. **多摄像头推理**。分别基于多图和视频进行训练；在推理时，要求对多摄像头驾驶场景进行推理。模型正确整合了多视图，尽管在训练中从未见过这种确切格式。
2. **标记集提示**。用户用编号标记注释图像中的物体；模型推理“标记3相对于标记7在做什么”。既未训练标记，也未训练注释；从空间定位（Spatial Grounding）和多图引用的组合中学习。
3. **iPhone截图代理**。用户提供iPhone屏幕截图并询问下一步点击计划。基于UI截图、用户工作流程视频以及多图前后对比对进行训练。泛化到代理用例。

这些不是训练任务；它们是从课程的组合结构中涌现出来的。

### 视觉令牌池化

令牌预算需要池化。OneVision在二维补丁网格上使用双线性插值：24x24 = 576个补丁变成12x12 = 144（2倍因子）或8x8 = 64（3倍因子）。池化在补丁网格空间中进行，而不是令牌空间，以保持局部性。

每个场景的池化因子选择本身就是一个超参数。更少池化 = 更多令牌 = 更丰富的表示。更多池化 = 更少令牌 = 可以容纳更多帧/图像。

### LLaVA-OneVision-1.5

2025年的后续版本（LLaVA-OneVision-1.5，arXiv 2509.23661）在训练数据、模型权重和代码上“完全开放”。在某些基准测试上缩小了与专有模型的差距，并使配方民主化。相同的课程，更多数据，更好的基础LLM。架构没有变化。

### 与Qwen2.5-VL的对比

Qwen2.5-VL（课程12.09）做出了不同的选择。它使用M-RoPE和动态帧率（FPS）而不是固定池化。其预算随输入缩放——1分钟视频比5秒视频使用更多令牌。LLaVA-OneVision固定预算并缩放池化。两者都有效；它们在可配置性与可预测性之间做了权衡。

## 使用

`code/main.py` 是一个用于OneVision风格VLM的课程和预算规划器。给定每个样本的令牌预算和目标场景混合（例如40%单图、30%多图、30%视频），它将：

- 为每个场景分配分辨率、池化因子和帧数。
- 检查每个场景是否都在共享预算内。
- 报告预期令牌数、LLM FLOPs以及哪些场景令牌不足。
- 打印分阶段训练计划。

用它来规划OneVision微调，或检查VLM部署的每请求成本是否合理。

## 交付

本课程生成 `outputs/skill-onevision-budget-planner.md`。给定目标任务分布和每个样本预算，它将输出AnyRes因子、每帧池化、视频帧数和课程阶段权重。每当训练或微调统一场景VLM时，请使用此输出。

## 练习

1. 你的产品支持80%单图、10%多图（2-4张图像）、10%视频（8-16帧）。设计令牌预算。你将把从非重度多图中节省的额外预算放在哪里？
2. 阅读LLaVA-OneVision第4.3节（涌现能力）。提出第四个课程很可能解锁但论文未报告的涌现技能。
3. 交换课程顺序——先训练多图，然后单图，最后视频。预测哪些基准测试会下降以及原因。
4. 论文报告的视频基准测试仅训练每个样本8帧。这在推理时能否泛化到30秒视频？首先失效的是什么——令牌预算还是时间推理？
5. 将24x24补丁双线性池化为12x12是每个维度的4倍缩减。在Python标准库中实现池化，并验证每个2x2块的平均值是否与双线性输出匹配。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|----------|----------|
| OneVision场景 | "单图、多图或视频" | 统一VLM处理的三种输入形状之一；预算跨场景保持恒定 |
| 令牌预算 | "每个样本多少个令牌" | LLM在每个训练/推理样本中看到的视觉令牌总数，通常为3000-4000 |
| 课程 | "训练顺序" | 为了涌现迁移而选择的阶段顺序（单图 → 多图 → 视频） |
| 双线性池化 | "令牌缩小" | 对补丁网格（2D）应用双线性插值以减少令牌数量，同时保持局部性 |
| 涌现技能 | "未训练但仍有效" | 在推理时出现的能力，无需匹配的训练数据，源于课程组合 |
| AnyRes-k | "k块设置" | k个固定分辨率的子块加一个缩略图，典型k ∈ {4, 9} |
| 任务迁移 | "跨场景泛化" | 在单图上学习的技能通过共享主干应用于视频（反之亦然） |
