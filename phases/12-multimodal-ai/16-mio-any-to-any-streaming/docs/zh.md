# 16 · MIO 与任意到任意（Any-to-Any）流式多模态模型

> GPT-4o 交付了一款大多数开源模型无法复刻的产品：一个能听语音、看视频、并实时开口回应的智能体。截至 2024 年末，开源生态给出的答案是 MIO（Wang 等人，2024 年 9 月）。MIO 把文本、图像、语音和音乐都「词元化（tokenize）」，在交错排列的序列上训练一个单一的因果 Transformer，并可从任意模态生成到任意模态。AnyGPT（Zhan 等人，2024 年 2 月）是概念验证；MIO 是规模化升级版；Unified-IO 2（Allen AI，2023 年 12 月）是带有视觉 + 动作接地（action grounding）的近亲。本课解读任意到任意的范式——四个分词器（tokenizer）、一个 Transformer、对流式解码友好。

**类型：** 学习
**语言：** Python（标准库，四模态词元分配器 + 流式解码循环）
**前置：** 阶段 12 · 11（Chameleon）、阶段 6（语音与音频）
**时长：** 约 120 分钟

## 学习目标

- 设计一个共享词表（vocabulary），同时容纳文本、图像、语音和音乐词元而不发生冲突。
- 在压缩与重建的权衡上，对比 SEED-Tokenizer（图像）与 SpeechTokenizer 残差向量量化（residual-VQ，语音）。
- 解释逐步构建出任意到任意生成能力的四阶段课程（curriculum）。
- 说出三种开源任意到任意方案及其主要权衡：MIO、AnyGPT、Unified-IO 2。

## 问题所在

统一多模态模型容易宣称，却很难在规模上做出来。截至 2024 年，大多数「任意到任意」系统都是流水线式的：视觉模型 → 文本表示 → 语音模型 → 音频。每一跳都会损失信息、增加延迟，并使训练复杂化。GPT-4o 的演示视频展示了一种亚秒级响应的单模型替代方案；开源系统落后了数月。

工程上的挑战：

- 每个模态都必须有分词器，压缩到「足够无损」以支持重建，并以 Transformer 能够消费的速率产出词元。
- 单一词表必须为文本（32k+）、图像（16k+）、语音（4k+）、音乐（8k+）分配空间。最少需要四万多个条目。
- 训练数据必须覆盖每一种输入-输出配对（text→image、image→speech、speech→image 等），否则模型必须靠组合（compose）来实现。
- 推理时必须以足够快的速度流式输出词元，以满足对话延迟要求（首字节音频时间 < 500ms）。

## 核心概念

### 四个模态对应四个分词器

MIO 的分词器栈：

- 文本：标准 BPE，词表约 32000。
- 图像：SEED-Tokenizer（2023）——带离散码本（codebook）的量化 VAE，4096 个条目，每张图像 32x32 个词元。
- 语音：SpeechTokenizer 残差向量量化（residual-VQ，2023）——将 16kHz 波形编码为 8 个分层码本；第一层是粗粒度内容，后续各层补充韵律（prosody）和说话人身份。
- 音乐：类似的残差向量量化（Meta 的 MusicGen / Encodec 系列），4-8 个码本。

每个模态都产出整数词元。这些词元在共享词表中获得互不相交的 ID 区间：

```
text:   0..31999
image:  32000..36095  (4096 image tokens)
speech: 36096..40191  (4096 speech base tokens, plus residual layers)
music:  40192..48383  (8192 music tokens)
sep:    48384..48390  (<image>, <speech>, <music>, </...>, etc.)
```

总计：约 48k 词表。输入嵌入（embedding）和输出投影（projection）都横跨整个词表。

### 流式解码

语音生成使用残差向量量化。Transformer 预测基础（第 0 层）语音词元；一个并行解码的残差量化器预测后续各层。每个第 0 层词元在 16kHz 下大约对应 50ms 的音频。

流式范式：

1. 用户对着麦克风说话；实时音频分词器每 50ms 发出一批语音词元。
2. MIO 在词元到达时即消费它们（提示词预填充 prefill + 增量前向）。
3. 输出词元随生成而流式输出；一个并行语音解码器以约 50-150ms 的延迟将其转换为音频样本。
4. 首字节音频时间：MIO 论文中约 300-500ms，接近 GPT-4o 的约 250ms。

Mini-Omni（arXiv:2408.16725）、GLM-4-Voice（arXiv:2412.02612）和 Moshi（arXiv:2410.00037）是互补的流式语音-大模型（speech-LLM）设计。其中 Moshi 在单块 GPU 上实现了 160ms 的往返延迟。

### 四阶段课程

MIO 的训练课程：

1. 阶段 1——对齐（alignment）。大规模模态配对语料：text-image、text-speech、text-music。每一种配对使用自己的词元词表段。训练共享词表。
2. 阶段 2——交错（interleaved）。多模态交错文档（带图像 + 视频的博客、带文字稿的播客等）。训练跨模态上下文。
3. 阶段 3——语音增强（speech-enhanced）。补充额外的音频数据以提升语音质量，同时不损失文本能力。
4. 阶段 4——SFT（监督微调）。跨模态的指令微调：VQA、图像描述（captioning）、解说（narration）、语音到语音对话。

缺失某一阶段会削弱特定能力：跳过阶段 2，模型会丧失跨模态上下文；跳过阶段 3，语音质量会很差。

### 视觉思维链（chain-of-visual-thought）

MIO 引入了视觉思维链：模型把中间图像词元作为一个推理步骤发出。对于「猫是不是在爬树？」这一问题，模型会：

1. 发出 `<image>` 词元来渲染场景（来自输入图像或一张草图）。
2. 发出分析该草图的文本。
3. 发出最终答案。

渲染出的中间图像充当了一块草稿板（scratchpad）。在空间推理任务上的基准表现得到提升。这一思路对应于文本推理中的思维链（chain-of-thought）。

### 任意到任意领域的竞争者

- AnyGPT（arXiv:2402.12226）：4 个模态（文本、图像、语音、音乐），设计类似。
- Unified-IO 2（arXiv:2312.17172）：增加了视觉动作输出、深度（depth）、法线（normals）。任务多样性更高，规模更小。
- NExT-GPT（arXiv:2309.05519）：大模型 + 模态专用的扩散解码器。不是单模型方案。
- CoDi（arXiv:2305.11846）：可组合扩散（composable diffusion）；通过共享隐空间实现任意到任意。

MIO 最接近纯词元的任意到任意。AnyGPT 是它的概念上的先祖。

### 延迟预算

对于一款对话式产品，每个组件的延迟都很重要：

- 麦克风到音频词元：约 50ms。
- 预填充（音频词元 + 历史记录）：在 8B 模型上约 100ms。
- 首个输出词元：约 50ms。
- 并行残差向量量化 + 语音解码器：约 100-150ms。

首字节音频时间总计：最少约 300ms。GPT-4o 宣称约 250ms。Moshi 宣称 160ms。根据公开基准，MIO/AnyGPT 在 400-600ms 区间。

### 任意到任意为何始终很难

即便到了 2026 年，开源任意到任意模型在两个维度上仍落后于闭源模型：

- 语音质量。残差向量量化分词器是有损的；与 ElevenLabs 级别的语音相比，对话式语音听起来很机械。
- 跨模态推理。让模型「为你看到的东西唱首歌」，其失败率仍高于纯视觉任务。

这些都是尚未解决的研究难题。Qwen3-Omni（课程 12.20）是 2025 年最先进的开源尝试。

## 动手用

`code/main.py`：

- 定义四模态词表分配并打印出来。
- 把一组多模态输入（文本、图像、音频片段、音乐）经过分词器路由器（tokenizer router）进行路由。
- 模拟一段文本到语音响应的流式解码，并进行延迟计数。
- 在给定编码器、预填充和解码器延迟的情况下，计算预期的首字节音频时间。

## 练习

1. 你的产品接受语音输入并返回语音输出。端到端延迟预算目标是多少？列出耗时的各个组件。

2. SpeechTokenizer 残差向量量化使用 8 个码本。请论证为什么需要对残差各层进行并行解码（而非顺序解码），以及它带来了哪些延迟节省。

3. 你的词表有 32k 文本 + 4k 图像 + 4k 语音。再加上 8k 音乐和约 10 个分隔符。在隐藏维度 4096 下，嵌入矩阵的参数代价是多少？

4. 视觉思维链会发出一张中间图像。哪类问题会从中受益？哪类问题会因额外的词元而受损？

5. 阅读 Moshi（arXiv:2410.00037）。描述其「内心独白（inner monologue）」技术，并与 MIO 的视觉思维链进行对比。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------------|------------------------|
| 任意到任意（Any-to-any） | 「多模态输入/输出」 | 一个单一模型，能在任意方向上接受并发出文本、图像、语音和音乐 |
| 残差向量量化（Residual-VQ） | 「语音分词器栈」 | 多码本词元化，每一层补充信息；基础层是内容，后续各层是韵律 |
| SEED-Tokenizer | 「图像编码」 | MIO 使用的离散图像分词器，带 4096 个条目的码本 |
| 视觉思维链（Chain-of-visual-thought） | 「视觉草稿板」 | 模型在给出最终答案前，生成一张中间图像作为推理步骤 |
| 首字节音频时间（Time-to-first-audio-byte） | 「TTFAB」 | 从用户语音到首段音频输出的延迟；对话感要求 < 500ms |
| 四阶段课程（Four-stage curriculum） | 「训练配方」 | 按此顺序：对齐 -> 交错 -> 语音增强 -> SFT |
