# MIO 与任意到任意流式多模态模型

> GPT-4o 交付了一个大多数开放模型无法复制的产品：一个听到语音、看到视频并实时回话的代理。到 2024 年底，开放生态系统的答案是 MIO（Wang 等人，2024 年 9 月）。MIO 将文本、图像、语音和音乐分词，在交错序列上训练一个因果 transformer，并在任何模态之间生成任何模态。AnyGPT（Zhan 等人，2024 年 2 月）是概念验证；MIO 是规模扩展；Unified-IO 2（Allen AI，2023 年 12 月）是具有视觉 + 动作基础的表亲。本课解读任意到任意模式——四个分词器，一个 transformer，流式友好解码。

**类型：** Learn
**语言：** Python（stdlib，四模态 token 分配器 + 流式解码循环）
**前置知识：** Phase 12 · 11（Chameleon），Phase 6（语音和音频）
**时间：** ~120 分钟

## 学习目标

- 设计一个共享词汇，容纳文本、图像、语音和音乐 token 而不冲突。
- 在压缩 + 重建权衡上比较 SEED-分词器（图像）和 SpeechTokenizer 残差-VQ（语音）。
- 解释构建任意到任意生成的四阶段课程。
- 命名三个开放任意到任意配方及其主要权衡：MIO、AnyGPT、Unified-IO 2。

## 问题所在

统一多模态模型容易声称但难以大规模构建。直到 2024 年，大多数"任意到任意"系统都是流水线式的：视觉模型 → 文本表示 → 语音模型 → 音频。每次跳跃丢失信息，增加延迟，并使训练复杂化。GPT-4o 的演示视频展示了单模型替代方案，响应时间低于一秒；开放系统落后数月。

工程挑战：

- 分词器必须存在于每种模态，压缩到足以无损重建的程度，并以 transformer 可以消费的速率产生 token。
- 单一词汇必须为文本（32k+）、图像（16k+）、语音（4k+）、音乐（8k+）分配空间。最少四万多个条目。
- 训练数据必须覆盖每个输入-输出对（文本→图像、图像→语音、语音→图像等），或者模型必须组合。
- 推理必须流式输出 token 足够快以满足对话延迟（<500ms 首音频字节时间）。

## 核心概念

### 四种模态的四个分词器

MIO 的分词器栈：

- 文本：标准 BPE，词汇约 32000。
- 图像：SEED-分词器（2023）——具有离散码本的量化 VAE，4096 个条目，每张图像 32x32 个 token。
- 语音：SpeechTokenizer 残差-VQ（2023）——将 16kHz 波形编码为 8 个分层码本；第一层是粗略内容，后续层添加韵律和说话人身份。
- 音乐：类似残差-VQ（Meta 的 MusicGen / Encodec 家族），4-8 个码本。

每种模态产生整数 token。Token 在共享词汇中获得不相交的 ID 范围：

```
text:   0..31999
image:  32000..36095  (4096 图像 token)
speech: 36096..40191  (4096 语音基础 token，加残差层)
music:  40192..48383  (8192 音乐 token)
sep:    48384..48390  (<image>, <speech>, <music>, </...>, 等)
```

总计：约 48k 词汇。输入嵌入和输出投影跨越所有词汇。

### 流式解码

语音生成使用残差-VQ。Transformer 预测基础（第 0 层）语音 token；并行解码的残差量化器预测后续层。每个第 0 层 token 在 16kHz 下大约是 50ms 音频。

流式模式：

1. 用户对着麦克风说话；实时音频分词器每 50ms 发出语音 token。
2. MIO 在 token 到达时消费它们（提示词预填充 + 增量前向）。
3. 输出 token 在生成时流式输出；并行语音解码器以约 50-150ms 延迟将它们转换为音频样本。
4. 首音频字节时间：MIO 论文中约 300-500ms，接近 GPT-4o 的约 250ms。

Mini-Omni（arXiv:2408.16725）、GLM-4-Voice（arXiv:2412.02612）和 Moshi（arXiv:2410.00037）是互补的流式语音-LLM 设计。Moshi 特别实现了单 GPU 上 160ms 往返。

### 四阶段课程

MIO 的训练课程：

1. 第 1 阶段——对齐。大规模模态对语料库：文本-图像、文本-语音、文本-音乐。每对使用自己的词汇段。训练共享词汇。
2. 第 2 阶段——交错。多模态交错文档（带图像 + 视频的博客、带转录的播客等）。训练跨模态上下文。
3. 第 3 阶段——语音增强。额外音频数据以在不损失文本能力的情况下提升语音质量。
4. 第 4 阶段——SFT。跨模态指令调优：VQA、描述、旁白、语音到语音对话。

缺少一个阶段会降低特定能力：跳过第 2 阶段，模型失去跨模态上下文；跳过第 3 阶段，语音质量差。

### 视觉思维链

MIO 引入视觉思维链：模型发出中间图像 token 作为推理步骤。对于"猫在爬树吗？"，模型：

1. 发出 `<image>` token 渲染场景（来自输入图像或草图）。
2. 发出文本分析草图。
3. 发出最终答案。

渲染的中间图像充当草稿纸。空间推理任务的基准测试改善。这个想法类似于文本推理的思维链。

### 任意到任意的竞争者

- AnyGPT（arXiv:2402.12226）：4 种模态（文本、图像、语音、音乐），类似设计。
- Unified-IO 2（arXiv:2312.17172）：添加视觉动作输出、深度、法线。更多任务多样性，更小规模。
- NExT-GPT（arXiv:2309.05519）：LLM + 模态特定扩散解码器。不是单模型方法。
- CoDi（arXiv:2305.11846）：可组合扩散；通过共享潜在变量的任意到任意。

MIO 最接近纯 token 任意到任意。AnyGPT 是其概念祖先。

### 延迟预算

对于对话产品，每个组件的延迟都很重要：

- 麦克风到音频 token：约 50ms。
- 预填充（音频 token + 历史）：8B 模型上约 100ms。
- 首个输出 token：约 50ms。
- 并行残差-VQ + 语音解码器：约 100-150ms。

首音频字节总时间：最少约 300ms。GPT-4o 声称约 250ms。Moshi 声称 160ms。MIO/AnyGPT 根据公开基准在 400-600ms 范围内。

### 为什么任意到任意仍然困难

即使在 2026 年，开放任意到任意模型在两个轴上仍落后封闭模型：

- 语音质量。残差-VQ 分词器是有损的；对话语音与 ElevenLabs 级声音相比听起来机械。
- 跨模态推理。要求模型"唱出你所看到的"仍然比纯视觉任务更频繁地失败。

这些是开放研究问题。Qwen3-Omni（第 12.20 课）是 2025 年最先进的开放尝试。

## 使用它

`code/main.py`：

- 定义四模态词汇分配并打印它。
- 通过分词器路由器路由多模态输入列表（文本、图像、音频片段、音乐）。
- 模拟文本到语音响应的流式解码，带延迟计数。
- 给定编码器、预填充和解码器延迟，计算预期的首音频字节时间。

## 交付它

本课产出 `outputs/skill-any-to-any-pipeline-auditor.md`。给定对话产品规格（输入模态、输出模态、延迟目标），它审计 MIO 家族的设计选择并计算延迟预算。

## 练习

1. 你的产品接受语音输入并返回语音输出。端到端延迟预算目标是什么？列出花费时间的组件。

2. SpeechTokenizer 残差-VQ 使用 8 个码本。提出为什么并行解码残差层是必要的（vs 顺序）以及它带来什么延迟节省。

3. 你的词汇有 32k 文本 + 4k 图像 + 4k 语音。添加 8k 音乐和约 10 个分隔符。在隐藏维度 4096 下，嵌入矩阵参数成本是多少？

4. 视觉思维链发出中间图像。什么问题受益？什么类型被额外 token 伤害？

5. 阅读 Moshi（arXiv:2410.00037）。描述其"内心独白"技术并与 MIO 的视觉思维链比较。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------------|------------------------|
| 任意到任意 | "多模态输入/输出" | 接受和发出文本、图像、语音和音乐任何方向的单一模型 |
| 残差-VQ | "语音分词器栈" | 多层码本分词，每层添加信息；基础层是内容，后续层是韵律 |
| SEED-分词器 | "图像代码" | MIO 使用的具有 4096 条目码本的离散图像分词器 |
| 视觉思维链 | "视觉草稿纸" | 模型在最终答案之前生成中间图像作为推理步骤 |
| 首音频字节时间 | "TTFAB" | 从用户语音到首个音频输出的延迟；<500ms 用于对话感觉 |
| 四阶段课程 | "训练配方" | 对齐 -> 交错 -> 语音增强 -> SFT，按此顺序 |
