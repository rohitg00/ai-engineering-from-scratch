# Chameleon 与早期融合纯令牌多模态模型

> 我们迄今所见的每个视觉语言模型（VLM）都将图像和文本分开处理。视觉令牌来自视觉编码器，流入投影器，然后在大语言模型（LLM）内部与文本汇合。视觉和文本词汇表从未重叠。Chameleon（Meta，2024年5月）提出了一个问题：如果它们重叠了呢？训练一个VQ-VAE，将图像转换为来自共享词汇表的离散令牌序列。现在每个多模态文档都是一个序列——文本令牌和图像令牌交织在一起，使用单一的自回归损失。副作用：模型可以生成混合模态的输出——在单次推理调用中交替出现文本和图像令牌。本课将解读早期融合论文，并从头到尾构建一个玩具版本。

**类型：** 构建
**语言：** Python（标准库，VQ-VAE分词器 + 交织解码器）
**前置知识：** 阶段 12 · 05，阶段 8（生成式 AI）
**时间：** 约 180 分钟

## 学习目标

- 解释为什么共享词汇表 + 单一损失会改变模型的能力。
- 描述VQ-VAE如何将图像分词为与Transformer的下一令牌目标兼容的离散序列。
- 列举Chameleon的训练稳定性技巧：QK-Norm、丢弃（dropout）放置、层归一化（LayerNorm）顺序。
- 比较Chameleon与BLIP-2的Q-Former方法，并描述何时选择哪种更合适。

## 问题

基于适配器的VLM（如LLaVA、BLIP-2、Qwen-VL）将文本和图像视为两种不同的东西。文本令牌经过`embed(text_token)`；图像经过`visual_encoder(image) → projector → ... pseudo_tokens`。模型有两个输入路径，在中间某处合并。

三个后果：

1. LLM只能消费图像，不能生成图像。输出仅是文本。
2. 混合模态文档（如文章中交替的段落和图像）处理起来很尴尬——要么在模型外部解析多模态输入，要么进行链式生成。
3. 分布不匹配。视觉令牌和文本令牌位于隐藏空间的不同区域，产生细微的对齐问题。

Chameleon拒绝了这一前提：图像只是来自共享词汇表的离散令牌序列。在交织文档上训练模型，单一损失，单一自回归解码器，就能免费获得混合模态生成。

## 概念

### VQ-VAE作为图像分词器

分词器是一个向量量化变分自编码器。架构如下：

- 编码器：CNN + ViT，将图像映射到空间特征图，例如32x32个256维特征。
- 码本：一个包含K个向量的学习词汇表（Chameleon使用8192），也是256维。
- 量化：对于每个空间特征，通过L2距离查找最近的码本条目。用整数索引替换连续特征。
- 解码器：CNN，将量化特征还原为像素。

训练：VAE重建损失 + 承诺损失（commitment loss） + 码本损失。码本索引构成了图像的离散字母表。

对于Chameleon：一张图像变成32*32 = 1024个令牌，这些令牌来自一个大小为8192的词汇表。与文本令牌（来自LLM的BPE词汇表，约32000个）连接。最终词汇表：40192个。Transformer看到一个序列，一个损失。

### 共享词汇表

Chameleon的词汇表结合了文本令牌、图像令牌和模态分隔符。每个令牌都有一个唯一的ID。输入嵌入层将每个ID映射到D维隐藏向量。输出投影将隐藏向量映射回词汇表logits。Softmax选出下一个令牌，无论是什么模态。

分隔符很重要：`<image>`和`</image>`标签包围着图像令牌序列。在生成时，如果模型发出`<image>`，下游软件就知道接下来的1024个令牌是VQ索引，需发送给解码器进行像素渲染。

### 混合模态生成

推理是在共享词汇表中的下一个令牌预测。示例提示："画一只猫并描述它。" Chameleon生成：

```
<image> 4821 1029 2891 ... (1024个图像令牌) </image>
这只猫是橙色的，坐在窗台上……
```

模型自主选择顺序——它可能先生成图像再生成文本，或者先文本后图像，或者交织。相同的解码器，相同的损失。

与适配器VLM（只生成文本）相比，Chameleon重新开放了模型输出模态的问题。

### 训练稳定性——QK-Norm、丢弃（dropout）、层归一化（LayerNorm）顺序

早期融合训练在大规模下不稳定。Chameleon的论文记录了三个技巧：

- QK-Norm。在注意力机制中，对查询（query）和键（key）应用层归一化，再进行点积。防止深度下logit幅值爆炸。2024年后的多个大型模型都使用了此技巧。
- 丢弃放置。在每个残差加法之后都进行丢弃，而不仅仅是在注意力机制和MLP之后。当图像令牌的梯度可能占主导时，需要更多正则化。
- 层归一化顺序。残差分支上的Pre-LN（标准做法），加上在最后一个块的跳跃连接上额外增加一个LN。稳定了最后一层的梯度流。

没有这些技巧，34B参数的Chameleon训练在多个检查点处发散。有了它们，则收敛。训练配方与架构本身贡献相当。

### 分词器的重建上限

VQ-VAE是有损的。当码本大小为8192，每张512x512图像使用1024个令牌时，重建峰值信噪比（PSNR）上限约为26-28 dB。这对于可识别的图像生成来说足够了，但明显劣于连续空间扩散（Stable Diffusion 3达到32+ dB）。

分词器是瓶颈。更好的分词器（MAGVIT-v2、IBQ、SBER-MoVQGAN）提高了上限。Emu3（课程 12.12）仅通过更好的分词器就实现了SDXL级别的生成质量。

### Chameleon vs BLIP-2 / LLaVA

Chameleon（早期融合，共享词汇表）：
- 单一损失，单一解码器。
- 生成混合模态输出。
- 分词器是质量上限。
- 昂贵：推理路径上每张生成的图像都需要VQ-VAE解码器。

BLIP-2 / LLaVA（晚期融合，独立塔）：
- 只有视觉输入，文本输出。
- 重用预训练的LLM。
- 理解任务中没有分词器瓶颈。
- 便宜：单次前向传播。

根据任务选择。如果需要图像生成，选Chameleon家族。如果只需要理解，适配器VLM更简单且能重用更多预训练计算。

### Fuyu 与 AnyGPT

Fuyu（Adept，2023）是一种相关方法：完全跳过独立的视觉编码器，将原始图像块通过LLM的输入投影直接作为令牌输入，不使用分词器。比Chameleon更简单，但失去了共享词汇表的输出生成能力。

AnyGPT（Zhan等人，2024）将Chameleon扩展到四种模态：文本、图像、语音、音乐。每种模态都采用相同的VQ-VAE技巧，共享Transformer。任意到任意生成。在课程 12.16 中有更详细的内容。

## 使用它

`code/main.py` 构建了一个玩具级端到端早期融合模型：

- 一个微型的VQ-VAE风格量化器，将8x8图像块映射到码本索引（K=16）。
- 一个共享词汇表：（文本ID 0..31）+（图像ID 32..47）+（分隔符48, 49）。
- 一个玩具级自回归解码器（双字母组合表），在合成标题+图像令牌序列上训练。
- 采样循环，根据提示交替生成文本+图像令牌。

代码特意将Transformer保持得很小（双字母组合），以便你能端到端地追踪信号流。

## 练习

1. Chameleon使用K=8192个码本条目，每张512x512图像使用1024个令牌。估算与24位RGB图像相比的压缩比。它是有损的吗？有多大的损失？

2. 一张4K图像（3840x2160）在相同的VQ-VAE密度下会产生多少个图像令牌？一个Chameleon风格的模型能在单次推理调用中生成一张4K图像吗？首先打破的是哪个——上下文、分词器质量，还是KV缓存？

3. 用纯Python实现QK-Norm。给定一个64维的查询（query）和键（key），展示层归一化前后的点积。为什么在深度情况下幅值控制很重要？

4. 阅读Chameleon第2.3节关于训练稳定性的内容。描述论文观察到的34B模型在没有QK-Norm时的确切失败模式。"范数爆炸"的信号是什么？

5. 扩展玩具解码器，使其能够在给定纯文本提示的情况下生成混合模态响应。在训练数据分布为60%文本优先/40%图像优先的情况下，测量模型选择图像优先与文本优先的频率。

## 关键术语

| 术语 | 人们通常说 | 实际含义 |
|------|------------|----------|
| 早期融合 (Early fusion) | "统一令牌" | 图像被转换为离散令牌，从一开始就与Transformer共享词汇表 |
| VQ-VAE | "图像分词器" | CNN + ViT + 码本，将图像映射为Transformer可以预测的整数索引 |
| 共享词汇表 (Shared vocabulary) | "一个词典" | 单一的令牌ID空间，覆盖文本+图像+模态分隔符 |
| QK-Norm | "注意力稳定器" | 在查询和键进行点积之前应用层归一化，防止范数爆炸 |
| 混合模态生成 (Mixed-modality generation) | "文本+图像输出" | 推理过程中自主地在一轮生成中产生交织的文本和图像令牌 |
| 码本大小 (Codebook size) | "K个条目" | VQ-VAE可以量化的离散向量数量；在压缩和保真度之间权衡 |
| 分词器上限 (Tokenizer ceiling) | "重建极限" | 解码VQ令牌所能达到的最佳PSNR；限制了模型的图像质量 |
