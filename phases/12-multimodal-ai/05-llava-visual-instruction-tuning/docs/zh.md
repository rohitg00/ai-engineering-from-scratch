# LLaVA 与视觉指令微调

> LLaVA（2023 年 4 月）是全球被复制最多的多模态架构。它用 2 层 MLP 替代了 BLIP-2 的 Q-Former，用朴素的 token 拼接替代了 Flamingo 的门控交叉注意力，并在 158k 条由 GPT-4 根据纯文本描述生成的视觉指令轮次上进行了训练。任何在 2023 年至 2026 年间构建 VLM 的从业者都构建过某种 LLaVA 变体。LLaVA-1.5 增加了 AnyRes。LLaVA-NeXT 提升了分辨率。LLaVA-OneVision 在单一方案中统一了图像、多图像和视频。本课将解读该方案，实现投影器，并解释为什么“更简单的方案赢了”。

**类型：** Build
**语言：** Python（stdlib，投影器 + 指令模板构建器）
**前置知识：** Phase 12 · 02（CLIP），Phase 11（LLM 工程——指令微调）
**时间：** ~180 分钟

## 学习目标

- 构建一个 2 层 MLP 投影器，将 ViT 的 patch 嵌入（维度 1024）映射到 LLM 的嵌入维度（维度 4096）。
- 掌握 LLaVA 的两阶段方案：（1）在 558k 描述对上对齐投影器，（2）在 158k 条 GPT-4 生成的轮次上进行视觉指令微调。
- 构建一个包含图像 token 占位符、系统提示词以及用户/助手轮次的 LLaVA 格式提示词。
- 解释为什么社区从 Q-Former 转向了 MLP，尽管 Q-Former 在 token 预算上具有优势。

## 问题所在

BLIP-2 的 Q-Former（第 12.03 课）将图像压缩为 32 个 token。简洁、高效、在基准测试中表现良好。但它有两个问题。

首先，Q-Former 是可训练的，但其损失函数并非最终任务。第一阶段训练 ITC+ITM+ITG。第二阶段训练 LM 损失。查询向量学习的是某种中间表示，然后由 LLM 进行解码。信息在瓶颈处丢失了。

其次，Q-Former 占用 1.88 亿参数，在 LLaVA 所处的 2023 年规模下，你必须与目标 LLM 共同设计它。更换 LLM，就得重新训练 Q-Former。更换视觉编码器，也得重新训练。每一种组合都是一个独立的研发项目。

LLaVA 的答案简单得令人尴尬：取 ViT 的 576 个 patch token，将每个 token 通过一个 2 层 MLP（`1024 → 4096 → 4096`），然后将全部 576 个 token 丢进 LLM 的输入序列。没有瓶颈。没有在奇怪目标函数上的第一阶段预训练。只需在直接的 LM 损失上训练 MLP。

数据从何而来？LLaVA 的第二个洞见：使用 GPT-4（纯文本）生成指令数据。向 GPT-4 提供图像的 COCO 描述和边界框数据，让它生成对话、描述和复杂的推理问题。免费获得 158k 条指令-回复轮次。无需人工标注。

结果是：一个 VLM 在 8 块 A100 上运行一天，在 MMMU 上击败了 Flamingo，并发布了一个可供社区扩展的开源检查点。到 2023 年底，它已经衍生出 50 多个分支。

## 核心概念

### 架构

LLaVA-1.5 13B 版本：
- 视觉编码器：CLIP ViT-L/14 @ 336（第一阶段冻结，第二阶段可选择性地解冻）。
- 投影器：带 GELU 激活函数的 2 层 MLP，`1024 → 4096 → 4096`。
- LLM：Vicuna-13B（后来是 Llama-3.1-8B）。

图像 + 文本提示词的前向传播过程：

```
图像 -> ViT -> 576 个维度为 1024 的 patch
patch -> MLP -> 576 个维度为 4096 的 token
提示词：系统提示词 + "<image>" 占位符 + 用户问题
将 <image> token 替换为 576 个投影后的 token
将完整序列输入 LLM
解码回复
```

图像在 LLM 上下文中占据 576 个 token。在 2048 的上下文长度下，这留给文本 1472 个 token。在 32k 的上下文长度下，这只是一个微不足道的零头。

### 第一阶段：投影器对齐

冻结 ViT。冻结 LLM。仅训练 2 层 MLP。数据集：558k 图像-描述对（LAION-CC-SBU）。损失函数：以投影后的图像 token 为条件的描述语言建模。

在 batch size 为 128 的情况下，单轮 epoch 只需几个小时即可完成。投影器学习将 ViT 空间映射到 LLM 空间。没有特定于任务的监督。

### 第二阶段：视觉指令微调

解冻投影器（仍然可训练）。解冻 LLM（通常是全部解冻，有时使用 LoRA）。在 158k 条视觉指令轮次上进行训练。

指令数据是诀窍所在。Liu 等人通过以下方式生成数据：
1. 取一张 COCO 图像。
2. 提取文本描述（5 条人工描述 + 边界框列表）。
3. 发送给 GPT-4，并附带三个提示词模板：
   - 对话：“生成一段关于这张图像的用户与助手之间的来回对话。”
   - 详细描述：“给出该图像丰富、详细的描述。”
   - 复杂推理：“提出一个需要对图像进行推理的问题，然后回答它。”
4. 将 GPT-4 的输出解析为（指令，回复）对。

这一切都没有直接接触图像——只有文本描述。GPT-4 会幻觉出看似合理的图像内容。存在一些噪声，但它奏效了：158k 轮次足以解锁对话能力。

### 为什么社区复制了这个方案

- 没有需要微调的第一阶段特定损失。全程使用 LM 损失。
- 投影器在数小时内完成训练，而非数天。
- LLM 可以更换（LLaVA-Llama2、LLaVA-Mistral、LLaVA-Llama3），只需重新训练投影器。
- 视觉指令数据流水线使用 GPT-4，并且针对新领域重新生成的成本很低。

### LLaVA-1.5 和 LLaVA-NeXT

LLaVA-1.5（2023 年 10 月）增加了：
- 学术任务数据（VQA、OKVQA、RefCOCO）混合到指令微调中。
- 更好的系统提示词。
- 上下文长度从 2048 扩展到 32k。

LLaVA-NeXT（2024 年 1 月）增加了：
- AnyRes：将高分辨率图像分割成 2x2 或 1x3 的 336x336 网格，外加一个全局低分辨率缩略图。每个裁剪块变成 576 个 token；每张图像总共约 2880 个视觉 token。OCR 和图表任务性能大幅提升。
- 使用 ShareGPT4V（高质量 GPT-4V 描述）进行更好的指令数据混合。
- 更强大的基础 LLM（Mistral-7B、Yi-34B）。

### LLaVA-OneVision

第 12.08 课深入介绍 OneVision。简而言之：相同的投影器，但使用涵盖单图像、多图像和视频的课程进行训练，共享视觉 token 预算。

### 与 Q-Former 的比较

| | Q-Former (BLIP-2) | MLP (LLaVA) |
|---|---|---|
| 每张图像的视觉 token | 32 | 576（基础）或 2880（AnyRes） |
| 可训练参数 | 1.88 亿 + LLM | 4000 万 + LLM |
| 第一阶段损失 | ITC+ITM+ITG | 仅 LM |
| LLM 即插即用 | 需要重新训练 | 更换时只需最少的重新训练 |
| 多图像 | 笨拙 | 自然（拼接） |
| 视频 | 笨拙 | 自然（每帧拼接） |
| Token 预算 | 小 | 大 |

MLP 在简单性和 token 灵活性上获胜。Q-Former 在 token 预算上获胜。到 2023 年底，token 预算不再是瓶颈（LLM 上下文增长到 32k-128k+），简单性占据了主导地位。

### 提示词格式

```
一个好奇的人类与人工智能助手之间的对话。助手对人类的问题给出有帮助、详细且礼貌的回答。USER: <image> 详细描述这张图像。ASSISTANT: 这张图像展示了……
```

`<image>` 是一个占位符 token。在分词之前，它被替换为 576 个视觉 token（或使用 AnyRes 时为 2880 个）。分词器看到的序列比它训练时的稍长，但由于第一阶段的训练，LLM 能够处理这种新颖的输入。

### 参数经济性

LLaVA-1.5-7B 的细分：
- CLIP ViT-L/14 @ 336：3.03 亿（第一阶段冻结，第二阶段通常解冻）。
- 投影器（2x 线性层）：约 2200 万可训练参数。
- Llama-7B：70 亿。
- 总计：73 亿参数。第二阶段可训练：完整的 70 亿 + 2200 万投影器。

第二阶段的训练成本：在 8xA100 上约 20 小时。这是关键数字——一天，一个节点，可复现。这就是 LLaVA 传播开来的原因。

## 使用它

`code/main.py` 实现了：

1. 2 层 MLP 投影器（玩具规模下维度为 16 → 32 → 32），使用纯 Python。
2. 提示词构建流水线：系统提示词 + 用 N 个投影 token 替换 `<image>` + 用户轮次 + 助手生成占位符。
3. 一个可视化工具，展示 576 个 token 的视觉块在 LLM 上下文中的样子（占 2k / 32k / 128k 上下文的百分比）。

## 交付它

本课产出 `outputs/skill-llava-vibes-eval.md`。给定一个 LLaVA 家族的检查点，它运行一个包含 10 个提示词的 vibe 评估套件（3 个描述、3 个 VQA、2 个推理、2 个拒绝），并报告一份人类可读的记分卡。不是基准测试；而是一个烟雾测试，用于确认投影器和 LLM 连接良好。

## 练习

1. 计算 `1024 → 4096 → 4096` 的 2 层 MLP 投影器的可训练参数数量。包含 GELU 和偏置，它占 LLaVA-13B 的多少比例？

2. 为“拒绝”案例构建一个 LLaVA 提示词——图像中包含一个私人个体。写出预期的助手回复。为什么 LLaVA 应该在零样本情况下拒绝，以及需要什么训练数据来强化这种拒绝？

3. 阅读 LLaVA-NeXT 博客的 AnyRes 部分。计算 1344x672 图像在 AnyRes 下的视觉 token 数量。与 336x336 下的基础 576 个 token 进行比较。

4. LLaVA 第一阶段的投影器使用描述上的 LM 损失进行训练。如果你跳过第一阶段直接进入第二阶段（视觉指令微调）会发生什么？引用 Prismatic VLMs 消融实验（arXiv:2402.07865）来回答。

5. LLaVA-Instruct-150k 使用带有 COCO 描述的 GPT-4 生成指令。对于一个新领域（医学 X 光、卫星图像），描述生成领域指令的四步数据流水线。每一步可能出什么问题？

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|----------------|------------------------|
| 投影器 | "MLP 桥梁" | 带 GELU 的 2 层 MLP，将 ViT 维度映射到 LLM 维度 |
| 图像 token | "<image> 占位符" | 在推理前被 N 个投影视觉 token 替换的提示词标记 |
| 视觉指令微调 | "LLaVA 第二阶段" | 在 GPT-4 生成的（图像，指令，回复）三元组上进行训练 |
| 第一阶段对齐 | "投影器预训练" | 冻结 ViT 和 LLM，使用描述上的 LM 损失训练投影器 |
| AnyRes | "多裁剪平铺" | 将高分辨率图像分割成瓦片网格，并拼接每个瓦片的视觉 token |
| LLaVA-Instruct | "GPT-4 生成" | 从 COCO 描述 + GPT-4 合成的 158k 条指令-回复对 |
| 视觉编码器冻结 | "骨干网络锁定" | CLIP 权重在第一阶段不更新，有时在第二阶段也不更新 |
| ShareGPT4V | "更好的描述" | 由 GPT-4V 生成的 100 万条密集描述，用于更高质量的对齐 |
| VQA | "视觉问答" | 回答关于图像的自由形式问题的任务 |
| Prismatic VLMs | "设计空间论文" | Karamcheti 2024 年系统测试投影器和数据选择的消融实验 |
