# Flamingo 和门控交叉注意力（Gated Cross-Attention）用于少样本视觉语言模型（Few-Shot VLM）

> DeepMind 的 Flamingo（2022）率先做到了两件事。它展示了一个单一模型能够处理任意交错排列的图像、视频和文本序列。它还展示了视觉语言模型（VLM）可以上下文学习（in-context learning）——给出一个包含三个示例（图像、描述）对的少样本提示（few-shot prompt），模型无需任何梯度步骤就能为一张新图像生成描述。其机制是：门控交叉注意力层（gated cross-attention layers），插入到冻结的大型语言模型（LLM）的现有层之间，带有一个从零开始学习的 tanh 门控，因此 LLM 的文本能力在初始化时得以保留。本课将讲解 Flamingo 的感知器重采样器（Perceiver resampler）和门控交叉注意力架构——这是 Gemini 的交错输入（interleaved inputs）和 Idefics2 的视觉标记（visual tokens）的祖先。

**类型：** 学习
**语言：** Python（标准库，门控交叉注意力 + 感知器重采样器演示）
**前提条件：** 阶段12 · 03（BLIP-2 Q-Former）
**时间：** ~120分钟

## 学习目标

- 解释门控交叉注意力如何通过 tanh(门) = 0 在初始化时保留冻结 LLM 的文本能力。
- 讲解感知器重采样器：通过交叉注意力从 N 个图像块生成 K 个固定的“潜在”查询（latent queries）。
- 描述 Flamingo 如何处理具有因果掩码（causal masking）的交错图像-文本序列，该掩码尊重图像的放置位置。
- 复现一个少样本多模态提示结构（3个图像-描述示例，然后一个查询图像）。

## 问题

BLIP-2 将 32 个视觉标记输入到冻结的 LLM 的输入层。这适用于每个提示一张图像的情况。但如果想输入 *多张* 与文本交错的图像，例如“这是图像A，描述它；这是图像B，描述它；现在这是图像C，描述它”呢？LLM 的自注意力将需要在单个流中处理图像标记和文本标记，并且哪些位置可以关注哪些图像的问题会变得很棘手。

Flamingo 的解决方案：完全不要改变 LLM 的输入流。在现有的 LLM 块之间插入额外的交叉注意力层。文本标记照常通过 LLM 的因果自注意力。每隔几个 LLM 块，文本标记还会通过一个新的门控层交叉关注图像特征。门控（初始化为零）意味着在初始步骤，新层是空操作（no-ops）——模型的行为与预训练的 LLM 完全相同。随着训练的进行，门控打开，视觉信息开始流动。

Flamingo 回答的第二个问题是：如何处理每个提示中可变数量的图像（0、1 或多张）？一个感知器重采样器——一个小的交叉注意力模块，它接受任意数量的块并产生固定数量的视觉潜在标记。无论提示中有多少张图像，LLM 交叉注意力层看到的形状都是相同的。

## 概念

### 冻结的 LLM

Flamingo 从一个冻结的 Chinchilla 70B LLM 开始。所有 70B 权重保持不变。现有的文本自注意力和前馈神经网络正常运作。

### 感知器重采样器

对于提示中的每张图像，ViT 产生 N 个块标记。感知器重采样器有 K 个固定的可学习潜在标记（Flamingo 使用 K=64）。每个重采样器块有两个子步骤：

1. 交叉注意力：K 个潜在标记关注 N 个块标记（查询来自潜在标记，键/值来自块标记）。
2. 潜在标记内部的自我注意力 + 前馈神经网络。

经过 6 个重采样器块后，输出是 K=64 个维度为 1024 的视觉标记，无论 ViT 产生了多少个块。一张 224x224 的图像（196 个块）和一张 480x480 的图像（900 个块）都作为 64 个重采样器标记退出。

对于视频，重采样器在时间维度上应用：每一帧的块产生 64 个潜在标记，时间位置编码使模型能够区分 t=0 与 t=N。整个视频变成 T * 64 个视觉标记。

### 门控交叉注意力

在冻结 LLM 的每隔 M 层（Flamingo 使用 M=4），插入一个新的门控交叉注意力块：

```
x_after_llm_block = llm_block(x_before)
cross = cross_attn(x_after, resampler_output)
gated = tanh(alpha) * cross + x_after
x_before_next_block = gated
```

- `alpha` 是一个可学习的标量，初始化为零。
- `tanh(0) = 0`，因此在初始化时门控分支贡献为零。
- 随着 `alpha` 偏离零，交叉注意力的贡献平滑增长。
- 残差连接意味着即使门控完全打开，也不会覆盖 LLM 的文本表示；它只是在之上添加了视觉信息。

这是 Flamingo 中最重要的设计选择：视觉条件化是加性的、门控的，并且在初始化时为零。第 0 步的 Flamingo 在纯文本输入上是一个完美的 Chinchilla 70B。

### 用于交错输入的掩码交叉注意力

在像 `<图像A> 描述A <图像B> 描述B <图像C> ?` 这样的提示中，每个文本标记应该只看到序列中位于它之前的图像。交叉注意力掩码强制执行：位置 `t` 的文本标记只关注图像索引 `i < i_t` 的图像重采样器标记，其中 `i_t` 是位置 `t` 之前最近的图像。“只看到前一张图像”或“看到所有前面的图像”都是有效的选择；Flamingo 选择了前者。

### 上下文少样本学习

一个 Flamingo 提示看起来像：

```
<图像1> 一张猫的照片。 <图像2> 一张狗的照片。 <图像3> 一张
```

模型看到补全模式并输出“鸟”（或图像3显示的任何内容）。没有梯度步骤。冻结的 LLM 的上下文学习能力通过门控交叉注意力传递——这是论文的要点，也是它重要的原因。

### 训练数据

Flamingo 在三个数据集上训练：

1. 多模态大规模网络（MultiModal MassiveWeb, M3W）：4300 万张带有交错图像和文本的网页，重构了阅读顺序。
2. 图像-文本对（ALIGN + LTIP）：44 亿对。
3. 视频-文本对（Video-Text Pairs, VTP）：2700 万个短视频剪辑。

OBELICS（2023）是一个开放复制的交错网络语料库，Idefics、Idefics2 和大多数开放的“类 Flamingo”模型都在其上训练。

### OpenFlamingo 和 Otter

OpenFlamingo（2023）是开放复制的版本。架构相同（感知器重采样器 + 冻结的 LLaMA 或 MPT 上的门控交叉注意力）。检查点有 3B、4B、9B 大小。由于基础 LLM 较小且数据较少，质量落后于 Flamingo。

Otter（2023）在 OpenFlamingo 的基础上进行指令微调，使用 MIMIC-IT（一个多模态指令数据集），表明门控交叉注意力也适用于指令跟随。

### 后续发展

- Idefics / Idefics2 / Idefics3：Hugging Face 的门控交叉注意力系列，逐步简化（Idefics2 放弃了重采样器，改用带有自适应池化的直接块标记）。
- Flamingo 到 Chameleon 的过渡：到 2024 年，许多团队转向早期融合（第 12.11 课）；在需要冻结骨干网络的生产环境中，Flamingo 风格的门控交叉注意力仍然在使用。
- Gemini 的交错输入：概念上继承了 Flamingo 交格式的灵活性，尽管具体机制是专有的。

### 与 BLIP-2 的比较

| 特性 | BLIP-2 | Flamingo |
|---|---|---|
| 视觉桥接 | 输入处一次 Q-Former | 每隔 M 层一次门控交叉注意力 |
| 视觉标记数 | 每张图像 32 个 | 每张图像每个交叉注意力层 64 个 |
| 冻结 LLM | 是 | 是 |
| 少样本上下文学习 | 弱 | 强——论文的核心 |
| 交错输入 | 原生不支持 | 是，设计目标 |
| 训练数据 | 1.3 亿对 | 13 亿对 + 4300 万交错页面 |
| 参数量 | 1.88 亿训练 | 约 100 亿训练（交叉注意力层） |
| 计算资源 | 8 张 A100 数天 | 数千张 TPUv4 数周 |

预算有限时，为单图像 VQA 选择 BLIP-2。需要交错、少样本或多图像推理时选择 Flamingo/Idefics2。

## 使用它

`code/main.py` 演示了：

1. 一个感知器重采样器，输入 36 个假块标记，有 8 个可学习潜在标记（纯 Python 交叉注意力）。
2. 一个门控交叉注意力步骤，`alpha = 0` → 输出等于输入（LLM 不变），然后 `alpha = 2.0` → 视觉贡献混合进去。
3. 一个交错掩码构建器，为 "(图像 1) (文本 1) (图像 2) (文本 2)" 序列生成 2D 注意力掩码。

## 交付它

本课生成 `outputs/skill-gated-bridge-diagnostic.md`。给定一个开放 VLM 的配置（是否有重采样器、交叉注意力频率、门控方案），它识别 Flamingo 系列元素并解释冻结策略。对于调试为什么微调会降低文本性能很有用（答案：门控开得太快太宽）。

## 练习

1. 计算 Flamingo-9B 的视觉参数数量：9B LLM + 1.4B 门控交叉注意力层 + 64M 重采样器。训练的参数占总参数的多少比例？

2. 在 PyTorch 中实现门控残差 `y = tanh(alpha) * cross + x`。通过实验证明当 `alpha=0` 时，初始化时 `y==x` 严格成立。

3. 阅读 OpenFlamingo 第 3.2 节（arXiv:2308.01390），了解当每个提示的图像数量不同时，它们如何处理一个批处理中的多张图像。描述填充策略。

4. 为什么 Flamingo 的交叉注意力掩码让文本标记只关注 *最前面的* 前一张图像，而不是所有前面的图像？阅读 Flamingo 论文第 2.4 节并解释折衷。

5. 上下文少样本：为一个新的 Flamingo 变体构建一个包含 4 个示例的提示，示例格式为“图像 → 主要物体颜色”。描述将示例数量从 0 变为 8 时预期的准确率模式。

## 关键术语

| 术语 | 通常说法 | 实际含义 |
|------|----------|----------|
| 感知器重采样器 | “固定潜在交叉注意力” | 从可变数量的输入块中产生 K 个固定标记的模块 |
| 门控交叉注意力 | “Tanh 门控桥” | 残差层 `y = tanh(alpha)*cross + x`，可学习 alpha，初始化为 0 |
| 交错输入 | “混合序列” | 按阅读顺序自由混合图像和文本的提示格式 |
| 冻结 LLM | “无 LLM 梯度” | 文本 LLM 的权重不更新；仅训练重采样器 + 交叉注意力层 |
| 少样本 | “上下文示例” | 在提示中给出几个（图像，答案）对；模型无需微调即可泛化 |
| OBELICS | “交错网络语料库” | 包含 1.41 亿网页的开放数据集，图像和文本按阅读顺序排列 |
| Chinchilla | “70B 冻结基础模型” | Flamingo 的冻结文本 LLM，来自 DeepMind 的 Chinchilla 论文 |
| 门控调度 | “alpha 如何变化” | 训练期间交叉注意力门控打开的速度 |
| 交叉注意力频率 | “每隔 M 层” | 每隔多少层插入一个门控交叉注意力块；Flamingo 使用 M=4 |
| OpenFlamingo | “开放复制” | MosaicML/LAION 的开放检查点，3-9B；架构与 Flamingo 相同 |
