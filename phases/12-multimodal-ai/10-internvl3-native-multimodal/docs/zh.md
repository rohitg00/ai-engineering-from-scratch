# InternVL3：原生多模态预训练

> 在 InternVL3 之前，所有开放视觉语言模型（VLM）都遵循相同的三步方案：先在大规模文本令牌上进行预训练的文本 LLM，再外挂一个视觉编码器，最后微调接缝处。这种方法可行，但存在对齐欠债（alignment debt）——文本 LLM 将整个预训练预算都花在了纯文本上，本身并不理解视觉令牌。当事后添加视觉时，LLM 必须重新学习如何将视觉输入与文本推理联系起来，同时还不能忘记文本。InternVL3（Zhu 等人，2025 年 4 月）拒绝了这种事后做法：一次预训练运行，从一开始就将文本和多模态交织在一起。最终结果在 MMMU-Pro 上以 78B 参数开源模型匹配了 Gemini 2.5 Pro。本节内容将剖析原生预训练的理由以及做出这一改变后带来的差异。

**类型：** 学习  
**语言：** Python（标准库，训练语料混合器）  
**前提条件：** 阶段12 · 05，阶段12 · 07（方案）  
**时间：** ≈120分钟

## 学习目标

- 解释为什么事后 VLM 训练会累积对齐欠债，并列举三个可测量的症状（灾难性遗忘、答案漂移、视觉-文本不一致）。
- 描述 InternVL3 的原生预训练语料混合比例，以及为什么文本：交织：描述的比例很重要。
- 比较 V2PE（可变视觉位置编码）与 Qwen2-VL 的 M-RoPE。
- 说出视觉分辨率路由器（ViR）和解耦视觉-语言部署（DvD）的部署优化方法。

## 问题

事后 VLM 训练是默认做法。LLaVA、BLIP-2、Qwen-VL、Idefics 等模型都采用了一种已经预训练好的 LLM（如 Llama、Vicuna、Qwen、Mistral），然后添加视觉能力。训练阶段通常如下：

1. 冻结 LLM + 冻结视觉编码器 + 可训练投影器，在描述对（caption pairs）上训练以对齐嵌入。
2. 解冻 LLM，在指令数据（如 LLaVA-Instruct、ShareGPT4V）上训练。
3. 可选的特定任务微调。

对齐欠债的三个症状显现出来：

- **灾难性遗忘**。事后 VLM 忘记了纯文本能力。GSM8K 分数下降 5-10 分。HellaSwag 分数下降。纯文本智能体出现退化。
- **答案漂移**。对同一视觉问题的不同细微措辞会得到不同答案。视觉编码器与 LLM 之间的绑定比 LLM 自身令牌之间的绑定更弱。
- **视觉-文本不一致**。VLM 能够正确描述图像，然后回答一个与其描述相矛盾的问题。视觉令牌没有像文本令牌那样参与到 LLM 的内部一致性检查中。

这些症状有充分文献记载。MM1.5 第4节对其进行了量化。LLaVA-OneVision 的消融实验也暗示了这些症状。原生预训练就是答案。

## 概念

### 原生多模态预训练

InternVL3 从零开始训练，其语料从一开始就是原生多模态的。混合比例为：

- 40% 纯文本数据（FineWeb、Proof-Pile-2 等）
- 35% 交织图像-文本数据（OBELICS、类似 MMC4 的风格）
- 20% 成对图像-描述数据
- 5% 视频-文本数据

视觉令牌、文本令牌和跨模态交互从一开始就参与了相同的损失函数。没有对齐预训练，没有投影器冻结阶段，也没有需要从中恢复的灾难性遗忘。

对于基础模型，训练只有单一阶段。之后进行指令微调，但基础模型已经将视觉令牌作为一等公民来理解。

### V2PE（可变视觉位置编码）

Qwen2-VL 使用具有固定轴分配的 M-RoPE。InternVL3 引入了 V2PE：位置编码根据模态类型（文本、图像、视频）变化，并带有可学习的缩放。在实践中：

- 文本令牌获得一维位置（文本索引）。
- 图像块获得二维位置（行，列）。
- 视频帧获得三维位置（时间，行，列）。

三者共享相同的 RoPE 频率基，但每个频带（band）的隐藏维度分配是学习的参数，而不是固定的分割。这使得在预训练过程中可以在时间分辨率与空间频率分辨率之间自由权衡。

V2PE 的消融实验声称：在相同计算量下，相比 M-RoPE，在视频基准上提升 1-2 个百分点。并非革命性变化，但更加干净。

### 视觉分辨率路由器（ViR）

部署优化。并非所有图像都需要全分辨率编码。一张包含单个物体且细节较少的照片，如果以 1280px 原生分辨率编码会浪费大量令牌。ViR 是一个小型分类器，在编码前预测回答问题所需的最小分辨率。

路由分为三个层级：低分辨率（256 令牌）、中分辨率（576）、高分辨率（2048+）。在生产流量中，60% 的查询只需要低或中分辨率。净效果：在同等质量下吞吐量提升 2-3 倍。

### 解耦视觉-语言部署（DvD）

当服务一个大 VLM 时，视觉编码器每张图像运行一次，但 LLM 需要为每个输出令牌进行自回归计算。这两个组件的瓶颈不同（视觉 = 卷积和注意力所需的 GPU 内存带宽；LLM = KV 缓存）。DvD 将它们分开部署到不同的 GPU 上，并在它们之间进行流式传输。

对于 8B + 400M 编码器的模型，DvD 相比共置部署，每个节点的吞吐量大约翻倍。

### 单阶段 vs 多阶段质量

InternVL3 的主要基准宣称：在 78B 参数下，匹配 Gemini 2.5 Pro 的 MMMU-Pro 成绩。在 38B 下，匹配 GPT-4o。在 8B 下，领跑开放 8B 排行榜。所有成绩均基于单阶段预训练加指令微调方案。

对齐欠债假设是可测量的：InternVL3-8B 在文本基准（MMLU、GSM8K）上的分数损失少于 Qwen2.5-VL-7B 每单位视觉基准提升对应的损失。该模型更具通才特性，因为训练是一个整体，而不是两个部分。

### InternVL3.5 与 InternVL-U

InternVL3.5（2025 年 8 月）扩展了该方案。相同的原生预训练方法，更多的数据，更多的参数。MMMU 的提升是渐进的。

InternVL-U（2026）增加了统一生成——通过在同一主干上添加 MMDiT 头实现图像输出。“U”代表“理解+生成”（Understanding + generation），追赶 Transfusion 风格的统一模型（第 12.13 课）。相同的原生预训练主干同时支持理解头和生成头。

### 原生预训练的权衡

原生预训练并非没有代价：

- **计算成本**。从头训练一个新 VLM 的成本与训练一个文本 LLM 相当——数百万 GPU 小时。事后适配则重复使用现有的 LLM 权重，节省了大部分成本。
- **数据**。大规模的交织图像-文本语料很稀缺。OBELICS 有 1.41 亿文档；MMC4 有 5.71 亿。纯文本则有 15 万亿令牌。多模态预训练数据的稀缺是一个硬约束。
- **基础 LLM 复用**。原生预训练放弃了稍后更换新 LLM 的选项。事后适配可以通过仅重新训练适配器，将 Llama-3.1 替换为 Llama-4。

InternVL3 的赌注是：对齐欠债比复用损失更严重。基准数据支持这一说法。高昂的生产成本阻碍了其他实验室低成本复制。事后 VLM 将继续存在，因为对大多数项目来说它们仍然更便宜。

## 使用它

`code/main.py` 是一个训练语料混合器和 ViR 路由器模拟器。它：

- 接受目标语料混合比例（%文本、%交织、%描述、%视频），并计算每种模态的预期步数。
- 模拟一批查询上的 ViR 路由（分布：50% 低细节，30% 中细节，20% 高细节），并报告平均令牌数。
- 根据编码器与 LLM 的 FLOPs 报告 DvD 吞吐量估算。
- 以参数、计算量、数据和预期对齐欠债症状等维度，并排对比事后与原生预训练。

## 交付

本课程产出 `outputs/skill-native-vs-posthoc-auditor.md`。给定一个提议的 VLM 训练计划，它会审计是选择原生还是事后方案，标记对齐欠债风险，并推荐语料混合比例。当你正在规划一个新的开放 VLM 项目并需要选择训练策略时使用它。

## 练习

1. 估算 InternVL3-8B（原生预训练）与 LLaVA-OneVision-7B（事后）之间的计算量差异。GPU 小时数的比例大致是多少？是什么导致了这一差距？

2. InternVL3 报告了 40% 文本 / 35% 交织 / 20% 描述 / 5% 视频。如果你的目标任务偏重视频，请提出一个新的比例，并论证为什么基础模型仍然需要大量的文本和描述数据。

3. 阅读 MM1.5 第4节关于遗忘的内容。说出事后训练表现出最大退化的确切基准。退化程度是多少？

4. ViR 将 60% 的流量路由到低分辨率编码。它会在哪些查询上误路由（即在需要高分辨率时发送到低分辨率）？提出三种路由器失效模式。

5. DvD 将视觉和 LLM 分别部署到不同的 GPU 上。在哪种流量模式下，DvD 反而会降低吞吐量而不是提升？

## 关键术语

| 术语 | 通常说法 | 实际含义 |
|------|---------|----------|
| 原生多模态预训练 | “从头一起训练” | 文本、图像和视频令牌从第一步起就参与损失函数，而非后续外挂 |
| 对齐欠债 | “事后惩罚” | 外挂视觉到冻结 LLM 上导致的可测量的文本技能退化和答案一致性下降 |
| V2PE | “可变视觉位置编码” | 每种模态可学习的位置编码分配；InternVL3 对 M-RoPE 的继承者 |
| ViR | “分辨率路由器” | 一个小型分类器，在编码前选择每个查询所需的最小分辨率，从而节省推理令牌 |
| DvD | “解耦部署” | 视觉编码器在一个 GPU 上，LLM 在另一个 GPU 上，通过流式传输交接；对大型 VLM 可提升吞吐量 2 倍 |
| InternVL-U | “统一理解+生成” | 2026 年的后续工作，在原生的预训练主干上增加图像生成头 |
| 交织语料 | “OBELICS / MMC4” | 文本和图像按自然阅读顺序排列的文档；原生预训练的原材料 |
