# InternVL3：原生多模态预训练

> 在 InternVL3 之前的每个开源 VLM 都遵循相同的三步配方：取一个在数万亿文本 token 上训练的文本 LLM，加装一个视觉编码器，然后微调接缝。这有效但会产生对齐债务——文本 LLM 将其完整的预训练预算花在了纯文本上，并不原生理解视觉 token。当你事后添加视觉时，LLM 必须重新学习如何将视觉输入与其文本推理关联起来，同时不遗忘文本。InternVL3（Zhu 等人，2025 年 4 月）拒绝了事后方法：一次预训练运行，文本和多模态从第一步就交错进行。结果在 78B 参数开源模型上匹配 Gemini 2.5 Pro 的 MMMU-Pro。本课解读原生预训练的案例以及当你采用它时会发生什么变化。

**类型：** Learn
**语言：** Python（stdlib，训练语料混合器）
**前置知识：** Phase 12 · 05，Phase 12 · 07（配方）
**时间：** ~120 分钟

## 学习目标

- 解释为什么事后 VLM 训练会积累对齐债务，引用三个可测量症状（灾难性遗忘、答案漂移、视觉-文本不一致）。
- 描述 InternVL3 的原生预训练语料混合以及为什么文本 : 交错 : 描述的比例很重要。
- 将 V2PE（可变视觉位置编码）与 Qwen2-VL 的 M-RoPE 进行比较。
- 命名视觉分辨率路由器（ViR）和解耦视觉-语言（DvD）部署优化。

## 问题所在

事后 VLM 训练是默认做法。LLaVA、BLIP-2、Qwen-VL、Idefics——都取一个已经预训练的 LLM（Llama、Vicuna、Qwen、Mistral）并添加视觉。训练阶段通常如下：

1. 冻结 LLM + 冻结视觉编码器 + 可训练投影器，在描述对上训练以对齐嵌入。
2. 解冻 LLM，在指令数据（LLaVA-Instruct、ShareGPT4V）上训练。
3. 可选任务特定微调。

三个对齐债务症状出现：

- 灾难性遗忘。事后 VLM 遗忘纯文本技能。GSM8K 分数下降 5-10 点。Hellaswag 分数下降。纯文本代理退化。
- 答案漂移。同一视觉问题的微小措辞变化得到不同答案。视觉编码器与 LLM 的连接比 LLM 自己的 token 更弱。
- 视觉-文本不一致。VLM 可以正确描述图像，然后回答与其自身描述矛盾的问题。视觉 token 不像文本那样参与 LLM 的内部一致性检查。

这些症状有充分记录。MM1.5 第 4 节对它们进行了量化。LLaVA-OneVision 的消融暗示了它们。原生预训练是答案。

## 核心概念

### 原生多模态预训练

InternVL3 从第一步就在原生多模态语料上从头训练。混合比例是：

- 40% 纯文本数据（FineWeb、Proof-Pile-2 等）
- 35% 交错图像-文本数据（OBELICS、MMC4 风格）
- 20% 成对图像-描述数据
- 5% 视频-文本数据

视觉 token、文本 token 和跨模态交互都从第一个梯度步骤参与相同的损失。没有对齐预训练，没有投影器冻结阶段，没有需要恢复的灾难性遗忘。

基础模型是单阶段训练。指令微调随后进行，但基础模型已经将视觉 token 理解为一等公民。

### V2PE（可变视觉位置编码）

Qwen2-VL 使用固定轴分配的 M-RoPE。InternVL3 引入 V2PE：位置编码按模态类型（文本、图像、视频）变化，带有可学习缩放。实践中：

- 文本 token 获得 1D 位置（文本索引）。
- 图像 patch 获得 2D 位置（行、列）。
- 视频帧获得 3D 位置（时间、行、列）。

三者共享相同的 RoPE 频率基础，但每频带的隐藏维度分配是可学习参数而非固定分割。在预训练期间自由权衡时间 vs 空间频率分辨率。

V2PE 的消融声明：在相同计算下，视频基准上比 M-RoPE 高 1-2 点。不是革命，但更干净。

### 视觉分辨率路由器（ViR）

部署优化。并非所有图像都需要全分辨率编码。一个低细节单物体的照片在 1280px 原生编码时浪费 token。ViR 是一个小型分类器，在编码前预测回答问题所需的最小分辨率。

路由有三个层级：低分辨率（256 token）、中分辨率（576）、高分辨率（2048+）。在生产流量的 60% 查询中，低或中分辨率足够。净效果：在同等质量下 2-3 倍吞吐量。

### 解耦视觉-语言部署（DvD）

当你服务大型 VLM 时，视觉编码器每张图像运行一次，但 LLM 为每个输出 token 自回归运行。两个组件有不同的瓶颈（视觉 = conv + 注意力的 GPU 内存带宽；LLM = KV 缓存）。DvD 将它们拆分到不同 GPU 上，之间流式传输。

对于 8B + 400M 编码器模型，DvD 相比共置部署大致将每节点吞吐量翻倍。

### 单阶段 vs 多阶段质量

InternVL3 的主要基准声明：78B 参数下匹配 Gemini 2.5 Pro 的 MMMU-Pro。38B 下匹配 GPT-4o。8B 下领先开源 8B 排行榜。全部基于单阶段预训练 + 指令微调配方。

对齐债务假设是可测量的：InternVL3-8B 每单位视觉基准增益损失的文本基准分数（MMLU、GSM8K）比 Qwen2.5-VL-7B 少。该模型更像通才，因为训练是一体的，而非两阶段。

### InternVL3.5 和 InternVL-U

InternVL3.5（2025 年 8 月）扩展了配方。相同的原生预训练方法，更多数据，更多参数。MMMU 改进是增量式的。

InternVL-U（2026）添加统一生成——通过同一骨干上的 MMDiT 头进行图像输出。"U"代表"理解 + 生成"，追逐 Transfusion 风格的统一模型（第 12.13 课）。相同的原生预训练骨干支持理解和生成头。

### 原生预训练的权衡

原生预训练并非免费：

- 计算。从头训练新 VLM 的成本与训练文本 LLM 相同——数百万 GPU 小时。事后适配重用现有 LLM 权重，节省大部分成本。
- 数据。大规模交错图像-文本语料很少。OBELICS 是 1.41 亿文档；MMC4 是 5.71 亿。纯文本以 15T token 传输。多模态预训练数据稀缺是一个硬约束。
- 基础 LLM 重用。原生预训练放弃了以后更换新 LLM 的选项。事后方法允许你通过仅重新训练适配器将 Llama-3.1 换成 Llama-4。

InternVL3 的赌注：对齐债务比重用损失更糟。基准支持这一声明。生产成本阻碍了未来实验室廉价复制。事后 VLM 将继续存在，因为它们对大多数项目来说仍然更便宜。

## 使用它

`code/main.py` 是训练语料混合器和 ViR 路由器模拟器。它：

- 获取目标语料混合（%文本、%交错、%描述、%视频）并计算每模态预期步数。
- 在批量查询上模拟 ViR 路由（分布：50% 低细节、30% 中等、20% 高细节）并报告平均 token 数量。
- 给定编码器 vs LLM FLOPs，报告 DvD 吞吐量估计。
- 打印事后 vs 原生预训练在参数、计算、数据和预期对齐债务症状方面的并排对比。

## 交付它

本课产出 `outputs/skill-native-vs-posthoc-auditor.md`。给定提议的 VLM 训练计划，它审计是走原生还是事后路线，标记对齐债务风险，并推荐语料混合。在为新的开源 VLM 项目调整大小时使用它，需要选择训练策略。

## 练习

1. 估计 InternVL3-8B（原生预训练）和 LLaVA-OneVision-7B（事后）之间的计算差异。GPU 小时比率大约是多少？什么解释了差距？

2. InternVL3 报告 40% 文本 / 35% 交错 / 20% 描述 / 5% 视频。如果你的目标任务视频密集型，提出新比例并论证为什么基础模型仍然需要大量文本和描述数据。

3. 阅读 MM1.5 第 4 节关于遗忘。命名事后训练显示最大回归的确切基准。回归代价多少？

4. ViR 将 60% 流量路由到低分辨率编码。它误路由了哪些查询（发送到低分辨率时需要高分辨率）？提出三种路由器故障模式。

5. DvD 将视觉和 LLM 拆分到不同 GPU。在什么流量模式下 DvD 损害而非提升吞吐量？

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------------|------------------------|
| 原生多模态预训练 | "从头一起训练" | 文本 + 图像 + 视频 token 从第 1 步就参与损失，而非后来加装 |
| 对齐债务 | "事后惩罚" | 将视觉加装到冻结 LLM 上带来的文本技能和答案一致性的可测量回归 |
| V2PE | "可变视觉位置编码" | 每模态可学习位置编码分配；InternVL3 的 M-RoPE 继任者 |
| ViR | "分辨率路由器" | 在编码前为每个查询选择最小分辨率的小型分类器，节省推理 token |
| DvD | "解耦部署" | 视觉编码器在一个 GPU 上，LLM 在另一个上，带流式切换；大型 VLM 的吞吐量翻倍 |
| InternVL-U | "统一理解 + 生成" | 2026 年后续，向原生预训练骨干添加图像生成头 |
| 交错语料 | "OBELICS / MMC4" | 自然阅读顺序中带文本和图像的文档；原生预训练的原材料 |
