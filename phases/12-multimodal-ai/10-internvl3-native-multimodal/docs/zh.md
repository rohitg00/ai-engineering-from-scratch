# 10 · InternVL3：原生多模态预训练

> 在 InternVL3 之前，几乎每一个开源视觉语言模型（VLM）都遵循同一套三步配方：拿来一个已经在数万亿文本 token 上训练好的文本大语言模型（LLM），外接一个视觉编码器，再对接缝处做微调。这套方法行得通，但欠下了「对齐债（alignment debt）」——文本 LLM 已经把全部预训练预算花在了纯文本上，并不能原生理解视觉 token。当你事后再加上视觉时，LLM 必须在不遗忘文本的前提下，重新学会如何把视觉输入与它的文本推理关联起来。InternVL3（Zhu 等人，2025 年 4 月）摒弃了这种事后拼接（post-hoc）的做法：一次预训练，从第一步起文本与多模态就交错进行。其结果是：在 78B 参数的开源规模上，于 MMMU-Pro 基准上与 Gemini 2.5 Pro 持平。本课将解读原生预训练（native pretraining）的论据，以及采用它之后会发生哪些变化。

**类型：** 学习
**语言：** Python（标准库，训练语料混合器）
**前置：** 阶段 12 · 05，阶段 12 · 07（配方）
**时长：** 约 120 分钟

## 学习目标

- 解释为什么事后 VLM 训练会累积对齐债，并列举三个可量化的症状（灾难性遗忘、答案漂移、视觉-文本不一致）。
- 描述 InternVL3 的原生预训练语料配比，以及「文本 : 交错 : 字幕」的比例为何重要。
- 将 V2PE（可变视觉位置编码，variable visual position encoding）与 Qwen2-VL 的 M-RoPE 做对比。
- 说出视觉分辨率路由器（Visual Resolution Router, ViR）与视觉-语言解耦部署（Decoupled Vision-Language, DvD）这两项部署优化。

## 问题

事后 VLM 训练是默认做法。LLaVA、BLIP-2、Qwen-VL、Idefics——它们全都拿一个已经预训练好的 LLM（Llama、Vicuna、Qwen、Mistral）再加上视觉。其训练阶段通常如下：

1. 冻结的 LLM + 冻结的视觉编码器 + 可训练的投影器（projector），在字幕配对数据上训练以对齐嵌入。
2. 解冻 LLM，在指令数据（LLaVA-Instruct、ShareGPT4V）上训练。
3. 可选的特定任务微调。

对齐债会表现出三个症状：

- 灾难性遗忘（catastrophic forgetting）。事后 VLM 会遗忘纯文本技能。GSM8K 分数下降 5-10 分。Hellaswag 分数下降。纯文本智能体出现退化。
- 答案漂移（answer drift）。同一个视觉问题的不同措辞会得到不同的答案。视觉编码器与 LLM 的绑定弱于 LLM 自身的 token。
- 视觉-文本不一致（visual-text inconsistency）。VLM 能够正确描述一张图像，却又给出与自己描述相矛盾的回答。视觉 token 不像文本那样参与 LLM 内部的一致性校验。

这些症状已有充分的文献记载。MM1.5 第 4 节对其进行了量化。LLaVA-OneVision 的消融实验也暗示了它们。原生预训练正是答案。

## 概念

### 原生多模态预训练

InternVL3 在一个从第一步起就原生多模态的语料上从零训练。其配比为：

- 40% 纯文本数据（FineWeb、Proof-Pile-2 等）
- 35% 交错图文数据（OBELICS、MMC4 风格）
- 20% 成对图像-字幕数据
- 5% 视频-文本数据

从第一步梯度起，视觉 token、文本 token 以及跨模态交互就全部参与同一个损失。没有对齐预训练，没有投影器冻结阶段，也没有需要去恢复的灾难性遗忘。

基础模型的训练是单一阶段的。之后会进行指令微调，但基础模型早已把视觉 token 当作一等公民来理解。

### V2PE（可变视觉位置编码）

Qwen2-VL 使用带固定轴分配的 M-RoPE。InternVL3 引入了 V2PE：位置编码按模态类型（文本、图像、视频）变化，并带有可学习的缩放。具体来说：

- 文本 token 获得 1D 位置（文本索引）。
- 图像 patch 获得 2D 位置（行、列）。
- 视频帧获得 3D 位置（时间、行、列）。

三者共享同一个 RoPE 频率基（frequency base），但每个频段（band）的隐藏维度分配是一个可学习参数，而非固定切分。这赋予了在预训练期间自由权衡时间频率与空间频率分辨率的能力。

V2PE 的消融实验论断：在相同算力下，视频基准上相比 M-RoPE 高出 1-2 分。算不上革命，但更干净。

### 视觉分辨率路由器（ViR）

部署优化。并非所有图像都需要全分辨率编码。一张只有一个低细节物体的照片，若以 1280px 原生分辨率编码就浪费了 token。ViR 是一个小型分类器，在编码之前预测回答问题所需的最低分辨率。

路由分三档：低分辨率（256 token）、中分辨率（576）、高分辨率（2048+）。对生产流量中 60% 的查询而言，低档或中档就足够了。净效果：在同等质量下吞吐量提升 2-3 倍。

### 视觉-语言解耦部署（DvD）

当你服务一个大型 VLM 时，视觉编码器每张图像只运行一次，而 LLM 则要为每个输出 token 自回归地运行。这两个组件有着不同的瓶颈（视觉 = 卷积与注意力的 GPU 显存带宽；LLM = KV 缓存）。DvD 把它们拆分到不同的 GPU 上，并在两者之间进行流式传输。

对于一个 8B + 400M 编码器的模型，相比共置部署，DvD 大致能把单节点吞吐量翻一倍。

### 单阶段与多阶段的质量对比

InternVL3 的主要基准论断：在 78B 参数下，于 MMMU-Pro 上与 Gemini 2.5 Pro 持平。在 38B 下，与 GPT-4o 持平。在 8B 下，领跑开源 8B 排行榜。这些全部基于单阶段预训练 + 指令微调的配方。

对齐债假说是可量化的：每获得一个单位的视觉基准提升，InternVL3-8B 损失的文本基准分数（MMLU、GSM8K）少于 Qwen2.5-VL-7B。该模型更像一个通才，因为它的训练是一个整体，而非两段拼接。

### InternVL3.5 与 InternVL-U

InternVL3.5（2025 年 8 月）扩展了这套配方。同样的原生预训练方法，更多数据，更多参数。MMMU 的提升是渐进式的。

InternVL-U（2026）加入了统一生成——在同一骨干网络之上通过 MMDiT 头实现图像输出。「U」代表「Understanding + generation（理解 + 生成）」，追逐 Transfusion 风格的统一模型（第 12.13 课）。同一个原生预训练骨干同时支持理解头与生成头。

### 原生预训练的权衡

原生预训练并非没有代价：

- 算力。从零训练一个全新的 VLM 与训练一个文本 LLM 成本相当——数百万 GPU 小时。事后适配复用现有的 LLM 权重，省下了大部分成本。
- 数据。大规模的交错图文语料很稀缺。OBELICS 有 1.41 亿篇文档；MMC4 有 5.71 亿篇。而纯文本动辄 15T token。多模态预训练数据的稀缺是一个硬约束。
- 基础 LLM 复用。原生预训练放弃了日后替换新 LLM 的选项。事后方法则让你只需重训适配器（adapter）就能把 Llama-3.1 换成 Llama-4。

InternVL3 押下的赌注是：对齐债比放弃复用带来的损失更糟。基准结果支撑了这一论断。其生产成本之高，让未来的实验室难以廉价复制。事后 VLM 仍会继续存在，因为对大多数项目而言它们依然更便宜。

## 动手用

`code/main.py` 是一个训练语料混合器兼 ViR 路由器模拟器。它会：

- 接收一个目标语料配比（%text、%interleaved、%caption、%video），并计算各模态的预期步数。
- 在一批查询上模拟 ViR 路由（分布：50% 低细节、30% 中等、20% 高细节），并报告平均 token 数。
- 在给定编码器与 LLM 的 FLOPs 的情况下，报告 DvD 吞吐量估算。
- 在参数量、算力、数据以及预期对齐债症状这几个维度上，并排打印事后预训练与原生预训练的对比。

## 交付物

本课产出 `outputs/skill-native-vs-posthoc-auditor.md`。给定一份提议的 VLM 训练计划，它会审计应当走原生路线还是事后路线，标记对齐债风险，并推荐一个语料配比。当你在为一个新的开源 VLM 项目做规模评估、需要挑选训练策略时，可以使用它。

## 练习

1. 估算 InternVL3-8B（原生预训练）与 LLaVA-OneVision-7B（事后）之间的算力差距。GPU 小时数之比大约是多少？是什么造成了这一差距？

2. InternVL3 报告的配比为 40% 文本 / 35% 交错 / 20% 字幕 / 5% 视频。如果你的目标任务以视频为主，请提出一个新的配比，并论证为什么基础模型仍然需要大量的文本与字幕数据。

3. 阅读 MM1.5 第 4 节关于遗忘的内容。说出事后训练表现出最大退化的那个确切基准。这一退化的代价有多大？

4. ViR 把 60% 的流量路由到低分辨率编码。它会把哪些类型的查询路由错误（在需要高分辨率时却发往低分辨率）？请提出三种路由器失败模式。

5. DvD 把视觉与 LLM 拆分到不同的 GPU 上。在什么样的流量模式下，DvD 反而会损害吞吐量而非提升它？

## 关键术语

| 术语 | 人们的说法 | 实际含义 |
|------|-----------------|------------------------|
| 原生多模态预训练 | 「从零一起训练」 | 文本 + 图像 + 视频 token 从第 1 步起就参与损失，而非后期拼接 |
| 对齐债 | 「事后惩罚」 | 把视觉拼接到冻结的 LLM 上所带来的、可量化的文本技能与答案一致性退化 |
| V2PE | 「可变视觉位置编码」 | 按模态可学习的位置编码分配；InternVL3 对 M-RoPE 的继任者 |
| ViR | 「分辨率路由器」 | 在编码前为每个查询挑选所需最低分辨率的小型分类器，节省推理 token |
| DvD | 「解耦部署」 | 视觉编码器在一块 GPU 上，LLM 在另一块上，两者间流式交接；为大型 VLM 把吞吐量翻倍 |
| InternVL-U | 「统一理解 + 生成」 | 2026 年的后续工作，在原生预训练骨干上加入图像生成头 |
| 交错语料 | 「OBELICS / MMC4」 | 文本与图像按自然阅读顺序排列的文档；原生预训练的原材料 |
