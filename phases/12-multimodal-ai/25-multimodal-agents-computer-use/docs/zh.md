# 多模态代理与计算机使用（顶点）

> 2026 年前沿产品是一个多模态代理，读取屏幕截图、点击按钮、导航网页 UI、填写表单并端到端完成工作流。SeeClick 和 CogAgent（2024）证明了 GUI 接地原语。Ferret-UI 添加了移动端。ChartAgent 引入了图表的视觉工具使用。VisualWebArena 和 AgentVista（2026）是前沿追逐的基准——甚至 Gemini 3 Pro 和 Claude Opus 4.7 在 AgentVista 的困难任务上得分约 30%。这个顶点将 Phase 12 的每条线索汇聚在一起：感知（高分辨率 VLM）、推理（带工具使用的 LLM）、接地（坐标输出）、长程记忆和评估。

**类型：** Capstone
**语言：** Python（stdlib，动作模式 + 代理循环骨架）
**前置知识：** Phase 12 · 05（LLaVA），Phase 12 · 09（Qwen-VL JSON），Phase 14（代理工程）
**时间：** ~240 分钟

## 学习目标

- 设计多模态代理循环：感知 → 推理 → 行动 → 观察 → 重复。
- 构建 GUI 接地输出模式（点击坐标、输入文本、滚动、拖动），VLM 可以将其作为 JSON 发出。
- 比较仅截图代理、可访问性树代理和混合代理。
- 在小型 VisualWebArena 切片上设置多模态代理基准评估。

## 问题所在

预订网站工作流："给我找 4 月 15 日去东京的航班，过道座位，800 美元以下，预订它。"

多模态代理需要：

1. 截取浏览器屏幕截图。
2. 将屏幕截图 + URL + 目标解析为计划。
3. 发出结构化动作：点击（在 x,y）、输入"Tokyo"（在元素 E）、向下滚动、选择（单选按钮）。
4. 将动作应用于浏览器。
5. 观察新状态（下一个屏幕截图）。
6. 重复直到任务完成。

每一步都是多模态 VLM 调用。VLM 输出必须是可解析的 JSON。错误在步骤间累积，因此恢复很重要。

## 核心概念

### GUI 接地——原语

GUI 接地是：给定屏幕截图和自然语言指令，输出要点击的 (x, y) 坐标（或其他动作）。

SeeClick（arXiv:2401.10935）是首个大规模开放结果：在合成 + 真实 GUI 数据上微调 VLM，将坐标作为纯文本 token 输出。有效。

CogAgent（arXiv:2312.08914）为密集 UI 添加了 1120x1120 高分辨率编码。分数：网页导航约 84%。

Ferret-UI（arXiv:2404.05719）专注于移动 UI，与 iOS 可访问性数据集成。

输出格式通常是 JSON：

```json
{"action": "click", "x": 384, "y": 220, "element_desc": "Search button"}
```

`element_desc` 帮助恢复：如果坐标在屏幕截图间漂移，语义提示让系统重新接地。

### 动作模式

典型动作模式有 6-10 种动作类型：

- `click`: (x, y)
- `type`: (text, x?, y?)
- `scroll`: (direction, amount)
- `drag`: (x0, y0, x1, y1)
- `select`: (option_index)
- `hover`: (x, y)
- `navigate`: (url)
- `wait`: (ms)
- `done`: (success, explanation)

代理每步发出一个动作。浏览器包装器执行并返回新状态。

### 仅截图 vs 可访问性树

两种输入模式：

- 仅截图：完整图像，无结构信息。最通用；适用于任何应用。
- 可访问性树：结构化 DOM / iOS 可访问性信息。接地更可靠；在树可用的地方有效。
- 混合：两者兼有，树作为原子动作的可靠接地器，截图用于语义上下文。

生产代理在可能时使用混合。浏览器自动化（Selenium + 可访问性）总是有树；桌面应用有时有。

### 长程记忆

20 步工作流生成 20 个屏幕截图。VLM 的上下文快速填满。三种压缩策略：

- 摘要链：每 5 步后总结发生了什么，丢弃旧屏幕截图。
- 跳帧：保留第一个、最后一个和每第 3 个屏幕截图。
- 工具记录日志：执行动作，保留所做内容的文本日志；不重新查看旧屏幕截图。

Claude 的计算机使用 API 使用日志模式。更简单，更可靠。

### 视觉工具使用

ChartAgent（arXiv:2510.04514）引入了图表理解的视觉工具使用：裁剪、缩放、OCR、调用外部检测。代理可以输出"裁剪到区域 (100, 200, 300, 400) 然后调用 OCR"作为工具调用。工具返回文本；VLM 继续推理。

这种模式泛化：标记集提示、区域注释和外部检测工具都适用于相同的"输出工具调用，接收结构化响应"模式。

### 2026 年基准

- ScreenSpot-Pro。约 1k 网页屏幕截图上的 GUI 接地。开放 SOTA Qwen2.5-VL-72B 约 85%。前沿约 90%。
- VisualWebArena。端到端网页任务（购物、论坛、分类信息）。开放 SOTA 约 20%。Gemini 3 Pro 约 27%。
- AgentVista（arXiv:2602.23166）。2026 年最困难基准。跨 12 个领域的真实工作流。前沿模型得分 27-40%；开放模型 10-20%。
- WebArena / WebShop。较旧基准；被前沿饱和。

### 为什么仍然困难

代理性能瓶颈：

1. 细粒度视觉接地。"点击小 X"在移动分辨率下经常失败。
2. 长程规划。10 个动作后，代理偏离目标。
3. 错误恢复。当点击失败（错误按钮）时，检测 + 恢复很少是训练数据。
4. 跨页面上下文。在标签页或长表单间跳转丢失状态。

研究方向：记忆架构、显式重新规划、多模态验证（屏幕截图匹配动作成功）。

### 顶点构建

顶点任务：构建一个计算机使用代理，它：

1. 读取预订网站模拟页面的 HTML + 屏幕截图。
2. 规划多步序列：搜索 → 选择 → 填写表单 → 提交。
3. 发出匹配动作模式的 JSON 动作。
4. 在固定 10 任务切片上评估。

本课提供易于扩展到真实浏览器的脚手架代码。

## 使用它

`code/main.py` 是顶点脚手架：

- 动作模式 JSON 定义（10 个动作）。
- 模拟浏览器状态作为字典。
- 代理循环骨架：接收状态，发出动作，应用，循环。
- 10 任务迷你基准（合成页面）以测量端到端成功率。
- 动作失败时的错误恢复钩子。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------------|------------------------|
| GUI 接地 | "点击坐标" | 模型在屏幕截图上输出指令目标的 (x,y) |
| 动作模式 | "工具定义" | 有效动作的 JSON 描述（点击、输入、滚动、拖动） |
| 可访问性树 | "结构化 DOM" | 来自浏览器/iOS API 的机器可读 UI 层次结构 |
| 混合代理 | "截图 + 树" | 同时使用图像和结构化信息；比单独任一更可靠 |
| 视觉工具使用 | "缩放/裁剪/检测" | 代理在计划中途调用外部视觉工具（OCR、检测） |
| 摘要链 | "记忆压缩" | 定期文本摘要替换长屏幕截图历史 |
| VisualWebArena | "端到端网页基准" | 2024 年端到端网页任务基准 |
| AgentVista | "2026 年困难基准" | 12 领域真实工作流；甚至 Gemini 3 Pro 得分约 30% |
