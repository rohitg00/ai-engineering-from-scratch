# 多模态智能体与计算机使用（顶点项目）

> 2026年的前沿产品是一款多模态智能体，能够读取截图、点击按钮、导航Web界面、填写表单并端到端完成工作流。SeeClick和CogAgent（2024）证明了GUI定位（GUI grounding）这一基础能力。Ferret-UI增加了对移动端的支持。ChartAgent引入了针对图表的可视化工具使用。VisualWebArena和AgentVista（2026）是前沿模型竞相追逐的基准——即便是Gemini 3 Pro和Claude Opus 4.7在AgentVista的困难任务上也仅取得约30%的分数。本顶点项目将第12阶段的所有线索整合在一起：感知（高分辨率VLM）、推理（具备工具使用能力的LLM）、定位（坐标输出）、长期记忆以及评估。

**类型：** 顶点项目  
**语言：** Python（标准库，动作模式 + 智能体循环框架）  
**先修知识：** 阶段12 · 05（LLaVA），阶段12 · 09（Qwen-VL JSON），阶段14（智能体工程）  
**耗时：** 约240分钟

## 学习目标

- 设计多模态智能体循环：感知 → 推理 → 行动 → 观察 → 重复。
- 构建GUI定位输出模式（点击坐标、键入文本、滚动、拖拽），使VLM能够以JSON格式输出。
- 对比纯截图智能体、无障碍树智能体以及混合智能体。
- 在VisualWebArena的小规模子集上搭建多模态智能体基准评估。

## 问题描述

一个预订网站的工作流：“帮我找4月15日去东京的航班，靠过道座位，价格低于800美元，并完成预订。”

多模态智能体需要：

1. 对浏览器进行截图。
2. 将截图、URL和目标解析为一个计划。
3. 输出结构化动作：点击（在x,y位置）、键入“Tokyo”（在元素E上）、向下滚动、选择（单选按钮）。
4. 在浏览器中执行该动作。
5. 观察新状态（下一张截图）。
6. 重复直到任务完成。

每一步都是一次多模态VLM调用。VLM的输出必须是可解析的JSON。错误会随着步骤累积，因此恢复能力至关重要。

## 核心概念

### GUI定位——基础能力

GUI定位指的是：给定一张截图和一条自然语言指令，输出要点击的（x，y）坐标（或其他动作）。

SeeClick（arXiv:2401.10935）是首个大规模开源成果：通过在合成+真实GUI数据上微调VLM，将坐标作为普通文本令牌输出。方法可行。

CogAgent（arXiv:2312.08914）引入了1120x1120的高分辨率编码，用于处理密集UI。评分：约84%的网页导航成功率。

Ferret-UI（arXiv:2404.05719）专注于移动端UI，并与iOS无障碍数据集成。

输出格式通常为JSON：

```json
{"action": "click", "x": 384, "y": 220, "element_desc": "Search button"}
```

`element_desc`有助于恢复：如果截图之间坐标发生偏移，语义提示可让系统重新定位。

### 动作模式（Action Schema）

典型的动作模式包含6-10种动作类型：

- `click`：（x, y）
- `type`：（text, x?, y?）
- `scroll`：（direction, amount）
- `drag`：（x0, y0, x1, y1）
- `select`：（option_index）
- `hover`：（x, y）
- `navigate`：（url）
- `wait`：（ms）
- `done`：（success, explanation）

智能体每一步发出一个动作。浏览器包装器执行该动作并返回新状态。

### 纯截图 vs 无障碍树

两种输入模式：

- 纯截图：完整图像，无结构信息。最通用；适用于任何应用。
- 无障碍树：结构化DOM / iOS无障碍信息。对于定位而言更可靠；适用于可以获取树的场景。
- 混合模式：两者结合，以树作为原子动作的可靠定位器，以截图作为语义上下文。

生产环境中的智能体在可能的情况下使用混合模式。浏览器自动化（Selenium + 无障碍）始终能获取树；桌面应用有时也能获取。

### 长期记忆

一个20步的工作流会产生20张截图。VLM的上下文会迅速填满。三种压缩策略：

- 摘要链（Summary-chain）：每5步后总结已发生的事情，丢弃旧截图。
- 跳帧（Skip-frame）：保留第一张、最后一张以及每第三张截图。
- 工具记录日志（Tool-recorded log）：执行动作，保留文本日志记录已完成的操作；不再重新查看旧截图。

Claude的计算机使用API采用了日志模式。更简单、更可靠。

### 可视化工具使用

ChartAgent（arXiv:2510.04514）引入了针对图表理解的可视化工具使用：裁剪、缩放、OCR、调用外部检测。智能体可以输出“裁剪到区域（100，200，300，400）然后调用OCR”作为工具调用。工具返回文本；VLM继续推理。

这种模式可以推广：标记提示（Set-of-mark prompting）、区域标注和外部检测工具都符合相同的“输出工具调用，接收结构化响应”模式。

### 2026年的基准测试

- ScreenSpot-Pro。约1000张网页截图上的GUI定位。开源SOTA Qwen2.5-VL-72B约85%。前沿模型约90%。
- VisualWebArena。端到端网页任务（购物、论坛、分类广告）。开源SOTA约20%。Gemini 3 Pro约27%。
- AgentVista（arXiv:2602.23166）。2026年最具挑战性的基准测试。跨12个领域的真实工作流。前沿模型得分27-40%；开源模型10-20%。
- WebArena / WebShop。较早期的基准测试；已被前沿模型饱和。

### 为什么仍然困难

智能体性能的瓶颈：

1. 精细尺度的视觉定位。“点击那个小X”在移动分辨率下经常失败。
2. 长期规划。10个动作后，智能体偏离目标。
3. 错误恢复。当点击失败（按错按钮）时，检测并恢复的行为很少出现在训练数据中。
4. 跨页面上下文。在标签页之间跳转或填写长表单时丢失状态。

研究方向：记忆架构、显式重规划、多模态验证（通过截图匹配验证动作成功与否）。

### 顶点项目构建任务

顶点项目任务：构建一个计算机使用智能体，能够：

1. 读取预订网站模拟页面的HTML和截图。
2. 规划多步骤序列：搜索 → 选择 → 填写表单 → 提交。
3. 输出符合动作模式的JSON动作。
4. 在固定的10个任务子集上进行评估。

本课程提供了脚手架代码，易于扩展为真实浏览器环境。

## 关键术语

| 术语 | 人们常说的 | 实际含义 |
|------|------------|----------|
| GUI定位（GUI grounding） | "点击坐标" | 模型输出指令目标在截图上的（x，y）坐标 |
| 动作模式（Action schema） | "工具定义" | 有效动作的JSON描述（点击、键入、滚动、拖拽） |
| 无障碍树（Accessibility tree） | "结构化DOM" | 来自浏览器/iOS API的机器可读UI层次结构 |
| 混合智能体（Hybrid agent） | "截图+树" | 同时使用图像和结构化信息；比单独使用任何一种更可靠 |
| 可视化工具使用（Visual tool use） | "缩放/裁剪/检测" | 智能体在计划过程中调用外部视觉工具（OCR、检测） |
| 摘要链（Summary-chain） | "记忆压缩" | 定期用文本摘要替代长截图历史 |
| VisualWebArena | "端到端网页基准" | 2024年端到端网页任务基准测试 |
| AgentVista | "2026年困难基准" | 12个领域的真实工作流；即使是Gemini 3 Pro也仅得分约30% |
