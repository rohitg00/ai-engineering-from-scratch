# 25 · 多模态智能体与计算机操作（综合实战）

> 2026 年的前沿产品是一个多模态智能体（multimodal agent），它能读取截图、点击按钮、在网页 UI 中导航、填写表单，并端到端地完成工作流。SeeClick 与 CogAgent（2024）验证了 GUI 接地（GUI-grounding）这一基本能力。Ferret-UI 补上了移动端。ChartAgent 为图表引入了视觉工具使用（visual tool-use）。VisualWebArena 与 AgentVista（2026）是前沿模型追逐的基准——即便是 Gemini 3 Pro 和 Claude Opus 4.7，在 AgentVista 的高难任务上也只能拿到约 30% 的分数。这一综合实战汇集了第 12 阶段的每一条线索：感知（高分辨率 VLM）、推理（带工具使用的 LLM）、接地（坐标输出）、长程记忆（long-horizon memory）以及评估。

**类型：** 综合实战
**语言：** Python（标准库，动作模式 + 智能体循环骨架）
**前置：** 第 12 阶段 · 05（LLaVA）、第 12 阶段 · 09（Qwen-VL JSON）、第 14 阶段（智能体工程）
**时长：** 约 240 分钟

## 学习目标

- 设计一个多模态智能体循环：感知 → 推理 → 行动 → 观察 → 重复。
- 构建一个 GUI 接地输出模式（点击坐标、输入文本、滚动、拖拽），让 VLM 能以 JSON 形式输出。
- 比较纯截图智能体、无障碍树（accessibility-tree）智能体与混合智能体（hybrid agent）。
- 在一个小型 VisualWebArena 子集上搭建多模态智能体的基准评估。

## 问题

一个订票网站的工作流：「帮我订一张 4 月 15 日飞往东京、价格低于 800 美元的靠过道座位的机票。」

一个多模态智能体需要：

1. 对浏览器截图。
2. 把截图 + URL + 目标解析成一个计划。
3. 输出一个结构化动作：在 (x,y) 处点击、在元素 E 处输入 "Tokyo"、向下滚动、选中（单选按钮）。
4. 把该动作施加到浏览器上。
5. 观察新状态（下一张截图）。
6. 重复直到任务完成。

每一步都是一次多模态 VLM 调用。VLM 的输出必须是可解析的 JSON。错误会跨步骤累积，所以恢复（recovery）很重要。

## 概念

### GUI 接地——基本能力

GUI 接地是指：给定一张截图和一条自然语言指令，输出要点击的 (x, y) 坐标（或其他动作）。

SeeClick（arXiv:2401.10935）是首个规模化的开源成果：在合成 + 真实 GUI 数据上微调一个 VLM，把坐标作为纯文本 token 输出。可行。

CogAgent（arXiv:2312.08914）加入了 1120x1120 的高分辨率编码，以应对密集 UI。得分：网页导航约 84%。

Ferret-UI（arXiv:2404.05719）聚焦移动端 UI，整合了 iOS 无障碍数据。

输出格式通常是 JSON：

```json
{"action": "click", "x": 384, "y": 220, "element_desc": "Search button"}
```

其中 `element_desc` 有助于恢复：如果坐标在两张截图之间发生偏移，这个语义提示能让系统重新接地（re-ground）。

### 动作模式

一个典型的动作模式（action schema）包含 6-10 种动作类型：

- `click`：(x, y)
- `type`：(text, x?, y?)
- `scroll`：(direction, amount)
- `drag`：(x0, y0, x1, y1)
- `select`：(option_index)
- `hover`：(x, y)
- `navigate`：(url)
- `wait`：(ms)
- `done`：(success, explanation)

智能体每一步输出一个动作。浏览器封装层执行该动作并返回新状态。

### 纯截图 vs 无障碍树

两种输入模式：

- 纯截图：完整图像，没有结构化信息。最通用；可用于任何应用。
- 无障碍树：结构化的 DOM / iOS 无障碍信息。接地可靠得多；在能拿到该树的场景下可用。
- 混合：两者都用，把无障碍树作为原子动作（atomic action）的可靠接地依据，把截图作为语义上下文。

生产级智能体在可能时会使用混合模式。浏览器自动化（Selenium + 无障碍接口）总能拿到这棵树；桌面应用有时也行。

### 长程记忆

一个 20 步的工作流会生成 20 张截图。VLM 的上下文很快被塞满。三种压缩策略：

- 摘要链（Summary-chain）：每 5 步之后，总结一下已经发生的事，丢掉旧截图。
- 跳帧（Skip-frame）：保留第一张、最后一张以及每隔 3 张的截图。
- 工具记录日志（Tool-recorded log）：执行动作，保留一份关于已做操作的文本日志；不再回看旧截图。

Claude 的计算机操作（computer-use）API 使用日志模式。更简单，更可靠。

### 视觉工具使用

ChartAgent（arXiv:2510.04514）为图表理解引入了视觉工具使用：裁剪、缩放、OCR、调用外部检测。智能体可以把「裁剪到区域 (100, 200, 300, 400)，然后调用 OCR」作为一次工具调用输出。工具返回文本；VLM 继续推理。

这一模式可以泛化：标记集合提示（set-of-mark prompting）、区域标注以及外部检测工具，都契合同一个「输出一次工具调用，接收一个结构化响应」的模式。

### 2026 年的基准

- ScreenSpot-Pro。在约 1k 张网页截图上的 GUI 接地。开源 SOTA 为 Qwen2.5-VL-72B 约 85%。前沿模型约 90%。
- VisualWebArena。端到端的网页任务（购物、论坛、分类信息）。开源 SOTA 约 20%。Gemini 3 Pro 约 27%。
- AgentVista（arXiv:2602.23166）。2026 年最难的基准。覆盖 12 个领域的真实工作流。前沿模型得分 27-40%；开源模型 10-20%。
- WebArena / WebShop。较老的基准；已被前沿模型刷满。

### 为什么它仍然很难

智能体性能瓶颈：

1. 精细尺度上的视觉接地。在移动端分辨率下，「点击那个小 X」经常失败。
2. 长程规划。在 10 个动作之后，智能体会偏离目标。
3. 错误恢复。当一次点击失败（点错按钮）时，检测 + 恢复几乎没有训练数据。
4. 跨页面上下文。在标签页之间跳转或填写长表单时会丢失状态。

研究方向：记忆架构、显式重新规划（explicit replanning）、多模态验证（用截图匹配来判断动作是否成功）。

### 综合实战动手做

综合实战任务：构建一个计算机操作智能体，它能：

1. 读取订票网站模拟页面的 HTML + 截图。
2. 规划一个多步序列：搜索 → 选择 → 填表 → 提交。
3. 输出符合动作模式的 JSON 动作。
4. 在一个固定的 10 任务子集上进行评估。

本课提供了易于扩展为真实浏览器的脚手架代码。

## 动手用

`code/main.py` 是综合实战的脚手架：

- 动作模式的 JSON 定义（10 个动作）。
- 以 dict 表示的模拟浏览器状态。
- 智能体循环骨架：接收状态、输出动作、施加、循环。
- 一个 10 任务的迷你基准（合成页面），用于测量端到端成功率。
- 当动作失败时的错误恢复钩子（error-recovery hook）。

## 练习

1. 用一个 `screenshot_region` 工具（裁剪 + 缩放）扩展动作模式。哪些任务会从中受益？

2. 阅读 AgentVista（arXiv:2602.23166）。描述最难的任务类别，以及前沿模型为何仍然失败。

3. 长程记忆压缩：设计一个摘要链，使其同时存活的截图不超过 4 张，但可记录任意数量的日志。

4. 构建一个错误恢复钩子：当动作失败（找不到按钮）时，智能体下一步该做什么？

5. 在 10 个网页任务上，比较纯截图的 Claude 4.7 与混合（截图 + 无障碍树）的 Qwen2.5-VL。在哪些任务上谁胜出？

## 关键术语

| 术语 | 人们常说 | 实际含义 |
|------|-----------------|------------------------|
| GUI 接地（GUI grounding） | 「点击坐标」 | 模型针对截图上某条指令的目标输出 (x,y) |
| 动作模式（Action schema） | 「工具定义」 | 对合法动作（click、type、scroll、drag）的 JSON 描述 |
| 无障碍树（Accessibility tree） | 「结构化 DOM」 | 来自浏览器/iOS API 的机器可读 UI 层级结构 |
| 混合智能体（Hybrid agent） | 「截图 + 树」 | 同时使用图像与结构化信息；比单独任一种都更可靠 |
| 视觉工具使用（Visual tool use） | 「缩放/裁剪/检测」 | 智能体在规划途中调用外部视觉工具（OCR、检测） |
| 摘要链（Summary-chain） | 「记忆压缩」 | 用周期性的文本摘要替代冗长的截图历史 |
| VisualWebArena | 「端到端网页基准」 | 2024 年的端到端网页任务基准 |
| AgentVista | 「2026 年高难基准」 | 12 个领域的真实工作流；连 Gemini 3 Pro 也只得约 30% |
