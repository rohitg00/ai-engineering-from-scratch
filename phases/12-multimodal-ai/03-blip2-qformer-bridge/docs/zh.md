# 从CLIP到BLIP-2 — 将Q-Former作为模态桥梁

> CLIP对齐图像和文本，但无法生成描述、回答问题或进行对话。BLIP-2（Salesforce, 2023）通过一个可训练的小型桥梁解决了这个问题：32个可学习的查询向量（learnable query vectors）通过交叉注意力（cross-attention）关注冻结的ViT特征，然后直接嵌入到冻结的LLM输入流中。188M参数的桥梁连接了一个11B的LLM和一个ViT-g/14。到2026年为止，每个基于适配器的视觉语言模型（VLM）——MiniGPT-4、InstructBLIP、LLaVA的衍生模型——都是它的后代。本课程解读Q-Former的架构，解释其两阶段训练，并构建一个玩具版本来将视觉令牌输入到冻结的文本解码器中。

**类型：** 构建
**语言：** Python（标准库，交叉注意力+可学习查询演示）
**前置知识：** 阶段12·02（CLIP），阶段7（Transformer）
**时间：** 约180分钟

## 学习目标

- 解释为什么在冻结的视觉编码器和冻结的LLM之间使用可训练的瓶颈比端到端微调在成本和稳定性上更优。
- 实现一个交叉注意力模块，其中一组固定的可学习查询关注外部图像特征。
- 理解BLIP-2的两阶段预训练：表示学习（ITC + ITM + ITG）然后是生成式学习（带冻结解码器的语言模型损失）。
- 比较Q-Former与LLaVA中使用的更简单的MLP投影器，并讨论何时哪种选择更胜一筹。

## 问题

你有一个冻结的ViT，每张图像产生256个维度为1408的块令牌。你有一个冻结的7B LLM，期望输入维度为4096的令牌嵌入。显而易见的桥梁——从1408到4096的线性层——可以工作，但将所有256个块令牌喂入LLM的上下文需要每张图像额外256个令牌。在一个32张图像的批次中，仅视觉模态就消耗了8192个令牌。

BLIP-2的问题：能否将256个令牌的图像表示压缩成更少的令牌（例如32个），同时为LLM保留足够的信息以进行描述、回答问题和推理？并且能否在不触及冻结主干的情况下训练这个桥梁，将训练成本仅限制在桥梁的参数上？

答案：Q-Former。32个可学习的“查询”向量通过交叉注意力关注ViT的块令牌，产生一个32令牌的视觉摘要供LLM使用。总共188M参数。在触及LLM之前，使用对比、匹配和生成式目标进行训练。

## 概念

### 可学习查询

Q-Former的核心技巧：不让LLM的文本令牌关注图像块，而是引入一组新的32个可学习查询向量`Q`，让*它们*来关注图像块。查询是模型的参数——它们在学习过程中被训练，并且对所有图像使用相同的32个查询。

经过交叉注意力后，每个查询都持有图像的一个压缩摘要——“描述主要对象”、“描述背景”、“数出对象数量”等。查询并非真正按语义标签专门化；它们会学习任何能使下游损失降低的编码。

### 架构

Q-Former是一个小型Transformer（12层，约100M参数），具有两条路径：

1. 查询路径：32个查询向量经过自注意力（彼此之间），然后与冻结的ViT的块令牌进行交叉注意力，再经过前馈网络（FFN）。
2. 文本路径：一个类似BERT的文本编码器与查询路径共享自注意力和FFN的权重。对于文本路径，交叉注意力被禁用。

训练时两条路径同时运行。查询和文本通过共享的自注意力进行交互，这意味着查询可以根据任务需要（ITM、ITG）以文本为条件。在用于VLM的推理时，只有查询路径运行，产生32个视觉令牌。

### 两阶段训练

BLIP-2进行两阶段预训练：

阶段1：表示学习（无LLM）。三种损失：
- ITC（图像-文本对比学习）：类似CLIP的对比损失，在池化的查询令牌和文本CLS令牌之间进行。
- ITM（图像-文本匹配）：二元分类器——这个图像-文本对是否匹配？采用硬负样本挖掘。
- ITG（基于图像的文本生成）：以查询为条件的因果语言模型头。迫使查询编码可生成文本的内容。

只有Q-Former进行训练。ViT冻结。不涉及LLM。

阶段2：生成式学习。连接一个冻结的LLM（如OPT-2.7B或Flan-T5-XL等）。通过一个小型线性层将32个查询输出投影到LLM的嵌入维度。将它们前置到文本提示之前。仅训练线性投影和Q-Former，使用连接后的提示+图像+描述序列的语言模型损失。

阶段2之后，Q-Former + 投影构成完整的视觉适配器。推理时：图像 → ViT → Q-Former → 线性投影 → 前置到文本 → 冻结的LLM生成输出。

### 参数经济学

BLIP-2使用ViT-g/14（1.1B，冻结）+ OPT-6.7B（6.7B，冻结）+ Q-Former（188M，训练）= 总计8B，训练188M。仅Q-Former约占整个堆栈参数的2.4%。训练成本反映了这一点：在少量A100上训练数天，而端到端训练需要数周。

质量：BLIP-2在零样本VQA上匹配或超越Flamingo-80B，但体积小50倍。桥梁有效。

### InstructBLIP与指令感知Q-Former

InstructBLIP（2023）通过额外的输入扩展了Q-Former：指令文本本身。在交叉注意力时，查询现在可以同时访问图像块和指令。查询可以针对每条指令进行专门化（“数出汽车数量”、“描述情绪”），而不是学习一个固定的摘要。在保留任务上的基准测试中取得了提升。

### MiniGPT-4与仅投影器方法

MiniGPT-4保留了Q-Former，但仅训练输出线性投影，同时冻结所有其他部分。成本低，但代价是质量——查询是BLIP-2的，而不是你自己的。适合快速迭代，但并非最佳架构。

### LLaVA为何选择更简单的方案

LLaVA（2023，课程12.05）用简单的2层MLP替代了Q-Former，该MLP将每个ViT块令牌投影到LLM空间——对于24x24的网格，每张图像产生576个令牌，全部送入LLM。压缩率更差，但让LLM能够关注原始块。当时这种做法有争议；到2023年底它成为主流，因为视觉指令数据（LLaVA-Instruct-150k）证明MLP能够训练以保留足够的信号。权衡：LLaVA的上下文更快填满，但它自然地扩展到多图像和视频。

到2026年，领域出现分化：在令牌预算重要的情况下（长视频、多图像）Q-Former存活；在每令牌原始质量优先的情况下，MLP投影器占主导。

### 门控交叉注意力：Flamingo，祖先

Flamingo（课程12.04）早于BLIP-2，使用了相同的交叉注意力思想，但在每个冻结的LLM层都进行，而不是作为一个单独的桥梁。BLIP-2表明你可以只在输入层进行压缩，并且仍然有效。Gemini和Idefics结合了两者：交错的输入令牌加上可选的用于上下文少量样本的门控交叉注意力。

### 2026年的后代

- Q-Former：BLIP-2、InstructBLIP、MiniGPT-4，以及大多数视频语言模型（因令牌预算原因）。
- Perceiver重采样器：Flamingo的变体（课程12.04）；Idefics系列、Eagle、OmniMAE。
- MLP投影器：LLaVA、LLaVA-NeXT、LLaVA-OneVision、Cambrian-1。
- 注意力池化：VILA、PaliGemma。

这四种都有效。决定性问题在于你是受令牌预算约束还是受每令牌质量约束。

## 使用它

`code/main.py` 构建了一个类似Q-Former风格的交叉注意力的stdlib实现：

1. 模拟256个图像块令牌（维度128）。
2. 实例化32个可学习查询（维度128）。
3. 运行缩放点积交叉注意力（Q来自查询，K/V来自块）。
4. 通过线性层投影到LLM维度（512）。
5. 输出32个LLM就绪的视觉令牌。

所有数学运算采用纯Python（向量上的嵌套循环）。虽然是玩具版本但形状正确。注意力权重矩阵会被打印出来，这样你可以看到每个查询从哪些块中提取了信息。

## 关键术语

| 术语 | 人们常说的 | 实际含义 |
|------|--------------|----------|
| Q-Former | "查询变换器" | 小型Transformer，包含32个可学习查询向量，通过交叉注意力关注冻结的ViT特征 |
| 可学习查询 | "视觉的软提示" | 一组固定的参数，作为交叉注意力的查询方；每个模型学习一次，所有输入共享 |
| 交叉注意力 | "查询来自这里，键/值来自那里" | 查询、键、值来自不同源的注意力；查询如何从ViT块中提取信息 |
| ITC | "图像-文本对比学习" | 将CLIP风格的损失应用于Q-Former池化查询与文本CLS之间 |
| ITM | "图像-文本匹配" | 对硬负样本挖掘对进行二分类；迫使查询区分细粒度不匹配 |
| ITG | "基于图像的文本生成" | 因果语言模型损失，文本在查询条件下生成；迫使查询编码可解码为文本的内容 |
| 两阶段预训练 | "先表示后生成" | 阶段1仅训练Q-Former（ITC/ITM/ITG）；阶段2连接冻结的LLM，仅训练投影器和Q-Former |
| 冻结主干 | "不微调" | 视觉编码器和LLM的权重固定；仅桥梁训练 |
| 投影头 | "线性到LLM维度" | 最终的线性层，将Q-Former输出映射到LLM的嵌入维度 |
| Perceiver重采样器 | "Flamingo的版本" | 类似的可学习查询交叉注意力，Flamingo在每一层使用，而不是作为单个桥梁 |
