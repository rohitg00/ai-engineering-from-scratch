# 具身 VLA：RT-2、OpenVLA、π0、GR00T

> 模型首次从网站上读取食谱并在厨房机器人中执行是 RT-2（Google DeepMind，2023 年 7 月）。RT-2 将动作离散化为文本 token，在网页数据加机器人动作数据上共微调 VLM，并证明网页规模的视觉-语言知识可以迁移到机器人控制。OpenVLA（2024 年 6 月）发布了开放 7B 参考。Physical Intelligence 的 π0 系列（2024-2025）添加了流匹配动作专家。NVIDIA 的 GR00T N1（2025 年 3 月）为大规模人形机器人提供了双系统（系统 1 / 系统 2）控制。VLA 原语——视觉-语言-动作，一个能看、读、动的单一模型——是本阶段理解模型与 Phase 15 自主系统之间的桥梁。

**类型：** Learn
**语言：** Python（stdlib，动作 token 化器 + VLA 推理骨架）
**前置知识：** Phase 12 · 05（LLaVA），Phase 15（自主系统，引用）
**时间：** ~180 分钟

## 学习目标

- 描述动作 token 化：离散分箱编码（RT-2）、FAST 高效动作 token、连续流匹配动作（π0）。
- 解释为什么在网页 + 机器人数据上共微调能保留对新任务的通用知识迁移。
- 在同一机器人任务上比较 OpenVLA（开放 7B Llama+VLM）、π0（流匹配）和 GR00T N1（双系统）。
- 命名 Open X-Embodiment 数据集及其作为 RT-X 训练语料库的角色。

## 问题所在

从自然语言指令执行家务的机器人自 1970 年代以来一直是研究目标。2020 年代的答案是：视觉-语言-动作（VLA）模型。用于 VQA 的相同 VLM 架构，但输出是动作（关节扭矩、末端执行器姿态、离散命令）而非文本。

VLA 特有的挑战：

1. 动作空间是连续的（关节角度、力）且高维（7-DOF 臂 + 3-DOF 夹爪 = 30 Hz 下 10 维）。
2. 机器人特定训练数据稀缺。Open X-Embodiment 有约 100 万轨迹；网页文本-图像有 50 亿+。
3. 控制频率很重要。30 Hz 控制循环意味着每个动作 33 毫秒预算。
4. 安全。错误动作会损坏硬件、人类或财产。

## 核心概念

### 动作 token 化（RT-2）

RT-2 的技巧：将每个关节目标表示为量化文本 token。将归一化 [-1, 1] 范围离散化为 256 个 bin，将每个 bin 映射到词汇 ID。10-DOF 动作在每个控制步骤变为 10 个 token。

在混合上共微调 PaLM-X VLM：

- 网页图像-文本对（描述、VQA）。
- 机器人演示，动作作为 token。

模型看到"捡起红色方块"（语言）→ 图像（视觉）→ 10-token 动作序列（离散关节目标）。网页预训练保留通用知识迁移：RT-2 可以遵循"向快速移动物体移动"，即使"快速移动"不在训练数据中。

RT-2 论文中推理为 3-5 Hz，受限于 VLM 自回归解码。

### OpenVLA——开放 7B 参考

OpenVLA（Kim 等人，2024 年 6 月）是开放权重的 RT-2 等效物。7B Llama 骨干，DINOv2 + SigLIP 双视觉编码器，256 bin 上的动作 token 化。

在 Open X-Embodiment（跨 22 个机器人的 97 万轨迹）上训练。附带 LoRA 微调支持以适应新机器人。

推理：A100 上量化后 4-5 Hz。对慢速操作足够快，对高频控制不够。

### FAST token 化器——更快的动作解码

Pertsch 等人（2024）表明离散 bin token 化效率低下——大多数动作聚集在 bin 空间的小区域。FAST（频域动作序列 Token 化器）通过 DCT 压缩动作序列并量化系数。

30 步动作轨迹变为约 10 个 FAST token，而非 300 个离散 bin token。推理速度提升 3-5 倍，无质量损失。

### π0 和流匹配动作

Physical Intelligence 的 π0（Black 等人，2024 年 10 月）用流匹配动作专家替换离散动作 token：

- 小型动作 transformer 读取 VLM 的隐藏状态，通过整流流输出连续 50 步动作序列。
- 动作头用流匹配损失训练；VLM 预训练保持不变。
- 推理：完整动作序列在约 5 个去噪步骤中发出，有效 50 Hz 控制。

π0 声称：在广泛的操纵任务套件上击败 OpenVLA 和 Octo。连续动作公式保留了离散化破坏的平滑性。

π0.5 和 π0-FAST 是增量升级。π0-FAST 将 FAST token 化与流匹配结合。

### GR00T N1——人形双系统

NVIDIA 的 GR00T N1（2025 年 3 月）为 人形机器人（>30 DOF，全身）构建：

- 系统 2：大型 VLM 读取场景 + 指令，以约 1 Hz 产生高级子目标。
- 系统 1：小型动作头 transformer 以子目标为条件产生低级 50-100 Hz 关节命令。

分离映射到卡尼曼的快慢思考：系统 2 规划，系统 1 行动。好处：慢速 VLM 大小的规划不会阻塞快速控制；系统 1 保持小型以降低延迟。

GR00T N1.7（2025 年末）改进数据扩展。GR00T 使用 Omniverse 的仿真到真实数据进行微调。

### Open X-Embodiment

训练数据。RT-X（2023 年 10 月）组装了覆盖 22 个机器人 100 万轨迹的 22 个数据集。Open X-Embodiment 是每个人使用的语料库：

- ALOHA / Bridge V2 / Droid / RT-2 Kitchen / Language Table。
- 每个样本：（机器人状态、摄像头视图、指令、动作序列）。
- 训练卫生：统一动作空间，归一化关节范围，调整摄像头大小。

OpenVLA 和 π0 在 Open X-Embodiment 上训练。到任何特定机器人的领域差距通过 100-1000 个任务特定演示的 LoRA 微调来关闭。

### 共微调 vs 仅机器人

共微调混合网页 VQA 数据与机器人轨迹。比例很重要：太多 VQA 模型会忘记动作；太多机器人数据模型会丢失通用知识。

RT-2 的比例：约 1:1。OpenVLA：约 0.5:1 网页对机器人。π0：类似。精确比例是每个数据集大小需要调整的超参数。

仅机器人训练产生在分布外指令上失败的任务特定模型。共微调是"捡起红色方块（演示中）"与"捡起左边第三大的物体（新措辞）"之间的区别。

### 安全和动作限制

每个生产 VLA 都配备：

- 硬关节限制（不能超过规格扭矩）。
- 速度限制（软裁剪）。
- 工作空间边界（末端执行器不能离开桌子）。
- 新任务的人工在环审批。

这些位于 VLA 外部作为控制层检查。VLA 的输出是建议，而非命令。

## 使用它

`code/main.py`：

- 实现 256-bin 动作 token 化和反 token 化。
- 基于 DCT + 量化的 FAST token 化器草图。
- 比较（离散 bin、FAST、连续流）每个动作步骤的 token 数量。
- 打印 RT-2 → OpenVLA → π0 → GR00T 的谱系摘要。

## 交付它

本课产出 `outputs/skill-vla-action-format-picker.md`。给定机器人任务（操纵、导航、人形全身），在离散 bin + RT-2、FAST + OpenVLA、流匹配 + π0 或双系统 + GR00T 之间选择。

## 练习

1. 30 Hz 控制率下的 10-DOF 臂。256 bin 的离散 bin token 化每秒发出多少 token？7B VLM 能跟上吗？

2. FAST token 化将 30 步轨迹压缩到约 10 个 token。如果轨迹有高频运动（例如击鼓），用户会失去什么？

3. π0 的流匹配头在约 5 步中去噪。与 OpenVLA 的 4-5 Hz 自回归解码比较吞吐量。

4. GR00T 的系统 1 / 系统 2 分离映射到卡尼曼。提出一个不同的分离（系统 3？）可能有助于双足行走。

5. 阅读 Open X-Embodiment 第 4 节关于数据集策划。命名防止领域泄漏的三个策划规则。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------------|------------------------|
| VLA | "视觉-语言-动作" | 接收图像 + 指令并输出动作命令的模型 |
| 动作 token 化 | "离散 bin" | 将连续关节目标量化为每维 256 个 bin，每个 bin 一个词汇 ID |
| FAST token 化器 | "频域动作 token" | DCT + 量化将 30 步轨迹压缩到约 10 个 token |
| 共微调 | "混合网页 + 机器人" | 在网页 VQA 数据与机器人演示上训练以保留通用知识 |
| 流匹配动作头 | "π0 连续输出" | 通过整流流输出 50 步动作序列的小型 transformer |
| 系统 1 / 系统 2 | "双系统控制" | 大型 VLM 慢速规划，小型动作头快速行动；GR00T 模式 |
| Open X-Embodiment | "RT-X 数据集" | 100 万轨迹跨机器人数据集；训练语料库 |
