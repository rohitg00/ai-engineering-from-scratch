# 具身VLA：RT-2、OpenVLA、π0、GR00T

> 首个从网站读取食谱并在厨房机器人中执行的模型是RT-2（Google DeepMind，2023年7月）。RT-2将动作离散化为文本词元（Token），在网页数据加机器人动作数据上共同微调视觉-语言模型（VLM），并证明了网页规模的视觉-语言知识能够迁移到机器人控制中。OpenVLA（2024年6月）提供了开源的7B参考实现。Physical Intelligence的π0系列（2024-2025年）增加了流匹配（Flow-Matching）动作专家模块。NVIDIA的GR00T N1（2025年3月）为大规模人形机器人提供了双系统（System 1 / System 2）控制。VLA原语——视觉-语言-动作（Vision-Language-Action），即一个能看、能读、能动作的单一模型——是当前阶段理解模型与阶段15（Phase 15）自主系统之间的桥梁。

**类型：** 学习
**语言：** Python（标准库，动作词元器 + VLA推理骨架）
**前置知识：** 阶段12·05（LLaVA），阶段15（自主系统，参考）
**时间：** 约180分钟

## 学习目标

- 描述动作分词化（Action Tokenization）：离散箱编码（RT-2）、FAST高效动作词元、连续流匹配动作（π0）。
- 解释为什么在网页+机器人数据上共同微调能够将通用知识迁移到新任务。
- 对比OpenVLA（开源7B Llama + VLM）、π0（流匹配）和GR00T N1（双系统）在相同机器人任务上的表现。
- 说出Open X-Embodiment数据集及其作为RT-X训练语料库的作用。

## 问题

自20世纪70年代以来，让机器人根据自然语言指令做家务一直是一个研究目标。2020年代的答案：视觉-语言-动作（VLA）模型。使用与VQA相同的VLM架构，但输出是动作（关节扭矩、末端执行器位姿、离散命令）而非文本。

VLA特有的挑战：

1. 动作空间是连续的（关节角度、力）且高维（7自由度臂 + 3自由度夹爪 = 10维，30 Hz）。
2. 机器人专用训练数据稀缺。Open X-Embodiment约有100万条轨迹；网页文本-图像数据超过50亿。
3. 控制频率至关重要。30 Hz的控制回路意味着每个动作的预算为33毫秒。
4. 安全性。错误动作可能损坏硬件、人类或财产。

## 概念

### 动作分词化（Action Tokenization，RT-2）

RT-2的技巧：将每个关节目标表示为量化后的文本词元。将归一化的[-1, 1]范围离散化为256个箱子（Bin），每个箱子映射到一个词汇ID。一个10自由度的动作在每个控制步骤变成10个词元。

在PaLM-X VLM上混合共同微调：

- 网页图像-文本对（描述、VQA）。
- 机器人演示，动作用词元表示。

模型看到"拿起红色立方体"（语言）→ 图像（视觉）→ 10词元动作序列（离散化关节目标）。网页预训练保留了通用知识迁移：RT-2可以执行"朝向快速移动的物体移动"这样的指令，即使"快速移动"不在训练数据中。

在RT-2论文中推理速度为3-5 Hz，受限于VLM自回归解码。

### OpenVLA——开源7B参考

OpenVLA（Kim等人，2024年6月）是开源权重的RT-2等价物。采用7B Llama骨干、DINOv2 + SigLIP双视觉编码器，使用256箱子的动作分词化。

在Open X-Embodiment（22个机器人上的97万条轨迹）上训练。支持通过LoRA微调适配新机器人。

推理速度：量化后在A100上为4-5 Hz。足以应付慢速操控，但不适用于高频控制。

### FAST词元器——更快的动作解码

Pertsch等人（2024年）指出，离散箱子分词化效率低下——大多数动作集中在箱子空间的一个小区域。FAST（Frequency-domain Action Sequence Tokenizer）通过离散余弦变换（DCT）压缩动作序列，并量化系数。

一个30步的动作轨迹变为约10个FAST词元，而不是300个离散箱子词元。推理速度提升3-5倍，且不损失质量。

### π0和流匹配动作

Physical Intelligence的π0（Black等人，2024年10月）用流匹配动作专家模块替代了离散动作词元：

- 一个小型动作Transformer读取VLM的隐含状态，并通过整流流（Rectified Flow）输出一个连续的50步动作序列。
- 动作头部使用流匹配损失训练；VLM的预训练保持不变。
- 推理：整个动作序列在大约5步去噪步骤中生成，有效控制频率达50 Hz。

π0的论断：在广泛的操控任务上优于OpenVLA和Octo。连续动作公式保留了离散化会破坏的平滑性。

π0.5和π0-FAST是增量升级。π0-FAST将FAST分词化与流匹配结合。

### GR00T N1——人形机器人的双系统

NVIDIA的GR00T N1（2025年3月）专为人形机器人（>30自由度，全身）构建：

- 系统2（System 2）：一个大型VLM读取场景+指令，以约1 Hz的频率产生高级子目标。
- 系统1（System 1）：一个小型动作头部Transformer，根据子目标生成低频的50-100 Hz关节命令。

这种划分对应了卡尼曼（Kahneman）的快慢思维：系统2规划，系统1执行。优势：慢速VLM级规划不会阻塞快速控制；系统1保持小型以实现低延迟。

GR00T N1.7（2025年末）改进了数据规模。GR00T通过来自Omniverse的仿真到现实（Sim-to-Real）数据进行微调。

### Open X-Embodiment

训练数据。RT-X（2023年10月）汇集了22个数据集，覆盖22个机器人上的100万条轨迹。Open X-Embodiment是大家通用的语料库：

- ALOHA / Bridge V2 / Droid / RT-2 Kitchen / Language Table。
- 每个样本：（机器人状态、相机视图、指令、动作序列）。
- 训练流程：统一动作空间、归一化关节范围、调整相机尺寸。

OpenVLA和π0在Open X-Embodiment上训练。通过LoRA在100-1000个特定任务演示上微调，即可缩小到任意特定机器人的领域差距。

### 共同微调 vs 纯机器人训练

共同微调混合了网页VQA数据和机器人轨迹。比例很重要：VQA过多则模型遗忘动作；机器人数据过多则模型丧失通用知识。

RT-2的比例：约1:1。OpenVLA：约0.5:1（网页:机器人）。π0：类似。精确比例是一个需要根据数据集大小调整的超参数。

纯机器人训练会产生任务专用模型，难以应对分布外的指令。共同微调是"拿起红色立方体（演示中见过的）"与"拿起左边数第三个最大的物体（新颖表述）"之间的区别。

### 安全与动作限制

每个生产级VLA都附带：

- 硬关节限制（扭矩不能超出规格）。
- 速度限制（软裁剪）。
- 工作空间边界（末端执行器不能离开桌面）。
- 新任务需人类在环（Human-in-the-loop）批准。

这些作为控制层检查位于VLA外部。VLA的输出是建议，而非命令。

## 使用

`code/main.py`：

- 实现256箱动作分词化与反分词化。
- 基于DCT+量化勾勒FAST词元器。
- 比较三种方式（离散箱子、FAST、连续流）每动作步骤的词元数量。
- 输出RT-2 → OpenVLA → π0 → GR00T的谱系摘要。

## 部署

本课程产出 `outputs/skill-vla-action-format-picker.md`。给定一个机器人任务（操控、导航、人形全身），选择离散箱子+RT-2、FAST+OpenVLA、流匹配+π0或双系统+GR00T。

## 练习

1. 一个10自由度机械臂以30 Hz控制频率运行。离散箱子分词化（256箱子）每秒产生多少个词元？一个7B VLM能否跟上？
2. FAST分词化将30步轨迹压缩到约10个词元。如果轨迹包含高频运动（例如击鼓），用户会损失什么？
3. π0的流匹配头部在大约5步内去噪。将其吞吐量与OpenVLA的4-5 Hz自回归解码进行比较。
4. GR00T的系统1/系统2划分对应卡尼曼理论。提出一个可能有助于双足行走的不同划分（系统3？）。
5. 阅读Open X-Embodiment第4节关于数据集整理的内容。列举防止领域泄漏的三条整理规则。

## 关键术语

| 术语 | 人们所说的 | 实际含义 |
|------|-----------|----------|
| VLA | "Vision-Language-Action" | 接收图像+指令并输出动作命令的模型 |
| Action tokenization | "Discrete bins" | 将连续关节目标每维量化为256个箱子，每个箱子对应一个词汇ID |
| FAST tokenizer | "Frequency action tokens" | 通过DCT+量化将30步轨迹压缩到约10个词元 |
| Co-fine-tune | "Mix web + robot" | 在网页VQA数据与机器人演示上共同训练，以保留通用知识 |
| Flow-matching action head | "π0 continuous output" | 通过整流流输出50步动作序列的小型Transformer |
| System 1 / System 2 | "Dual-system control" | 大型VLM缓慢规划，小型动作头部快速执行；GR00T模式 |
| Open X-Embodiment | "RT-X dataset" | 跨机器人100万轨迹数据集；训练语料库 |
