# Vision Transformers 与补丁-词元（Patch-Token）原语

> 在多模态之前，图像必须首先转换为Transformer可处理的词元序列。2020年的ViT论文通过16×16像素的补丁、线性投影和位置嵌入解决了这个问题。五年后的2026年，所有前沿模型（Claude Opus 4.7原生支持2576px、Gemini 3.1 Pro、Qwen3.5-Omni）仍然以此方式起步——编码器从ViT变为DINOv2再到SigLIP 2，添加了寄存器词元（Register Tokens），位置方案改为2D-RoPE，但原语保持不变。本课将从头到尾解读补丁-词元（Patch-Token）流水线，并用标准库Python构建它，以便第12阶段的其余部分对"视觉词元"有一个具体的思维模型。

**类型：** 学习  
**语言：** Python（标准库，补丁词元化器 + 几何计算器）  
**前置条件：** 阶段7（Transformer）、阶段4（计算机视觉）  
**时长：** 约120分钟

## 学习目标

- 将H×W×3图像转换为具有正确位置编码的补丁词元序列。
- 计算给定（补丁大小、分辨率、隐藏维度、深度）的ViT的序列长度、参数量和FLOPs。
- 说出使ViT从2020年研究走向2026年生产的三个升级：自监督预训练（DINO / MAE）、寄存器词元（Register Tokens）和原生分辨率打包（Native-Resolution Packing）。
- 为下游任务在CLS池化（CLS Pooling）、平均池化（Mean Pooling）和寄存器词元（Register Tokens）之间做出选择。

## 问题

Transformer作用于向量序列上。文本已经是序列（字节或词元）。图像是一个具有三个颜色通道的2D像素网格——不是序列。如果你展平每个像素，一张224×224的RGB图像将变成150,528个词元，而该长度下的自注意力是不可行的（序列长度的平方复杂度）。

2020年之前的方法在Transformer前接入一个CNN特征提取器：ResNet生成一个7×7的2048维向量特征图，将这些49个词元送入Transformer。这有效但继承了CNN的偏置（平移等变性、局部感受野），并丧失了Transformer对规模扩展的偏好。

Dosovitskiy等人（2020年）提出了一个直白的问题：如果我们跳过CNN会怎样？将图像分割成固定大小的补丁（例如16×16像素），对每个补丁进行线性投影得到一个向量，添加位置嵌入，并将序列送入一个普通Transformer。这在当时是异端——没有卷积的视觉。有了足够的数据（JFT-300M，之后是LAION），它在ImageNet上击败了ResNet并持续改进。

到2026年，ViT原语已成为不可动摇的基础。每个开源权重VLM的视觉塔（Vision Tower）都是某种衍生物（DINOv2、SigLIP 2、CLIP、EVA、InternViT）。问题不再是"我们应该用补丁吗？"，而是"用什么补丁大小、什么分辨率调度、什么预训练目标、什么位置编码。"

## 概念

### 补丁作为词元

给定一个形状为`(H, W, 3)`的图像`x`和一个补丁大小`P`，你将图像切割成一个`(H/P) x (W/P)`的非重叠补丁网格。每个补丁是一个`P x P x 3`的像素立方体。将每个立方体展平成一个`3 P^2`的向量。应用一个形状为`(3 P^2, D)`的共享线性投影`W_E`，将每个补丁映射到模型的隐藏维度`D`。

以ViT-B/16的经典配置为例：
- 分辨率224，补丁大小16 → 网格14x14 → 196个补丁词元。
- 每个补丁是`16 x 16 x 3 = 768`个像素值，投影到`D = 768`。
- 添加一个可学习的`[CLS]`词元 → 序列长度为197。

补丁投影在数学上等同于一个卷积核大小为`P`、步长为`P`、输出通道数为`D`的2D卷积。这就是生产代码实际实现它的方式——`nn.Conv2d(3, D, kernel_size=P, stride=P)`。"线性投影"是概念性的框架；卷积核框架是高效的。

### 位置嵌入

补丁没有固有的顺序——Transformer将它们视为一个袋子。早期的ViT添加了一个可学习的1D位置嵌入（每个位置一个768维向量，共197个）。有效，但将模型与训练分辨率绑定：在推理时，如果你改变网格，你必须插值位置表。

现代视觉骨干使用2D-RoPE（Qwen2-VL的M-RoPE、SigLIP 2的默认）或因子化2D位置。2D-RoPE根据补丁的（行，列）索引旋转查询和键向量，因此模型从旋转角度推断相对2D位置。没有位置表。模型在推理时处理任意网格大小。

### CLS词元、池化输出和寄存器词元

图像级表示是什么？存在三种选择：

1. `[CLS]`词元。在补丁序列前添加一个可学习向量。经过所有Transformer块后，CLS词元的隐藏状态就是图像表示。继承自BERT。由原始ViT、CLIP使用。
2. 平均池化（Mean Pool）。对补丁词元的输出隐藏状态取平均。由SigLIP、DINOv2以及大多数现代VLM使用。
3. 寄存器词元（Register Tokens）。Darcet等人（2023年）观察到，没有显式汇点词元（Sink Token）训练的ViT会发展出高范数（High-Norm）的"伪影"补丁，这些补丁劫持了自注意力。添加4-16个可学习的寄存器词元吸收这个负载，并提高密集预测（分割、深度）的质量。DINOv2和SigLIP 2都内置了寄存器。

对于下游任务，选择很重要。CLS适用于分类。对于将补丁词元馈入LLM的VLM，你完全跳过池化——每个补丁都成为LLM的输入词元。寄存器在交接前被丢弃（它们是脚手架，不是内容）。

### 预训练：监督、对比、掩码、自蒸馏

2020年的ViT在JFT-300M上用监督分类进行预训练。很快被以下方法取代：

- CLIP（2021年）：在400M对图像-文本对上使用对比学习。第12.02课。
- MAE（2021年，He等人）：掩码75%的补丁，重建像素。自监督，适用于纯图像。
- DINO（2021年）/ DINOv2（2023年）：学生-教师自蒸馏，无标签，无描述。2023年的DINOv2 ViT-g/14是最强的纯视觉骨干，也是"密集特征"用例的默认选择。
- SigLIP / SigLIP 2（2023年，2025年）：使用Sigmoid损失和NaFlex实现原生宽高比的CLIP。2026年开源VLM（Qwen、Idefics2、LLaVA-OneVision）中的主导视觉塔。

你选择的预训练决定了骨干的强项：CLIP/SigLIP擅长与文本的语义匹配，DINOv2擅长密集视觉特征，MAE作为下游微调的起点。

### 缩放定律

ViT缩放（Zhai等人，2022年）确立了ViT的质量在模型大小、数据大小和计算资源方面遵循可预测的定律。在固定计算资源下：
- 更大的模型 + 更多数据 → 更好的质量。
- 补丁大小是序列长度与保真度之间的杠杆。补丁14（DINOv2/SigLIP SO400m的典型值）每张图像产生的词元比补丁16多；对OCR和密集任务更好，但对速度更差。
- 分辨率是另一个重要杠杆。从224提高到384再到512几乎总是有帮助，但代价是FLOPs的二次增长。

ViT-g/14（1B参数，补丁14，分辨率224 → 256个词元）和SigLIP SO400m/14（400M参数，补丁14）是2026年开源VLM的两个主力编码器。

### ViT的参数量计算

完整计算在`code/main.py`中。以224分辨率的ViT-B/16为例：

```
patch_embed = 3 * 16 * 16 * 768 + 768  =  591k
cls + pos    = 768 + 197 * 768          =  152k
block        = 4 * 768^2 (QKVO) + 2 * 4 * 768^2 (MLP) + 2 * 2*768 (LN)
             = 12 * 768^2 + 3k          =  7.1M
12 blocks    = 85M
final LN    = 1.5k
total       ≈ 86M
```

在加载检查点之前，用这种方式估算每个ViT。骨干大小决定了你在任何下游VLM中的VRAM下限。

### 2026年生产配置

2026年大多数开源VLM搭载的编码器是SigLIP 2 SO400m/14，采用原生分辨率（NaFlex）。它具有：
- 400M参数。
- 补丁大小14，默认分辨率384 → 每张图像729个补丁词元。
- 平均池化用于图像级任务；所有729个补丁流入LLM进行VQA。
- 4个寄存器词元，在LLM交接前丢弃。
- 用于原生宽高比的带图像级缩放的2D-RoPE。

该配置中的每个决策都可以追溯到你可以阅读的论文。

## 使用它

`code/main.py`是一个补丁词元化器和几何计算器。它接受（图像H，W，补丁P，隐藏维度D，深度L）并报告：

- 补丁化后的网格形状和序列长度。
- 一个合成8x8像素玩具图像的词元序列（逐步演示展平+投影路径）。
- 按补丁嵌入、位置嵌入、Transformer块和头部划分的参数量。
- 目标分辨率下每次前向传播的FLOPs。
- 跨ViT-B/16 @ 224、ViT-L/14 @ 336、DINOv2 ViT-g/14 @ 224、SigLIP SO400m/14 @ 384的对比表。

运行它。将参数量与已发布数字匹配。尝试不同的补丁大小和分辨率，感受词元数量的成本。

## 交付

本课生成`outputs/skill-patch-geometry-reader.md`。给定一个ViT配置（补丁大小、分辨率、隐藏维度、深度），它生成一个带有理由说明的词元计数、参数量和VRAM估计。每当你在VLM中选择视觉骨干时使用此技能——它可以防止"词元爆炸导致我的LLM上下文被填满"的惊喜。

2. 一张1080p帧（1920x1080）在补丁14下产生多少个词元？以30 FPS播放5分钟的视频，总共多少个视觉词元？哪种节约成本最有效：池化、帧采样还是词元合并？

3. 在纯Python中实现基于补丁词元的平均池化。验证对DINOv2输出的196个词元进行平均池化是否与模型在请求池化嵌入时返回的结果一致。

4. 阅读"Vision Transformers Need Registers"（arXiv:2309.16588）的第3节。用两句话描述寄存器吸收了什么伪影，以及为什么这对下游密集预测很重要。

5. 修改`code/main.py`以支持"补丁打包"（Patch-n'-Pack）：给定一个不同分辨率的图像列表，生成一个单一的打包序列和块对角注意力掩码。在到达第12.06课时进行验证。

## 关键术语

| 术语 | 人们说的意思 | 实际含义 |
|------|----------------|------------------------|
| 补丁 (Patch) | "16x16像素方块" | 输入图像中固定大小的非重叠区域；成为一个词元 |
| 补丁嵌入 (Patch Embedding) | "线性投影" | 一个共享的学习矩阵（或步长为P的Conv2d），将展平的补丁像素映射到D维向量 |
| CLS词元 (CLS Token) | "类别词元" | 预置的可学习向量，其最终隐藏状态代表整个图像；2026年可选 |
| 寄存器词元 (Register Token) | "汇点词元" | 额外的可学习词元，用于吸收ViT在预训练期间产生的高范数注意力伪影 |
| 位置嵌入 (Position Embedding) | "位置信息" | 每个位置的向量或旋转，使序列感知顺序；2D-RoPE是现代默认 |
| 网格 (Grid) | "补丁网格" | 给定分辨率和补丁大小下的(H/P) x (W/P) 2D补丁数组 |
| NaFlex | "原生灵活分辨率" | SigLIP 2特性：单一模型支持多种宽高比和分辨率，无需重新训练 |
| 骨干 (Backbone) | "视觉塔" | 预训练的图像编码器，其补丁词元输出馈入VLM中的LLM |
| 池化 (Pooling) | "图像级摘要" | 将补丁词元转换为一个向量的策略：CLS、平均、注意力池化或基于寄存器的 |
| 补丁14 vs 16 | "更细 vs 更粗的网格" | 补丁14每张图像产生更多词元，对OCR保真度更好，但速度更慢；补丁16是经典默认 |
