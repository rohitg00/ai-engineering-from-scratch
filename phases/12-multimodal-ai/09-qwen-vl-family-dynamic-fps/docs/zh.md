# Qwen-VL 家族与动态FPS视频

> Qwen-VL 家族——Qwen-VL (2023)、Qwen2-VL (2024)、Qwen2.5-VL (2025)、Qwen3-VL (2025)——是 2026 年最具影响力的开源视觉语言模型（Vision-Language Model, VLM）系列。每一代都做出了一次决定性的架构赌注，并在十二个月内被整个开源生态复制：通过 M-RoPE 实现原生动态分辨率、通过绝对时间对齐实现动态FPS采样、在 ViT 中使用窗口注意力（Window Attention）、以及结构化智能体输出格式。到了 Qwen3-VL，配方已经稳定：一个具有原生宽高比输入的 2D-RoPE-ViT 编码器、一个 MLP 投影器嵌入大型 Qwen3 语言基座，以及将 OCR、定位和智能体行为作为一等目标的训练阶段。本课按时间顺序阅读整个家族，让你理解每个旋钮为何处于现在的位置。

**类型：** 学习
**语言：** Python (stdlib, M-RoPE 编码器 + 动态FPS采样器)
**前置条件：** Phase 12 · 06 (patch-n'-pack)
**时间：** ~120 分钟

## 学习目标

- 计算 M-RoPE 的三轴旋转（时间、高度、宽度），并解释为什么需要所有三个轴。
- 为视频选择动态FPS采样策略，并推理每秒钟的token数与事件检测准确率之间的权衡。
- 按顺序说出 Qwen-VL 四代升级的名称以及每代实现了什么。
- 编写 Qwen2.5-VL 风格的 JSON 智能体输出格式，并从 VLM 响应中解析结构化工具调用。

## 问题

Qwen-VL 于 2023 年 8 月发布，直接回应了 LLaVA-1.5 和 BLIP-2。Qwen 团队瞄准的差距有三方面：分辨率、视频和结构化输出。

**分辨率：** LLaVA-1.5 运行在 336x336。对照片来说不错，但对于中文发票或密集的电子表格截图毫无用处。Qwen-VL 的第一个创新是 448x448 和带定位的边界框输出，让模型能够指向事物。

**视频：** Video-LLaMA 堆叠了逐帧编码器并将其输入 LLM。它适用于短视频片段，但不适用于时间轴本身就是信号的多分钟视频。Qwen 团队希望有一个能理解时间的单一编码器。

**结构化输出：** LLaVA 输出自由格式文本。而智能体需要 JSON。Qwen-VL 在显式的 JSON 输出格式上进行了训练，包括以文本形式输出的边界框坐标。

每一代 Qwen-VL 都在扩展这三个轴之一。

## 概念

### Qwen-VL (2023 年 8 月)

第一代：OpenCLIP ViT-bigG/14 编码器（2.5B 参数），兼容 LLama 的 Q-Former（1步，256 查询），Qwen-7B 基座。贡献：

- 448x448 分辨率（当时开源 VLM 的最新水平）。
- 定位：在带有显式坐标token输出的图文对上训练。"猫在 <box>(112, 204), (280, 344)</box>"。
- 从一开始就进行中英文多语言训练。

当时的基准测试：在英文上与 GPT-4V 竞争，在中文上占主导地位。定位监督才是真正的亮点。

### Qwen2-VL (2024 年 9 月) — M-RoPE 与原生分辨率

Qwen2-VL 用原生动态分辨率的 ViT 编码器取代了固定分辨率 + Q-Former 的堆叠。关键变化：

- **原生动态分辨率。** ViT 接受任何能被 28 整除的 H×W（patch 14，2x 空间合并）。1120x672（40x24 合并 patch）的图像产生 960 个视觉 token。无需调整大小、无需分块、无需缩略图。
- **M-RoPE（多模态 RoPE）。** 每个 token 携带一个 3D 位置 (t, h, w) 而不是 1D。对于图像 t=0，对于视频 t=帧索引。RoPE 在每个轴上以不同的频率旋转查询/键向量。没有位置嵌入表。
- **MLP 投影器。** 丢弃 Q-Former；在合并后的 patch token 上使用 2 层 MLP。
- **支持动态 FPS 的视频。** 默认以 1-2 FPS 采样视频，但模型接受任意帧数。

结果：Qwen2-VL-7B 在多个多模态基准测试中与 GPT-4o 持平，并在 DocVQA 上击败了它（94.5 vs 88.4）。架构变化是决定性的举措。

### Qwen2.5-VL (2025 年 2 月) — 动态 FPS + 绝对时间

Qwen2.5-VL 的重大转变在于视频。动态 FPS 不仅仅是“在需要时采样更多帧”。论文形式化了：

- **绝对时间 token。** 不使用位置索引（第 0、1、2 帧...），而是使用实际时间戳。“在 0:04 时，猫跳了起来。”模型看到与帧 token 交织的 `<time>0.04</time>` token。
- **动态 FPS。** 对于慢速素材以 1 FPS 采样，对于动作场景以 4+ FPS 采样。由用户或训练者选择；M-RoPE 适应性调整。
- **ViT 中的窗口注意力。** 空间注意力是窗口化的（块内局部）以提高吞吐量；每隔几层进行一次全局注意力。
- **显式的 JSON 输出格式。** 在工具调用数据上进行训练：`{"tool": "click", "coords": [380, 220]}`。开箱即用，适合智能体。
- **MRoPE-v2 缩放。** 位置随最大输入大小缩放，因此 10 分钟的视频不会超出频率范围。

基准测试：Qwen2.5-VL-72B 在大多数视频基准测试中击败了 GPT-4o，在文档上与 Gemini 2.0 持平，并在 GUI 定位（ScreenSpot: 84% 准确率 vs GPT-4o 的 38%）上设立了开源模型最佳水平。

### Qwen3-VL (2025 年 11 月)

Qwen3-VL 是一次增量升级，它巩固而非重新发明：更大的 LLM 基座（Qwen3-72B）、扩展的训练数据、改进的 OCR、通过 Qwen3 的“思考模式”实现的更强推理。ViT 和 M-RoPE 保持不变。论文侧重于数据和训练改进而非架构。

整个系列的要点：到 2025 年，Qwen-VL 架构已经稳定。后续几代扩展的是计算和数据，而非基本元素。

### M-RoPE 的数学原理

经典 RoPE 通过配对坐标将维度为 `d` 的查询向量 `q` 按位置 `m` 旋转：

```
q_rot[2i]   = q[2i]   * cos(m * theta_i) - q[2i+1] * sin(m * theta_i)
q_rot[2i+1] = q[2i]   * sin(m * theta_i) + q[2i+1] * cos(m * theta_i)
theta_i     = 10000^(-2i/d)
```

M-RoPE 将隐藏维度分成三个频带。假设 `d = 96`。分配 32 维给时间，32 维给高度，32 维给宽度。每个频带由其自身轴的位置旋转。位于 (t=5, h=10, w=20) 的 patch 分别对其三个频带应用旋转 `R_t(5)`、`R_h(10)`、`R_w(20)`。

文本 token 使用 `t = text_index, h = 0, w = 0`（或某种归一化选择），保持兼容性。视频帧使用 `t = frame_time, h = row, w = col`。单张图像使用 `t = 0`。

优点：一种位置编码处理文本、图像和视频，无需分支代码或不同的位置表。

### 动态FPS采样逻辑

给定时长为 `T` 秒的视频和目标 token 预算 `B`：

1. 计算你可负担的最大 FPS：`fps_max = B / (T * tokens_per_frame)`。
2. 从 `{1, 2, 4, 8}` 中选择一个满足 `fps <= fps_max` 的目标 FPS。
3. 如果运动量高（光流启发式或显式用户请求），选择更高的 FPS。如果运动量低，选择更低的 FPS。
4. 以选定的 FPS 均匀采样；在帧之间插入 `<time>t</time>` token。

Qwen2.5-VL 隐式地训练了这种逻辑；推理时用户通过 `fps` 参数控制。一段 60 秒的动作序列以 4 FPS 采样，每帧 81 个 token = 19440 个 token，在 32k 上下文中可以处理。

### 结构化智能体输出

Qwen2.5-VL 的智能体训练明确针对结构化工具调用：

```
{
  "tool": "mouse_click",
  "coords": [1024, 512],
  "button": "left",
  "modifier": null
}
```

解析是确定性的：对模型输出运行 JSON.parse。与需要正则表达式和歧义处理的自由格式 "click at (1024, 512)" 相比，这一转变解释了为什么 Qwen2.5-VL 的 ScreenSpot 分数从 Qwen2-VL 的 55% 跃升至 84%。

## 关键术语

| 术语 | 人们常说的 | 实际含义 |
|------|-----------|----------|
| M-RoPE | "多模态 RoPE" | 3D 旋转位置嵌入，在隐藏维度上具有时间、高度和宽度频带 |
| 动态 FPS | "智能采样" | 根据运动量、时长和 token 预算，为每个视频选择的帧采样率 |
| 绝对时间 token | "时间戳 token" | 序列中交错的 `<time>t</time>`，使模型看到实际秒数而不是帧索引 |
| 窗口注意力 | "局部注意力" | 限制在小窗口内的空间自注意力以提高速度；定期添加全局注意力 |
| 结构化智能体输出 | "JSON 模式" | 训练数据监督，教导 VLM 输出带坐标和工具名称的可解析 JSON |
| min_pixels / max_pixels | "分辨率边界" | 每次请求 Qwen2.5-VL 控制的总像素数，从而控制 token 数 |
| 定位 | "指向它" | 以文本 token 形式输出边界框坐标；自 Qwen-VL v1 起使用 |
