# 多模态RAG与跨模态检索

> 视觉原生文档RAG只是其中一部分。生产级多模态RAG的范围更广——在文本、图像、音频和视频之间进行检索，应用于行程规划（"帮我找一家安静、有自然光的纯素早午餐餐厅"）、医疗分诊（"这张照片加上这些笔记对应什么伤情"）、电商（"与我这张自拍照相似、符合我尺寸的服装"）和现场服务（"根据这个发动机声音和零件照片进行诊断"）等工作流。2025年的三篇综述——Abootorabi等人、Mei等人、Zhao等人——系统化了子问题：跨模态检索、检索融合、生成溯源、多模态评估。本课将阅读这些综述并设计一个生产级流水线。

**类型：** 构建
**语言：** Python（标准库，带融合机制的跨模态检索器 + 带溯源的生成器）
**前置条件：** 阶段12·23（ColPali），阶段11（RAG基础）
**时间：** 约180分钟

## 学习目标

- 设计跨模态检索：文本→图像、图像→文本、音频→视频等。
- 比较三种融合策略：分数融合、基于注意力的融合、MoE融合。
- 解释生成溯源：当来源是多种模态的混合时，"引用你的来源"是什么样子。
- 说出2025年三篇经典的多模态RAG综述及其子问题分类。

## 问题

单模态RAG是一个已解决的模式：嵌入查询、嵌入分块、检索、填充到LLM。多模态RAG需要：

1. 多个检索头（每种模态需要在兼容空间中拥有嵌入）。
2. 跨模态的检索结果融合。
3. 跨模态引用来源的生成溯源。
4. 涵盖跨模态信号的评估指标。

2025年的综述都得出了相同的分类。

## 概念

### 跨模态检索

给定模态A的查询，检索模态B的文档。三种模式：

1. 共享嵌入空间。CLIP和CLAP在共享空间中生成文本+图像/文本+音频嵌入。跨模态的余弦相似度可直接使用。仅限于CLIP训练过的配对。

2. 逐模态编码器+翻译。文本编码器+图像编码器+一个在空间之间映射的小型翻译模块。Gupta等人的Sen2Sen及其他2024年的设计。灵活但增加了复杂性。

3. VLM作为编码器。使用VLM的隐藏状态作为检索表示。VLM支持的任何模态均可工作。质量更高，成本更高。

选择：对于文本+图像使用CLIP/SigLIP 2；对于文本+音频使用CLAP；对于前沿质量的跨模态使用VLM隐藏状态。

### 融合策略

你检索了10个结果：5张图像、3段文本、2个音频片段。如何合并？

分数融合（最便宜）。每种模态有自己的检索器，各自返回分数。在模态内部归一化分数然后求和。简单，通常有效。

基于注意力的融合。将所有检索项拼接起来，让一个小型注意力网络为它们加权。需要训练。

MoE融合。门控网络将查询路由到特定模态的专家。不同的查询类型路由不同——视觉问题给图像更高的权重。

生产默认设置：分数融合，并轻微偏向查询的主要模态。如果A/B测试在你的领域显示明显优势，则升级到MoE。

### 生成溯源

LLM应引用哪个检索项支持了每个断言。对于多模态：

- 文本来源：标准引用 `[1]`。
- 图像来源：`[img 3]` 并附简短标题。
- 音频：`[audio 2 at 0:34]`。

使用溯源感知数据训练生成器：训练目标中的每个断言都标记有来源索引。推理时，模型自然会产生引用。

### 2025年综述

Abootorabi等人（arXiv:2502.08826，"以任意模态提问"）：多模态RAG的分类法。涵盖检索、融合、生成。覆盖范围最广。

Mei等人（arXiv:2504.08748，"多模态RAG综述"）：侧重于子任务基准和失败模式。对评估设计有用。

Zhao等人（arXiv:2503.18016）：视觉聚焦的综述。在ColPali系列工作方面很强。

阅读这三篇可以了解截至2025年春季的最新技术水平。大多数子问题仍然开放。

### MuRAG——基础论文

MuRAG（Chen等人，2022）是第一个多模态RAG。从多模态知识库中检索图像+文本，生成答案。在VLM浪潮之前展示了可行性。现代系统（REACT、VisRAG、M3DocRAG）建立在它之上。

### 生产级行程规划器示例

查询："帮我找一家安静、有自然光的纯素早午餐餐厅。"

流水线：

1. 分解查询。"安静" → 音频/评论关键词；"纯素早午餐" → 菜单项；"自然光" → 图像特征。
2. 逐模态检索：
   - 对评论进行文本检索："纯素早午餐，安静氛围。"
   - 对餐厅照片进行图像检索："自然光，通风。"
   - 对环境声音片段进行音频检索："低分贝，无音乐。"
3. 融合分数。每家餐厅都有一个综合分数。
4. 前k家餐厅 → VLM生成器，包含所有证据 → 带引用的答案。

这远远超出了文本RAG。每种模态都增加了文本单独无法捕捉的信号。

### 智能体多模态RAG

多跳：如果第一次检索没有返回高置信度答案，LLM会重新表述并再次检索。阶段14中的智能体RAG模式在此适用。示例：

- 检索初始前10项 → LLM询问"太嘈杂，过滤出<40 dB" → 重新检索。
- 检索图像 → LLM看到其中一个有菜单 → 检索菜单文本 → 回答。

增加了复杂性，但能处理单次检索无法处理的查询。

### 评估

跨模态评估仍然不成熟。常见替代指标：

- 每模态的Recall@k。
- 融合后的前k准确率。
- 人工判断的端到端满意度。
- 任务特定（预订完成、购买完成）。

没有标准基准涵盖所有模态。大多数论文在领域特定任务上进行评估。

## 使用它

`code/main.py`：

- 三个模拟检索器（文本、图像、音频）在共享的餐厅语料库上运行。
- 分数融合，将模态分数与可配置权重结合。
- 一个生成器桩，输出带有引用的最终答案。
- 一个简单的智能体循环，在置信度低时重新表述查询。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|----------|
| 跨模态检索 | "用一种模态查询，检索另一种" | 文本查询检索图像；图像查询检索文本；需要共享空间或翻译器 |
| 分数融合 | "合并分数" | 每模态检索分数的加权和；最简单的融合 |
| MoE融合 | "模态路由专家" | 门控网络为每个查询选择信任哪种模态的分数 |
| 溯源生成 | "引用你的来源" | 答案中的每个断言标记有来源索引 |
| MuRAG | "第一个多模态RAG" | 2022年建立了多模态RAG模式的论文 |
| 智能体多跳 | "重新表述并重试" | 当第一次通过置信度低时，LLM重新查询检索器 |
