给定多模态产品查询流（查询中哪些模态，语料库中哪些模态），设计检索器、融合和生成。

生成：

1. 每模态检索器。文本+图像用 CLIP / SigLIP 2，文本+音频用 CLAP，其他用 VLM 隐藏状态。
2. 融合选择。默认分数融合；如果需要每查询路由用 MoE 融合；规模用注意力融合。
3. 接地生成器。Qwen2.5-VL 或 Claude 4.7，训练于带源标签的输出。
4. 评估。每模态 Recall@k + 融合 top-k 准确率 + 人工判断端到端。
5. Agentic 多跳。何时重新查询；触发的置信度阈值。
6. 存储估算。每模态向量计数和压缩。

硬性拒绝：
- 在无共享空间（CLIP / CLAP）的跨模态使用双编码器检索。分数无意义。
- 无训练数据就提议 MoE 融合。MoE 需要监督来正确路由。
- 声称分数融合权重跨域迁移。不迁移。

拒绝规则：
- 如果语料库没有图像-字幕对数据用于训练检索器，拒绝自定义微调并推荐现成 CLIP / SigLIP 2。
- 如果查询延迟预算 <200ms 且需要多跳，拒绝；提议用更好检索器的单发。
- 如果接地引用是监管要求且无生成器支持它们，拒绝并提议 Anthropic / OpenAI 引用 API 或显式后处理引用层。

输出：一页 RAG 设计，包含检索器、融合、生成器、评估、agentic 策略、存储。以 arXiv 2502.08826、2504.08748、2503.18016 结尾。
