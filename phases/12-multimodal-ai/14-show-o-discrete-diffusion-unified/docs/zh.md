# Show-o 与离散扩散统一模型

> Transfusion 混合了连续和离散表示。Show-o（Xie 等人，2024年8月）则走了另一条路：文本 token 使用因果式下一个 token 预测，图像 token 使用基于 MaskGIT 精神的掩码离散扩散（Masked Discrete Diffusion）。两者共存于一个具有混合注意力掩码的 Transformer 中。其结果是在单一骨干网络、每种模态一个分词器、单一损失公式（将下一个 token 预测扩展到掩码预测）上统一了 VQA、文生图、图像修复和多模态生成。本课程将讲解 Show-o 的设计——为什么掩码离散扩散是一种并行、少步数的图像生成器——并与 Transfusion 和 Emu3 进行对比。

**类型：** 学习
**语言：** Python（标准库，掩码离散扩散采样器）
**前置知识：** 阶段 12 · 13（Transfusion）
**时长：** 约 120 分钟

## 学习目标

- 解释掩码离散扩散：一种调度策略，均匀地掩码 token，然后让 Transformer 恢复它们。
- 比较并行图像解码（Show-o, MaskGIT）与自回归图像解码（Chameleon, Emu3）在速度和质量上的差异。
- 列举 Show-o 在单个检查点中处理的三个任务：T2I、VQA、图像修复。
- 选择一种掩码调度（余弦、线性、截断）并分析其对样本质量的影响。

## 问题

Transfusion 的双损失训练虽然有效，但动力学更为复杂——连续扩散损失的数值尺度与离散的 NTP 损失不同。平衡损失权重需要进行超参数搜索。该架构有效但复杂。

Show-o 的解决方案：将两种模态都保持为离散（类似 Chameleon），但通过掩码离散扩散并行生成图像，而非顺序生成。训练目标变为单一的掩码 token 预测，自然地泛化了下一个 token 预测。

## 概念

### 掩码离散扩散（MaskGIT）

原始的 Chang 等人（2022）MaskGIT 技巧非常优雅。从完全被掩码的图像开始（每个 token 都是特殊的 `<MASK>` id）。每一步，并行预测所有被掩码的 token，然后保留最置信的前 K 个预测，并重新掩码其余部分。经过约 8-16 次迭代，所有 token 都被填充。每一步取消掩码的 token 数量由调度决定——余弦调度效果良好。

训练很简单：从 [0, 1] 均匀采样一个掩码比例，将其应用于图像的 VQ token，训练 Transformer 恢复被掩码的 token。这正是 BERT 对文本所做的，但已扩展到图像生成。

### Show-o：一个 Transformer，混合掩码

Show-o 将 MaskGIT 置于一个因果式语言模型 Transformer 中。注意力掩码是：

- 文本 token：因果式（标准 LLM）。
- 图像 token：在图像块内部是全双向的（这样被掩码的 token 在预测时可以看到所有其他图像 token）。
- 文生图：文本关注之前的图像，图像关注之前的文本。

训练在以下之间交替进行：
1. 文本序列的标准 NTP。
2. T2I 样本：文本 → 带有掩码图像 token 的图像，使用掩码 token 预测损失。
3. VQA 样本：图像 → 带有掩码文本 token 的文本（实际上就是 NTP）。

统一的损失是在 `<MASK>` token 上的交叉熵，它涵盖了文本 NTP（只有最后一个 token 被“掩码”）和图像掩码扩散（随机子集被掩码）。

### 并行采样

Show-o 在大约 16 步内生成一张图像，而不是大约 1000 步（逐 token 自回归）或大约 20 步（扩散）。每一步，并行预测所有被掩码的 token；提交最置信的前 K 个；重复。

比较：
- Chameleon / Emu3（对 token 自回归）：需要 N_tokens 次前向传播，每张图像通常为 1024-4096 次。
- Transfusion（连续扩散）：约 20 步，每一步都是完整的 Transformer 前向传播。
- Show-o（掩码离散扩散）：约 16 步，每一步都是完整的 Transformer 前向传播。

Show-o 在相似规模的模型上比 Chameleon 更快，步数与 Transfusion 大致相当，但每步成本更低（离散词表 logits 与连续 MSE 损失）。

### 单个检查点中的任务

Show-o 在推理时支持四个任务，通过提示格式选择：

- 文本生成：标准自回归文本输出。
- VQA：图像输入，文本输出。
- T2I：文本输入，通过掩码离散扩散输出图像。
- 图像修复：图像中部分 token 被掩码，进行填充。

图像修复能力来自掩码预测训练。对 VQ token 网格的一个区域进行掩码，输入其余部分加上文本提示，预测被掩码的 token。

### 掩码调度

每步取消掩码的 token 数量调度会影响质量。Show-o 推荐余弦调度：

```
mask_ratio(t) = cos(pi * t / (2 * T))   # t = 0..T
```

第 0 步时，所有 token 被掩码（比例为 1.0）。第 T 步时，无掩码。余弦调度将质量集中在预测最具信息量的中等比例范围。线性调度也可以，但会更快地达到平台期。

### Show-o2

Show-o2（2025 年后续工作，arXiv 2506.15564）对 Show-o 进行了扩展：更大的 LLM 基础、更好的分词器、改进的掩码调度。架构模式相同。

### Show-o 的定位

在 2026 年的分类中：

- 离散 token + NTP：Chameleon, Emu3。简单但推理慢。
- 离散 token + 掩码扩散：Show-o, MaskGIT, LlamaGen, Muse。并行采样，仍受分词器有损影响。
- 连续 + 扩散：Transfusion, MMDiT, DiT。质量最高，训练更复杂。
- 连续 + 流匹配在 VLM 中：JanusFlow, InternVL-U。最新。

根据任务选择：当您希望在一个开放模型中获得 T2I + 图像修复 + VQA 且速度合理时，选择 Show-o；当质量最重要且您可以承受双损失管道的复杂性时，选择 Transfusion。

## 使用它

`code/main.py` 模拟了 Show-o 采样：

- 一个由 16 个 VQ token 组成的玩具网格。
- 一个“模拟” Transformer，它根据提示和当前未掩码的 token 预测 logits。
- 采用余弦调度，在 8 步内进行并行掩码采样。
- 打印中间状态（掩码模式演变）和最终 token。

运行它，观察掩码逐步溶解。

2. 图像修复对于掩码扩散来说是免费的。请提出一个实际或假设的产品用例，其中 Show-o 的图像修复优于专用模型。

3. 余弦调度与线性调度：针对 T=8，追踪每步未掩码 token 的数量。哪个更平衡？

## 关键术语

| 术语 | 人们通常的说法 | 实际含义 |
|------|-----------------|------------------------|
| 掩码离散扩散 | "MaskGIT 风格" | 训练预测掩码 token；推理时，迭代地取消最置信的预测的掩码 |
| 余弦调度 | "取消掩码调度" | 推理步骤中掩码比例的衰减；将置信度增长集中在中程范围内 |
| 并行解码 | "一次性所有 token" | 每一步通过一次前向传播预测完整的掩码 token 序列，然后提交前 K 个 |
| 混合注意力 | "因果 + 双向" | 在文本 token 上是因果的，在图像块内部是双向的掩码 |
| 图像修复 | "填充生成" | 以具有部分掩码 token 的图像为条件，预测缺失的部分；来自训练目标的免费能力 |
| 提交率 | "每步前 K 个" | 每次迭代中声明为“完成”的 token 数量；控制推理速度与质量的权衡 |
